## 배경 및 문제 정의

기존 `/결정` 스킬의 아쉬운 점:
- 결론 카드가 너무 짧고 단조로움 (3줄)
- 대화 중 응답이 단답형 → 유저가 스스로 고민하고 답을 이끌어내기에 부족
- 질문만 던지고 경우의 수/방향에 대한 발판 없음

원하는 경험:
- 초반엔 거울(내 말을 반영해 스스로 답 발견) + 후반엔 코치(못 보는 각도 짚어주기)
- 질문하면서 생각해볼 수 있는 경우의 수와 해결 방향을 발판으로 같이 제시
- 턴 수 제한 없음 (자연스러운 흐름 우선)

## 설계 방향

### 구조: GROW 뼈대 + 유형별 프레임워크 + 반영→질문+발판 패턴

**GROW (B)**: 모든 고민에 적용되는 대화 뼈대  
**유형별 프레임워크 (C)**: 고민 유형에 맞는 심층 질문 방법론  
**반영→질문+발판**: 매 턴 공통으로 적용되는 대화 패턴

---

## 대화 흐름

### Step 0 — 접수 및 반영

고민을 받으면 **질문 금지**. 반영 한 줄 먼저.

형식:
> "[고민 핵심 요약] — 맞아요?"

목적: 유저가 "내 말을 들었다"는 신뢰를 갖게 함. 고민을 외부에서 보는 효과.

MI 양면 반영 적용 (방향형 고민):
> "하고 싶은 마음도 있고, 멈추고 싶은 마음도 동시에 있는 거군요."

---

### Step 1 — G턴 (목표 명확화)

진짜 원하는 게 뭔지 확인. **발판 옵션 3개** 포함.

형식:
> "[오픈 질문] — [예시 A]인지, [예시 B]인지, 아니면 완전히 다른 뭔가인지."

예시:
> "이 결정에서 진짜 원하는 게 뭐예요? — 완성하는 것인지, 지금 이 상태를 바꾸는 것인지, 아니면 완전히 다른 뭔가인지."

---

### Step 2 — R턴 (현실 파악)

상황을 있는 그대로 파악. 이 단계에서 내부적으로 **고민 유형 판단**.

- **선택형**: 선택지가 이미 있는 고민 ("A vs B 중 뭐가 나을까")
- **방향형**: 계속/그만 여부를 모르는 고민 ("이걸 계속 해야 하나")
- **우선순위형**: 순서를 모르는 고민 ("뭘 먼저 할지 모르겠어")

예시:
> "지금 어디까지 왔어요? 막히는 지점도요."

---

### Step 3 — O턴 (가능성 탐색)

유형에 따라 다른 프레임워크로 깊이 탐색. 매 질문에 발판 옵션 3개 + 열린 끝 유지.

#### 선택형 → 소크라테스 가정 탐색 + 10/10/10

가정 탐색:
> "두 선택지 중 어느 게 낫냐는 질문 이전에 — '좋은 선택'이라고 보는 기준이 뭐예요? — 오래 남는 것인지, 지금 당장 편한 것인지, 아니면 다른 뭔가인지."

10/10/10:
> "그 선택을 했다고 상상하면 10분 뒤 첫 감정이 뭐예요? 10년 뒤에 돌아보면요?"

#### 방향형 → Pre-mortem + 후회 최소화

Pre-mortem:
> "그 방향으로 갔다가 6개월 뒤 후회하고 있다면 이유가 뭐였을 것 같아요? — 너무 일찍 멈춘 건지, 억지로 밀어붙인 건지, 아니면 다른 뭔가인지."

후회 최소화:
> "80세에 돌아볼 때 지금 이걸 안 한 걸 후회할 것 같아요?"

#### 우선순위형 → 아이젠하워 2축 + 2차 사고

아이젠하워:
> "이것들 중에서 지금 안 하면 실제 타격이 오는 것 vs 나중에 해도 되는 것 — 어떻게 나뉘어요?"

2차 사고:
> "제일 먼저 처리하면 그 다음에 뭐가 풀려요? — 시간이 생기는 건지, 에너지가 생기는 건지, 아니면 다른 뭔가인지."

---

### Step 4 — 닫기 전 확인

결론 카드 출력 전 마지막 점검.

> "지금까지 보면 [방향]으로 기울어진 것 같은데 — 아직 걸리는 게 있어요? 없으면 여기서 닫을게요."

유저가 "없어요" 또는 침묵하면 → 결론 카드 출력.  
유저가 새 걱정을 꺼내면 → O턴으로 돌아가서 처리 후 재확인.

---

### Step 5 — 결론 카드

```
📌 결정: [선택한 것]

🎯 기준
- [핵심 기준 1 — 유저가 대화 중 쓴 언어 그대로]
- [기준 2 — 대화에서 두 번째 기준이 언급됐을 때만 추가, 없으면 생략]

⚠️ 알고 하는 것: [가장 큰 리스크 or 포기하는 것]

👣 첫 걸음: [결정 후 첫 번째 행동]

📅 날짜: YYYY-MM-DD
```

**필수 조건:**
- 🎯 기준은 유저가 대화 중 실제로 쓴 단어/표현 사용
- ⚠️ 항목: 재확인 질문 없이 Claude가 대화에서 파악한 내용으로 채움
- 👣 항목: GROW의 Will 단계 — 결정이 행동으로 이어지게

## 핵심 규칙

| 규칙 | 내용 |
|------|------|
| 반영 우선 | 매 턴 질문 전에 반영 한 줄 먼저 |
| 발판 3개 | 오픈 질문 뒤 예시 옵션 3개 + "아니면 다른 뭔가?" |
| 옵션은 발판 | "예를 들면"으로 프레이밍 — 선택지 아님 |
| 파고들기 | 유저가 발판 중 하나를 고르면 "그게 어느 쪽인지 더 말해줄 수 있어요?"로 깊이 탐색 |
| 질문 1개 | 1턴에 질문 1개만 |
| 유저 언어 사용 | 결론 카드에 유저가 쓴 단어 그대로 |
| 턴 제한 없음 | 자연스러운 흐름이 되면 3턴 이상도 가능 |
| 말투 | 짧고 직접적. "~해보면 어떨까요?" 같은 컨설턴트 말투 금지 |

## 연구 기반

**1차 리서치 (코칭 방법론 + 결정 프레임워크):**
- GROW 모델 → 대화 뼈대
- Motivational Interviewing (MI) → 반영/요약 기법
- 소크라테스식 질문법 → 가정 탐색
- 10/10/10, 후회 최소화, Pre-mortem, 아이젠하워, 2차 사고 → 유형별 질문

**2차 리서치 (질문+옵션 패턴):**
- MI 메뉴 제공 → 발판 옵션 3개 + "아니면 다른 뭔가?"
- 앵커링 회피 원칙 → 질문 먼저, 옵션은 "예를 들면"으로
- 탐색형 질문 패턴 → 닫힌 질문 → 열린 질문으로 전환
