# 나를 알아가는 내 OS — 설계 문서

**날짜:** 2026-07-13
**코어 OS:** 나를 알아가는 내 OS (세컨브레인 · 라이프 컨설턴트)
**저장 위치:** `C:\Users\USER\mybrain\` (로컬 개인 폴더, 클럽 repo에 절대 미포함)
**버전:** v1 (첫 슬라이스 = 캡처 + 온디맨드 반추)

---

## 배경 및 문제 정의

스폰지클럽 6주 동안 여러 OS를 시도하되, "코어 1개 + 곁가지" 구조로 간다.
코어는 **허브 모양**이어야 곁가지가 자연스럽게 꽂히는데, 4개 후보(AI공부·오픽비즈니스AI회사·오픽컨텐츠자동화·나를아는OS) 중 유일하게 데이터를 빨아들이는 허브 모양이 **나를 알아가는 내 OS**다.

목표: 텔레그램으로 내 관심·시간·정보를 계속 기록해, "실제 쓰는 시간 vs 하고 싶다던 것"의 격차를 마주하고, 50대 중반 이후 라이프를 새로 설계하는 동반자를 만든다.

**최대 리스크:** 이 OS는 "매일 쓰는 습관"이 안 붙으면 죽는다. 그래서 첫 슬라이스는 기능이 아니라 **매일 할 행동(텔레그램에 던지기)** 위에 설계한다.

---

## 전체 그림 — 3층

1. **기록층 (Capture)** — 텔레그램에 던지면 자동 태깅되어 쌓임 ← **이번 슬라이스**
2. **반추층 (Reflect)** — 쌓인 걸 보고 "너 이런 사람이야 / 여기 시간 쏠렸어" 되돌려줌 ← **이번 슬라이스(온디맨드)**
3. **논의층 (Consult)** — 그 위에서 라이프·사업을 밀착 논의 ← 데이터 쌓인 뒤(향후)

곁가지(AI공부 #1, 오픽비즈니스 #2·#3)는 나중에 기록층에 **입력원**으로 꽂는다.
예: 기존 `ai-tool-briefing` 스킬이 만든 요약 → `🔗인풋` 카테고리로 자동 유입.

## 데이터 모델 (이 OS의 심장)

### 2층 분류: 카테고리(형식) + 태그(주제)

한 기록은 **카테고리 1개**(무슨 종류의 기록이냐) + **태그 여러 개**(무슨 주제냐)로 분류한다.
이 분리가 "추후 추가 정리"를 가능하게 한다 — 나중에 `태그=오픽`만, 또는 `카테고리=시간 & 태그=AI공부`만 자유롭게 잘라볼 수 있음.

### 카테고리 (닫힌 목록, 6개 + 안전망)

| 카테고리 | 코드 | 담는 것 |
|---|---|---|
| 💭 생각 | `thought` | 아이디어·고민·깨달음 |
| ⏱️ 시간 | `time` | "뭘 하며 시간 썼다" (시간량 포함) |
| 🔗 인풋 | `input` | 읽은 것·링크·영상·정보 |
| 🎯 바람 | `want` | 하고 싶다·되고 싶다·언젠가 |
| 😌 상태 | `mood` | 기분·에너지·컨디션 |
| 📥 기타 | `misc` | 분류 애매한 것 (메시지를 절대 안 버리는 안전망) |

- `😌상태`: "나를 알아가는" OS의 무기 — 언제 에너지가 좋고 언제 무너지나 패턴. 라이프 재설계의 근거.
- `📥기타`: 봇이 애매하면 여기로. 데이터 손실 방지.
- `할일·약속`은 의도적으로 제외 (할 일 관리 앱의 몫, 넣으면 스코프가 커짐).

### 태그 (열린 목록, 주제/영역)

`#오픽` `#AI공부` `#가족` `#건강` `#돈` `#유튜브` 등. 봇이 던질 때 자동 부여.
`meta/tags.md`에 정규화 규칙 관리(예: "AI공부"="AI학습"을 하나로).

### 원본 레코드 (JSONL 한 줄 = 한 기록)

```json
{
  "id": "20260713-2130-01",
  "ts": "2026-07-13T21:30",
  "occurred": "2026-07-13",
  "cat": "time",
  "tags": ["오픽", "기획"],
  "text": "오전에 오픽 컨텐츠 기획",
  "dur_min": 180,
  "raw": "오전에 오픽 컨텐츠 기획 3시간 했어"
}
```

- `ts`: 던진 시각 / `occurred`: 실제 있었던 날(어제 걸 오늘 적을 때 대비)
- `dur_min`: `time` 카테고리일 때만
- `raw`: 원문 보존 (태깅이 틀려도 원본에서 재분류 가능)

### 폴더 구조 — "원본 로그 + 사람용 카드" 2단

```
C:\Users\USER\mybrain\
├─ log\2026-07.jsonl      # 원본. append만, 절대 수정 안 함 = 진실의 근원
├─ daily\2026-07-13.md    # 사람이 읽는 하루 카드 (원본에서 파생)
├─ weekly\2026-W28.md     # 주간 반추 (파생)
└─ meta\tags.md           # 태그 사전 / 정규화 규칙
```

원본 JSONL은 append만 하므로 봇이 망가뜨릴 일이 없고, 구조화돼 있어 어떤 각도로든 다시 잘라볼 수 있다. `.md` 카드는 언제든 원본에서 다시 뽑는 파생물.

## 첫 슬라이스 동작

### 1) 캡처 (매일 하는 유일한 행동)

- 사용자가 텔레그램 봇에 **아무거나** 던짐 (생각/시간로그/링크/바람/상태).
- 봇(Claude)이:
  1. 카테고리 1개 + 태그 부여, `occurred`·`dur_min` 추출
  2. `log/YYYY-MM.jsonl`에 한 줄 append
  3. 짧게 확인 답장: `⏱️시간 · #오픽 #기획 · 180분으로 기록했어요`

### 2) 분류 교정 루프 (데이터 품질의 핵심)

- 봇이 매긴 분류를 답장으로 보여줌.
- 사용자가 👎 하거나 "그거 시간 아니고 바람이야" 한마디 → 해당 줄만 고쳐 재기록.
- 정정 자체도 학습 재료(자주 틀리는 패턴은 `meta/tags.md`에 반영).

### 3) 온디맨드 반추 카드

- 사용자가 "오늘 어때?" / "이번 주 어때?" 하면 원본에서 파생 카드 생성:
  - 오늘/이번 주 기록 요약
  - **시간이 어디에 쏠렸나** (카테고리·태그별)
  - **"하고 싶다던 것(want) vs 실제 쓴 시간(time)" 격차 한 줄** ← 이 OS의 심장
  - 내일/다음 주를 위한 질문 1개
- 저녁 자동 발송(스케줄)은 첫 슬라이스 이후 스트레치.

## 성공 기준 (첫 슬라이스)

1. 텔레그램에 던진 메시지가 올바른 카테고리·태그로 `jsonl`에 쌓인다.
2. 잘못 분류된 걸 한마디로 정정하면 그 줄이 고쳐진다.
3. "오늘 어때?"에 시간 쏠림 + want↔time 격차가 담긴 카드가 나온다.
4. (습관 지표) 며칠 연속 실제로 던지게 되는가 — 이게 진짜 성공 신호.
