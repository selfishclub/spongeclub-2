# 🧽 가상 고객 피드백 스킬 (virtual-customer-feedback)

> 내 프로덕트를 진짜 유저에게 내놓기 전에, **통계에 물린 한국인 페르소나**에게 먼저 부딪혀 보는 Claude 스킬.

스폰지클럽 2기 W4(실제 유저가 쓰는 프로덕트) 실습용으로 만들었어요.
NVIDIA가 공개한 [Nemotron-Personas-Korea](https://huggingface.co/datasets/nvidia/Nemotron-Personas-Korea) 데이터셋(CC BY 4.0)을 기반으로 합니다.

---

## 이게 왜 필요한가요

Claude에게 그냥 "30대 워킹맘인 척하고 피드백 줘"라고 하면, AI가 상상한 그럴싸한 30대 워킹맘이 나와요.
대체로 친절하고, 대체로 칭찬해 주고, 대체로 당신의 프로덕트를 쓰겠다고 합니다. **그런 피드백은 쓸모가 없어요.**

이 스킬은 KOSIS·대법원·국민건강보험공단 통계에 정렬된 실제 페르소나 데이터를 씁니다.
그래서 이런 사람들이 등장해요:

- 종이 신문을 구독하는 76세 부산 어르신 — 당신 앱의 온보딩 최종 보스
- 약관을 실제로 끝까지 읽는 64세 퇴직 행정 전문가
- 스마트스토어를 운영하며 아이템스카우트로 키워드 분석하는 34세 1인 사업자
- 리뷰를 집요하게 정독한 뒤에야 배달을 시키는 22세 취준생

여기에 **아첨 금지 · 안 쓸 이유 먼저 · 없는 정보 지어내지 않기** 규칙을 걸어두었습니다.

---

## 설치 방법 (복붙 2분)

### 방법 1 — 폴더 복붙 (클로드코드, 권장)

1. 이 폴더(`skills/virtual-customer-feedback/`)를 통째로 복사
2. 내 `~/.claude/skills/virtual-customer-feedback/` 에 붙여넣기
3. 끝! 클로드코드에서 바로 쓰여요

```bash
# 레포를 이미 클론했다면
cp -r skills/virtual-customer-feedback ~/.claude/skills/
```

### 방법 2 — .skill 파일 업로드 (클로드 웹·앱)

1. 이 폴더의 **`virtual-customer-feedback.skill`** 파일을 다운로드
   (파일명 클릭 → 우측 상단 **Download** 아이콘)
2. [claude.ai](https://claude.ai) → **설정(Settings)** → **Capabilities** → **Skills**
3. **Upload skill** → 받은 `.skill` 파일 선택
4. 목록에 `virtual-customer-feedback`가 보이면 완료

### 작동 확인

새 대화에서 이렇게 말해보세요.

```
30대 워킹맘을 위한 주간 반찬 구독 서비스를 만들었어. 가상 고객한테 피드백 받아줘.
```

Claude가 페르소나 3~4명을 불러 인터뷰를 시작하면 정상 작동입니다.

## 어떻게 쓰나요

스킬 이름을 부를 필요는 없어요. **"고객 반응이 궁금하다"는 맥락이면 알아서 발동합니다.**

### 5가지 모드

| 모드 | 이렇게 말하면 됩니다 | 뭘 알게 되나요 |
|---|---|---|
| ① **패널 인터뷰** | "가상 고객한테 피드백 받아줘" | 이걸 쓸까, 왜 안 쓸까 |
| ② **5초 테스트** | "내 랜딩페이지 카피 테스트해줘" | 3초 안에 뭔지 이해되는가 |
| ③ **비고객 스트레스 테스트** | "누가 이걸 절대 안 쓸까?" | 경계 밖 사람이 거절하는 이유 |
| ④ **온보딩 워크스루** | "가입 흐름 어디서 이탈할까?" | 어느 단계에서 막히는가 |
| ⑤ **가격 수용도** | "월 9,900원이면 살까?" | 지불 의사와 그 근거 |

### 실제 사용 예시

```
내 상세페이지 카피야: "당신의 하루를 바꾸는 AI 습관 코치, 지금 무료로 시작하세요."
5초 테스트 해줘.
```

```
1인 자영업자용 리뷰 관리 툴인데 월 19,000원이면 살까?
그리고 누가 절대 안 쓸지도 궁금해.
```

```
회원가입 → 프로필 설정 → 첫 목표 등록 순서인데, 어디서 사람들이 나갈 것 같아?
```

### 타깃을 아직 못 정했어도 괜찮아요

프로덕트 설명만 던지면 스킬이 **타깃 후보 2~3개를 근거와 함께 제안**합니다.
그중에서 고르거나, 전혀 다른 타깃을 말해도 돼요. **최종 선택은 항상 여러분 몫입니다.**

## 세션이 끝나면 — 이기적공유 카드

어떤 모드를 쓰든 마지막에 조별 공유용 카드가 나옵니다. 발표 1~2분 분량이에요.

```
## 🧽 가상 고객 피드백 이기적공유 카드
- 프로덕트: (한 줄)
- 만난 가상 고객: (이름·나이·직업)
- 가장 의외였던 반응: (페르소나 발언 인용)
- 그래서 바꾸기로 한 것: (한 가지)
- 아직 확인 못한 것: (실제 고객에게 물어볼 질문)
```

**"가장 의외였던 반응"이 공유회에서 제일 좋은 소재예요.**
예상대로였던 반응 말고, 예상을 깬 반응을 골라 담아보세요.

---

## ⚠️ 꼭 기억할 것

> **가상 피드백은 실제 고객 검증의 전 단계입니다.**

이 스킬은 진짜 고객을 대신하지 않아요. 실제 유저를 만나기 전에 프로덕트를 다듬는 용도예요.
특히 **가격 수용도**는 방향 감각용일 뿐, 진짜 가격 검증은 실제 결제로만 확인됩니다.

## 페르소나가 부족할 때

기본 제공되는 스타터 팩은 30명(5개 세그먼트)입니다.
특정 직업·지역·조건의 페르소나가 더 필요하면 추출 스크립트를 쓰세요. Claude Code 등 로컬 환경이 필요해요.

```bash
pip install datasets

# 마케팅 직군 5명
python scripts/extract_personas.py --occupation 마케팅 --n 5

# 서울 거주 25~39세 8명
python scripts/extract_personas.py --age-min 25 --age-max 39 --province 서울 --n 8

# 서술에 '배달'이 등장하는 사람 5명
python scripts/extract_personas.py --keyword 배달 --n 5
```

추출된 `.md` 파일을 대화에 첨부하면 Claude가 그 페르소나로 인터뷰합니다.
