# 제출 (스폰지클럽 2기)

멤버가 과제나 인사이트를 올릴 때 **PR·브랜치·머지를 신경 쓰지 않고** 한 마디로 제출하게 해주는 스킬.

## 언제 쓰나
멤버가 이렇게 말할 때 발동:
- "나는 3조 미미야, 제출해줘"
- "과제 제출해줘", "인사이트 올려줘", "submit"

## 1단계 — 멤버 파악
1. 메시지에서 **조 번호**와 **닉네임**을 읽는다. (예: "3조 미미" → 조=`3`, 닉네임=`미미`)
   - 안 주어졌으면 한 번만 묻는다: **"몇 조의 누구신가요? (예: 3조 미미)"**
2. 본인 폴더를 찾는다. `1_mission/{조}조/` 와 `2_insights/{조}조/` 안에서:
   - `{닉네임}(` 으로 **시작하는** 폴더 (예: `미미(박미영)`), 또는
   - 닉네임이 없는 분은 **이름과 정확히 일치**하는 폴더 (예: `곽동욱`)
   - 못 찾으면 해당 조 폴더 목록을 보여주고 확인받는다.

## 2단계 — 제출 절차 (순서대로 실행)
```bash
# 1) 최신화 (충돌 예방)
git pull --no-edit

# 2) 본인 폴더 변경분만 stage (존재하는 경로만) — ⚠️ 본인 폴더 밖은 절대 add 금지
git add "1_mission/{조}조/{내폴더}/" "2_insights/{조}조/{내폴더}/" 2>/dev/null

# 3) 변경 없으면 멈춤
#    git diff --cached --quiet 면 "변경된 내용이 없어요. 파일을 저장하셨나요?" 안내 후 종료

# 4) 브랜치 (ASCII로 — 한글 ref 회피)
TS=$(date +%Y%m%d-%H%M%S)
git checkout -b "submit/jo{조}-$TS"

# 5) 커밋
git commit -m "{닉네임}({이름}) 제출"

# 6) push
git push -u origin "submit/jo{조}-$TS"

# 7) PR 생성·머지 (정리용 — 멤버에겐 안 보임)
gh pr create --base main --title "{닉네임} 제출" --body "자동 제출" 2>&1
gh pr merge --merge --delete-branch 2>&1

# 8) 동기화
git checkout main && git pull --no-edit
```

## 3단계 — 보고
멤버에게는 딱 이렇게만:
> ✅ 제출 완료! 반영됐어요 🎉

브랜치·PR·머지·명령어는 **언급하지 않는다.** (멤버가 신경 안 쓰는 게 목적)

## 폴백 (막힐 때)
- **`gh` 없음/인증 안 됨**: 브랜치 push는 됐으니 `gh api`로 PR 생성·머지 시도 → 그래도 안 되면 `git checkout main && git merge "submit/jo{조}-$TS" && git push origin main` (main은 안 잠겨 있음).
- **push 자체가 안 됨(네트워크)**: "업로드가 막혀 있어요. 회사/카페 와이파이가 git 업로드를 차단하는 경우가 많아요. **폰 핫스팟**으로 바꿔서 다시 시도해보세요." 안내.
- **충돌(conflict)**: 멈추고 상황을 설명. 본인 폴더만 다루므로 충돌은 드물다.

## 안전 규칙
- 본인 폴더 밖 파일은 **절대 커밋하지 않는다.**
- 비밀번호·API 키 등 민감 정보가 보이면 경고하고 멈춘다.
- 제출 전 항상 `git pull`로 최신화한다.
