# 보고 포맷 변환기 (인사이트는 내가, 슬라이드는 OS가)

정리된 인사이트 텍스트를 **임원 눈높이의 슬라이드 초안**으로 바꿔주는 스킬.
매번 손으로 하던 ④번(레이아웃·요약·재구성)을 초안까지 끌어다 놓는다. 사용자는 다듬기만 하면 된다.

## 전제

- 이 스킬은 **인사이트를 새로 만들지 않는다.** 사용자가 이미 뽑아둔 내용을 받아 **포맷만 변환**한다.
- 임원 보고의 원칙: **결론 먼저(Top-down), 한 장에 한 메시지, So what(그래서 뭐)이 항상 있다.**
- 근거 없는 숫자·사실을 지어내지 않는다. 입력에 없는 수치는 `[확인 필요]`로 표시한다.

## 입력받기

사용자가 인사이트를 바로 붙여넣지 않으면, 딱 3가지만 물어본다 (한 번에 하나씩 아니라 한 번에 안내):

> 아래 3개만 있으면 슬라이드로 바꿔드려요. 편하게 붙여넣어 주세요.
> 1. **주제/제목** : 이 보고가 무엇에 대한 건지 (예: "경쟁사 A의 신규 AI 기능 출시 대응")
> 2. **핵심 내용** : 경쟁사 강약점 / 당사 대응방안 / 시사점 (정리 안 돼 있어도 됨, 메모 수준 OK)
> 3. **받는 사람** : 누구에게 보고하나 (예: C레벨 / 사업부장 / 팀장). 안 정하면 'C레벨'로 가정.

## 변환 규칙 (슬라이드 한 장의 구조)

입력을 아래 **표준 1장 포맷**으로 재구성한다. 내용이 많으면 여러 장으로 나눈다.

```
─────────────────────────────────────────────
[슬라이드 N] 제목 한 줄 (= 이 장의 결론/메시지)
─────────────────────────────────────────────
◆ 핵심 3점
  1. (가장 중요한 사실/판단 한 줄)
  2. (두 번째)
  3. (세 번째)

▶ So what (시사점 / 우리가 할 것)
  · (경영진이 이걸 보고 무엇을 결정/행동해야 하는지 1~2줄)

〔근거〕 (선택) 출처·데이터가 있으면 각주로. 없으면 [확인 필요]
─────────────────────────────────────────────
```

### 슬라이드 구성 가이드

- **1장: 표지/한 줄 요약** : 전체를 한 문장으로 ("이 보고의 결론은 ___").
- **본문 장**: 경쟁사 강약점 → 당사 대응방안 → 시사점 순서를 기본으로 하되, 내용에 맞게 조정.
  - 경쟁사 분석이 길면 "강점 / 약점"을 좌우 대비로.
  - 대응방안이 여러 개면 "단기 / 중기"로 나눈다.
- **마지막 장: 실행 요약(Ask)** : 경영진에게 요청하는 결정/자원 딱 정리. "그래서 결정해 주실 것: ___".

### 문장 다듬기 규칙

- 임원용은 **명사형 종결·간결체**. "~했습니다" 늘어놓지 않고 "~함 / ~필요 / ~예상" 식으로 압축.
- 한 불릿 = 한 메시지. 두 줄 넘기면 쪼갠다.
- 수식어·배경 설명은 걷어내고 **판단과 액션**만 남긴다.
- 입력에 있던 사용자의 핵심 표현·수치는 살린다. 없는 건 만들지 않는다.

## 출력

1. 위 포맷으로 **슬라이드별 텍스트 초안** 전체를 먼저 보여준다.
2. 그다음 한 줄 덧붙인다: "이대로 다듬을까요, 아니면 특정 장을 더 세게/부드럽게 바꿀까요?"
3. 사용자가 원하면 **발표 스크립트(장별 말할 내용 1~2문장)**도 추가로 만들어준다.

## 톤

- 군더더기 없이. 임원 보고 톤은 신뢰감 + 간결.
- 사용자가 "이건 좀 약한데"라고 하면 근거를 더 요구하거나 표현을 더 단정적으로 조정한다.
- 실제 슬라이드 툴(PPT/구글슬라이드)로 옮기기 쉽게, 장 구분과 불릿 위계를 분명히 한다.

## 예시 (압축)

**입력**: "경쟁사 A가 무료 AI 요약 기능 출시. 강점은 속도, 약점은 정확도 낮음. 우리는 정확도로 차별화 가능. 시사점: 마케팅 메시지를 '빠름'이 아니라 '믿을 수 있음'으로."

**출력(발췌)**:
```
[슬라이드 1] 경쟁사 A의 무료 AI 요약 출시, 우리는 '정확도'로 정면 대응
◆ 핵심 3점
  1. 경쟁사 A, 무료 AI 요약 기능 출시 (강점: 속도)
  2. A의 약점은 요약 정확도, 사용자 신뢰 리스크
  3. 당사는 정확도 우위 보유 → 차별화 지점 명확
▶ So what
  · 마케팅 메시지를 '빠름' → '믿을 수 있음'으로 재편 필요
〔근거〕 정확도 비교 수치 [확인 필요]
```
