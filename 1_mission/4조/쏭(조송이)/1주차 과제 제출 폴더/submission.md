# 1주차 — 나만의 OS 만들기 🛠️

> 미션을 진행하며 **과정과 결과물**을 기록해주세요. (다 못 채워도 OK, 한 것 위주로!)

## 🎯 미션 1. 내 OS 재료 찾기
> 인터뷰 스킬(아이데이션)로 "내 삶에 필요한 게 뭔지" 찾기

- **과정 (어떻게 찾았나):**
  인터뷰 스킬로 시작했는데, 나는 이미 몇 달째 짓고 있는 앱(**Tally** — 저널·투두·캘린더가 하나로 얽힌 라이프-OS)이 있어서 인터뷰가 *거꾸로* 굴렀다. "뭘 만들지 찾기"가 아니라 **"내가 왜 이걸 만드는지 다시 맞추기."** 지금 나를 미는 건 *"머릿속 생각을 다 꺼내 절대 안 잃고, 정리되고, 보이게"* 였다. 이걸 Tally의 밑바닥 원칙(#1 어제의 나와 비교)에 겹쳐보니 — 어긋난 게 아니라 **한 스택**이었다: 캡처(안 잃어버림)는 *바닥*, 성장(어제보다 나아짐)은 *목적*. **"어제와 비교하려면 어제가 기록돼 있어야 한다"** — 그래서 지금은 바닥(캡처)을 짓는 시기라는 게 또렷해졌다.

- **결과 — 🟡 내 OS 재료 카드:**

  | 항목 | 내용 |
  |---|---|
  | **영역** | 일 + 삶 둘 다 — Tally는 안 쪼개고 하나의 저널로 받는 "삶 전체 OS" |
  | **걸리는 지점** | 머릿속 생각이 흩어져 사라짐. 폰에선 담을 데가 없고, 정리(parse)는 돈 내는 API에 의존 |
  | **지금은** | Tally는 있지만 — 폰 캡처 없음 · 매 parse마다 metered API(Sonnet+Haiku) · "내 두뇌"는 아직 없음 |
  | **OS가 된다면** | 어디서든 텔레그램으로 툭 던지면 즉시 안전히 저장(안 잃음) → 이미 가진 클로드 코드가 정리 → 결국 완전히 내 것인 로컬 두뇌로 |
  | **한 문장** | "던지는 순간 안 잃고, 정리는 알아서 — 그리고 결국 완전히 내 것" |
  | **첫 한 걸음** | 텔레그램 봇으로 raw 캡처(durable-first · allowlist) + parse 두뇌를 클로드 코드로 (metered API 삭제) |

- **느낀 점:**
  "딱히 안 불편한데"가 아니라, 이미 있는 앱의 *빠진 조각*(폰에서 못 담음 · 정리는 돈 내는 API)이 진짜 재료였다. 파고들다 크게 두 개가 남았다 — ① **캡처가 성장의 전제조건**이라는 것(잃어버린 데이터 위엔 어떤 성찰도 못 돈다), ② **채팅이 곧 저널을 폰으로 옮긴 것**이라는 것(정리 안 하고 쏟는, 마찰 0의 감각). 그리고 내 두 파서가 *둘 다* 돈 내는 API를 쓰고 있었다는 걸 코드에서 확인했을 땐 살짝 억울했지만 — 사실은 "이미 가진 걸 재활용"이라는 최적화를 발견한 거였다.

## 🧩 미션 2. 내 OS 기획
> 인터뷰 결과 + 세션 내용(흐민·배짱·키노) 활용해 기획

- **세션 소스에서 가져온 것 (흐민):**
  **흐민의 텔레그램↔클로드 코드 연결** — 폰에서 봇한테 말하면 컴퓨터의 클로드 코드가 받아 처리하는 방식. → 이게 내 캡처 문제의 **열쇠**였다. 앱을 새로 안 지어도, *이미 있는 텔레그램* + *이미 쓰는 클로드 코드*를 선으로 잇기만 하면 폰 캡처가 된다("허들 최소" 원칙 그대로).

- **기획 내용 — 핵심은 "두뇌의 여정":**
  Tally의 *정리하는 두뇌*를 **빌린 것(metered API) → 이미 가진 것(클로드 코드) → 완전히 내 것(로컬 모델)** 으로 옮겨간다. 그 밑엔 흔들리지 않는 약속: **던지는 순간 저장은 이미 끝난다.**
  - **캡처:** 폰 → 텔레그램 봇 → **raw 즉시 저장(durable-first)**. 정리는 뒤에서 따라옴 → 두뇌가 느리든/꺼졌든 아무것도 안 잃는다.
  - **정리(임시 다리):** 돈 내던 API를 *지우고*, 이미 구독으로 쓰는 클로드 코드를 정리 두뇌로. **클로드 코드가 정리한 모든 게 곧 로컬을 가르칠 "정답"으로 쌓인다** — 쓰는 값이 곧 자립으로의 저축.
  - **정리(엔드게임):** 그 정답 코퍼스로 **로컬 모델**을 길러, 결국 API도 구독도 없이 **비공개·공짜로 내 컴퓨터에서 도는 두뇌**로 넘긴다.
  - **넛지:** Tally가 **매달**(시간 기반) "한 달치 쌓였어요 — 같이 정리하고 로컬 가르칠까요?" 하고 **초대**. 추궁 아닌 초대(guilt machine 금지).
  - **3단계 사다리:** ①텔레그램 raw 캡처 → ②클로드 코드 정리 + 월간 넛지로 코퍼스 쌓기 → ③로컬로 넘김(완전 자립).

- **막혔던 점 / 어떻게 풀었나:**
  1. **24시간 vs 안 잃어버림.** 텔레그램 봇 큐는 ~24h만 보관 → 노트북 며칠 꺼지면? → **캡처를 파싱과 분리**(raw 먼저 저장) + **부팅 자동 폴러** + "그래도 놓치면 대화창에 영원히 남으니 forward" 백스톱. → *유실*이 아니라 *정리 지연*으로 바뀜.
  2. **"클로드 코드로 API 대체?" 비전 충돌처럼 보임.** 비전이 "API 임시 → 로컬 대체"인데 세 번째 두뇌를 끼우는 것 같았다. → **"로컬 될 때까지의 임시 다리"**로 프레이밍하니, *임시 두뇌만 바꾸는 작은 스왑*이었다(엔드게임 그대로).
  3. **API를 fallback으로 남길까 → 아예 삭제.** 클로드 코드가 죽으면 저품질 파서로 얼버무리지 않고 **"지금 정리 못 해요, 곧" 하고 정직하게 기다림**(raw는 안전하니까). — *"소터는 내 데이터에 대해 거짓말하지 않는다"* 원칙 구현.
  4. **착각 정정 — 클로드가 내 문체를 몰래 배우는 게 아님.** 모델은 고정. 월간 의식은 *내가 실제로 쓴 글*(세션 기록·확정 항목)을 **분석**하는 것.
  - **+ 검증:** 커밋 전에 **4명짜리 에이전트 council**(비전정렬·기술·레드팀·대안)로 이 방향을 압박 검증한 뒤 정리 기록으로 남김.
