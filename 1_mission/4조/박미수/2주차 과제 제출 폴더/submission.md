# 2주차 — 내 OS 구현하기 🚀

> 미션을 진행하며 **기획 → 구현 → 삽질 → 결과물 → 인사이트** 를 상세히 기록해주세요.

## 🎯 미션 1. 내 OS 만들기

**✅ 선택:** 콘텐츠 OS (자연치유 건강 리딩 웹서비스 — 누구나 쓰고 공유하는 형태)

### 📐 기획
> 무엇을, 왜, 어떻게

- **무엇:** 타고난 **얼굴형으로 형상체질(목·화·토·금·수)** 을 판정하고, **오행 생극(生克) 기반 건강 리딩**(약한 장부·좋은 음식·추천 활동·경락점·행운의 색)을 주는 **점신 스타일 공개 웹서비스.** 결과 화면에서 **자연치유사 상담 신청**까지 이어진다.
- **왜:** 나는 간호사 출신으로 카인드닥터 CPO이자 자연치유를 공부하는 사람이다. 자연치유는 체계·근거가 약해 보일 수 있는데, **재미있는 문진으로 접근성을 높이고 → 근거를 정직하게 표기하며 → 실제 상담으로 연결**하고 싶었다. 대학원 동료·자연치유협회 사람들이 함께 쓸 수 있는 도구를 목표로 했다.
- **어떻게:** 클로드 코드로 **HTML 웹페이지(정적)** + **Supabase(DB)** + **Vercel(배포)**. 로그인·서버 없이 가장 작게 시작.

### ⚙️ 구현
> 실제로 만든 것

- **🔗 라이브 URL:** https://natural-care-cyan.vercel.app/
- **구성:**
  - **프론트:** 얼굴형 결정트리 문진 → 체질 판정 / 오늘 끌리는 색 리딩 / 생년월일 기반 행운의 색
  - **처음이에요 / 다시 왔어요** 분기 (재방문은 `localStorage`로 체질 기억)
  - **생극 로직:** 강한 장부(체질 그 자체) ↔ 극(克) 관계의 **약한 장부**를 계산해, 약한 장부를 돕는 색·음식·경락점을 추천
  - **상담 신청 폼:** 이름·연락처 + **[필수] 개인정보 수집·이용 동의** → **Supabase DB 저장** (RLS로 "신청은 되지만 남의 정보는 못 읽게" 보안)
  - **디자인:** Pretendard 폰트 + 브랜드 3색(그린·브라운·살구빛 팝), 라이트/다크 대응
- **스택:** 정적 HTML/CSS/JS · Supabase(Postgres + REST API) · Vercel 배포

### 🧗 과정에서의 삽질
> 막혔던 지점, 어떻게 풀었는지 솔직하게

1. **얼굴형을 눌러도 안 넘어감** — 자바스크립트 변수 이름을 `history`로 썼는데, 브라우저 내장 `window.history`와 충돌해서 클릭 시 에러가 났다. → **변수명을 `navStack`으로 변경**해 해결. (제일 오래 헤맨 부분)
2. **오행 생극 로직을 반대로 넣음** — 처음엔 체질의 '자기 장부'를 약한 곳으로 표시했는데, 실제로는 **강하게 타고난 장부**다. 극(克) 관계인 장부가 약한 장부. → 화형=심·소장 강, 폐·대장/신·방광 약으로 **도메인 검증 후 교정**. 잘 맞는 색·음식도 '약한 장부를 돕는' 방향으로 바로잡음.
3. **Supabase 지역이 도쿄로 생성됨** — 서울로 바꾸려 했으나 **생성 후 지역 변경 불가.** 도쿄도 한국에서 충분히 빨라 그대로 진행.
4. **새 API 키 체계 혼동** — `publishable`(공개용) vs `secret`(비밀) 중 실수로 secret을 쓸 뻔. → **publishable 키만** 프론트에 사용, 연결 전 테스트 삽입으로 작동 확인(HTTP 201).
5. **GitHub repo가 Vercel 목록에 안 보임** — 권한 이슈. → **Vercel의 폴더 드래그 배포**로 우회해서 바로 URL 생성.

### ✅ 결과물

- **작동하는 공개 웹서비스:** https://natural-care-cyan.vercel.app/
- 문진 → 건강 리딩 → 상담 신청 → **DB 저장까지 실제 작동** 확인
- (스크린샷은 `이미지첨부/` 폴더에 추가 예정)

### 💡 알게 된 인사이트 & 공유하고 싶은 내용
- **개발자가 아니어도** 클로드 코드와 대화만으로 **프론트 + DB + 배포**가 있는 진짜 웹서비스를 만들 수 있다. (제품을 지휘하던 CPO가 처음 직접 만들어봄)
- **근거의 정직함이 신뢰다.** 전통(오행·황제내경)과 현대 과학 근거를 구분해 표기하고, 불확실하면 "확인 필요"로 드러냈다. 지어낸 근거는 오히려 브랜드를 해친다.
- **개인정보(건강 관련)를 다루면 동의·보안이 필수.** 재미 요소 뒤에 안전장치(RLS·동의)를 꼭 넣어야 한다.
- 흐민님 공유회의 **"개인 OS → 팀 OS"** 확장을 실제로 경험했다. (내 플래너 → 동료·협회를 돕는 도구로)
