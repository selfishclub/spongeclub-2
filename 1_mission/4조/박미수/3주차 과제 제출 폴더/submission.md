## 결과물

이번 주에 **골프랭크(Golf Rank)** 라는 골프 모임 관리 웹앱을 만들었다. 라운드 스코어를 기록하면 자동으로 랭킹이 매겨지고, 사진·일정·공지까지 모임 하나 안에서 관리된다.

- 🔗 **배포 주소(폰에서 바로 사용):** https://golf-rank-beta.vercel.app
- 📱 **SNS 후기:** https://lnkd.in/p/gUK8m8zm
- 💻 **소스:** 이 폴더 전체 (바닐라 HTML/CSS/JS · 페이지별 단일 파일 구조)

**주요 기능**
- **캘린더 대시보드** — 월별 달력에 라운드(초록 점)·예정 일정 표시, 올해 총 라운드·이번 달 라운드·이번 달 MVP 요약, 골프장(CC)별 필터
- **라운드 기록 3가지 방식** — ① 총타수 빠르게 ② 홀별 직접 입력 ③ **스코어 사진 자동 인식(OCR)**. SmartScore 사진을 찍으면 이름·전반·후반·합계를 읽어옴
- **랭킹 시스템** — 매 라운드 순위를 정규화해 시즌 누적, *출석을 많이 한다고 유리하지 않도록* 평균 기반 + 베이지안 보정 적용. 월간 MVP는 평균 스코어로 별도 선정. 핸디캡 자동 계산(최근 3라운드)
- **멤버·게스트 관리** — 닉네임/핸디/구력/라베/티박스/목표스코어/프로필 사진, 권한 이모지(👑모임장·⭐운영진), 게스트는 순위 제외
- **갤러리 / 일정 / 개인 리포트 / 소셜(공지·댓글)**

**연동한 것들 (이번 과제의 핵심)**
- **Supabase(클라우드 DB)** — 데이터를 브라우저가 아닌 서버에 저장 → 모든 멤버가 같은 데이터를 실시간 공유·자동 동기화
- **이메일 로그인/계정** — 로그인하면 내 모임이 계정을 따라와 폰·PC 어디서든 동일
- **Vercel 배포** — 실제 주소로 접속해 친구들이 폰으로 바로 사용
- **개인 초대 링크 + 이름 자동 매칭** — 미리 추가한 멤버와 나중에 가입한 사람이 '한 사람 = 한 명'으로 연결되어 중복이 생기지 않게 설계

## 삽질 과정

- **스코어 사진 인식이 처음엔 엉망이었다.** SmartScore 캡처의 이름·전반·후반·합계를 제대로 못 읽어서, Tesseract OCR 결과를 우리 포맷(한글 이름 + 합계 우선)에 맞게 파싱하는 로직을 여러 번 다듬었다.
- **랭킹이 "많이 나온 사람이 이기는" 구조였다.** 그냥 누적 점수로 하면 출석왕이 1등이 된다. 순위를 정규화해 점수로 바꾸고, 평균 기반에 베이지안 보정(shrinkage)을 얹어서 *실력*이 드러나게 바꿨다.
- **카카오 로그인에서 막혔다.** account_email 동의 항목이 비즈앱 심사를 요구해서(KOE205) 계속 실패 → 깔끔하게 **이메일 로그인**으로 방향을 틀었다.
- **친구 화면에선 내 모임이 안 보였다.** localStorage는 브라우저마다 따로 논다는 걸 체감하고, **Supabase 공유 DB**로 옮겨 저장 방식을 통째로 바꿨다.
- **멤버가 중복으로 뜨는 문제.** 내가 이름으로 미리 추가한 멤버와, 그 사람이 직접 가입한 계정이 따로 생겼다. 개인 초대 링크로 특정 슬롯을 '이어받고(claim)', 이름이 같으면 자동 매칭을 제안하는 방식으로 "한 사람 = 한 명"을 보장했다. 저장도 전체 삭제→삽입에서 **병합(upsert)** 으로 바꿔 남의 데이터가 지워지지 않게 했다.
- **모바일에서 하단 바가 스크롤할 때 같이 흔들리고 달력 줄 간격이 틀어졌다.** 원인은 `position: fixed` + 모바일 주소창, 그리고 요일 헤더와 날짜가 한 그리드에 섞인 것. 레이아웃을 **"화면 높이 고정 + 본문만 스크롤"** 구조로 재설계하고, 달력 요일 헤더를 분리해 행 높이를 고정했다.

## 인사이트

혼자 쓰는 앱과 **"같이 쓰는 앱"** 은 완전히 다른 문제였다. 예쁜 화면보다 로그인·공유 DB·중복 방지 같은 **'연동'** 이 진짜 핵심이었고, 사람이 실제로 쓰기 시작하면 데이터가 어긋나지 않게 지키는 설계가 가장 어렵고 중요했다.
