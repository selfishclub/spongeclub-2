## 🎯 미션 1. 내 OS 만들기
> **[ 내 삶을 돕는 OS ]** 또는 **[ 콘텐츠 OS ]** 중 하나를 선택해 완성해주세요.

**✅ 선택:** 내 삶을 돕는 OS — **「아벨OS」** (Claude Code 기반 개인 비서실)

> 늘 "지금 뭐부터 해야 하지"가 안 잡혔다. 그래서 콘텐츠 OS가 아니라, **내 일·삶 전체를 아는 비서실**을 만들기로 했다.

### 📐 기획
> 무엇을, 왜, 어떻게 만들지

**한 줄 정의**
> 아벨OS = 회사와 개인 삶·학습 전체를 아는 **비서실**. 총괄비서가 내 말을 받아 분류·라우팅하고, 정리된 결과만 돌려준다.

**왜 만들었나 (pain point)**
- 도메인이 셋(회사 A / 회사 B / 개인)인데 머릿속에서만 굴리니 우선순위가 늘 흐렸다.
- 아이디어·자료·링크가 노션·메모·카톡에 흩어져 "어디 뒀더라"가 반복됐다.
- 지금은 **학습이 중요한 단계**인데, 학습을 설계하고 리마인드해주는 장치가 없었다.
- 혼자 판단하니 방향이 맞는지 검증받을 곳이 없었다.

**어떻게 (설계 원칙)**
1. **에밀리님 OS를 참고**하되 내 차별점 두 개를 추가 — 🎓 개인코치(학습 설계·리마인드), 🦉 멘토/VC(진척 평가·조언).

### ⚙️ 구현
> 실제로 만든 것 (링크·스크린샷 — 이미지는 `이미지첨부/` 폴더에)

**① 헌법 (CLAUDE.md)** — OS의 뿌리
정체성·목소리·라우팅 규칙·루틴을 담은 헌법. "매 턴 작동 방식"까지 규정해서, 세션을 열면 총괄비서가 상태를 브리핑하고 → 담당 직원 이모지로 답하고 → 남길 가치가 있으면 파일로 기록하고 → `▶ 다음:` 한 줄로 닫는다.

**② 비서팀 10인** (`.claude/agents/`)
| 직원 | 하는 일 |
|---|---|
| 🎩 총괄비서 | 분류·라우팅·보고 (헌법 자체) |
| 📅 스케줄러 | 구글캘린더 연동 + 아침/저녁/위클리 브리핑 |
| 🧠 기록담당 | 던지는 걸 제자리에 저장 (판단 안 함) |
| 📊 PMO | 전 프로젝트 진척·우선순위 |
| 🎓 개인코치 | 학습 설계·자료 제작·리마인드 |
| 💚 상담사 | 컨디션·감정 체크인, 건강 |
| 🧭 시니어 디렉터 | 20년차 PM/BD 눈으로 실무 조언 |
| 🦉 멘토/VC | 위클리에서 목표 대비 평가·조언 |
| 🔍 리서처 | 자료·시장 조사 (출처 필수) |
| 🧩 인사 | OS에 뭐가 더 필요한지 스스로 제안 |

**③ 도메인 폴더 구조**
`00_비서실`(브리핑·체크인·위클리 아카이브) / `10_타인AI` / `20_라이프클래스` / `30_학습` / `40_개인` / `90_인박스` / `99_레퍼런스`.

**④ 루틴 3종 + 슬래시 명령어**
`/브리핑`(아침) · `/체크인`(저녁) · `/위클리`(월요일) · `/동기화`(git). 아침 브리핑은 일정·리마인드·프로젝트 진척·오늘의 학습을 한 장으로 뽑는다.

**⑤ 자동화 — 무인 브리핑 & 텔레그램**
launchd 예약작업으로 아침 07:30 브리핑을 **헤드리스로 생성**해 폰으로 push, 저녁 21:00 체크인 리마인드. 텔레그램 Channels 플러그인으로 폰에서 던진 말도 헌법 라우팅을 그대로 탄다.

**⑥ 학습 퍼널** — RSS 24개 소스 수집 → `/다이제스트`로 나에게 맞는 데일리 다이제스트 생성(52건→10건 큐레이션).

**⑦ 멀티 디바이스 동기화** — GitHub 비공개 저장소로 회사 맥북·집 데스크톱·개인 노트북 3대를 오간다. *(이 과제도 개인 노트북에서 작성 중.)*

### 🧗 과정에서의 삽질
> 막혔던 지점, 시도한 방법, 어떻게 풀었는지 솔직하게

- **비서가 "범용 도우미 말투"로 돌아갔다.** 헌법에 팀을 정의해놨는데도, 실제로는 그냥 친절한 Claude처럼 설명만 늘어놨다. → 헌법에 `§0 매 턴 작동 방식`(목소리·라벨·기록 반사·닫기 리추얼)을 못박고 **9인 목소리 블록**을 심어서 "수술"했다. *기대만으론 안 되고, 규칙을 명시적으로 강제해야 한다는 걸 배움.*
- **맥·윈도우 혼용에서 줄바꿈이 깨졌다.** 셸 스크립트가 CRLF/LF로 꼬여 자동화가 실패. → `.gitattributes`로 셸 스크립트 LF 고정.
- **무인 브리핑이 안 도는 날이 있었다.** 헤드리스 실행/예약작업(launchd) 실패. → 실패 시 셸이 직접 실패 핑을 보내도록, 그리고 기기별로 자동화 주체를 분리(자동화는 맥북 담당). *3대를 쓰다 보니 "어느 기기에서 뭐가 도는가"를 명확히 해야 했다.*
- **"채팅에만 남고 파일로 안 남는" 함정.** 좋은 결정을 대화로 내리고 세션이 끝나면 사라졌다. → 헌법에 `기록 반사`·`세션 마무리 규칙`을 넣어 "대화는 그 컴퓨터에만, 파일은 모든 컴퓨터로 따라온다"를 원칙화.
- **자료 이관(AX화)이 제일 지루하고 오래 걸렸다.** 흩어진 자료를 도메인 폴더로 모으는 건 지금도 상시 진행 중. 한 번에 안 끝난다는 걸 인정하고 `90_인박스` → 주기적 분류 흐름으로 풀었다.

### ✅ 결과물
> 완성한 것 / 작동 화면

- **작동 중인 아벨OS** — 헌법 + 비서팀 10인 + 도메인 폴더 + 루틴 3종 + 무인 브리핑/텔레그램 + 학습 퍼널까지 로드맵 4단계 전부 가동.
- 매일 아침 브리핑·저녁 체크인이 파일로 쌓이고(`00_비서실/`), 3대 기기가 git으로 동기화된다.
- 실제로 이 OS 위에서 회사 프로젝트와 개인 학습이 함께 굴러가고 있다.
- **로드맵 완주:** 1단계 뼈대 → 2단계 루틴 → 3단계 확장(멘토·리서처·인사) → 4단계 모바일(텔레그램). ✅
- *(스크린샷: 세션 시작 브리핑 화면 / 폴더 트리 / 텔레그램 브리핑 push — `이미지첨부/`에 추가 예정)*

### 💡 알게 된 인사이트 & 공유하고 싶은 내용
> 하면서 깨달은 것, 크루들과 나누고 싶은 것

1. **OS의 핵심은 도구가 아니라 "헌법"이다.** 에이전트 여러 명보다, "이 저장소를 여는 AI는 이렇게 행동한다"를 한 파일에 못박는 게 훨씬 큰 차이를 만들었다.
2. **파일 = 기억.** 채팅은 그 컴퓨터에만 남고 사라진다. 결정을 파일로 남기는 순간, 3대 기기 어디서 열어도 이어진다. "기록 반사"를 습관이 아니라 규칙으로 강제한 게 신의 한 수.
3. **최소 3인부터.** 에밀리님 조언대로 스케줄러·기록담당·PMO 3명으로 시작해 단계적으로 늘렸다. 처음부터 10명 다 만들었으면 무너졌을 것.
4. **기대하지 말고 강제하라.** AI가 알아서 잘하겠지 → 안 된다. 말투도, 기록도, 자동화 실패 대응도 전부 명시적 규칙으로 박아야 원하는 대로 굴러간다.
5. (공유) **"내 삶을 아는 비서실"은 생각보다 빨리 만들 수 있다.** 어려운 건 세팅이 아니라, 매일 실제로 던지고 쌓는 습관이었다.
