## 🎯 미션 1. 내 OS 만들기

**✅ 선택: 내 삶을 돕는 OS**

<br>

### 📐 기획 — 무엇을 · 왜 · 어떻게

- **무엇을** — 나루(iny-OS), 삶의 여러 역할을 돕는 개인 비서
  - 입구는 텔레그램 하나 → 안에서 나루(PO)가 분류·라우팅
- **왜** — 인입 정보를 편하게 보고 → 자산으로 쌓고 → 커리어로 잇기
  - 1주차 "배움 휘발" → 2주차 "삶 전체 역할"로 확장
- **어떻게** — 0→100 금지. 8영역 지도부터 그리고, 한 칸씩 진행

<br>

### ⚙️ 구현 — 이번 주 한 것

| 영역 | 한 것 |
|------|------|
| 🗺️ 사용 지도 | 8영역 라우팅 구조 확정·문서화 + 대시보드 구조도 뷰 |
| 📅 구글 통합 | 일정·메일·드라이브 통합, 캘린더 읽기·쓰기 실검증 |
| 🌙 모드 빌드 | 상담·건강 모드 + 수면 자동기록, 폰 텔레그램 실검증 |
| 🗄️ 아카이빙 | 1기·로컬 옛 자료 소화 → 자산화 |

> 📊 **완결도 41/52 (79%)**

<br>

### 🧗 과정에서의 삽질

**🌅 아침 인사, 3전 4기 끝에 성공**
- 외부에서 나루 깨우기 → 통로 없어 폐기
- 나루 스스로 알람 → 됐지만 재시작하면 소멸
- → **launchd 독립잡**으로 근본 해결 (+ 낡은 캐시는 강제 새로고침)
- 💡 *재시작에도 살아남아야 진짜 자동화*

**📱 수면 자동기록, 아이폰 함정 3개**
- '자동화' 탭에서 만들어야 저절로 돎
- '즉시 실행'이어야 방해금지가 안 막음
- 방해금지 혼자 켜지는 루프 = 단축어 속 '켜기' 동작이 범인 → 삭제
- 💡 *기록만 할 자동화에 '상태 바꾸는 동작'을 섞으면 루프가 생긴다*

**🔐 구글 캘린더, 내 계정인데 접근 차단(403)**
- OAuth '테스트' 상태 → 테스트 사용자에 내 gmail 등록
- 기본 권한이 읽기전용 → 쓰기 403 → 전체 권한으로 재인증
- 💡 *새 구글 연동은 테스트 사용자 + 권한 범위부터*

**💬 텔레그램 봇이 답을 안 함 (두 번)**
- `--channels` 플래그 누락 → 켜니 해결
- 맥 절전에 연결 끊김 → 재시작 + 절전 방지
- 💡 *상주 봇은 채널 모드로, 맥 절전은 연결을 끊는다*

**🪞 대시보드가 낡은 채 '거짓말'**
- 두 파일 수기 이중관리로 어긋남
- → `board.json` 하나만 원본, 커밋 훅이 나머지 자동 생성
- 💡 *진실의 원본은 딱 하나만*

**🧩 자잘한 함정들**
- `git add -A`로 엉뚱한 파일 커밋할 뻔 → 필요한 것만 지정
- Next 16이 middleware→proxy 개명 → 문서부터 확인
- zsh가 빈 검색에 명령을 죽임 → `find`로 교체
- 발표 PDF는 `pdftotext`로 글자만 추출
- 계획은 역량 5분류였지만 실제 데이터는 8개 → 데이터부터 세기
