# 리뷰 자동 분류 채널 (리뷰분류)

스마트스토어에서 다운로드한 리뷰 엑셀(.xlsx)을 넣으면 두 갈래로 분류해서 담당자에게 넘길 리스트를 만든다.

## ⚠️ 고객정보 보호 (제일 중요)
- 입력 엑셀에는 등록자·주문번호 등 **고객정보가 들어있다. 절대 이 저장소(git) 안에 두지 않는다.**
- 파일은 저장소 밖(예: `~/Desktop/`)에 두고 **경로만** 전달한다.
- 결과물(일간 리스트)도 고객정보를 포함하므로 저장소 밖(`--out ~/Desktop/...`)에 저장한다. 제출·공개할 땐 값을 가린 스크린샷만 사용한다.

## 실행 방법

```bash
# 일간 모드 — 환불·CS 대상 / 칭찬 대상 두 표 (매일 아침용)
python3 triage.py <리뷰.xlsx> --mode daily --out ~/Desktop/리뷰분류_일간.md --top 20

# 월간 모드 — 경영진 CRM 보고용 집계 리포트
python3 triage.py <리뷰.xlsx> --mode monthly
```
- `--top`: 칭찬 대상 상위 몇 명까지 뽑을지 (기본 20)
- pandas/openpyxl 등 설치 불필요 (파이썬 표준 라이브러리만 사용)

## 분류 규칙

**⚪ 체험단·협찬 리뷰 제외 (분석 전 단계)**
- 리뷰 내용에 "업체로부터 제공받아 작성", "체험단", "협찬", "소정의 원고료" 등이 있으면 진짜 고객 반응이 아니므로 분석에서 제외한다. 제외 건수는 리포트 맨 위에 표시(몰래 빼지 않음).


**🔴 환불·CS 대상 (고객만족보장제도)** — 놓치면 안 되는 쪽
1. 평점 1~2점 → 무조건 대상
2. 평점 3점 이상이라도 **명백한 부정 신호**(환불·반품·파손·오배송·이물질·곰팡이·불량·유통기한 등)가 있으면 대상 → *별점 5점인데 내용은 불만인 리뷰를 여기서 잡는다*
3. 애매한 단어(별로·냄새·늦게 등)는 3점 이하일 때만 인정 (긍정 문맥 오탐 방지)
- 각 행에 **배송 출발지**(풀필먼트사 값 있으면 대한통운 풀필먼트 / 없으면 자체 음성공장 도착보장) 표시 → 오배송·파손 책임소재 파악
- 사유별(배송지연·파손·오배송/누락·상품불만/변질·기타불만) 자동 분류

**🟢 칭찬 대상 (팬 만들기)** — 5점 3천여 건을 소수로 좁힘
- 5점 + 부정 신호 없음 + 정성 신호(사진·영상 / 장문 / 리뷰도움수)
- 월간 '우수 리뷰'는 더 엄격(사진+100자 이상 / 300자 이상 / 도움수 3+) → 쿠폰·꽃바구니·손편지 발송 검토

## 클로드 최종 확인 (선택 · 정밀도 ↑)
스크립트는 키워드 기반 1차 분류다. 더 정확히 하려면, 분류된 '환불·CS 대상' 표의 `리뷰 요약`을 클로드가 직접 읽고 오탐(긍정 문맥에 부정어가 섞인 경우)을 걸러낸다. 반대로 놓친 은근한 불만도 추가한다.

## 한계 / 다음 단계
- **환불 금액**: 리뷰 파일엔 금액이 없다. 월간 리포트는 인원·건수까지 채우고, 금액은 주문번호로 정산 데이터를 조회해 붙인다.
- **VIP·재구매 판별**: 이 파일엔 구매 이력이 없어 '자주 사는 분'을 못 거른다. 구매이력 데이터를 붙이면 칭찬 대상에 반영된다.
- **자동 스케줄(월~금 8:30)·담당자 전달**: 2단계. `schedule`(크론) 채널이나 알림 연동으로 확장한다.
