# 1주차 — 나만의 OS 만들기 🛠️

> 미션을 진행하며 **과정과 결과물**을 기록해주세요. (다 못 채워도 OK, 한 것 위주로!)

## 🎯 미션 1. 내 OS 재료 찾기
> 인터뷰 스킬(아이데이션)로 "내 삶에 필요한 게 뭔지" 찾기
- **과정 (어떻게 찾았나):** 인터뷰 스킬로 하루를 돌아봤다. 일 자체엔 큰 이슈가 없었는데, 10년 동안 네이버 블로그·브런치·구글 Keep에 써온 글들이 흩어져 있고 콘텐츠로 못 뽑아내고 있다는 걸 발견했다. 미팅 녹음 관리도 안 된다는 것도 나왔는데, 더 당기는 건 글 쪽이었다.
- **결과:** 걸리는 지점 = 글마다 Claude에 복붙 → 스타일 설명 → 채널 특성 알려주고 → 그제야 하나 완성. 매번 반복하는 구조.
- **느낀 점:** "딱히 불편한 게 없다"고 생각했는데, 각도를 바꿔보니 10년치 재료가 잠들어 있었다.

## 🧩 미션 2. 내 OS 기획
> 인터뷰 결과 + 세션 내용(흐민·배짱·키노) 활용해 기획
- **기획 내용:**

  **OS 이름:** 마크의 콘텐츠 OS

  **OS 정의:** 10년치 글·경험·목소리라는 자원을 LinkedIn·스레드·블로그 채널에 맞게 연결하고 조율하는 개인 운영 체계. OS의 핵심은 자동화가 아니라 연결과 조율이며, AI(Claude)가 그 과정을 매번 0부터 하지 않아도 되게 해준다.

  **한 문장:** "글 쓸 때마다 0부터 시작하지 않아도 된다 — 그리고 앞으로도 계속"

  **채널 구조:**

  | 채널 | 역할 | 비고 |
  |------|------|------|
  | 네이버 블로그 | 긴 글, 검색 유입, 수익화(애드포스트) | 메인 채널 |
  | LinkedIn | 커리어/인사이트, 전문가 네트워크 | 전문적 톤 |
  | 스레드 | 짧은 생각, 일상 인사이트 | 가볍고 솔직 |
  | 유튜브 | 영상 대본 기반 콘텐츠 | 말하듯이 |
  | 브런치 | 과거 아카이브 (신규 없음) | 유지만 |

  **3단계 구조:**

  | 단계 | 내용 | 시점 |
  |------|------|------|
  | 1단계 (이번 주) | 네이버 블로그·브런치·Keep 글을 옵시디언으로 통합 | 지금 |
  | 2단계 | 글 스타일 SKILL 제작 + 기존 글 카테고리화(5~10개) | 다음 |
  | 3단계 | 카테고리 기반 주제 제안 → 선택 → 채널별 자동 발행 | 그다음 |

  **1단계 구체 실행:**
  - 네이버 블로그: 관리자 → 백업 → zip 받기 → Claude Code로 HTML→md 변환 스크립트 → 옵시디언 저장
  - 브런치: Claude Code로 크롤링 스크립트 짜서 md로 저장
  - 결과: Keep(완료) + 블로그 + 브런치 = 옵시디언 한 통

- **막혔던 점 / 어떻게 풀었나:** 브런치 공식 내보내기가 없어서 크롤링이 필요하다는 걸 알았다. Claude Code로 스크립트 짜기로 방향 잡음.

## ⚙️ 미션 3. 내 OS 구현
> 실제로 만들어본 것 (클로드코드 '채널' 기능 활용 OK)

- **무엇을 만들었나:**
  "흐민님의 0 to 100 하지 말고 가장 작은 것부터"를 그대로 적용해, **콘텐츠 창고 구축 + 초안 생성 + 검토 대시보드**까지 한 주에 다 달렸다.

- **결과물 1 — 콘텐츠 창고 (옵시디언 Vault):**
  10년치 글을 하나의 폴더로 통합했다.
  - 네이버 블로그: Playwright 크롤링 스크립트(Python)로 **657개** md 파일 수집
  - 브런치: 크롤링 스크립트로 **132개** md 파일 수집
  - Google Keep: Takeout(HTML+JSON) → Python 변환 스크립트로 **2,137개** md 파일 수집
  - 총 **2,926개** 글이 옵시디언 Vault 한 곳에 모임

- **결과물 2 — 마크 글스타일 SKILL 파일:**
  2,926개 글에서 샘플을 뽑아 문체·구조·패턴을 분석, `마크_글스타일.md` SKILL 파일 제작.
  ㅡ 짧은 문장, 줄바꿈 자주, 개인 경험 먼저
  ㅡ 숫자로 말하기, ㅡ 형식 리스트, 단정적 결론
  ㅡ 채널별(블로그/LinkedIn/스레드) 경어·평어 구분 기준 포함
  Claude가 이 파일을 참조하면 마크 목소리로 글을 쓴다.

- **결과물 3 — 콘텐츠 OS 대시보드 (Next.js + Supabase):**
  기존 JJH Agent 관리 앱에 `/content` 페이지를 추가해 콘텐츠 검토 대시보드를 구현했다.
  - Claude API가 마크 스타일로 카테고리별 초안 자동 생성 → Supabase DB 저장
  - 대시보드에서 날짜별로 초안 목록 확인 → 클릭하면 본문 펼쳐짐
  - **확정 / 수정 요청 / 건너뜀 / 초안 복구** 버튼으로 검토
  - **제목+본문 복사** 버튼으로 각 채널에 바로 붙여넣기
  - 확정 → 발행완료 상태로 이력 관리

- **콘텐츠 카테고리 (확정):**
  | 카테고리 | 독자 | 채널 |
  |---------|------|------|
  | 커리어·이직 | 이직 고민 직장인 | 블로그, LinkedIn |
  | HR·AX | 기업 임원, HR 담당자 | LinkedIn |
  | AI·바이브코딩 | AI 써보고 싶은 직장인 | 블로그, LinkedIn |
  | 에세이 | 3040 여성 | 스레드 |

- **동작 방식:**
  매주 토요일 오전 9시, Claude API가 카테고리별 초안 생성 → Supabase 저장 → 대시보드에 표시 → 마크가 검토 후 각 채널에 직접 발행

- **막혔던 점 / 어떻게 풀었나:**
  ㅡ 네이버 블로그: JS 렌더링 구조라 requests로 안 됨 → Playwright로 해결
  ㅡ 크롤러가 Claude Code 백그라운드에서 CPU 고정 현상 → 터미널에서 직접 실행으로 해결
  ㅡ Gemini 대화 기록 Takeout 불가 → 수동 방식으로 전환
  ㅡ Supabase 프로젝트 제한(무료 2개) → 기존 프로젝트에 테이블 추가로 해결
