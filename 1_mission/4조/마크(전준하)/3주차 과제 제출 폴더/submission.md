## 결과물

디스코드 채널에 글이나 음성 메시지를 올리면, 자동으로 5~10분 분량의 유튜브 스크립트를 생성해서 저장해주는 파이프라인을 완성했다.

- 채널에 "유튜브"(또는 "YouTube") 키워드가 포함된 메시지를 올리면 → Claude API가 스크립트를 자동 생성 → 파일로 저장 → 디스코드에 완료 메시지와 리액션까지 자동으로 붙는다.
- 실제로 "최용수 감독 사주 분석" 주제로 던진 메시지 하나로, 오프닝-본론 3~4개-마무리 구조를 갖춘 완성된 스크립트가 나오는 것까지 확인했다.
- 원래 만들려던 n8n 기반 "콘텐츠 OS" 워크플로우는 결국 폐기했다. 실사용 빈도가 낮았고, 이미 있던 파이썬 디스코드 봇에 조건문 몇 줄 추가하는 것으로 똑같은 결과를 훨씬 빠르게 얻을 수 있었기 때문이다.

## 삽질 과정

- 처음엔 n8n으로 "Discord 트리거 → 조건 분기 → Claude 호출 → 파일 저장" 워크플로우를 GUI로 만들려고 했다. 브라우저 자동화로 노드를 하나씩 클릭해서 만드는 게 생각보다 훨씬 오래 걸렸고, import 기능도 눈에 잘 안 띄어서 계속 헤맸다.
- 나중에야 알았는데, 애초에 이 n8n 인스턴스엔 "Discord Trigger"라는 네이티브 트리거 노드 자체가 없었다 (Schedule/Webhook만 지원). 처음부터 불가능한 방향으로 시간을 쓰고 있었던 셈이다.
- "n8n이 자동화를 만드는 유일한 방법은 아니다"라는 걸 깨닫고 나서, 이미 잘 돌아가고 있던 기존 파이썬 디스코드 봇(메시지 감지 → 텍스트 저장 로직은 이미 있었음)에 "유튜브 키워드 감지 → 스크립트 생성" 조건 하나만 추가하는 쪽으로 방향을 완전히 틀었다.
- 막상 방향을 틀고 나서도 삽질은 계속됐다: 봇이 켜져 있는 줄 알았는데 실제로는 꺼져 있어서 처음 올린 메시지가 처리되지 않았고, 뒤늦게 채널 히스토리를 직접 조회해서 수동으로 복구 처리했다.
- 키워드 매칭이 "유튜브"(한글)만 잡도록 짜여 있어서, "YouTube"(영문)로 말한 메시지는 안 걸리는 버그도 발견해서 대소문자 무관하게 고쳤다.
- 콘솔에서 한글/이모지 출력할 때 cp949 인코딩 에러로 봇이 죽는 문제, 스크립트가 중간에 뚝 끊기는 문제(알고보니 글자 수가 아니라 토큰 제한 2000에 걸린 것)도 하나씩 원인을 찾아 고쳤다.

## 인사이트

자동화 도구(n8n 같은)는 자동화를 만드는 여러 방법 중 하나일 뿐이지 목적이 아니다 — 이미 동작하는 코드가 있다면 조건문 몇 줄 추가하는 게 새 워크플로우를 처음부터 만드는 것보다 훨씬 빠르고 확실할 때가 많다.
