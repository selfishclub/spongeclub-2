# 1주차 — 나만의 OS 만들기 🛠️

> 미션을 진행하며 **과정과 결과물**을 기록해주세요. (다 못 채워도 OK, 한 것 위주로!)

## 🎯 미션 1. 내 OS 재료 찾기
> 인터뷰 스킬(아이데이션)로 "내 삶에 필요한 게 뭔지" 찾기

- **과정 (어떻게 찾았나):**
  인터뷰 스킬 5스텝을 따라가며 하루를 되짚었다.
  - 시간을 많이 쓴 일: ① 이직 준비(이력서·경력기술서용 경험 정리) ② 구글시트 데일리 리포트 작성
  - '아 짜증' 포인트: 데일리 리포트가 **매체별 숫자만 수기로** 들어가고 세부(로)데이터가 없어서, 월간 리뷰 쓸 때 **로데이터를 처음부터 다시 취합**해야 하는 이중 작업.
  - 일 밖 영역: 이직 준비(친구 내부 추천으로 급하게 시작).
  - 무게 재기: 지금 시간은 이직 준비에 제일 많이 쓰지만, **수요일 월간리뷰 미팅**이 임박해 데일리 리포트를 먼저 다루기로 함. (이직 준비 자동화는 다음 단계로 남김)

- **결과: 🟡 내 OS 재료 카드**
  - **영역**: 일(업무) — 데일리/월간 리포트
  - **걸리는 지점**: 데일리 리포트가 매체별 숫자만 수기로 들어가서, 월간 리뷰 쓸 때 로데이터(매체별 시트 + GA)를 다시 취합해야 함
  - **지금은**: 매체별 시트에 로데이터만 쭉 누적(지난 달 행은 숨김) → 월말에 매체별로 다시 긁어모아 월간 리뷰 작성 (이중 작업)
  - **OS가 된다면**: 데일리 리포트 형식 자체를 로데이터가 잘 쌓이게 재설계 → 월간 리뷰 땐 확정 데이터만 한 번 넣으면 리뷰가 바로 써짐
  - **한 문장**: *"매일 더 자세하게 쌓아두면, 월말에 다시 리뷰용 데이터를 쌓느라 시간 낭비할 필요가 없다"*
  - **첫 한 걸음**: 데일리 리포트 시트 형식을 '월간 리뷰까지 재사용 가능한 구조'로 다시 설계하기 (매체 데이터 + GA를 한 뼈대에)

- **느낀 점:**
  "딱히 불편한 건 없다"고 생각했는데, 하루를 스텝별로 되짚으니 매일 조금씩 쌓여 월말에 터지는 반복 작업이 명확히 보였다. 기능부터 떠올리지 않고 "어디가 걸리는가"에서 출발하니 만들 것이 저절로 정해졌다.

## 🧩 미션 2. 내 OS 기획
> 인터뷰 결과 + 세션 내용(흐민·배짱·키노) 활용해 기획

- **기획 내용:**

  전체 그림을 3단계로 나누고, 이번 주는 **1단계**에 집중한다.
  - **1단계 (이번 주)**: 데일리 리포트 시트 뼈대를 '월간 리뷰까지 재사용되는 구조'로 재설계
  - **2단계**: 확정 데이터 한 번 넣으면 월간 요약(합계·증감·매체 비중)이 자동 계산
  - **3단계**: 이직 준비 경험 정리까지 묶어 '내 업무 OS'로 확장

  **1단계 설계 — 통합 로데이터 테이블**
  지금은 "매체별 시트에 숫자만" 쌓이는데, 이걸 **하나의 로데이터 테이블(한 뼈대)**로 바꾼다. 핵심은 행이 아니라 **열(칼럼)을 제대로 잡는 것**:

  | 날짜 | 월 | 매체 | 캠페인 | 노출 | 클릭 | 비용 | 전환(매체) | 세션(GA) | 유입(GA) | 전환(GA) | 확정비용 | 메모 |
  |------|-----|------|--------|------|------|------|-----------|----------|----------|----------|----------|------|
  | 2026-07-01 | 2026-07 | 메타 | 여름_리타겟 | 12,400 | 320 | 85,000 | 14 | 298 | 메타/cpc | 12 | | |
  | 2026-07-01 | 2026-07 | 구글 | 브랜드검색 | 8,100 | 540 | 120,000 | 22 | 511 | 구글/cpc | 20 | | |
  | 2026-07-01 | 2026-07 | 네이버 | 파워링크 | 5,600 | 180 | 45,000 | 9 | 172 | 네이버/cpc | 8 | | |

  - **`월`** = `=TEXT(날짜, "YYYY-MM")` 자동 추출 → 행 숨김 대신 이 칼럼으로 필터. 지난 달을 지울 필요 없음.
  - **매체 데이터 + GA 데이터가 같은 행**에 나란히 → 취합 시 두 시트를 오갈 일이 없고, "매체 전환 vs GA 전환" 차이도 한눈에.
  - **`확정비용`** = 평소 비워두고, 월간 리뷰 때 소폭 차이나는 값만 확정 입력. 원본 `비용`은 보존.
  - **`캠페인`·`메모`** = 월간 리뷰에서 성과 맥락을 붙일 자리.

  **월간 리뷰는 이렇게 끝난다** (새로 취합 X):
  1. `월` 칼럼을 해당 월로 필터
  2. 피벗 테이블로 매체별 합계 한 번에
  3. 소폭 차이나는 값만 `확정비용`에 채우기 → 끝

  → "월말에 다시 리뷰용 데이터를 쌓느라 시간 낭비할 필요가 없다"가 이 표 하나로 실현된다.

- **막혔던 점 / 어떻게 풀었나:**
  처음엔 "매체별 취합 자동화" 같은 부분 자동화로 생각했으나, 인터뷰 중 GA 데이터도 따로 들어간다는 점이 드러나면서 **형식 자체를 다시 설계**하는 방향으로 잡혔다. 매체·GA를 한 행에 합치고 `월`·`확정비용` 칼럼을 추가하는 것으로, 취합 작업 자체를 없애는 구조에 도달했다.
