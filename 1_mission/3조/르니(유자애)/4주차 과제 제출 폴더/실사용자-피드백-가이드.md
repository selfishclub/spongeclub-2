# 결정 OS — 실사용자 피드백 가이드

> 나중에 실제로 2명 이상 만날 때 그대로 들고 가서 쓰는 가이드.
> 목적은 칭찬받는 게 아니라 **내가 세운 가정 중 뭐가 틀렸는지 확인하는 것.**

---

## 0. 누구에게 보여줄까 (섭외 기준)

PRD ②에서 좁힌 타깃: **클로드/AI 도구를 이미 편하게 쓰지만, 평소엔 결정을 잘 하는 편이고 가끔 무거운 결정 앞에서만 못 정하는 사람.**

- 가능하면 **1명은 이 타깃에 가까운 사람**(AI 도구 익숙 + 최근에 "이거 결정 못 하겠다" 싶었던 경험 있는 사람), **1명은 타깃 밖**(AI 낯설거나, 결정을 잘 미루지 않는 사람)으로 섭외하면 좋아요. 타깃 안/밖 반응이 갈리는지가 가장 좋은 데이터예요.
- 가상 패널이 이미 "김재성·김세원 유형"에서 긍정 신호를 냈으니, 그 유형(직장인/자영업/사이드프로젝트 하는 2030~3040, AI 도구 사용 경험 있음)에 가까운 지인·수강생이 1순위 후보예요.

## 1. 뭘 보여줄까 (세팅)

- **가짜 시나리오 주지 말기.** "이런 상황이라 치고 써보세요" 대신, **그 사람이 지금 진짜 고민하고 있는 것**을 가져오게 하세요. (없다고 하면 "요즘 뭐 정하기 애매한 거 없어요?"로 살짝만 유도)
- 화면을 같이 보되, **손은 그 사람이 직접 움직이게** 하세요. 대신 타이핑해주지 않기.
- 클로드 실행부터 `/결정` 켜는 것까지 **처음부터** 시켜보세요 — 여기서부터가 검증 대상이에요 (가상 패널에서 "켜는 게 일"이라는 반응이 나왔던 지점).

## 2. 뭘 관찰할까 (말보다 행동)

말로 나오는 반응보다, 옆에서 **조용히 지켜보며** 아래를 체크하세요. 멈칫하는 순간이 정보예요.

| 관찰 포인트 | 왜 보는가 |
|---|---|
| `/결정` 켜기 전까지 걸린 시간 / 헤맴 여부 | 진입장벽 검증 (가상 패널: "켜는 게 일") |
| 첫 질문(G턴)에 바로 답하는지, 멈칫하는지 | 질문 방식이 자연스러운지 |
| "발판 3개(예: A인지 B인지…)" 중 하나를 그대로 고르는지, 무시하고 자기 말로 답하는지 | 발판이 도움되는지 방해되는지 |
| 중간에 스마트폰 보거나 딴짓하는 순간 | 흐름이 늘어지는 지점 |
| 결론 카드 보고 나온 첫 표정·반응 (침묵/끄덕/헛웃음 등) | "결론이 났다"는 게 실제로 느껴지는지 |
| 결론 카드를 캡처하거나 다시 읽어보는지 | 결과물에 가치를 느끼는지 |
| "저장할까요?" 물었을 때 바로 답하는지, 망설이는지 | 통제권 설계(선택형 저장)가 맞았는지 |

## 3. 끝나고 뭘 물어볼까 (인터뷰)

**순서가 중요해요 — 안 좋았던 점부터 먼저.** 좋은 점 먼저 물으면 예의상 칭찬이 나와서 진짜 반응이 묻혀요.

1. "이거 쓰면서 제일 걸렸던 거, 혹은 불편했던 순간 있었어요?" (최소 1개는 나올 때까지 기다리기 — 없다고 해도 "진짜요? 하나만 짜내면요?"로 한 번 더)
2. "결론 카드 보고 어땠어요? 진짜 도움이 됐어요, 아니면 그냥 정리된 척이었어요?"
3. "다음에 비슷한 고민 생기면 또 쓸 것 같아요? 언제, 어떤 상황에요?" (구체적인 상황을 받아내기 — "네 쓸게요"로 끝내지 않기)
4. "이거 친구한테 설명한다면 뭐라고 할 것 같아요?" (한 문장으로)

## 4. 특히 검증하고 싶은 것 (가상 패널의 가정 vs 실제)

가상 패널에서 나온 예측이 실제로도 맞는지 꼭 확인하세요:

- [ ] **"켜는 게 귀찮아서 안 쓴다"**는 반응이 실제로도 나오는가?
- [ ] **디지털 능숙자는 "무거운 결정에만 쓴다"**는 조건이 실제로도 맞는가, 아니면 가벼운 결정에도 의외로 쓰는가?
- [ ] **"결론 카드가 내 상황을 뭘 안다고"** 하는 불신이 실제로도 나오는가?

## 5. 기록 방법

- 그 자리에서 스마트폰 메모나 종이에 **인용문 그대로** 짧게 적어두세요 (나중에 표현을 다듬지 말고 원문 그대로가 더 쓸모 있어요).
- 두 사람 다 끝나면 아래 형식으로 `submission.md`의 "실제 유저 피드백" 항목에 옮겨 적으면 돼요:

```
- OOO (관계·특징): [멈칫했던 지점] / [했던 말 인용] / [다시 쓸지 여부]
- OOO (관계·특징): [멈칫했던 지점] / [했던 말 인용] / [다시 쓸지 여부]
- 가상 패널 예측과 비교: [맞았던 것] / [틀렸던 것]
```
