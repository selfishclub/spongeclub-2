> 지금까지 공유하며 받은 **피드백을 반영해 최종 완성**!

- **완성한 것 (무엇을):** `/결정` 스킬에 **Step 6 — 기록 저장**과 **기록 삭제** 기능을 추가해 v2 → v3로 완성했다. 결론 카드가 나온 뒤 저장 여부를 물어보고(a: `decision_log`에 저장 / b: 저장 안 함), 저장을 고르면 `C:\Users\claude_projects\decision_log\`에 결론 카드 사본 + 원본 대화 위치가 기록된다. 이번 주엔 실제로 이 스킬을 써서 진짜 고민(프리랜서로서 인적자본을 어떻게 쌓을지)을 풀었고, 그 결정의 "첫 걸음"으로 나온 `아이디어-목록.md`(아이디어 14개)까지 만들었다.

- **피드백 반영한 점:** 크루 피드백보다는 직접 써보다가 부딪힌 내 불편함을 그대로 기능으로 옮겼다. VS Code에서 나눈 결정 대화가 앱에서 안 보여서 "대화가 사라진 줄" 알고 당황했던 게 계기였다. 처음엔 결론 카드가 나오면 무조건 자동 저장되게 만들었는데, "내가 저장할지 말지 고를 수 있게 해달라"고 스스로 요청해서 선택형으로 바꿨다. 저장 기능을 만들다 보니 "지우고 싶을 때도 있겠다"는 생각이 들어 삭제 기능까지 같이 설계했다.

- **결과물 (링크·스크린샷 — 이미지는 `이미지첨부/` 폴더에):**
  - 스킬 파일: `C:\Users\wwyut\.claude\skills\결정\SKILL.md` (제출 사본: 같은 폴더의 `결정.md`)
  - 기록 저장소: `C:\Users\claude_projects\decision_log\`(`README.md` + 과거 결정 8건 소급 기록)
  - (스크린샷은 이번 주 생략 — 대신 위 경로로 실제 동작 확인 가능)

- **알게 된 인사이트:**
  - 저장 기능을 만들면 삭제 기능도 짝으로 있어야 완전해진다. "남기고 싶다"만 생각했지 "지우고 싶을 수도 있다"는 안 떠올랐는데, 만들다 보니 자연스럽게 나온 요구였다.
  - 기록보다 통제권이 먼저였다. 자동 저장이 편해 보였지만 실제로 원했던 건 "저장할지 말지 내가 고르는 것"이었다.
  - 스킬을 고치는 것과 스킬을 실제로 쓰는 걸 같은 날 같이 하니 더 잘 만들어졌다. 추상적으로 설계할 때는 안 보이던 필요(경로 오타, 세션 ID 혼란)가 실사용 중에 바로바로 드러났다.
