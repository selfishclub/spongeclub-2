# 결정 OS 설계

## 목표

"이거 결정 못 하겠어"를 던지면, 고민이 결정으로 바뀌는 흐름을 만든다.

## 파일 위치

```
C:\Users\wwyut\.claude\plugins\결정\결정.md
```

전역(global) 위치에 저장해서 어느 프로젝트에서든 `/결정`으로 호출 가능.

## 대화 흐름 (3단계 적응형)

### 1단계 — 고민 유형 파악

고민을 받으면 자동으로 유형을 판단:

| 유형 | 예시 | 접근 |
|------|------|------|
| 선택형 | "A vs B 중 뭐가 나을까" | 선택지가 이미 있음 → 기준 질문으로 |
| 방향형 | "이걸 계속 해야 하나" | 선택지 먼저 펼쳐줌 |
| 우선순위형 | "뭘 먼저 할지 모르겠어" | 항목 정리 후 기준 질문 |

### 2단계 — 핵심 질문 (최대 3번)

유형에 따라 다른 질문 1개씩:
- 선택형 → "둘 중 어느 쪽이 더 오래 남는 선택이에요?"
- 방향형 → "지금 이게 부담인가요, 아니면 그냥 귀찮은 건가요?"
- 우선순위형 → "지금 당장 안 하면 제일 타격 큰 게 뭐예요?"

질문은 1회에 1개. 최대 3번 안에 결론으로 닫는다.
사용자가 "모르겠어" 해도 Claude가 방향 제시하고 닫는다.

### 3단계 — 결론 카드 출력

```
📌 결정: [선택한 것]
🎯 기준: [결정의 근거 한 줄]
📅 날짜: YYYY-MM-DD
```

재확인 질문 없이 바로 카드 출력.

## 톤

- 짧고 단도직입적
- "~해보면 어떨까요?" 같은 컨설턴트 말투 금지
- "지금 제일 걸리는 게 뭐예요?" 스타일
