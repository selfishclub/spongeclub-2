# 1주차 — 나만의 OS 만들기 🛠️

> 미션을 진행하며 **과정과 결과물**을 기록해주세요. (다 못 채워도 OK, 한 것 위주로!)

## 🎯 미션 1. 내 OS 재료 찾기
> 인터뷰 스킬(아이데이션)로 "내 삶에 필요한 게 뭔지" 찾기
- **과정 (어떻게 찾았나):**
  인터뷰 스킬로 5스텝 진행. ① 오늘 하루 떠올리기 → 혼신기록·단수끼리·주목습관 3개 브랜드를 동시에 굴리다 보니 병목이 많다는 게 먼저 나옴. ② 짜증 포인트 좁히기 → 단수끼리의 AI 캐릭터 일관성 문제(시간 아끼려다 오히려 배로 씀)와 혼신기록의 반복 작업이 후보로 나옴. ③ 일 밖 삶 영역 확인 → 임신 준비(시험관), 굿모닝 챌린지·시험관 히스토리 기록이 밀리고 있는 것도 나옴. ④ 무게 재기 → 삶 쪽 두 개는 마감이 있는 "밀린 걸 몰아서 끝내면 되는" 일회성 과제인 반면, 일 쪽은 매번 반복되는 진짜 마찰이라는 걸 구분하게 되면서 일 쪽으로 수렴. ⑤ 혼신기록 안에서도 "직접 초안 쓰기(의도적 선택)"와 "썸네일·발행 반복 작업(진짜 마찰)"을 구분해서, 후자를 재료로 확정.
- **결과:**
  🟡 내 OS 재료 카드
  - 영역: 일 — 크리에이터 콘텐츠 제작 (혼신기록 뉴스레터 + OSMU 확장)
  - 걸리는 지점: 매달 뉴스레터 발행 때마다, 아이디어를 초안으로 정리하고 타겟·메시지·기존 포맷에 맞춰 완성하는 과정 + 발행 후 인스타·유튜브에 따로 콘텐츠를 만들어 올리는 과정이 매번 반복되는 수작업
  - 지금은: 조각난 아이디어를 직접 초안으로 쓴 뒤 Claude에게 보완 요청 → 발행 → 인스타/유튜브용 콘텐츠는 따로 다시 만들어야 하는데, 그마저도 밀려서 인스타는 손도 못 대고 있음
  - OS가 된다면: 아이디어·키 컨셉만 넣으면 타겟/메시지/기존 포맷에 맞춘 뉴스레터 초안이 나오고(비히브 MCP 연동), 그 레터를 바탕으로 인스타 후킹 피드 콘텐츠 + 유튜브 커뮤니티 공유 콘텐츠까지 이어서 뽑아주는 파이프라인
  - 한 문장: "한 달에 한 번 레터 발행이 거의 하루 작업으로 끝나고, 레터 하나가 써지면서 인스타와 유튜브 콘텐츠까지 저절로 채워진다. 그리고 남은 시간을 단수끼리 콘텐츠 발행에 집중해서 쓸 수 있다."
- **느낀 점:**
  "반복되는 마찰"과 "마감 있는 일회성 캐치업"을 구분하는 게 OS 재료를 고르는 기준이 된다는 걸 알게 됨. 임신 준비 관련 기록들은 중요하지만 지금 단계에선 매일 자동화가 필요한 루틴이 아니라 밀린 걸 몰아서 끝내면 되는 일이라, OS보다는 그냥 처리할 일에 가깝다는 걸 정리하며 깨달음. 또 혼신기록 안에서도 "내가 지키고 싶은 과정(초안 직접 쓰기)"과 "없애고 싶은 마찰(반복 작업)"이 섞여 있었는데, 이걸 나누고 나니 뭘 자동화해야 할지 훨씬 또렷해짐.

## 🧩 미션 2. 내 OS 기획
> 인터뷰 결과 + 세션 내용(흐민·배짱·키노) 활용해 기획
- **기획 내용:**
  흐민 세션의 5단계 루프(인풋→저장→연결→제안→발행)와 배짱의 온맘(트리거 기반 자동 메시지) 사례를 참고해 3단계로 기획함.

  | 단계 | 핵심 | 참고한 사례 |
  |---|---|---|
  | 1 | 텔레그램으로 자연어 인풋 → 혼신기록 아이디어 자동 저장 | 흐민 (인풋·저장) |
  | 2 | 월 1회, 쌓인 아이디어를 모아 비히브 MCP로 초안 자동 생성 (타겟·메시지·기존 포맷 반영) | 흐민 (제안) |
  | 3 | 레터 발행 트리거 → 인스타 후킹 피드 콘텐츠 + 유튜브 커뮤니티 공유 콘텐츠 자동 파생 (OSMU) | 배짱 온맘 (트리거 자동화) |

  이번 주는 1단계(텔레그램 인풋 채널 구축)만 실제로 만들어보는 것을 목표로 함.
- **막혔던 점 / 어떻게 풀었나:**
  단수끼리(AI 캐릭터 일관성)와 혼신기록(발행 반복 작업) 중 어느 쪽을 먼저 OS로 만들지 결정하기 어려웠음. "지금까지 상반기 내내 손 못 댄 단수끼리를 더 밀어야 하지 않나" 하는 생각도 있었지만, 혼신기록 안에서 다시 나눠보니 "썸네일/발행 반복"이 단수끼리의 캐릭터 일관성 문제와 같은 성격(매번 반복되는 마찰)이라는 걸 발견 → 둘 다 후보로 열어두되, 비히브 MCP 연동이라는 구체적 실행 경로가 이미 보이는 혼신기록 쪽을 1단계로 먼저 선택함. 단수끼리는 다음 기획 대상으로 남겨둠.
