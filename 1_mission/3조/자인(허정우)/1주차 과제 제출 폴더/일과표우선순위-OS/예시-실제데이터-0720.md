# 실행 예시 — 실제 구글캘린더 데이터로 테스트 (2026-07-20, 월)

가상의 예시가 아니라, 실제 구글캘린더에 연결해서 오늘 일정을 그대로 읽어와 스킬을 돌린 기록.

## 입력 (구글캘린더에서 읽어온 오늘 일정 그대로)

```
2026년 7월 20일(월) 일정 7개
- 만만선생님 캐나다 (종일, 7/17~28 — 배경 정보, 할 일 아님)
- 07:30–08:45 박종웅 원장님 보험한약 강의
- 13:00–13:15 결산회의
- 13:30–14:00 전략회의
- 14:00–14:30 김현지님 치료
- 18:00–19:00 자연치유법인-세무자료 제출
- 21:00–22:00 [스폰지클럽] 이기적 클럽에서 이기적으로 사는 법

확인 결과:
- 오늘 캘린더 외 추가로 처리할 잡다한 할 일 없음
- 요즘 진료는 거의 안 봐서 빈 시간대는 실제로 비어 있음
- "세무자료 제출"은 18시가 실제 마감이 아니라, 저녁에 입력하다 보니 그 시간에 찍힌 것 — 진짜 마감은 이번 주 안
```

## 출력 (스킬이 재판단한 오늘의 실행 시간표)

| 순서 | 시간대 | 할 일 | 예상 소요 | 우선순위 근거 |
|---|---|---|---|---|
| 1 | 07:30–08:45 | 박종웅 원장님 보험한약 강의 | - | 고정 · 강사·수강생과 맞춘 시간 |
| 2 | 09:00–10:00 | 자연치유법인 세무자료 제출 준비 | 1시간(캘린더 입력 근거) | 이번 주 안에 · **원래 18시로 찍혀 있었지만 실제 고정 아님** → 오전 여유 블록으로 재배치 · 집중 |
| 3 | 13:00–13:15 | 결산회의 | - | 고정 · 동료와 맞춘 시간 |
| 4 | 13:30–14:00 | 전략회의 | - | 고정 · 동료와 맞춘 시간 |
| 5 | 14:00–14:30 | 김현지님 치료 | - | 고정 · 환자와 맞춘 시간 |
| 6 | 21:00–22:00 | [스폰지클럽] 세션 | - | 고정 |

**🎯 지금 할 일**: 강의 끝나면 바로 "세무자료 제출 준비"부터. 이번 주 안이라 오늘 안 해도 되지만, 오전에 큰 여유 블록이 있고 — 지금 캘린더를 보니 이런 행정 처리가 매번 저녁 시간에 몰려 처리되는 패턴이 반복되고 있어서, 여유 있을 때 먼저 처리하는 쪽을 추천.

**⚠️ 오늘 못 끝낼 수도 있는 것**: 없음 — 유동 항목이 하나뿐이라 오늘 안에 여유 있게 끝남.

**남는 시간(10:00–13:00, 14:30–21:00)**: 급한 유동 항목이 더 없어서, 구글 태스크에 쌓인 백로그(기본 목록 501개 이상 진행 중) 중 하나를 골라 처리하는 데 써도 좋다고 제안.

## 이 테스트로 알게 된 것

가짜 예시 데이터로 만들었을 때는 "고정 여부"를 그냥 캘린더에 시간이 있는지 없는지로만 판단했었다. 그런데 실제 데이터를 넣어보니, **캘린더에 시간이 찍혀 있어도 진짜 고정이 아닌 경우(세무자료 제출)** 가 있다는 걸 발견했다. 구글캘린더가 "입력한 시점"을 기준으로 시간을 저장하다 보니 생기는 문제였다. 그래서 처리 규칙 1번을 "상대방이 관여하는 일인지 아닌지"로 다시 정의해서 반영했다 — 이게 이번 다듬기의 핵심 변경점이다.

또 하나, 구글 태스크에 500개가 넘는 할 일이 별도로 쌓여 있다는 걸 확인했다. 오늘 하루만 정리해서는 안 되고, 이 백로그를 조금씩 녹이는 것도 이 OS가 결국 다뤄야 할 문제라는 걸 알게 되어 3단계 로드맵에 반영했다.
