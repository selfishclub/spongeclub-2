## 🎯 미션 1. 내 OS 만들기
> **[ 내 삶을 돕는 OS ]** 또는 **[ 콘텐츠 OS ]** 중 하나를 선택해 완성해주세요.

**✅ 선택:** 내 삶을 돕는 OS — **`Hair OS`** (1인 헤어/두피케어 브랜드 창업 운영 OS)

### 📐 기획
> 무엇을, 왜, 어떻게 만들지

- **왜:** 두피케어 브랜드를 혼자 창업 중이다. 시장조사·전략·마케팅·재무·PM을 전부 혼자 하다 보니,
  5월부터 쌓아온 리서치·의사결정이 계속 흩어지고 **같은 걸 매번 다시 조사하고 다시 추론**하게 됐다.
- **무엇을:** 브랜드 그 자체보다, **"브랜드를 만드는 과정"을 축적하는 OS**를 세우기로 했다.
  핵심 무기(moat)를 제품이 아니라 *"의사결정·실험·검증 이력이 LLM이 읽을 수 있는 형태로 쌓인 운영 원장"*으로 정의.
- **어떻게 (설계 3축):**
  1. **지식층** — git+markdown 위키. 모든 리서치에 TL;DR 의무, `[[교차참조]]`, 출처 라벨(`[공식]`/`[보도]`/`[추정]`) 강제.
  2. **AI 조직층** — 총괄 PM 에이전트 '헤일리' 아래 전략·제품·마케팅·재무·리뷰어·데이터수집 6개 부서 서브에이전트.
     각 부서는 자기 PRD만 보고 타 부서 PRD는 못 보게 격리(컨텍스트 오염 방지).
     위임 → 리뷰어 검토 → 총괄 점검 → 나에게 보고, 4단 오케스트레이션.
  3. **원장/대시보드층** — `roadmap.yaml` 하나가 PM 원장(epic→task→sub_task). 웹 대시보드에서 상태를 바꾸면
     yaml을 최소편집으로 패치하고 자동 git commit(이력이 곧 자산).
- **승인 게이트:** 리서치·위키 작성·상태 제안은 자동 진행, 비용 발생·외부 발송·전략 변경은 반드시 내 승인.
  애매하면 승인 쪽으로.

### ⚙️ 구현
> 실제로 만든 것 (링크·스크린샷 — 이미지는 `이미지첨부/` 폴더에)

이번 주(7/8~7/11) **커밋 130개**로 위 설계를 실제로 돌아가게 만들었다.

1. **지식 위키:** research 123 + strategy 57 등 **문서 180개 이상** + LLM용 인덱스(470줄) + append-only 로그(1,291줄).
   5월부터 쌓아온 리서치 자산을 새 구조로 전부 이관·시스템화.
2. **AI 부서 조직:** 헤일리(총괄 PM) + 6개 부서 서브에이전트 + 부서별 PRD 8개.
   7/10 스모크 테스트로 위임→재검토 루프→에스컬레이션→승인 게이트까지 **전 구간 동작 확인**.
   실전 투입 1건: 경쟁사 성분·가격 오기재를 11개 파일에서 찾아 정정(전략 부서 수행 → 리뷰어 2회 검토 통과).
3. **웹 대시보드**(Next.js): 간트·위키·결정·컨택트 4탭. 간트에서 태스크 상태 변경 → roadmap.yaml 자동 패치+커밋.
   **단위 테스트 100개 전부 통과.**
4. **Slack 봇:** `@헤일리`가 실제 채널에서 응답·공지 작동 중.
5. **합성 리서치 파이프라인:** 페르소나 12명(코어 5·이탈군 4·대조군 3) 합성 심층인터뷰를 설계·실행,
   브랜드 언어 가설 검증에 실사용 (아래 삽질 참조).

### 🧗 과정에서의 삽질
> 막혔던 지점, 시도한 방법, 어떻게 풀었는지 솔직하게

- **가상 FGD 파이프라인을 통째로 폐기했다.** 페르소나 매칭·패널 선정·프롬프트 생성까지 다 만들어 실행했는데,
  결과를 브랜드 판단 근거로 쓰기엔 신뢰할 수 없다고 결론. `DEPRECATED.md`로 "사용 금지"를 명시하고,
  반증 장치(대조군 포함)를 갖춘 **합성 심층인터뷰 방식으로 갈아엎었다.** 만든 걸 버리는 게 제일 아팠다.
- **내 브랜드 언어 가설이 내 OS한테 반증당했다.** 핵심어로 밀던 "맑음/맑다"가 인터뷰 12명 중 **발화 0명**
  (광고 클리셰로만 인지). "가볍다"도 카드에 미리 제시한 단어라 자연발생이 아닐 수 있어(priming 오염 의심) 승격 보류.
- **수요 thesis도 뒤집혔다.** "예방/longevity" 컨셉을 올리브영 리뷰 6,388건 전수 클러스터링 + 검색 데이터로
  검증했더니 수요 0.99% → **"현재형 페인 즉각 해소"로 방향 전환**하고 확정 판단으로 못박았다.
- **리뷰 크롤링 지옥:** 헤드리스 브라우저·표준 MCP 전부 WAF에 막힘 → 실제 Chrome + Shadow DOM 피어싱으로만 성공,
  이후 페이지네이션 버그까지 잡아서 파이프라인화.
- **에이전트 조직도 모순 발견:** 문서마다 비서 에이전트와 총괄 PM의 관계가 다르게 적혀 있었다(상하 vs 동급).
  '동급 peer'로 확정하고 문서 전체 정합성을 맞췄다 — 사람 조직처럼 AI 조직도 R&R 문서가 어긋나면 바로 사고가 난다.

### ✅ 결과물
> 완성한 것 / 작동 화면

- **돌아가는 Hair OS 한 세트:** 위키 180+ 문서 · AI 부서 6개(스모크 테스트 통과) · 대시보드(테스트 100개 통과) ·
  Slack 봇 라이브 · 합성 인터뷰 파이프라인.
- **OS가 실제로 사업 판단을 바꾼 것 3건:** 수요 thesis 전환(예방 → 현재형 페인), 브랜드 핵심어 반증("맑음" 폐기),
  경쟁사 데이터 오류 정정(11개 파일).
- 원장(`roadmap.yaml`)에는 코드가 아니라 **실제 GTM 진행 상황**(세그먼트 결정, GTM Phase 0~2, 예산 배분 초안)이 관리되고 있다.

### 💡 알게 된 인사이트 & 공유하고 싶은 내용
> 하면서 깨달은 것, 크루들과 나누고 싶은 것

- **OS의 가치는 편해지는 게 아니라 "내 가설이 반증당하는 것"이었다.** 이번 주 가장 큰 산출물은 기능이 아니라,
  내가 믿던 컨셉 2개가 데이터 앞에서 깨진 것. 혼자 창업하면 반대해주는 사람이 없는데, OS가 그 역할을 했다.
- **AI 에이전트 조직도 사람 조직처럼 굴려야 한다.** 부서별 R&R 문서(PRD), 타 부서 정보 격리, 리뷰 게이트,
  승인 라인 — 이걸 안 만들면 에이전트들이 서로 컨텍스트를 오염시킨다.
- **버리는 기록도 자산이다.** 폐기한 파이프라인에 DEPRECATED 사유를 남겨두니, 같은 실수로 되돌아가지 않는다.
