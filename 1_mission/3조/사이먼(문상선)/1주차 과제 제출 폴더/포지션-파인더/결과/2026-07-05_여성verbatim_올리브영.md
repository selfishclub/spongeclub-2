- **방법:** gstack 연결 Chrome + 올리브영 모바일 cursor API (실브라우저 동일오리진 fetch). 화해는 CloudFront가 자동화 차단 → 올영으로 대체.

## ⚡ 1차 데이터가 뒤집은 것 (웹 리서치 대비)
- 웹 리서치 판정: "라보에이치·닥터포헤어가 임상수치로 효능 체감을 준다 → 효능체감은 부분 빈땅."
- **실제 리뷰:** 그 임상 브랜드조차 **실사용자는 "효과는 모르겠지만 쿨링 좋아서 만족"**이라고 쓴다. 박스의 임상수치가 **체감으로 전환되지 않음.** → **효능 '체감'은 여전히 빈 땅.** (박스의 '클레임'과 손끝의 '체감'은 다르다)

## 판정 갱신
- **효능 '체감' = 빈 땅 [확실↑]** — 임상 리더 제품조차 실사용자는 효능을 체감 못 하고 쿨링으로 대체. 웹 리서치의 [가능성 높음] → 1차 데이터로 **[확실]에 근접**.
- **지성(열감·기름) + 숱 동시 = 실제 리뷰에서 확인** — 정량조사(설문)와 리뷰(자연발화) 양쪽이 일치. 신뢰도↑.
- **쿨링은 위생요인이자 함정** — 모두가 쿨링으로 "관리 중" 착각을 팔고 있음. 차별화는 쿨링이 아니라 **효능을 진짜 체감·확인시키는 것**.
