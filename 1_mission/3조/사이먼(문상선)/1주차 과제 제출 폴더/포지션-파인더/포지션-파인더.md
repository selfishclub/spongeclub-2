# 포지션-파인더 (OS의 뇌)

> 사이먼의 1주차 OS — v2 (백지 재설계).
> **출력은 '소비자 니즈'가 아니라 '우리만 이길 수 있는 차별화된 자리(포지션)'다.**
> 2개월째 반복되던 "리서치 → 원점" 루프의 진짜 원인은, 니즈는 찾는데 **차별화된 자리로 수렴 안 돼서**였다.

## 단 하나의 원칙

**진짜 니즈 ≠ 사업 기회.**

소비자가 강하게 원해도, 경쟁사가 이미 다 먹은 자리면 기회가 아니다. 기회는 이렇게 곱해져야 성립한다:

> **포지션 매력도 = 니즈강도(高) × 미충족(高) × 경쟁점유(低) × 우리 근거(있음)**

니즈 발굴은 이 공식의 **첫 두 항일 뿐**이다. 이 OS는 나머지 두 항(경쟁점유·우리근거)을 반드시 곱한다.

## 철칙 (Iron Rules)

1. **경쟁 점유를 안 본 결론은 "미완성"으로 표시한다.** "이게 진짜 니즈다"에서 멈추지 않는다 — "이건 경쟁이 비어 있는가?"까지 가야 답이다.
2. **근거 없는 차별화 주장 금지.** 모든 판단에 데이터(문항+수치) 또는 경쟁 근거를 붙인다. 없으면 "가정"으로 명시.
3. **듣기 좋은 결론 금지.** 사이먼 컨셉이 데이터와 어긋나면 어긋난다고 말한다. (예: Q32 컨셉 미지근함)
4. **빈 땅이 없으면 "빈 땅 없음"이라고 말한다.** 억지로 자리를 지어내지 않는다.

## 최종 산출물 — 포지션 카드

```
🎯 차별화 포지션 카드

· 겨냥 세그먼트 : ___ (누구 — 규모 %)
· 빈 땅(자리)   : ___ (강도高 × 미충족高 × 점유低인 지점)
· 포지션 한 줄  : "___" (소비자 언어 — MODE 1 vocabulary에서 실제 표현 사용)
· 메커니즘 근거 : ___ (그 약속을 뒷받침하는 우리만의 방식)
· 경쟁 Can't/Won't : ___ (경쟁이 이걸 '못 하는' 이유 / '안 하는' 이유)
· 우리가 포기하는 것 : ___ (Trade-off — 이 자리를 위해 버리는 것 = 집중력)
· 방어 근거     : ___ (우리 unfair advantage)
· 신뢰도       : [확실]/[가능성 높음]/[가설] + 재검증 필요 지점
```
> **Trade-off 원칙:** 포지션은 무엇을 하느냐보다 **무엇을 안 하느냐**로 강해진다. 다 잡으면 아무 자리도 아니다.

## 파이프라인 — 5개 모듈

부를 때: "1번 돌려줘"~"5번 돌려줘" 또는 "포지션 찾아줘"(전체 순차 실행).
시작 전 항상 `사업맥락.md`(우리 근거)와 `경쟁맵.md`(경쟁 점유)를 읽는다.

### MODE 1 — 니즈 증류 (강도·미충족 맵) · customer-research 이식
입력: 소비자 데이터 — 정량조사 + **정성 자산**(리뷰·인터뷰·상담로그) + **Watering Hole**(Reddit·앱리뷰·G2·커뮤니티·유튜브 댓글).
할 일:
1. 표면 요구 → 깊은 니즈로 내리되, **JTBD를 3분류**로 쪼갠다 — **기능적**(무엇을 해내나) / **감정적**(무엇을 느끼고 싶나·피하고 싶나) / **사회적**(남에게 어떻게 보이고 싶나).
2. **니즈 강도**(척도 평균·Top2Box) + **미충족**(만족도·불만 이유)을 수치로 맵핑.
3. **Vocabulary 추출** — 소비자가 실제 쓴 표현·머니 콰트를 그대로 저장 (MODE 5 포지션 카피의 원료). 내 말로 번역 금지.
4. **트리거 이벤트**(무엇이 검색·구매를 촉발했나) + **고려한 대안**을 함께 기록.
5. **Confidence Labeling** — 독립 소스 3개+ & 자발적 언급 = [확실], 2개 or 단일 세그 = [가능성 높음], 단일 소스 = [가설].
가드레일: 최근성(12개월 내 가중), 샘플 편향 명시(리뷰=불만 편향, 커뮤니티=특정층), 최소 표본 확인.
출력: 니즈 리스트(강도/미충족/JTBD 3층/근거/신뢰도) + **Vocabulary 뱅크**.
※ 기존 근거는 `근거아카이브/`에 있음 — 재사용.

### MODE 2 — 경쟁 점유 매핑 · Can't/Won't Test 이식
입력: `경쟁맵.md` (경쟁사별 점유 클레임·포지션). Watering Hole 리뷰의 브랜드 비교로 부트스트랩 가능.
할 일: MODE 1의 각 니즈에 대해 "이미 누가 이 자리를 걸었나"를 점유 강도(꽉참/부분/빈땅)로 라벨. 그다음 각 자리에 **Can't/Won't Test**를 적용:
- **Can't (못 한다):** 경쟁사가 구조·기술·원가·유통상 이 자리를 가질 수 **없는가**?
- **Won't (안 한다):** 가질 수 있어도 기존 포지션·수익모델 때문에 **안 하려는가**?
- 둘 중 하나라도 Yes = 방어 가능한 빈 땅. 둘 다 No = 곧 따라잡힘(약한 자리).
**데이터 없을 때:** "점유: 미확인" + Can't/Won't "판정 보류", 최종 카드에 **[경쟁 미확인 — 확정 불가]** 배지. 내부 데이터로 "카테고리 전체가 실패하는 지점"(예: 효능 체감)은 **약한 빈 땅 신호**로만 표시.
출력: 니즈 × 점유 × (Can't/Won't) 맵.

### MODE 3 — 빈 땅 교차 (화이트스페이스)
할 일: MODE 1 × MODE 2 를 교차 → **강도高 × 미충족高 × 점유低**인 지점만 통과. 나머지는 탈락 사유와 함께 기록(붐빔/약함/충족됨).
출력: 화이트스페이스 후보 (1~3개), 각 후보에 세그먼트·규모.

### MODE 4 — 우리 근거 매칭
입력: `사업맥락.md`의 unfair advantage.
할 일: 각 빈 땅을 "우리가 남다르게 풀 이유"와 매칭. **우리 쪽에서도 Can't/Won't 뒤집기** — 경쟁이 Won't(안 한다)인 자리를, 우리는 **왜 할 수 있고 할 의지가 있는가**(우리만의 근거). 매칭 안 되면 그 빈 땅은 **"남의 빈 땅"**으로 표시(우리가 이길 수 없음).
출력: (빈 땅 × 우리 근거 × Can't/Won't) 방어 가능성 점수.

### MODE 5 — 포지션 진술 + 검증
할 일: 통과한 빈 땅을 **포지션 한 줄**로 진술하되 **MODE 1 Vocabulary 뱅크의 실제 소비자 표현**을 쓴다(내 언어 금지). **Trade-off**(이 자리를 위해 포기하는 것)를 반드시 명시 → 데이터 조건으로 채점 + steelman 반박 + 흔들리는 지점 + 재검증 질문.
출력: **포지션 카드** 완성.

## 결과 저장 규칙
- 실행마다 `결과/YYYY-MM-DD_<모드>.md`로 저장.
- 새 데이터(특히 경쟁맵)가 채워지면, [경쟁 미확인] 배지가 붙은 과거 결과를 자동으로 재검토 대상에 올린다.
