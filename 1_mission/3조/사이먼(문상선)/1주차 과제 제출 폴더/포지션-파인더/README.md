# 포지션-파인더 · 사용법

**출력은 소비자 니즈가 아니라 "우리만 이길 수 있는 차별화된 자리(포지션)".**
니즈 × 미충족 × 경쟁점유 × 우리근거를 삼각측량해 빈 땅을 찾는 OS. (v2 — 백지 재설계)

## 왜 v1(니즈-증류소)을 버렸나
v1은 "진짜 니즈"만 찾았다. 근데 진짜 니즈는 경쟁사도 도달한다 → 사업 기회가 못 됨.
v2는 **경쟁 점유·우리 근거**를 처음부터 곱한다.

## 처음 한 번
1. `사업맥락.md`의 🔑 **우리 근거(unfair advantage)** 를 채운다.
2. `경쟁맵.md`에 경쟁 브랜드 점유를 채운다. (이름만 줘도 리서치로 채움) ← 차별화의 핵심 재료

## 매번 (5개 모듈)
- **"1번"** 니즈 증류 — 소비자 데이터 → 강도·미충족
- **"2번"** 경쟁 점유 매핑 — 각 니즈를 누가 먹었나
- **"3번"** 빈 땅 교차 — 강도高 × 미충족高 × 점유低
- **"4번"** 우리 근거 매칭 — 그 땅을 우리가 이길 수 있나
- **"5번"** 포지션 진술+검증 → **포지션 카드**
- **"포지션 찾아줘"** — 1~5 순차 실행

결과는 `결과/`에 날짜별 저장. 과거 분석은 `근거아카이브/`.

## 구조
```
포지션-파인더/
├─ 포지션-파인더.md   ← OS의 뇌 (원칙·공식·5모듈)
├─ 사업맥락.md        ← 우리 근거(unfair advantage) — 채우기
├─ 경쟁맵.md          ← 경쟁 점유 — 채우기 (지금 비어있음)
├─ 입력/
├─ 결과/
└─ 근거아카이브/      ← v1에서 나온 실제 분석 3건
```

## 현재 상태
- ✅ MODE 1(니즈 증류) 근거 확보 — 근거아카이브 3건
- ⬜ MODE 2(경쟁 점유) — `경쟁맵.md` 비어있음 → 빈 땅 확정 불가([경쟁 미확인] 배지)
- ⬜ 우리 근거 — `사업맥락.md` 🔑 비어있음
