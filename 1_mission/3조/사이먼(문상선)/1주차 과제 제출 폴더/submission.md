# 1주차 — 나만의 OS 만들기 🛠️

> 미션을 진행하며 **과정과 결과물**을 기록해주세요. (다 못 채워도 OK, 한 것 위주로!)

## 🎯 미션 1. 내 OS 재료 찾기
> 인터뷰 스킬(아이데이션)로 "내 삶에 필요한 게 뭔지" 찾기

- **과정 (어떻게 찾았나):**
  창업 2개월 차, 내 하루에서 계속 반복되는 마찰을 파고들었다. 나는 매일 뭔가를 하긴 한다 — 웹페이지를 리서치하고,
  생각을 메모하고, Slack·이메일로 소통한다. 그런데 그게 **전부 흩어진다.** (1) 직접 리서치한 웹페이지를 어디에
  저장할지 애매하고, (2) 내가 정리한 내용을 위키에 넣고 우선순위 매기기가 번거롭고, (3) Slack·메일로 오간 진행이
  로드맵에 반영이 안 된다. 결국 "지금 내 사업이 **어디까지 왔는지**"를 매번 처음부터 다시 짜맞춰야 했다.

- **결과:**
  🟡 내 삶을 돕는 OS · 재료 카드

  ```
  · 영역      : 일 (1인 창업 — 사업 운영·진행 관리)
  · 걸리는 지점 : 리서치·메모·소통이 다 흩어져서 '진행 상황'으로 안 모임
                → 매번 "어디까지 왔지?"를 다시 재구성하느라 앞으로 못 나감
  · 지금은    : 리서치/메모/소통이 브라우저·머릿속·Slack에 조각조각
  · OS가 된다면 : ① 캡처 — 흩어진 걸 던지면 요약·분류해 위키에 저장
                ② 로드맵 — 그게 사업 진행률(역할별)로 자동 연결
                ③ 매일 아침 리포트 — 진행 요약 + 다음 액션을 비서가 브리핑
  · 한 문장   : "내 사업을 함께 운영하고, 매일 아침 브리핑해주는 비서"
  · 첫 한 걸음 : 웹 대시보드 + Slack 아침 리포트로 '캡처 → 로드맵 → 브리핑' 한 바퀴
  ```

- **느낀 점:**
  병목이 '정보가 부족해서'인 줄 알았는데, 인터뷰를 따라가 보니 진짜 병목은 **흩어짐과 진행 추적**이었다.
  더 많이 모으는 게 아니라, 이미 한 걸 한자리에 모아 "어디까지 왔는지"를 늘 보이게 하는 게 필요했다.

## 🧩 미션 2. 내 OS 기획
> 인터뷰 결과 + 세션 내용(흐민·배짱·키노) 활용해 기획

- **기획 내용:**
  개인 운영 콕핏 **`Cockpit OS`**. 처음엔 'DB·로그인 붙은 풀 웹앱'으로 크게 가려다, 1인 콕핏엔 과하다고 판단해
  아래 원칙으로 **얇게** 좁혔다.

  - **데이터 = git으로 추적되는 파일(마크다운·yaml)** = 나만의 '운영 이력'. DB·인증 없음 → 이력 자체가 자산.
  - **지능 = 앱이 아니라 Claude 에이전트**가 담당(요약·분류·리포트·읽기). 앱은 '파일 읽고 캡처 큐잉'만 하는 바보.
  - **로드맵 = 역할 5축**(시장조사·전략·마케팅·재무·개발). 에이전트는 로드맵을 직접 못 고치고 **'제안 → 내 승인'** 게이트를 거친다(원장 신뢰성).
  - **캡처 3층:** inbox(받은편지함) → captures(처리대장) → pages(위키 본문). 지식은 중복 저장 안 함.
  - **산출물 = 웹 대시보드 + Slack 아침 리포트**(매일 08:30).

- **막혔던 점 / 어떻게 풀었나:**
  "0 to 100 하지 말고 가장 불편한 것 하나부터." 큰 걸 한 번에 안 짓고 **Phase 0(엔진) → 대시보드 → Phase 1**로
  쪼갰다. 덕분에 대시보드보다 먼저 'Slack 아침 리포트'라는 핵심 가치가 하루 만에 실제로 돌기 시작했다.

## ⚙️ 미션 3. 내 OS 구현
> 실제로 만들어본 것 (클로드코드 '채널' 기능 활용 OK)

- **무엇을 만들었나:** 클로드 코드로 **스펙 → 플랜 → 서브에이전트 TDD**로 실제로 돌아가는 `Cockpit OS`를 구축했다.
  (혼자 다 짜는 대신, 작업을 잘게 나눠 서브에이전트가 하나씩 구현하고 매 단계 리뷰 게이트를 통과시키는 방식.)

- **결과물 (오늘 하루, 스펙 → 구현 → 라이브까지):**
  1. **엔진**(bun + TypeScript): 로드맵 검증·역할별 진행률 집계·캡처 파일 기록·Slack 발송 — **단위테스트 총 30건 통과.**
  2. **Slack 아침 리포트 라이브:** 봇을 직접 만들어 `#all-on-grove` 채널로 진행 브리핑을 **실제 발송**(전체 진행률 62% + 역할별 하이라이트 + 다음 액션 3개).
  3. **웹 대시보드 3화면**(localhost:3000): 로드맵(5역할 진행률 + 마일스톤 카드)·오늘 리포트·캡처 박스.
     마일스톤을 **클릭하면 상세 설명·근거가 펼쳐지고**, 근거 문서를 누르면 위키 원문까지 대시보드 안에서 열린다.
  4. **캡처 한 바퀴 실증:** 멤버십이 걸린 트렌드 리포트(careet)를 브라우저로 읽어 → 요약·분류 → 위키에 편입 → 로드맵 근거로 연결(실제 1건).

- **링크 / 스크린샷:** 이미지는 `이미지첨부/` 폴더에
  - `로드맵.png` — 5역할 진행률 대시보드 (전체 62%)
  - `리포트.png` — 오늘 아침 리포트 (진행 요약 + 다음 액션 3)
  - `캡처.png` — 캡처 박스 + 인박스/처리된 캡처 목록

- **느낀 점:** "오늘 내가 한 걸 저장하기도 어렵다"는 통증으로 시작해, **그 통증을 해결하는 OS를 실제로 세웠다.**
  하루 만에 기획→구현→라이브까지 간 건, 내가 다 코딩해서가 아니라 **작게 쪼개 위임하고 매번 검증**하는 방식 덕이었다.
