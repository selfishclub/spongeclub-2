## 🎯 미션 1. 내 삶을 돕는 OS 최종 완성

> 지금까지 공유하며 받은 **피드백을 반영해 최종 완성**!

### ✅ 완성한 것 (무엇을)

**"OS = 컴퓨터 운영체제"가 아니라 "내 일상을 매일 같은 방식으로 굴려주는 시스템"으로 재정의**하고, 텔레그램 봇을 중심으로 3개 모듈을 완성했습니다.

**① 데일리 액션 트래커 (업무)**
- macOS `launchd`가 평일 10시 / 15시 / 18시에 텔레그램으로 **먼저 질문 발송**
- 답하면 클로드 코드 채널 세션이 자동으로 파싱 → 일별 JSON으로 저장
- 정리된 할 일이 **슬랙 팀 채널로 자동 공유** (팀원과 업무 싱크)

**② 바디 체인지 트래커 (건강) — 이번 주 핵심**
- 하루 5번(9:30 / 10:30 / 13:30 / 16:30 / 19:00) 봇이 먼저 물어봄
  → 공복 몸무게 · 아침 · 점심 · 운동 · 하루 마무리
- 답변을 봇이 **매크로(칼로리·단백질·탄수·지방)로 자동 환산**해 기록
- PT 선생님 식단표를 **내가 실제로 먹는 재료(현미밥·닭가슴살·계란흰자)로 대체 계산**
- 기초대사량(1,285) · 체성분 · 활동량을 기반으로 **목표 1,350kcal / 단백질 125g** 설정

**③ 나만의 트레이닝 센터 웹사이트 (아카이빙 & 공개)**
- 쌓인 기록을 자동으로 읽어 시각화 → **Vercel 배포**
- 탭 4개(홈 / PT / 식단 / 기록), 식단·운동 코치 캐릭터, PT 달력, 몸무게 곡선
- **PT 회원권 카운터** (20회 중 5회 사용 → 15회 남음, 이용권 D-168)
- PT 선생님이 준 운동 루틴과 코치 코멘트를 그대로 아카이빙 + 종목별 자세 영상 검색 링크

### 🔁 피드백 반영한 점

| 받은 피드백 | 반영한 것 |
|---|---|
| "화면이 너무 길어서 보기 힘들다" | 한 페이지에 다 쌓여 있던 걸 **탭 4개로 분리**, 스크롤 대폭 축소 |
| "달력이 월별로 다 나와서 복잡하다" | **◀ ▶ 월 선택형 달력**으로 변경, 오늘 날짜 강조 + 범례 추가 |
| "운동 루틴이 한 번에 다 펼쳐져 있다" | **아코디언(접기/펼치기)** 으로 변경, 최근 세션만 기본 펼침 |
| "모바일에서도 보고 싶다" | 620px 이하 전용 레이아웃 (카드 축소, 통계 2열, 탭 가로 스크롤) |
| "몸무게는 공개하기 부담스럽다" | **AES-256으로 암호화**해 비밀번호 아는 사람만 열람 가능하게 |
| "봇이 너무 사무적이다" | 봇 말투를 **따뜻한 친근 존댓말**로 재정의 (숫자 뒤에 항상 응원 한마디) |

### 💡 알게 된 인사이트

**1. "OS"의 본질은 반복되는 사이클을 자동으로 굴려주는 것**
컴퓨터 OS만 OS가 아니었습니다. 아침에 계획하고 → 중간에 점검하고 → 저녁에 마감하는 내 하루도 똑같은 구조라, 그 위에 시스템을 얹을 수 있었습니다.

**2. 트리거와 처리를 분리하면 비개발자도 만들 수 있다**
"몇 시에 알림 보내기"는 macOS 기본 스케줄러(launchd)가, "자연어를 이해하고 기록·계산하기"는 클로드가 담당했습니다. 둘이 만나는 지점만 짧은 스크립트 하나. 어려운 부분을 각자 잘하는 도구에 맡기니 만들 수 있었습니다.

**3. 자동화보다 '데이터 누적'이 더 큰 자산**
일주일 기록하니 패턴이 보였습니다. 단백질 88g 먹은 날 몸무게 최저(63.3kg), 외식한 날 +0.7kg. 일기보다 정확하고 분석 가능한 기록이 남습니다.

**4. 시스템 제약은 비개발자에게 더 큰 장벽 — 그런데 대부분 우회법이 있다**
macOS TCC가 자동 실행 프로세스의 특정 폴더 접근을 막아 알림이 실패했고, 노트북 절전 설정(1분) 때문에 봇이 계속 죽었습니다. "내가 뭘 잘못했나" 싶었지만 전부 시스템 제약이었고, 경로 변경·전원 설정으로 해결됐습니다.

**5. 공개할 것과 감출 것을 나누는 것도 설계다**
"화면에서 숨기기"와 "데이터를 빼기"는 다릅니다. 화면만 가리면 소스를 열어 볼 수 있어서, 배포용 데이터에서 몸무게를 아예 제거하고 별도로 암호화했습니다. 공유하고 싶은 마음과 지키고 싶은 정보는 같이 갈 수 있습니다.

**6. 완성은 한 번에 오지 않는다**
"메모장처럼 떠 있으면 좋겠어" 한마디를 구현하는 데 4단계(데이터 → 서버 → 화면 → 노출 방식)가 필요했고, 디자인은 3~4번 갈아엎었습니다. 한 번에 완성된다고 가정하지 않는 게 오히려 빨랐습니다.
