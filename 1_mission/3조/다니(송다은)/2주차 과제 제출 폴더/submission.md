# 2주차 — 내 OS 구현하기 🚀

> 미션을 진행하며 **기획 → 구현 → 삽질 → 결과물 → 인사이트** 를 상세히 기록해주세요.
> (다 못 채워도 OK, 한 것 위주로!)

## 🎯 미션 1. 내 OS 만들기
> **[ 내 삶을 돕는 OS ]** 또는 **[ 콘텐츠 OS ]** 중 하나를 선택해 완성해주세요.

**✅ 선택:** 콘텐츠 OS — **캐러셀(카드뉴스) 자동 제작 OS**

### 📐 기획
캐러셀 콘텐츠를 일관된 디자인·톤으로 빠르게 만드는 OS. 핵심은 **디자인 시스템**과 **콘텐츠 컨텍스트**를 각각 MD 파일로 정리해 Claude Code에 넣고, 그 위에서 캐러셀을 생성·배포하는 것.

### ⚙️ 구현
**1) 디자인 시스템 MD 만들기 — 3가지 방법**
- **기존 콘텐츠 분석**: 내가 만든 콘텐츠를 캡처 → Claude에게 분석시키고, SQL 퀘스천으로 애매한 부분을 보완 → MD 파일로 정리
- **레퍼런스 기반 분석**: Snippit(메타 광고·SNS 콘텐츠 아카이브, 셀피시클럽 무료 링크 제공) 또는 Behance에서 레퍼런스를 캡처·다운로드해 분석
- **겟디자인MD 활용**: BMW·에어비앤비·애플 등 대기업 디자인 시스템이 MD로 공개된 사이트(getdesignmd)에서 복사·가공

**2) OS 구축 & 배포**
- 디자인 시스템 MD + 콘텐츠 컨텍스트 MD **두 파일을 완성** → Claude Code 폴더에 넣어 OS 구축
- 배포: **Vercel**

**3) 심화**
- 캐릭터 생성: **Gemini API 키 연동 + 나노바나나2 모델** 활용
- **Supabase** DB 연동으로 **콘텐츠 보관함** 구축

### 🧗 과정에서의 삽질
> 막혔던 지점·해결 과정이 있으면 여기 추가해주세요

### ✅ 결과물
- 디자인 시스템 MD + 콘텐츠 컨텍스트 MD 기반 **캐러셀 OS** (Claude Code + Vercel 배포)
- 캐릭터 생성 기능 + Supabase 콘텐츠 보관함

### 💡 알게 된 인사이트 & 공유하고 싶은 내용
- 디자인 시스템·콘텐츠 컨텍스트를 **MD로 정리**해두면 OS의 재료로 재사용할 수 있다
- 레퍼런스는 **Snippit · Behance · getdesignmd**에서 확보 가능
- **API 키 보안**: `.env` + `.gitignore`로 외부 노출 방지, 주기적으로 재발급 권장
