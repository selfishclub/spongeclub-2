## 🎯 미션 1. 내 OS 만들기
> **[ 내 삶을 돕는 OS ]** 또는 **[ 콘텐츠 OS ]** 중 하나를 선택해 완성해주세요.

**✅ 선택:** 내 삶을 돕는 OS — **"적용까지 가는 AI 브리핑 OS"** (대시보드형)

### 📐 기획
> 무엇을, 왜, 어떻게 만들지

**문제**: 나(기획·마케터)는 빠르게 바뀌는 AI를 유튜브로만 따라잡았다. 검색하고 영상을 하나하나 보느라 시간을 다 쓰고, 정작 **내 업무 적용은 늘 겉핥기**로 끝났다.

**만들 것**: 정해둔 신뢰 소스에서 새 AI 툴 소식을 **영상 없이 요약**하고, 각 항목에 **우리 비즈니스 활용팁**을 붙여, **매일 여는 대시보드 페이지**에서 체크한다. 정보가 쌓이므로 **카테고리로 분류·정렬·삭제**까지 되는 DB형으로.

> 한 문장: **"영상 대신 요약으로 — 아낀 시간을 진짜 적용에 쓴다."**

### ⚙️ 구현
> 실제로 만든 것 (링크·스크린샷 — 이미지는 `이미지첨부/` 폴더에)

**① 엔진 — 클로드 코드 스킬 `ai-tool-briefing`**
- 신뢰 소스 3개(TLDR AI · The Rundown AI · GeekNews) 범위에서 **`WebSearch`로 최신 소식을 먼저 찾고** 본문만 `WebFetch`로 확인 → 요약 + 활용팁 생성.

**② 배달 — 로컬 HTML 대시보드 (DB형)**  `dashboard.html`
- **좌측 사이드바**: 대·중·소·세 **카테고리 트리 + 각 건수**. 클릭하면 해당 분류로 필터.
- **상단**: **최신 10개** 리스트 (전체 건수도 함께 표시).
- **정렬**: 최신순 / 오래된순 / 카테고리순.
- **각 항목**: 분류 경로 → 제목 → 요약 → **👉 우리 비즈니스 활용팁** → 출처 → **× 삭제**.
- **DB화**: 데이터·삭제·정렬 상태를 브라우저 **localStorage에 저장** → 새로고침해도 유지, "초기 데이터로 되돌리기"로 복구.

**③ 디자인 — Claude 디자인 시스템 적용**
- 저장소에 디자인 가이드 문서화(`docs/design/claude-design.md`) 후 대시보드에 적용: **크림 캔버스 · 코랄 활용팁 · EB Garamond 세리프 헤드라인 · 다크 네이비 푸터**.

### 🧗 과정에서의 삽질
> 막혔던 지점, 시도한 방법, 어떻게 풀었는지 솔직하게

- **배달 방식 전환**: 처음엔 텔레그램 온디맨드로 갔는데, "세션이 켜져 있어야만 동작"하는 한계가 걸렸다. → **매일 여는 대시보드 페이지**로 방향을 바꾸니 훨씬 내 습관에 맞았다.
- **소스 홈페이지 ≠ 뉴스**: `WebFetch`로 `tldr.tech/ai`를 열었더니 가입 페이지였다. → **`WebSearch`를 1차 취득 경로**로 바꿔 해결.
- **`file://`가 안 열림**: 만든 HTML을 브라우저 자동화로 확인하려니 로컬 파일 접근이 막혔다. → **`python -m http.server`로 로컬 서버**를 띄워 실제 렌더링·필터·정렬을 검증.
- **삭제 확인창**: 삭제(×)에 `confirm` 창을 달았더니 자동 브라우저 스크린샷이 잠깐 멈췄다(실사용에선 정상 — 실수 방지 장치).

### ✅ 결과물
> 완성한 것 / 작동 화면

**DB형 AI 브리핑 대시보드 완성.** 로컬 서버로 띄워 실제 동작 확인:
- 사이드바 카테고리 트리(전체 11 · 광고·마케팅 5 · 검색·GEO 2 …) + 건수 ✅
- 최신 10 리스트 + 정렬(카테고리순 재정렬 동작 확인) ✅
- 각 카드의 활용팁 + × 삭제 + localStorage 유지 ✅

- **작동 화면 스크린샷**: `이미지첨부/` 폴더 (대시보드 전체 뷰)
- **파일**: `.claude/skills/ai-tool-briefing/dashboard.html` (저장소는 `.claude/`를 gitignore → 로컬 전용)

### 💡 알게 된 인사이트 & 공유하고 싶은 내용
> 하면서 깨달은 것, 크루들과 나누고 싶은 것

- **"어디서 받느냐"가 습관을 만든다.** 텔레그램(알림)보다 **매일 여는 대시보드**가 내겐 더 맞았다. OS는 기능보다 "내 동선에 맞는 자리"가 중요하다는 걸 배움.
- **DB화의 힘**: 그냥 요약이 아니라 **쌓이고, 분류되고, 정렬·삭제되니** 정보가 '흐르지 않고 남는다'. 대·중·소·세 카테고리가 머릿속 정리까지 도와줌.
- 이 과정에서 **GEO(생성엔진 최적화)** 흐름을 처음 인지 — OS가 "내가 놓칠 뻔한 것"을 잡아준 순간.
- 다음 확장: 매일 아침 자동 갱신, 활용팁 → 실제 미니 실습 연결, 웹 배포.
