# 2주차 — 내 OS 구현하기 🚀

> 미션을 진행하며 **기획 → 구현 → 삽질 → 결과물 → 인사이트** 를 상세히 기록해주세요.
> (다 못 채워도 OK, 한 것 위주로!)

## 🎯 미션 1. 내 OS 만들기
> **[ 내 삶을 돕는 OS ]** 또는 **[ 콘텐츠 OS ]** 중 하나를 선택해 완성해주세요.

**✅ 선택:** **내 삶을 돕는 OS** — 매월 반복하던 업무 데이터 점검을 자동화한 브라우저 검증 앱 (1주차에 만든 것을 실무에 넘길 수 있는 수준으로 발전).

### 📐 기획
> 무엇을, 왜, 어떻게 만들지

1주차에 만든 건 **한 가지 업무(병원선정)**만 점검하는 앱이었습니다. 그런데 제가 매달 손보는 데이터는 **비슷하지만 다른 두 가지 업무**가 있었어요. 2주차 목표는 세 가지로 잡았습니다.

- **① 한 앱에서 두 업무를:** 두 번째 업무(전원)를 같은 앱에 얹어, **상단 토글 하나로 전환**하며 쓰게 만들기. 점검 규칙만 다르고 뼈대(엔진)는 같으니, 앱을 두 개 만드는 대신 **엔진은 공유하고 업무별 차이(시트·규칙)만 분리**하는 구조로.
- **② 나 말고 남도 쓰게:** 내 손을 떠나도 동료가 바로 쓸 수 있도록 **사용가이드**를 만들기. 코드를 못 봐도 "엑셀 올리기 → 결과 → 보고문"을 따라 하게.
- **③ 안전하게 넘기기:** 개인정보가 든 사내 도구라, **밖으로 새지 않게** + **함부로 뜯어고쳐 잘못 쓰지 않게** 하는 최소한의 방어선 두르기.

### ⚙️ 구현
> 실제로 만든 것 (링크·스크린샷 — 이미지는 `이미지첨부/` 폴더에)

- **한 앱, 두 모드:** 공통 엔진 + 업무별 설정만 분리하는 구조로 리팩터링해서, **토글 한 번으로 두 업무를 전환**하도록 완성. 새 업무 규칙은 목록에 **한 줄씩 추가**하면 늘어나게 설계(현재 두 업무 합쳐 100개 이상 점검 규칙).
- **사용가이드 제작:** 비개발자 동료용 슬라이드 가이드. **모드 선택 → 엑셀 올리기 → 결과 확인 → 보고문 만들기**를 STEP별로, "내 데이터는 안전한가요?" 안내까지. 화면 캡처의 **민감한 부분(시트 구조·접수번호 등)은 전부 블러 처리**해서, 가이드 자체로는 업무 구조가 새지 않게 했습니다.
- **실무 이관용 다듬기:** 저장되는 보고문 파일명을 **모드별로 자동 구분**, 화면·코드에 **"내부용·무단 수정/재배포 금지"** 안내 문구, 그리고 **누군가 앱을 뜯어고치면 "변조된 사본" 경고가 뜨게** 하는 감지 배너까지 넣었습니다.
- **동작 방식은 그대로:** 실제 데이터는 여전히 **브라우저 안에서만** 처리, 필요한 기능은 파일에 내장해 **인터넷 없이(오프라인)** 돕니다.

> ⚠️ 이 앱은 **개인정보가 포함된 사내 업무 도구**라, 컬럼·검증 규칙 구조가 외부에 드러나면 안 됩니다. 그래서 코드·구조는 공개하지 않고, **민감 부분을 가린(블러) 스크린샷·시연으로 갈음 인증**을 요청드립니다. (개발·테스트는 가짜 샘플로만, 실제 데이터는 회사 PC에서만 처리)
>
> _(가이드의 블러 처리 스크린샷을 `이미지첨부/` 폴더에 넣어 첨부 예정 — 채은 님 확인 후 추가)_

### 🧗 과정에서의 삽질
> 막혔던 지점, 시도한 방법, 어떻게 풀었는지 솔직하게

- **"남이 못 뜯어보게 완전히 잠글 수 있을까?"** — 이걸 붙잡고 한참 고민했습니다. 결론은 **브라우저에서 도는 앱은 원리상 완전 차단이 불가능**하다는 것. 실행하려면 읽을 수 있는 형태로 전달돼야 하니까요.
  - **난독화**는 사람은 헷갈려도 AI는 꽤 잘 복원하고, 무엇보다 **제가 나중에 유지보수를 못 하게** 됩니다.
  - **exe로 감싸기**도 결국 쉽게 풀려서 "자물쇠"가 아니라 "속도방지턱"일 뿐이었어요.
  - 그래서 방향을 틀어, **완벽 차단(불가능)** 대신 **현실적인 방어선**을 택했습니다 — ① 경고 문구로 인식·책임 소재를 분명히 하고, ② 파일을 아무에게나 주지 않는 배포 통제, ③ 손대면 티가 나는 **변조 감지 배너**. (정작 제일 중요한 개인정보 유출은 "외부 전송 0" 구조라 이미 안전했고요.)
- **두 업무를 한 앱에 합칠 때** 규칙이 뒤섞이지 않게, "엔진은 공통 / 업무별 차이는 설정 한곳"으로 경계를 나눈 게 핵심이었습니다.

### ✅ 결과물
> 완성한 것 / 작동 화면

- **토글 하나로 두 업무를 전환하는 단일 HTML 검증 앱** (설치 불필요·오프라인).
- **동료용 사용가이드** (STEP별, 민감 부분 블러).
- **실무 이관 준비 완료** — 파일명 자동 구분·경고 문구·변조 감지까지.
- 가짜 샘플 데이터로 자동 테스트를 돌려 **오탐 없이 통과** 확인.

### 💡 알게 된 인사이트 & 공유하고 싶은 내용
> 하면서 깨달은 것, 크루들과 나누고 싶은 것

- **"완벽히 막기"보다 "현실적으로 안전하게".** 코드를 100% 잠그려던 집착을 내려놓고, *무엇이 진짜 위험한가(개인정보 유출)* 를 기준으로 방어선을 다시 그으니 답이 나왔습니다. 보안은 이상이 아니라 **트레이드오프를 아는 것**이더라고요.
- **내 손을 떠날 걸 생각하면 만드는 게 달라진다.** "나만 쓰는 도구"에서 "동료도 쓰는 도구"로 목표가 바뀌자, 가이드·경고 문구·오작동 방지 같은 **내가 없을 때를 위한 장치**가 진짜 기능이 됐습니다.
- **하나 잘 만들면 둘째는 얹으면 된다.** 엔진을 공유하는 구조로 짜두니, 두 번째 업무는 규칙만 더해서 붙일 수 있었어요. OS는 한 번 만들고 끝이 아니라 **계속 얹어 키우는 뼈대**라는 걸 체감했습니다.
