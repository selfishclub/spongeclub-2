# 2주차 — 내 OS 구현하기 🚀

## 🎯 미션 1. 내 OS 만들기

**✅ 선택:** 콘텐츠 OS

### 📐 기획
1주차에 만든 `edufi-review`(홈페이지 후기 자동화)에서 한 걸음 더 나갔다.
내가 후기를 올리는 채널은 홈페이지 말고도 **인스타그램·네이버 블로그**가 있는데, 채널마다 규격도 글투도 달라서 매번 새로 만드는 게 일이었다.

그래서 이번엔 **"교육 하나 → 채널별 콘텐츠(캐러셀 이미지 + 피드 글 + 블로그 글)로 한 번에 뽑아주는 OS"** 를 목표로 잡았다.
벤치마킹은 스폰지타임즈 캐러셀 생성기(spongetimes-mvp-carousel.vercel.app)로, "왼쪽 미리보기 + 오른쪽 편집 + 이미지 저장" 구조를 그대로 참고했다.
핵심 원칙은 1주차와 같다 — **매번 같은 부분(브랜드·틀·문의처)은 고정, 매번 다른 부분(교육명·후기·사진)만 입력.**

### ⚙️ 구현
**Next.js로 만들어 Vercel에 배포**했다. 이제 폰·PC 어디서든 링크로 접속해서 쓴다.

- 🔗 **라이브: https://edufi-carousel.vercel.app**

만든 기능:
- **후기 캐러셀 편집기** — 표지·교육개요·강사소개·수강생후기·교육현장사진·한줄평·문의 7가지 슬라이드, 왼쪽 실시간 미리보기 + 오른쪽 편집
- **사이즈 2종** — 인스타 피드 4:5(1080×1350, 프로필 그리드에서 안 잘림) / 정사각형 1:1(홈페이지용)
- **얼굴 자동 모자이크(초상권)** — 사진 올리면 AI가 얼굴 인식해 자동 모자이크 + 자동이 놓친 얼굴은 마우스로 직접 가리는 **수동 모자이크**
- **인스타 피드 글 자동 생성** — 인사말·문의처 고정 + 본문 자동, 해시태그 인스타 정책(5개)에 맞춰 배치
- **네이버 블로그 글 자동 생성** — 우리 블로그 실제 글 형식(진단도구 설명 도입 → 소제목 → 참가자 반응 → 링크·유선문의) 그대로, SEO 제목 3후보
- **PNG/전체 ZIP 저장, 백업 내보내기·가져오기, EduFi 로고 자동 삽입**

> 💡 위 라이브 링크로 들어가면 실제 편집 화면과 작동을 바로 볼 수 있어요. (스크린샷을 넣고 싶으면 화면 캡처 후 `이미지첨부/` 폴더에 넣고 이 자리에 `![편집 화면](이미지첨부/파일명.png)` 형식으로 걸면 됩니다.)

### 🧗 과정에서의 삽질
- **벤치마킹 사이트 분석**: 자바스크립트로 그려지는 앱이라 주소만 봐선 내용이 안 보여서, 브라우저로 직접 열어 화면을 뜯어보며 구조를 파악했다.
- **얼굴 인식이 자꾸 놓침**: 처음 쓴 모델(blazeface)이 정면만 잘 잡고 여러 명·측면 얼굴을 놓쳤다. 워크숍 단체 사진에서 얼굴을 못 잡길래 **더 정확한 모델(SSD MobileNet)로 교체**하고 모델 파일을 앱에 내장했다. (같은 사진에서 1명 → 3명으로 인식 개선) 그래도 완전히 등진 얼굴은 못 잡아서 **수동 모자이크**를 따로 붙였다.
- **로고가 깨져 보임**: 로고를 텍스트+아이콘으로 흉내 냈더니 'i'와 와이파이가 분리돼 보였다. **실제 로고 이미지를 검정/흰색 두 버전으로 앱에 내장**해서 배경에 맞게 자동으로 쓰도록 고쳤다.
- **사이즈 두 개 대응**: 정사각형만 생각하고 만들었는데 인스타 그리드는 4:5라 잘렸다. 슬라이드 레이아웃을 **높이에 따라 자동 정렬되는 구조로 다시 짜서** 두 비율 모두 균형 있게 나오게 했다.
- **블로그 글투 맞추기**: 예전 후기 예시로 만들었더니 실제 최신 글과 인사말·마무리·구성이 달랐다. 실제 발행 글(DISC 교사 연수)을 기준으로 다시 맞췄다.

### ✅ 결과물
- 배포된 OS: **https://edufi-carousel.vercel.app** (지금 바로 들어가 후기 캐러셀 + 피드 글 + 블로그 글을 만들 수 있음)
- 매번 반복하던 "채널별로 이미지 만들고 → 얼굴 가리고 → 글 새로 쓰고" 과정을 **한 화면에서** 끝낼 수 있게 됐다.

### 💡 알게 된 인사이트 & 공유하고 싶은 내용
- OS를 만드는 건 결국 **"고정(틀)과 변수(내용)를 갈라내는 일"** 이라는 게 이번에 더 확실해졌다. 이 원칙만 잡으면 채널이 늘어도 **틀(슬라이드 타입·글 포맷)만 추가**하면 된다.
- **배포(Vercel)** 를 해보니 "내 컴퓨터에서만 도는 도구"가 "어디서든 쓰는 서비스"가 됐다. 비개발자도 클로드코드에 시키면 명령 몇 줄로 인터넷에 올릴 수 있다는 게 가장 큰 수확이었다.
- 초상권처럼 **"자동이 100%가 아닌 부분"은 수동 보정을 같이 붙여두는 게** 실무에서 안심하고 쓰는 방법이라는 것도 배웠다.
