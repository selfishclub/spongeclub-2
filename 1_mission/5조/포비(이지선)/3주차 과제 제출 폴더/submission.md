# 3주차 — 내 OS 최종 완성 🏁

## 🎯 미션 1. 내 삶을 돕는 OS 최종 완성

- **완성한 것 (무엇을): 에듀파이 상세페이지 OS**

  강사양성과정을 새로 론칭하거나 전자책·교구재를 팔 때마다 홈페이지(edufi.kr)에 올릴 **상세페이지 이미지를 매번 손으로 만들던 일**을 OS로 만들었다.

  먼저 운영 중인 상세페이지 5개를 전부 뜯어서 분석했더니, 이미 내가 감으로 "860×1100 카드를 세로로 쌓는 템플릿"을 쓰고 있었다. 여기에 동종 업계(패스트캠퍼스·클래스101·경쟁사) 상세페이지 트렌드 조사를 더해 **표준 카드 16종**(히어로 → 숫자 증명 → 후기 → 추천 대상 → 수강 전후 비교 → 핵심 포인트 → 자료 목업 → 혜택 → 커리큘럼 → 강사 소개 → 모집 일정 → 장소 → FAQ → 신청 방법 → 문의·환불)을 정의하고, 2주차 캐러셀 OS와 같은 구조(왼쪽 입력 폼 + 오른쪽 실시간 미리보기 + PNG 저장)로 구현했다.

  핵심 기능:
  - **입력하면 완성**: 과정명·후기·커리큘럼만 입력하면 카드 16장 + 목록용 썸네일(750×750)까지 자동 생성, ZIP 한 번에 저장 → 아임웹에 순서대로 업로드하면 끝
  - **고정 블록 재사용**: 강사 소개·교육 장소·신청 방법·환불 규정은 한 번 입력하면 모든 과정에서 공통 사용 (입력량 절반으로 감소)
  - **테마 8종 × 디자인 스킨 2종(클래식/오로라 글로우) × 썸네일 2종(강사양성형/판매상품형)**: 과정 성격에 맞춰 클릭 한 번으로 전체 분위기 전환
  - **무료 상업용 폰트 8종**: 프리텐다드·지마켓 산스·페이퍼로지 등을 제목/본문 따로 선택
  - **과정별 저장 슬롯**: 이전 과정을 불러와 제목·일정만 고쳐서 새 과정 제작

- **피드백 반영한 점 (직접 써보며 보완한 것):**
  - "장표마다 여백이 들쭉날쭉해서 배치를 만지고 싶다" → **카드 안 요소(제목·사진·뱃지 등)를 클릭·드래그로 옮기고 크기 조절**하는 요소 단위 배치 기능 추가
  - "내용이 비슷한 과정은 이전 작업을 그대로 불러오고 싶다" → **과정별 저장 슬롯 + JSON 파일 백업/복원** 추가
  - "기존 강사과정 색상만으로는 부족하다" → 테마 3종(네이비-골드·포레스트 그린·선셋 코랄)과 **트렌디한 오로라 글로우 스킨**, 판매상품형 썸네일 추가

- **결과물:**
  - 🔗 **라이브: https://edufi-detail.vercel.app** — 샘플 과정이 들어 있어 "PNG 전체 저장" 버튼만 눌러도 완성 이미지 17장을 받을 수 있다.
  - 📦 **깃허브: https://github.com/jisun5/edufi-detail** (비공개)
  - 이전에는 상세페이지 하나에 며칠 걸리던 디자인 작업이, 이제 **내용 입력 → 배치 조정 → ZIP 저장**으로 끝난다.

- **알게 된 인사이트:**
  - 반복 작업을 OS로 만들 때 제일 먼저 할 일은 기능 상상이 아니라 **내가 이미 하고 있던 것의 패턴 발견**이었다. 분석해보니 내 상세페이지는 이미 템플릿이었고, OS는 그걸 코드로 옮긴 것뿐이었다.
  - "매번 같은 것(고정 블록)"과 "매번 다른 것(과정 내용)"을 나누는 원칙이 1주차부터 3주차까지 모든 OS를 관통했다. 이 구분만 잘하면 어떤 반복 업무든 OS가 될 수 있다.
