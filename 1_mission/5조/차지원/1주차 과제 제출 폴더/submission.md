# 1주차 — 나만의 OS 만들기 🛠️

> 미션을 진행하며 **과정과 결과물**을 기록해주세요. (다 못 채워도 OK, 한 것 위주로!)

## 🎯 미션 1. 내 OS 재료 찾기
> 인터뷰 스킬(아이데이션)로 "내 삶에 필요한 게 뭔지" 찾기

- **과정 (어떻게 찾았나):**
  - "내 삶을 돕는 OS 인터뷰" 스킬로 하루 중 시간을 가장 많이 잡아먹는 일을 짚었고, **"똑같은 작업 반복"**에서 출발.
  - 매일 구글시트↔엑셀을 오가며 **수도 없이 복붙**하던 장기렌트/리스 DB 납품 업무가 가장 큰 마찰점으로 드러남.

- **결과 — 내 OS 재료 카드:**

  ```
  🟡 내 삶을 돕는 OS · 재료 카드

  · 영역      : 일 (장기렌트/리스 DB 납품 업무)
  · 걸리는 지점 : 디비 수집~분배 전 과정을 구글시트↔엑셀 오가며 손으로 복붙
  · 지금은    : ①수집 ②취합 ③정렬 ④중복검사(광고주별 이력 대조)
                ⑤중복없는 디비 분배 ⑥유입일시 보정 ⑦납품 ⑧이력갱신 — 전부 수작업
  · OS가 된다면 : 수집된 디비를 차수에 맞춰 자동으로 취합·중복검사·분배·시간보정 →
                광고주별 납품 엑셀이 지정 드라이브 폴더에 자동 생성
  · 한 문장   : "복붙 지옥 없이, 폴더만 열면 납품 파일이 준비되어 있다"
  · 첫 한 걸음 : 가장 안전한 토대 = '취합 + 차시내 중복제거'부터 앱스크립트로
  ```

- **느낀 점:**
  - "딱히 만들 게 없다"가 아니라, **매일 반복하는 복붙 자체가 가장 좋은 OS 재료**였다.
  - 처음엔 "기능 하나"를 떠올렸는데, 인터뷰를 통해 **하루 업무 전체가 통과하는 길(파이프라인)**로 그림이 커졌다.

## 🧩 미션 2. 내 OS 기획
> 인터뷰 결과 + 세션 내용 활용해 기획

- **기획 내용 — 기능 명세서:**

### 1. 시스템 한 줄 정의
> 수집된 DB를 **차수(새벽·실시간1·2·3차)에 맞춰 자동 취합 → 차시내중복제거 → 중복검사 → 우선순위 분배 → 타입별 파일분리 → 시간보정 → xlsx 생성 → 이력·정산 기록**까지 무인 처리. 사람은 충전·할당·우선순위만 수기 입력.

### 2. 시트 2개 + openById 연동
| 시트 | 역할 |
|---|---|
| **메인** (장기렌트 DB 유입 및 납품) | 납품 엔진 (Google Apps Script 탑재) |
| **외부** (수량 관리) | 타입별 충전·잔여 = 정산 두뇌 (수기, openById로 읽기/쓰기) |

### 3. 탭 구성 (메인 시트)
| 그룹 | 탭 | 핵심 |
|---|---|---|
| 🟢 수집 | 바이럴·메타·틱톡 | A~D 공통(유입일시·이름·전화·차종), **납품여부 미체크 = 납품 대상** (체크 열 위치만 탭별 매핑) |
| 🔴 검사 | 블랙리스트 / 분배 우선순위 / 중복검사 / 광고주 중복기준탭 | 제외명단 / 분배순서 / 작업허브 / 납품이력(날짜=B, 번호=D) |
| 🟠 분배·출력 | 렌트 납품수량 / [광고주]_차지원 | 목표·차수기록·계약시간대 / 최종 출력(E="네" 고정) |

### 4. 처리 파이프라인 (각 차수마다 실행)
1. **취합** — 수집탭 미체크 DB의 A:D → 중복검사 시트. **유입일시 오름차순(FIFO)**.
2. **차시내 중복제거** — 같은 번호 2건↑ → 최신 1건만 남기고 삭제. 차시 내 **1 DB = 1 광고주 소진**.
3. **중복검사** (시트 수식 계산, 스크립트는 읽기만)
   ```
   S    : =COUNTIFS(S!$D:$D,$C2, S!$B:$B, ">="&(TODAY()-365))   // 1년
   그 외 : =COUNTIFS(렌트런!$D:$D,$C2, 렌트런!$B:$B, ">="&(TODAY()-30)) // 1개월
   ```
   \+ 블랙리스트 번호 제외 → **카운트 0 = 그 광고주에 납품 가능**.
4. **우선순위 분배** (광고주-바깥루프, 1 DB 1 광고주 소진)
   ```
   for 광고주 in 분배우선순위(S→슈퍼맨→렌트런→굿차→리렌트레이드):
       오래된 DB부터 → 그 광고주 비중복 & 미소진 건을 차수 목표량까지 배정
   목표 = 새벽:D열 / 실시간:E열 (+ 새벽 부족분 이월)
   ```
5. **타입 분배·파일분리** — 외부 활성블록(할당일 최신) 잔여 기준, 차감순서 `정규 → 정규AS → 특별AS(슈퍼맨만) → 추가AS`로 쪼개 **타입별 물리 분리 파일** 생성.
6. **시간보정**

   | 차수 | 날짜 | 시간(랜덤, 초 유지) |
   |---|---|---|
   | 새벽 | D-1 (아니면 D-1로 변경) | 18:00–23:59 |
   | 실시간1차 | D-Day (D-1 잔여면 변경) | 06:00–10:45 |
   | 실시간2차 | D-Day | 11:00–14:45 |
   | 실시간3차 | D-Day | 15:00–16:00 |
   (시간대 값은 `렌트 납품수량` 탭 19~23행에서 읽음)
7. **출력**
   ```
   …/납품/{YYYYMMDD}/{차수}/{MMDD}_{광고주}{(타입)}_차지원_{건수}건[_실시간n차].xlsx
   타입라벨: 정규=없음 / (정규AS) / (특별AS) / (추가AS)
   ```
8. **기록 (무인)**
   - 메인: 광고주 중복기준탭 append + 수집탭 납품여부 체크 + 렌트납품수량 G/H/I 차수기록·J 잔여
   - 외부: 활성블록 날짜로그에 새 행 + **타입 열에 숫자만** 기입

### 5. 스케줄 (시간 트리거)
| 새벽 00:00 | 실시간1차 10:50 | 실시간2차 14:50 | 실시간3차 16:10 |
|---|---|---|---|
| 새벽수량(D) | 실시간 + 새벽 부족분 | 부족분 | 부족분, 못 채우면 그대로 마감 |

### 6. 가변성 원칙 (하드코딩 금지)
- 광고주 명단 · 우선순위 · 기준기간 · 시간대 · 타입을 전부 **시트 데이터로 분리**.
- → 광고주 들고남(예: 고그로스 퇴장), 특별AS(슈퍼맨), 우선순위 변경을 **코드 수정 없이** 대응.

### 7. 구현 직전 필요한 것
- 외부 시트의 **정확한 셀 좌표**(블록별 잔여·타입열·로그영역) + **두 시트 ID**

- **막혔던 점 / 어떻게 풀었나:**
  - **실행시간 제한 우려** → 중복검사를 앱스크립트 연산이 아니라 **시트의 `COUNTIFS` 수식**이 계산하게 해서 회피. ("시트=계산엔진, 스크립트=지휘자")
  - **광고주가 수시로 들고남** → 명단을 코드에 박지 않고 **시트에서 동적 로딩**하도록 설계.
  - **타입별 정산이 다른 시트에 있음** → 합치지 않고 **openById로 잔여를 읽고 실제 납품수를 회신**하는 연동 구조로 분리.
  - **로컬 폴더에 못 떨굼** → 앱스크립트 특성상 **구글 드라이브 폴더**로 출력하는 것으로 끝선 확정.

## ⚙️ 미션 3. 내 OS 구현
> 실제로 만들어본 것 (클로드코드 '채널' 기능 활용 OK)

- **결과물:** Apps Script로 1단계·3a까지 구현 (실시간 테스트 진행)
  - **1단계** `1_취합_중복제거.gs` — 미체크 DB 취합 → 전화번호 정규화/검증 → 블랙리스트 제외 → FIFO 정렬 → 차시내 중복제거 → `중복검사` 기록 (무효/블랙은 소스에서 체크+표시)
  - **2단계** 중복검사 COUNTIFS는 시트 수식이 계산(스크립트는 읽기만)
  - **3a** `2_분배.gs` — 차수별 **선입선출+균등 분배** → `광고주_차지원` 기록 + 유입일시 보정(가중) + 소스 체크(무효/블랙/중복/납품불가) + 실적 기록
  - **3c** — 분배 결과를 **광고주별 xlsx로 구글 드라이브에 자동 저장**(`…/YYYYMMDD/차수/`), 본시트 서식 그대로(copyTo)
  - **3b** — 외부 정산시트 잔여 읽어 **타입 분할(정규→정규AS→특별AS→추가AS) + 파일 분리 + 외부 로그 회신**
  - **3d** — 납품분을 광고주 이력 탭에 append → COUNTIFS **재납품 차단**
  - **트리거** — 새벽 00:00 / 실시간 10:50·14:50·16:10 자동 실행 → **무인 파이프라인 완성**
  - 코드/설명: 코드는 별도 프로젝트 `cpa-automation`으로 이전됨 (`cpa-automation/코드/` — `1_취합_중복제거.gs` · `2_분배.gs` · `README.md`)
