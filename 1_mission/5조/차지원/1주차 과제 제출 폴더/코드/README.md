# 장기렌트 DB 납품 자동화 OS — 코드

Google Apps Script(컨테이너 바인딩)로 만든 납품 엔진. **메인 시트(장기렌트 DB 유입 및 납품)** 에 붙여서 동작한다.

## 단계별 진행
| 단계 | 파일 | 내용 | 상태 |
|---|---|---|---|
| 1 | `1_취합_중복제거.gs` | 취합 → 정규화/검증 → 블랙리스트 제외 → FIFO 정렬 → 차시내 중복제거 → 중복검사 기록 | ✅ |
| 2 | (시트 수식) | 중복검사 COUNTIFS(S=1년·나머지=30일) — 시트가 계산, 스크립트는 읽기만 | ✅ |
| 3a | `2_분배.gs` | 차수별 우선순위 분배(1DB1광고주·FIFO) → 광고주_차지원 기록 + 시간보정 + 소스 체크 + 실적 기록 | ✅ |
| 3c | `2_분배.gs` | 분배 결과를 광고주별 xlsx로 드라이브 저장(…/YYYYMMDD/차수/) | ✅ |
| 3b | `2_분배.gs` | 외부 정산시트 잔여 읽기 → 타입 분할(정규→정규AS→특별AS→추가AS) → 파일 타입별 분리 + 외부 로그 회신 | ✅ |
| 3d | `2_분배.gs` | 납품분을 광고주 이력 탭(S·렌트런…)에 append → 중복검사 COUNTIFS가 당일/이후 재납품 차단 | ✅ |
| 마무리 | `2_분배.gs` | 차수별 시간 트리거(새벽 00:00 / 실시간 10:50·14:50·16:10) 자동 실행 | ✅ |

> **두 파일은 같은 Apps Script 프로젝트**에 둬야 함 (`CONFIG`·`_취합DB` 등 공유). 파일1을 새 버전으로 교체하고, 파일2를 새 파일로 추가.

## 1단계 설치 & 실행
1. 메인 시트 열기 → **[확장 프로그램] → [Apps Script]**
2. `1_취합_중복제거.gs` 내용을 붙여넣고 **저장** (프로젝트 이름은 자유)
3. 시트로 돌아와 **새로고침** → 상단에 **`🛠 납품자동화`** 메뉴 생성
4. **[1단계: 취합 + 중복제거]** 클릭 (최초 1회 권한 승인)
5. `중복검사` 탭 A:D에 결과가 채워지고, 우측 하단에 `취합 N건 → M건` 토스트 표시

## 3a 분배 실행 (`2_분배.gs`)
메뉴 `🛠 납품자동화` → **새벽 납품 / 실시간 1·2·3차** 중 하나 클릭.
- 각 차수: 취합 → 중복검사 → **선입선출(FIFO)+균등 분배** → `광고주_차지원` 탭에 기록(E="네") + 유입일시 보정 + 소스 체크 + 실적 기록
  - **DB를 오래된 순으로 하나씩** 처리(구멍 없음), 각 DB는 받을 수 있는(적격·목표미달) 광고주 중 **가장 적게 받은 쪽**에 배정(동률=우선순위)
  - 1DB1광고주 · COUNTIFS 0만 · 차수 목표까지
  - **남은 물량**(받을 곳은 있으나 목표 참) → 미체크로 다음 차수 대기
  - **납품 불가**(전 광고주에 이미 납품돼 COUNTIFS 전부 ≠0) → 체크 + 옆칸에 `납품 불가` 표시
- **차시내 중복**: 같은 번호 여러 건 → 최신 1건만 사용, **버려진 행은 체크 + 옆칸에 `중복` 표시**
- **실적열(D·G·H·I) 리셋 = '그날 첫 실행'에 자동** (DocumentProperties `운영일` 기준). 새벽을 건너뛰고 실시간1차부터 돌려도 됨 → 그날 첫 실행이면 D=0이라 **일일 전체(B+C)가 실시간 목표**가 됨
- **새벽 목표** = `floor(일일납품/2)` (단 일일납품 ≤ 3 이면 전량) → `D열`에 실적 기록
- **실시간 목표** = 현재 `J(잔여)` → `G·H·I`에 차수별 실적 기록
- **차지원 탭은 매 차수 시작 때 비움** (이번 차수 배치만 담음 → 3c에서 xlsx로 내보낸 뒤 다음 차수가 덮어씀)
- **시간보정(가중)**: 날짜는 새벽=D-1 / 실시간=D-Day. 시·분은 유입일시 순서로 균등 슬롯 배치(오래된=이른 시간, 슬롯 안 랜덤)
  - 실시간은 **'어제자 이전' DB를 `[시작, 시작+클램프분(기본 60분)]`로 제한**(1차=06:00~07:00) → 어제 DB가 10시대로 안 찍힘. 당일 DB는 전체 시간대. (`DIST.차수[*].클램프분`으로 조절)

### 3c 드라이브 파일 저장 (`2_분배.gs`)
- `DIST.드라이브_루트폴더ID`에 저장할 드라이브 폴더 ID를 넣으면(비우면 파일 생성 안 함), 차수마다 `…/{루트}/{YYYYMMDD}/{차수}/` 폴더를 만들어 광고주별 xlsx 저장
- 파일명: `{MMDD}_{광고주}_차지원_{건수}건[_실시간n차].xlsx` (날짜=납품일=D-Day)
- 내용 = 그 차수 배치. **차지원 탭을 통째로 복사(copyTo)**해서 헤더색·열너비·서식 그대로 유지 → xlsx export → 폴더 저장 → 임시 삭제 (데이터 아래 빈 행·열은 정리)
- 최초 실행 시 **드라이브 접근 권한** 승인 필요
- ⏳ 아직 **타입 분리 前** → 광고주당 1파일. 3b에서 정규/AS로 분리 예정

### 3b 타입 분리 + 외부 정산 연동 (`2_분배.gs`)
- `DIST.외부시트ID`의 수량 관리 시트를 `openById`로 열어, 광고주 탭의 **맨 오른쪽(활성) 할당 블록** 잔여를 타입별로 읽음
- 배정 건수를 **차감순서 정규→정규AS→특별AS(슈퍼맨만)→추가AS**로 분할 → **타입별 파일 분리**
  - `0701_S_차지원_5건_실시간1차.xlsx`(정규) + `0701_S(정규AS)_차지원_4건_실시간1차.xlsx` …
- 납품 후 외부 시트 **활성 블록 날짜 로그에 회신**(날짜열 첫 빈 행 = `YYYY-MM-DD[실시간N]` + 타입 열에 수량). 잔여는 시트 수식이 자동 차감
- 외부 잔여보다 많이 배정되면 **초과분은 정규로 처리 + 경고 로그**(잔여 관리로 예방)
- ⚠️ 최초 실행 시 **외부 시트 접근 권한** 승인 한 번 더 필요(openById)
- `외부시트ID`를 비우면 3b를 건너뛰고 광고주별 1파일(3c)로 동작

### 3d 광고주 이력 append (`2_분배.gs`)
- 납품분을 광고주 이력 탭(`S`·`렌트런`…, 중복검사 COUNTIFS 참조처)에 append: `A 납품시간대(예: 2026-07-01 실시간1차) / B 유입일시(보정값) / C 이름 / D 전화번호(텍스트) / E 차종`
- → 다음 차수·다음 날 COUNTIFS가 당일 납품분을 반영해 **같은 번호 재납품 차단**
- ⚠️ A열 라벨 형식은 `YYYY-MM-DD 차수`로 가정(기능엔 영향 없음, `DIST.차수[*].이력라벨`에서 조정). 기존 이력이 다른 형식이어도 무관 — COUNTIFS는 B(날짜)·D(번호)만 사용

## 자동 트리거 (마무리)
메뉴 `🛠 납품자동화` → **⏰ 자동 트리거 설치** 클릭 → 매일 자동 실행:
- 새벽 00:00 / 실시간1차 10:40 / 실시간2차 14:40 / 실시간3차 16:00 (실시간은 ±15분 오차 대비 10분 앞당김)
- 제거하려면 **⏰ 자동 트리거 제거**
- ⚠️ **주의 2가지**:
  1. 구글 시간 트리거는 지정 시각 **±15분 창**에서 실행됨(정확한 분 보장 X). 납품 파일의 유입시간은 시간대 규칙이 따로 잡으니 기능엔 무방
  2. 시각 기준 = **Apps Script 프로젝트 표준시**. 프로젝트 설정에서 **Asia/Seoul**인지 확인 (아니면 00:00이 한국 자정이 아님)

## 설정 (변경 시 `1_취합_중복제거.gs` 상단 `CONFIG` / `2_분배.gs` 상단 `DIST` 수정)
- **수집 탭 추가/삭제**: `수집탭`에 `'탭이름': 납품여부열번호` 추가/삭제
  - 현재: 바이럴 디비=G(7) / 성국광=G(7) / 최광철=H(8) / 틱톡=F(6)
- 모든 수집 탭은 **A=유입일시, B=이름, C=전화번호, D=희망차종** 공통 전제

## 동작 규칙
- **미체크(`납품여부` 체크박스 OFF)** = 납품 대상
- **전화번호 정규화**: 하이픈·공백 제거, `+82`/`82`/`0082` → `0` 복원
- **형식 검증**: `010/011/016/017/018/019` + 7~8자리만 통과, 지역번호·자릿수 오류는 제외
- **블랙리스트 제외**: 매 실행 시 `블랙리스트` 탭의 전화번호(B열)를 읽어 Set으로 만들고, 일치하는 DB는 제외 (열/수식 추가 없이 스크립트가 처리)
- **FIFO**: 유입일시 오름차순(오래된 것 먼저)
- **차시내 중복제거**: 같은 전화번호 2건↑ → 가장 최신 1건만 남김
- 정규화된 번호로 기록(C열은 텍스트 고정 → 앞자리 0 보존)
- **제외 행 처리**: 무효번호/블랙리스트로 걸린 행은 수집 탭에서 **납품여부 체크 + 옆칸(체크박스 오른쪽)에 사유 표시**(`제외됨(무효번호)` / `제외됨(블랙리스트)`) → 다음 실행 때 다시 안 잡힘
- 유효 DB는 아직 납품 전이라 **체크하지 않고 그대로 둠**(체크/이력 갱신은 납품 단계)
- 제외된 무효 번호 건수/샘플은 토스트 + Apps Script 로그(보기 > 실행 로그)에서 확인

## 주의
- **'결번(실제 끊긴 번호)'은 코드로 판별 불가** — 통신사 조회/외부 API 필요. 여기선 형식 검증까지만.
- `중복검사` 탭의 E:K(일일중복·광고주 COUNTIFS) 수식은 1단계가 건드리지 않는다. 데이터 행만큼 수식이 채워져 있어야 정상 계산됨(2단계에서 정비 예정).
- **2단계 매칭 주의**: 중복검사 COUNTIFS가 광고주 이력 탭(과거 번호)과 정확히 매칭되려면, 과거 탭 번호도 같은 정규화 형식이어야 함. 형식이 다르면 1회 정규화가 필요(2단계에서 다룸).
