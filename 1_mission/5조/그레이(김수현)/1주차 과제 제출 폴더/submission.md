# 1주차 — 나만의 OS 만들기 🛠️

> 미션을 진행하며 **과정과 결과물**을 기록해주세요. (다 못 채워도 OK, 한 것 위주로!)

## 🎯 미션 1. 내 OS 재료 찾기
> 인터뷰 스킬(아이데이션)로 "내 삶에 필요한 게 뭔지" 찾기

- **과정 (어떻게 찾았나):**
  Claude와 두 차례 인터뷰 → ChatGPT와 추가 대화를 통해 결론을 다듬었다.
  업무 패턴(콘텐츠 제작·검수 반복)과 일상 패턴(자료 저장만 하고 실행 안 함, 시작해도 흐름이 끊기면 재개 안 됨)을 함께 들여다봤다.

- **결과:**

  **반복해서 겪는 문제**
  업무에서는 작업지시서를 받아 콘텐츠를 만들고, 모든 산출물을 검수해 피드백을 주고, 판매 페이지를 세팅하는 과정이 무한 반복된다. 과거 상세페이지 일부 이미지를 자동화 시켜볼까 생각해보았지만 판매 상품이 매번 다르고 생성된 이미지가 기준에 못 미쳐 아직은 완전한 자동화가 어렵다고 생각한다.

  일상에서는 AI 정보나 해보고 싶은 것들을 카카오톡·사진첩·인스타그램 저장함·부계정 DM 등 여러 곳에 저장해 두지만, 저장만 하고 실행하지 않는 습관이 반복된다. [1언젠가 해야지 -> 2저장] 이 단계가 어느샌가 [1저장해야지 끝]으로 실행 단계가 사라졌다. 이 패턴은 맛집·책·배우고 싶은 것·자료 수집 전반에 걸쳐 나타난다.

  **핵심 문제**
  자료가 흩어진 것보다 **저장만 하고 행동으로 이어지지 않는 것**이 진짜 문제다. 더 정확하게는, 시작을 하더라도 외부 조건에 걸리면 흐름이 끊기고 재개가 안 된다. 어쩌다 AI에 흥미가 생겨 공부를 자발적으로 시작했지만 토큰이 떨어지는 상황이 반복되면서 흐름이 끊겼고, 외부 환경의 영향을 받아 관심 순위가 밀렸다. 시작은 하는데 지속이 안 되는 패턴이 반복된다.

  **내가 만들고 싶은 OS**
  단순히 자료를 한곳에 모으는 시스템이 아니라, 저장한 정보를 적절한 시점에 다시 꺼내주고 작은 행동으로 이어질 수 있게 도와주는 시스템. 흐름이 끊겼을 때 "여기까지 했었어요, 이번 주에 이거 하나만 해볼까요?" 하고 다시 붙잡아 주는 것. 업무와 일상을 따로 관리하는 게 아니라 **저장 → 실행 → 재개**라는 같은 패턴을 함께 개선하는 것이 핵심이다.

- **느낀 점:**
  인터뷰 전에는 자료를 잘 정리하는 시스템이 있으면 좋겠다고 생각했다. 그런데 인터뷰를 하면서 내가 정말 원하는 건 '정리'보다 '실행'이라는 걸 다시 확인하게 됐다.

  나는 시작을 안하는 것 보다는, **시작해도 흐름이 끊기면 다시 못 잡는다**는 게 좀 더 아쉬웠다. AI 공부도 직접 강의 보고 끄적여봤고, 업무 자동화도 직접 만들어봤다. 근데 토큰이 떨어지거나, 결과물이 기대에 못 미치거나, 새로운 관심사가 생기면 — 그냥 멈춰버렸다. 의지 문제가 아니라 흐름이 끊기는 구조 문제였던 것 같다.

  내가 만들고 싶은 OS는 **내 호기심을 실제 행동으로 이어주고, 끊긴 흐름을 다시 잡아주는 시스템**이었으면 좋겠다. 그런 시스템이 있다면, AI 공부뿐 아니라 일상에서 하고 싶었던 일들도 조금씩 실천하면서 지금보다 더 만족스러운 삶을 만들 수 있을 것 같다.

## 🧩 미션 2. 내 OS 기획
> 인터뷰 결과 + 세션 내용(흐민·배짱·키노) 활용해 기획

- **기획 내용:**

  **OS 이름: S box**
  텔레그램 채널 이름에서 따왔다. 내 관심사와 하고 싶은 것들이 들어오고 나가는 상자.

  **핵심 구조: 입력 → 저장 → 꺼내기**

  **1. 입력 (텔레그램 S box)**
  인스타 릴스, 유튜브, 출퇴근 중 떠오른 생각 등 언제 어디서든 텔레그램으로 던진다.
  주로 텍스트를 직접 쓰기보다 **링크 + 중요한 장면 캡처 1~2장 + 한 줄 메모**를 함께 보내는 방식이 편하다.
  예: 인스타 AI 릴스 링크 + 캡처 + "나중에 업무 자동화에 써볼 것 같음"
  (Claude는 링크만으로는 영상 내용을 볼 수 없으므로, 캡처와 메모로 내용을 파악해 정리한다.)

  **2. 저장 (맥북 폴더 — 카테고리별 MD 파일)**
  Claude Code가 받은 내용을 카테고리 판단 후 자동으로 분류·저장한다.
  - **#배우기** — AI 공부, 맥 사용 팁, 유튜브 강의 등 학습 관련
  - **#해보기** — 맛집, 야구, 보드게임, 가보고 싶은 곳 등 경험 관련
  - **#챙기기** — 건강검진, 병원 예약, 챙겨야 할 일
  - **#업무** — 디자인 자동화, 업무에 적용해볼 것들
  새 관심사가 생기면 태그를 새로 만들어 카테고리를 추가할 수 있다.

  **3. 꺼내기 (자동 추천 + 날짜 알림 + 검색)**

  꺼내기는 두 갈래로 나눴다. **"언젠가 할 것"은 텔레그램**이, **"정해진 날짜에 할 것"은 구글 캘린더**가 맡는다.

  - **매주 토요일 오전 10시 — 배우기 추천 (텔레그램)**:
    이번 주 저장한 것만 보지 않고, **최근에 자주 던진 관심사 흐름을 반영해** 지금 하기 좋은 것 1개를 골라 추천한다.
    "지난번에 『OO』 저장해뒀는데, 오늘 30분만 해볼까요?" → "오늘 하기 힘든 작업이야" 하면 **두 번째 후보**를 다시 제안한다.
    토요일에 일정이 있어 미뤄야 하면 **"구글 캘린더로 넘겨줘"** 라고 말한다. 그러면 원하는 날짜에 알림이 등록되고, 캘린더 알림이 오면 그때 텔레그램을 열어 이어서 실행한다. 미뤄도 흐름이 끊기지 않게 캘린더가 다시 붙잡아주는 것.

  - **날짜 기반 알림 (구글 캘린더)**:
    날짜가 있는 일은 텔레그램(맥북이 켜져 있어야 함) 대신 **구글 캘린더에 등록**해 핸드폰으로 알림받는다. 맥북이 꺼져 있어도 알림이 오는 게 장점. 네이버 캘린더를 쓰고 있었지만, Claude Code와 자동 연동이 가장 잘 되는 구글 캘린더로 새로 시작하기로 했다. 이런 알림들이 필요하다:
    - **챙기기**: 동생 생일(7/15), 가족 모임(7/11) 같은 걸 모임 전날/전전날에 미리 알림
    - **해보기**: 보드게임 모임 전에 "룰 미리 찾아보기" 리마인드
    - **업무**: 3분기 목표처럼 특정 시점부터 시작해야 하는 일을 "지금쯤 시작하라"고 알림
    (구글 캘린더 자동 등록 연결은 다음 단계 과제로 남겼다.)

  - **필요할 때 검색 (텔레그램)**: "AI 공부 관련 저장한 거 보여줘" 하고 물어보면 해당 카테고리를 훑어 꺼내준다.

  **세션에서 배운 것과의 연결**
  흐민님 OS에서 텔레그램 → Claude Code → 로컬 MD 파일 저장 구조를 그대로 가져왔다.
  차이점은 지식 자산화보다 **실행 재개**에 초점을 맞춘 것. 쌓는 게 목적이 아니라 토요일에 하나를 꺼내 실제로 해보는 것이 목적이다.

- **막혔던 점 / 어떻게 풀었나:**
  인스타 릴스를 어떻게 저장할지가 제일 막혔다. Claude가 링크만으로는 영상 내용을 볼 수 없다는 걸 알게 됐고, 링크 + 캡처 + 한 줄 메모 조합으로 해결했다. 완벽한 자동화보다 내가 조금 수고해서 더 정확하게 저장되는 쪽을 택했다.

## ⚙️ 미션 3. 내 OS 구현
> 실제로 만들어본 것 (클로드코드 '채널' 기능 활용 OK)

- **결과물: S box 1단계 — 텔레그램으로 던지면 자동 분류·저장**

  텔레그램 채널(S box) → Claude Code 연결을 세팅하고, `telegram_Sbox` 폴더에 실제로 작동하는 저장 시스템을 만들었다.

  **만든 것**
  1. **CLAUDE.md (OS의 두뇌)** — 텔레그램으로 뭔가 오면 어떻게 처리할지 규칙을 자연어로 작성. 카테고리 분류 기준, 저장 위치, 파일 형식, 토요일 추천 방식까지 정의.
  2. **폴더 구조 (저장소)**
     ```
     sbox/
     ├── 배우기/    ← 흐르는 것 (한 번 하고 끝)
     ├── 해보기/
     ├── 챙기기/
     ├── 업무/
     ├── 레퍼런스/  ← 고이는 것 (계속 관리하는 목록)
     └── _done/     ← 완료한 것
     ```
  3. **흐르는 것 / 고이는 것 구분** — 릴스·강의처럼 한 번 하고 끝나는 건 건건이 파일로, 디자인 레퍼런스 사이트처럼 계속 참조하는 건 하나의 목록 파일에서 추가·삭제·수정하며 관리하게 나눴다. (트렌드가 바뀌면 목록도 바뀌니까)

  **실제로 작동 확인한 것**
  - 핸드폰 텔레그램에서 인스타 릴스 링크 + 캡처 + "디자이너 레퍼런스 사이트 정리해줘" 전송
  - → Claude Code가 캡처 이미지를 읽어 사이트 4곳(Savee, It's Nice That, Designspiration, Awwwards)을 인식
  - → 카테고리 판단 후 `sbox/`에 MD 파일로 자동 저장하고 "저장했어요 👍" 답장
  - → "URL도 저장해야지"라고 피드백하니 파일에 URL 추가 + "앞으로 URL 필수로 챙길게요"라고 규칙까지 학습
  - → "이건 평생 쓸 목록이니 따로 관리하고 싶다"고 하니, '한 번 하고 끝나는 것(배우기·해보기)'과 '계속 관리하는 목록(레퍼런스)'을 구분하는 구조로 발전시킴. 디자인 사이트 4곳은 `레퍼런스/` 목록 파일로 이동.

  **막혔던 점 / 배운 것**
  - 처음엔 Claude가 캡처 내용을 답장으로만 써주고 파일 저장을 안 했다. → 받는 창이 CLAUDE.md를 읽기 전에 켜져 있어서였고, 껐다 켜니 규칙대로 저장됐다. **CLAUDE.md를 바꾸면 세션을 다시 켜야 반영된다**는 걸 배웠다.
  - 음성 메시지는 변환 도구(whisper 등)가 없어서 아직 안 된다. 텍스트+캡처로 충분해서 다음 단계로 미뤘다.

- **다음 단계 (2단계)**
  - 토요일 오전 10시 배우기 추천 자동화 (스케줄러 연결)
  - 날짜 알림 → 구글 캘린더 자동 등록
  - 음성 메시지 입력 지원

- **링크 / 스크린샷:** (이미지는 `이미지첨부/` 폴더에)

## 📱 미션 4. SNS 1주차 소감
> AI 도움 없이 직접 작성! (인증하면 셀 지급)
- **인증 링크:**
