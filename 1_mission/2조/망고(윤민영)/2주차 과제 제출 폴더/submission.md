### 📐 기획

#### 해결하고 싶었던 문제

1주차 인터뷰에서 나온 핵심 마찰: 성경 묵상을 할 때마다 **탭을 5개씩 열어야 했다.**

- 개역개정 탭, 새번역 탭, NKJV 탭, NASB 탭 — 번역본 비교만으로 4개
- 원어 사전 탭 (헬라어/히브리어 찾아보기)
- 그걸 복사해서 노트에 정리

매번 이 과정을 반복하다 보니 말씀 앞에 앉기도 전에 지쳐버리는 일이 생겼다. **도구가 묵상을 방해하고 있었다.**

#### OS의 방향

> 본문 하나 입력하면, 번역 비교 + 단어 분석 + 배경이 한 화면에 뜨고  
> 묵상 기록 → 저장 → 알림 → 공유까지 한 흐름으로 이어지는 것

단순 자동화가 아니라, **말씀 앞에 집중할 수 있는 환경** 자체를 만드는 것.

앱 이름은 처음에 "아침 묵상 OS"였다가, 아가서 2:10에서 오는 **Come Away** 로 결정.  
헤더에는 아가서 6:5 — *"Turn your eyes away from me, for they overwhelm me."* — 를 새겼다.

#### 기술 스택 선택 이유

- **Python (프레임워크 없이 BaseHTTPRequestHandler)**: 설치 없이 바로 실행, 가장 단순
- **Claude CLI (`claude -p`)**: Anthropic API 키 없이 이미 설치된 Claude를 subprocess로 호출
- **HTML/JS 단일 파일 임베드**: 별도 빌드 없이 Python 파일 하나로 전부 완결
- **ntfy.sh**: 계정 없는 무료 핸드폰 푸시 알림

   - ‹ › 버튼으로 월 이동

    - 서버가 실행 중일 때 30초마다 체크하는 백그라운드 데몬 스레드

**[디자인]**

- 파리 앤틱 스타일: 세로 줄무늬 헤더 (French ticking fabric), 양피지 배경 + 핀스트라이프

### 🧗 과정에서의 삽질

#### 삽질 1. Claude CLI가 subprocess에서 무한 대기

**상황:** Python에서 `subprocess.run(["claude", "-p", ...])` 를 실행하면 분석 버튼을 눌러도 아무 반응 없이 멈춰버림. 터미널에서 직접 실행하면 잘 되는데 코드에서만 안 됨.

**원인 파악:** `claude -p` 는 TTY(터미널 입력)를 기다리는데, subprocess에는 TTY가 없어서 입력을 무한정 기다리는 상태가 됨.

**시도한 것들:**
- `subprocess.run(..., timeout=30)` → TimeoutExpired 에러만 남
- `subprocess.Popen(..., stdin=subprocess.PIPE)` → 그래도 대기
- Claude Code 화면이 자동으로 뜨는 것 확인 → 방향이 잘못됐음을 인지

**해결:** `stdin=subprocess.DEVNULL` + `--dangerously-skip-permissions` 플래그 추가
```python
result = subprocess.run(
    ["claude", "-p", "--dangerously-skip-permissions", "--output-format", "text", prompt],
    capture_output=True, text=True, timeout=120,
    stdin=subprocess.DEVNULL
)
```
DEVNULL로 stdin을 막아버리니 TTY를 기다리지 않고 바로 실행됨.

#### 삽질 2. 포트 80은 sudo가 필요

**상황:** 처음에 `localhost:80` 또는 `localhost:3333`으로 만들었는데, 80포트는 관리자 권한 없이 실행 불가.

**해결:** 포트를 **1331**로 변경. 접속 주소는 `/etc/hosts`에 커스텀 도메인 추가로 해결.
```
127.0.0.1 psalm1331
```
접속: `http://psalm1331:1331`

처음에 포트 없이 `psalm1331`만 입력했더니 연결 안 됨 → **포트 번호를 같이 써야 한다**는 것을 알게 됨.

#### 삽질 3. 클립보드 API가 http에서 차단됨

**상황:** 카카오톡 공유 버튼을 눌러도 복사가 안 됨. 콘솔에 `NotAllowedError` 에러.

**원인:** `navigator.clipboard.writeText()`는 보안상 **HTTPS에서만 동작**. `http://psalm1331:1331`은 http라 막힘.

**해결:** 구형 방식인 `document.execCommand('copy')` 로 대체.
```javascript
const ta = document.createElement('textarea');
ta.value = text;
ta.style.cssText = 'position:fixed;opacity:0';
document.body.appendChild(ta);
ta.select();
document.execCommand('copy');
document.body.removeChild(ta);
```
지금도 정상 작동 중.

#### 삽질 4. 포트 충돌 (Address already in use)

**상황:** 코드 수정 후 서버를 다시 실행하면 `OSError: [Errno 48] Address already in use`.

**이유:** 이전 서버 프로세스가 백그라운드에서 계속 살아있음.

**해결:**
```bash
pkill -f "묵상-os.py"
lsof -ti :1331 | xargs kill -9
```
이 두 줄을 재시작 전에 항상 먼저 실행.

#### 삽질 5. Claude가 성경 번역본 출력을 거부

**상황:** 4개 번역본을 달라고 하면 "저작권이 있는 번역본을 인용할 수 없습니다"라고 거절.

**시도:** "인용" 대신 "공부 목적으로" 라고 표현 변경.

**해결:** 프롬프트에서 "인용"이라는 단어를 지우고 "공부" 맥락으로 바꾸니 바로 통과. `claude -p` (CLI)는 API보다 훨씬 유연하게 처리해줌.

#### 삽질 6. 품사 분류 오류

**상황:** "give a defense" 같은 동사구가 "명사"로 분류됨.

**원인:** 프롬프트에 품사 유형을 구체적으로 명시하지 않아서 Claude가 단순하게 분류.

**해결:** 프롬프트에 명시적으로 품사 목록 추가.
```
pos 필드: 명사/동사/형용사/부사/동사구/명사구/형용사구 중 하나
```
이후 "give a defense"는 정확하게 **동사구**로 분류됨.

#### 삽질 7. 중복 저장

**상황:** 묵상을 저장한 뒤 실수로 "기록 저장" 버튼을 또 누르면 같은 내용이 2~3번 저장됨.

**해결:** `isSaved` 플래그 추가.
```javascript
let isSaved = false;

async function saveEntry() {
    if (isSaved) { alert('이미 저장됐어요!'); return; }
    // ... 저장 로직 ...
    isSaved = true;
    saveBtn.textContent = '✅ 저장됨';
    saveBtn.disabled = true;
}
// 새 분석 시작할 때 isSaved = false 로 리셋
```

#### 삽질 8. 핸드폰 알림 — Telegram이냐 ntfy냐

**상황:** macOS 팝업(`osascript`)은 Mac이 켜져있을 때만 동작. 핸드폰으로 받고 싶었음.

**처음 고려:** Telegram Bot  
→ 이미 Bot Token은 있었으나 Chat ID를 얻으려면 사용자가 봇에 먼저 메시지를 보내야 하는 추가 단계 필요. 번거로움.

**최종 선택:** [ntfy.sh](https://ntfy.sh)  
- 계정 없음, API 키 없음
- 앱 설치 → 토픽 구독 → 끝
- 서버에서는 HTTP POST 한 줄로 발송:
```python
req = Request(
    f"https://ntfy.sh/mango-bible-morning-os",
    data=message.encode(),
    headers={"Title": title, "Tags": "bible"}
)
urlopen(req, timeout=5)
```
지금은 알림이 Mac 팝업 + 핸드폰 ntfy 푸시로 동시에 옴.

#### 삽질 11. "이 링크 공유하면 되겠다" — 그런데 아무도 못 들어왔다

**상황:** 앱을 다 만들고 나서 `http://psalm1331:1331` 링크를 친구에게 보내려다가 멈칫함. 이게 다른 사람한테도 열릴까?

**현실:** 안 열린다.
- `psalm1331`은 내 Mac의 `/etc/hosts`에만 존재하는 가짜 주소
- `localhost`와 마찬가지로 "내 컴퓨터 안에서만" 유효
- 외부에서는 그 이름이 어디를 가리키는지 알 수 없음

**구조적 원인:** 앱이 `claude -p` CLI(내 Mac에 설치된 Claude Code)를 subprocess로 직접 호출하는 구조라, 클라우드 서버에 그대로 올리는 것도 불가능. 클라우드에는 내 Claude Code가 없으니까.

**시도한 것들:**
- SSH 터널 (`ssh -R 80:localhost:1331 serveo.net`) → 임시 공개 URL 생성 가능하나 Mac이 켜져있어야 함
- npx localtunnel → npm 캐시 권한 오류로 실패
- 클라우드 배포 → `claude -p` 대신 Anthropic API 키가 필요해서 구조 전면 수정 필요

**최종 결정:** 파일 하나만 공유하는 방식.
- `묵상-os.py` + `시작하기.md` 두 파일을 카톡으로 전송
- 받는 사람은 Claude Code 설치 → 터미널 명령어 한 줄 → 자기 Mac에서 실행
- 커뮤니티 기능은 원래 만들어둔 **카카오톡 공유 버튼**으로 충분: 각자 묵상하고 결과를 카톡 그룹에 붙여넣기

> 생각해보면 이게 더 맞는 구조다. 묵상은 원래 혼자 하는 것이고, 나눔은 카톡으로 하는 것. 공유 링크 하나에 모두가 들어오는 구조가 아니어도 됐다.

### 💡 알게 된 인사이트 & 공유하고 싶은 내용

**1. 도구를 만드는 게 아니라 환경을 만드는 것**  
처음에는 "번역 비교 자동화 도구"를 만든다고 생각했는데, 완성하고 나서 보니 그게 아니었다. 말씀 앞에 온전히 앉을 수 있는 **환경**을 만든 것. 도구는 손이 안 보여야 잘 만든 거다.

**2. API 없이도 할 수 있는 게 많다**  
Anthropic API 키 없이 `claude -p` CLI로 같은 결과를 냈다. ntfy.sh는 계정조차 없다. Telegram bot도 고려했는데 더 복잡했다. 더 단순한 방법이 대부분 더 빠르다.

**3. 삽질이 실제로 제일 많이 배우는 구간이다**  
TTY 문제, 포트 문제, HTTPS 클립보드 문제 — 전부 "왜 안 되지?"에서 시작해서 그 원리를 이해하고 나서야 풀렸다. 검색보다 부딪히는 게 빠를 때가 있다.

**4. 이름을 찾는 데 생각보다 오래 걸렸다**  
기능보다 이름이 더 오래 걸렸다. "아침 묵상 OS"는 내가 만든 게 아니라 회사 제품 이름 같았다. **Come Away** 는 하나님이 먼저 부르신다는 신학적 의미와, 매일 아침 그 초대에 응한다는 관계성을 동시에 담고 있다. 이름이 정해지니 앱 전체가 달라 보였다.

**5. 크루들에게 공유하고 싶은 것**  
성경 묵상을 하면서 번역 비교가 얼마나 묵상을 깊게 만드는지 몰랐었다. 같은 구절을 개역개정, 새번역, NKJV, NASB로 나란히 놓으면 **번역자가 어떤 단어를 선택했는지**가 보이고, 그게 말씀 해석에 전혀 다른 빛을 비춰준다. 이 앱 없이도, 내일 묵상할 때 번역 두 개만 나란히 놓아보는 것부터 시작해볼 것.
