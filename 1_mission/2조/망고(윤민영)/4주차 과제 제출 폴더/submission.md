# 4주차 — 기획·구현 → 검증 → 기록 🧪

> 프로젝트: **Come Away — 작은 오병이어 상점** (성경 묵상 커뮤니티 웹앱)
> 라이브: **https://come-away-applemango-yoons-projects.vercel.app**

## 🎯 과제 1. PRD 기획 및 구현
> 주요 가치 · 페르소나 · 소구 방식을 담은 PRD → 구현까지

- **주요 가치:**
  - **깊이를 쉽게** — 성경 한 구절을 넣으면 AI가 ①번역본 4개(개역개정·새번역·NKJV·NASB) 비교, ②어려운 영어 핵심 단어 5개(뜻·뉘앙스), ③원어(헬라어/히브리어)+Strong's 번호 출처 링크, ④배경까지 정리. 신학을 몰라도 깊은 묵상 재료를 얻는다.
  - **함께 하는 지속성** — 묵상 기록을 서로 보는 '광장', 매일 갓 구운 빵(달란트)을 나누는 '오병이어' 게이미피케이션으로, 혼자서는 어려운 매일 묵상을 공동체가 붙잡아 준다.
  - **우리만의 안전한 공간** — 관리자 승인제(등록된 이름만 입장)로 아무나 못 들어오는 구독자 전용 공간.

- **페르소나:** 핵심 타깃은 *"매일 묵상하고 싶지만 혼자서는 잘 안 되는 신앙 공동체 구성원"*. 나이·디지털 숙련도 분포가 넓다.
  - 모임 리더(29·교사, 관리자) — 멤버 관리·출석/성취 트래킹·이벤트 운영
  - 워킹맘(42·간호사) — 새벽에 짧게, 빠르게 은혜받는 묵상 + 알림
  - 은퇴 장로(68, 스마트폰 서툼) — 진입 쉬움·큰 글씨·깊은 말씀
  - 청소년/청년(17~24, 디지털 네이티브) — 재미(캐릭터·달란트)·영어 학습

- **소구 방식:**
  - **진입 장벽 제거** — 회원가입·비밀번호 없이 "이름만" 입장, 링크 하나로 전 연령대 접속
  - **따뜻한 브랜딩** — 빵집🍞·물고기🐟 콘셉트 + 성경 속 동물·열매로 내 캐릭터 꾸미기 → 애착·재방문
  - **작은 성취의 반복** — 매일 나오는 빵을 나누고 모아 '공유회'를 여는 오병이어 시스템으로 참여 습관화
  - **리더 중심 확산** — 관리자(제빵실)가 멤버를 초대·관리하고 스페셜 공지로 독려 → 공동체 단위 확산

- **구현한 것 (링크·스크린샷):**
  - 라이브: https://come-away-applemango-yoons-projects.vercel.app (관리자가 등록한 이름으로 입장)
  - 기술 스택: 프론트(HTML/CSS/JS 단일 페이지) · 서버리스 함수(Python, Vercel) · DB(Supabase) · AI 분석(**Claude API**, 백업 Google Gemini)
  - 기능: 성경 분석(번역 4개 비교·핵심 영어 단어 5개·원어+Strong's 출처 링크·배경) / 묵상 기록·광장 커뮤니티·댓글·반응 / 오병이어 달란트 나눔·공유회 / 이름 기반 관리자 승인제 로그인·내 소개실(캐릭터·알림 메일·로그아웃) / 관리자 제빵실(멤버 관리·성취 트래킹·공지·물고기 지급) / 분석 결과 캐시로 재검색 즉시 응답
  - 스크린샷: `이미지첨부/` 폴더에 추가 예정

## 🗣 과제 2. 유저 피드백 받기
> 실제 2명 이상 + 플러스 알파로 가상 페르소나

- **실제 유저 피드백 (2명 이상):**
  <!-- 배포 링크를 공유하고 받은 실제 반응을 아래에 채워 주세요 -->
  - ① (닉네임/관계): 좋았던 점 · 아쉬운 점 · 요청 —
  - ② (닉네임/관계): 좋았던 점 · 아쉬운 점 · 요청 —

- **가상 페르소나 피드백:** 나이 17~73세, 8개 페르소나 베타 테스트 (평균 만족도 ★4.4/5)
  - 이지연(24·대학생) ★★★★★ — "톤이 따뜻하고 캐릭터 고르는 게 취향저격"
  - 정다은(17·고등학생) ★★★★★ — "빵 모아 공유회 신청하는 게 게임 같아 계속 하게 됨"
  - 윤가영(29·교사·리더) ★★★★★ — "제빵실에서 멤버 관리·성취 트래킹 다 돼서 리더에게 최고"
  - 김성호(68·은퇴 장로) ★★★★☆ — "코드 없이 이름만 치니 편해. 글씨만 조금 더 컸으면"
  - 박민호(35·개발자) ★★★★☆ — "이름만 로그인이라 편하지만 동명이인·보안은 고려 필요"
  - 최은주(42·간호사) ★★★★☆ — "새벽 묵상에 딱. 내가 등록돼야 들어오는 건 몰랐음"
  - 오세욱(50·자영업) ★★★★☆ — "광장에서 서로 나누는 게 모임 분위기와 잘 맞음"
  - 한영자(73·권사) ★★★☆☆ — "이름 넣어주셔서 들어왔어요. 표가 옆으로 넘어가 조금 힘들어"

- **베타 테스트에서 나온 고칠 점 (개선 우선순위):**
  - **[P1] "승인되지 않은 이름" 안내 불친절** — 새 사용자가 왜 막히고 어떻게 등록 요청하는지 설명이 약함(워킹맘·자영업 페르소나).
  - **[P1] 고령·저시력 가독성** — 본문 글씨가 작고, 번역 4개 비교표가 가로로 넓어 좌우 스크롤 필요(은퇴 장로·권사).
  - **[P2] 캐릭터·로그아웃 발견성** — 오른쪽 위 배지 한 곳에만 진입점이 있어 어르신들이 못 찾음.
  - **[P2] 첫 분석 콜드스타트 지연 체감** — 서버가 깨어나는 몇 초를 "멈췄나?"로 오해.
  - **[P2] 이름만 로그인의 보안·동명이인** — 비밀번호가 없어 도용·기록 혼선 여지(개발자 지적).
  - **[P3] 오병이어(달란트) 온보딩** — 빵/물고기/공유회 개념을 처음엔 이해 못 함.
  - **[P3] 멤버 많아질 때 관리 편의** — 멤버 목록 검색/정렬 없음(리더).
  - **[P3] 하이라이트 대비·원어 카드 난이도** — 번역 차이 하이라이트 색이 옅고, 원어 카드가 초신자·고령층엔 어려움.

- **피드백에서 알게 된 것 (그래서 무엇을 했나):**
  1. **진입 장벽 제거가 최고 호평** — 초기 '입장 코드'를 없애고 "이름만 로그인"으로 바꾼 결정이 전 연령대, 특히 고령층 진입을 크게 낮췄다.
  2. **가독성 지적 → 실제 반영** — "글씨가 작다"는 P1 피드백을 받아 본문·단어 뜻·원어·배경 글씨를 실제로 키웠다.
  3. **한글 이름 사용자 분석 버그 발견 → 수정** — 페르소나별로 다른 환경을 가정해 테스트한 덕에, 한글 이름이 요청 헤더에서 타입에러를 내던 문제를 찾아 인코딩 처리로 해결했다.
  4. **재미 요소가 지속성의 열쇠** — 청소년·청년은 오병이어(달란트)·캐릭터 꾸미기에서 재방문 동기를 얻음.
  5. **온보딩 안내는 다음 과제** — 미승인 안내·오병이어 첫 설명을 더 친절히 하면 이탈을 줄일 수 있음(구독자 전용이라 우선순위는 낮게 판단).

## 📣 과제 3. SNS 기록 남기기
> 기획 · 구현 · 피드백의 전 과정과 소회를 기록으로

- **기록 내용 (소회):**
  기획(PRD)부터 구현·디버깅·베타테스트·개선까지 전 과정을 직접 겪었다. 실제로 부딪힌 문제를 하나씩 해결 — 입장 코드 검증 오류→관리자 승인제(이름 로그인)로 구조 전환, 한글 이름이 요청 헤더에서 타입에러→인코딩/디코딩 처리, DB 저장 실패(RLS 권한)→정책 수정, AI 공급자 이슈(Groq 지역 차단·무료 한도)→결제한 Claude API 우선+캐시로 "여러 사람이 동시에 접속해도 빠르게".
  가장 기억에 남는 건 **"관리자인 나만 되고 다른 사람은 안 되는"** 버그의 진짜 원인이 *한글 이름의 헤더 인코딩*이었다는 걸 찾은 순간이다. 사용자마다 다른 환경에서 테스트하는 것이 왜 중요한지 체감했다. 완벽한 첫 버전보다 **빠르게 만들고 실제 반응으로 고쳐 나가는 것**이 제품을 살아 움직이게 한다는 걸 배웠다.
