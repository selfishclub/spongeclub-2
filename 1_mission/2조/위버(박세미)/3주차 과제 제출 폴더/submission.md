# 3주차 — 내 OS 최종 완성 🏁

> 미션을 진행하며 과정과 결과를 기록해주세요. (다 못 채워도 OK, 한 것 위주로!)

## 🎯 미션 1. 내 삶을 돕는 OS 최종 완성
> 지금까지 공유하며 받은 **피드백을 반영해 최종 완성**!

- **완성한 것 (무엇을):**

  2주차에 만든 **AI 공부 본진**(강의·자료·노트를 한 곳에 모으는 통합 학습 대시보드)을, 3주차엔 **매일 실제로 쓰이는 OS**로 굳혔다. 이번 주에 새로 한 것:

  - **① 텔레그램 노트 캡처 — 입력 통로를 폰에 붙였다.** 2주차 앱의 가장 큰 마찰은 *"본진을 켜고(.bat 실행) → 노트 뷰로 가서 → 입력"* 하는 단계였다. 생각날 때 바로 못 적으면 결국 안 쓴다. → 로컬 서버(`study-hub-server.js`)에 **`/api/note` 통로**를 새로 뚫어, **텔레그램에 "노트: …" 한 줄만 보내면 본진의 공부 노트로 자동 저장**되게 했다. 저장된 노트엔 `텔레그램` 태그가 자동으로 붙어 나중에 모아볼 수 있다. **실제로 텔레그램으로 던진 메모들이 본진에 쌓인 걸 확인**했다. (앱을 켜지 않아도 폰에서 바로 던지는 "30초 캡처"가 됐다.)
  - **② 매일 실사용으로 데이터가 5배로.** 2주차 제출(7/11) 이후에도 강의·자료·노트를 계속 넣으며 써서, 데이터가 **약 249KB → 1.18MB로 약 5배** 늘었다. "만들어 두고 안 쓰는 앱"이 아니라 **6주 캠프 내내 실제로 굴러가는 학습 본진**이 됐다.
  - **③ 다음 업그레이드 설계 완료 (아직 코드 반영 전).** 매일 쓰다 보니 실제로 걸리는 지점이 보여, 손대기 전에 **개선 계획서를 문서로 먼저 정리**했다 — ⓐ **데이터 안전장치**(자동 백업 로테이션·저장 크기 제한·깨진 JSON이 정상 파일을 덮어쓰지 않게 검증), ⓑ **이미지 용량 다이어트**(붙여넣는 이미지를 WebP로 더 압축 + 현재 데이터 용량 표시·경고). 데이터가 커질수록 "유실·손상" 위험이 진짜 리스크라, **다음 제출 때 반영 예정**이다.

- **피드백 반영한 점:**
  - ✍️ _이번 주 크루·유닛에서 받은 피드백과 반영한 점을 한두 줄로 채워주세요._

- **결과물 (링크·스크린샷 — 이미지는 `이미지첨부/` 폴더에):**
  - **AI 공부 본진** (로컬 단일 페이지 대시보드 + 로컬 저장 서버, 오프라인 동작) — `AI공부본진.html` · `study-hub-server.js`(+`/api/note` 텔레그램 통로) · `AI공부본진-데이터.json` · `AI공부본진 실행.bat`
  - 실사용 규모: 강의·자료·노트 등 실제 학습 기록 누적(데이터 약 1.18MB), 텔레그램 태그 노트 포함
  - ✍️ _텔레그램 캡처 화면 / 본진 노트에 쌓인 화면 캡처를 `이미지첨부/` 폴더에 넣어주세요._

- **알게 된 인사이트:**
  - **"기능"보다 "입력 통로"가 OS를 살린다.** 2주차엔 대시보드를 잘 만드는 데 집중했지만, 정작 매일 쓰게 만든 건 *텔레그램에 한 줄 던지기*라는 마찰 제거였다. 아무리 잘 만든 앱도 "켜서 입력"이 귀찮으면 안 쓴다 — 입력 통로를 **이미 매일 쓰는 앱(텔레그램)**에 붙이니 비로소 매일 쓰는 도구가 됐다.
  - **매일 쓰니 진짜 할 일이 보였다.** 안 쓰는 앱엔 개선 목록조차 안 생긴다. 실사용으로 데이터가 커지자 "백업·용량"이라는 다음 과제가 저절로 드러났고, 그게 다음 업그레이드 설계로 이어졌다. **실사용이 곧 로드맵.**
