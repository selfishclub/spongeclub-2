## 결과물

> 이번 주에 실제로 만든 것·완성한 것을 적어주세요. (링크, 스크린샷, 데모 등)

**"AI 공부 본진"** — AI 공부 자료를 한 곳에 모으는 **로컬 웹 대시보드**를 만들었다. 1주차 아이디어 창고(텔레그램 봇)와는 다른 새 방향으로, *"구글 시트·노션·슬랙·트렐로·줌 너무 많은 툴을 왔다갔다 하느라 정신없다. 6주 캠프 등록했는데 혼자 디벨롭하는 걸 기록하고 온라인에서 받은 자료도 다 모아보고 싶다"*는 데서 출발했다. `service-blueprint` 스킬을 GitHub에서 설치해 기획부터 시작했다.

**만든 것 (실제 동작 확인):**
- **`AI공부본진.html`** — 사이드바로 7개 뷰(홈·강의·자료·노트·주차·전체·설정)를 넘나드는 단일 페이지 대시보드.
- **`study-hub-server.js`** — 데이터를 브라우저가 아니라 옆 JSON 파일에 저장하는 로컬 서버(포트 8899). 실행하면 화면이 자동으로 열린다.
- **`AI공부본진 실행.bat`** — 더블클릭 한 번으로 서버를 켜고 대시보드를 여는 런처.
- **`AI공부본진-데이터.json`** — 진짜 데이터가 쌓여 있음. 현재 **13개 항목**(강의 12 + 과제 1). 6주 과정(2026-06-27 시작) 기준으로 "셋업데이 / 1주차 / 2주차 / 클로드코드 워크샵" 등 실제 수강 내용과 줌·구글 드라이브·커뮤니티 바로가기가 저장돼 있다. 백업본도 남겼다(`백업/데이터백업_20260711_0253.json`).

**주요 기능:** 주차 중심 구조 · 강의 영상 카드 리스트 · 라벨 붙는 자료 모음집(노션 링크 + 이미지/글 편집기 + PDF 아카이빙) · 공부 노트 · 줌/드라이브/슬랙 바로가기 채널 · 공개/나만 보기 분리(저작권 이슈 대비) · 웹에서 Ctrl+C·Ctrl+V로 노션 이미지까지 붙여넣기 · 뷰어/수정 모드 전환.

**완성 vs 다음:** 로컬 실행·저장·아카이빙까지 실제로 동작(데이터 13건 + 백업). Vercel 배포는 검토만 하고 로컬 서버 방식으로 남겨뒀고, "AI 후기 카드뉴스 구성을 대시보드 안에 스킬처럼 붙이기"는 다음 확장 아이디어로 남았다.

## 삽질 과정

> 막혔던 지점, 시도한 방법, 해결 과정을 솔직하게 적어주세요.

기능 만드는 것보다 **"내가 넣은 게 진짜 저장되고, 그 파일을 어디서 여느냐"**가 진짜 싸움이었다.

- **저장이 자꾸 날아감.** 처음엔 브라우저에만 임시 저장돼 새로고침하면 사라졌다. → 데이터를 옆 JSON 파일에 저장하는 로컬 서버(`study-hub-server.js`)를 만들어 해결.
- **파일을 어디서 열어야 할지 몰랐다.** *"파일이 안 열리는데 어디서 열면 되냐"* → 비개발자도 쓸 수 있게 `.bat` 런처를 만듦.
- **"파일모드" 함정.** HTML을 `file:///…`로 직접 열면 저장이 안 되고(파일모드), 서버(localhost:8899)로 열어야 저장된다는 걸 이해하는 데 시간이 걸렸다. 파일모드에서 링크를 잔뜩 넣었다가 다시 하는 상황도 있었다. → 화면에 "지금은 파일모드예요" 경고를 띄우고, 서버 실행을 기본 경로로 잡아 정리.
- **팝업 드래그 버그.** 팝업에서 마우스로 드래그하면 창이 닫히던 버그 → 수정.
- **노트 기능 범위 오해.** "공부 노트 만들어줘"를 새 프로젝트로 오해해 멈췄다가, "AI 공부 본진 안에 기능 추가"라고 정정해서 다시 진행.

## 인사이트

> 이번 주에 배운 것·깨달은 것을 한 줄로 정리해주세요.

- 비개발자에게 진짜 난관은 기능이 아니라 **"넣은 게 진짜 저장되는가, 그 파일을 어디서 여는가"** — 로컬 서버 + 원클릭 `.bat`로 풀렸다.
- service-blueprint의 **A/B/C 선택 강제** 방식이 "툴이 너무 많아 정신없다"는 막연한 불편을 "노션 대체용 통합 학습 대시보드"라는 구체적 결과물로 좁혀줬다.
