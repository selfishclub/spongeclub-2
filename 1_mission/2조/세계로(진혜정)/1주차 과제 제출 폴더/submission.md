# 1주차 — 나만의 OS 만들기 🛠️

> 미션을 진행하며 **과정과 결과물**을 기록해주세요. (다 못 채워도 OK, 한 것 위주로!)

## 🎯 미션 1. 내 OS 재료 찾기
> 인터뷰 스킬(아이데이션)로 "내 삶에 필요한 게 뭔지" 찾기
- **과정 (어떻게 찾았나):** Claude Code의 sponge-and-interview 스킬로 삶의 반복되는 불편함을 탐색. 가계부를 쓰려고 할 때마다 작심삼일이 되는 이유를 파고들었다.
- **결과:** 가계부가 실패하는 진짜 이유는 "입력이 번거롭기 때문". 카드 내역 긁어오기나 앱 로그인 없이, 지출하는 순간 텔레그램으로 말하면 바로 기록되는 OS가 필요하다는 결론.
- **느낀 점:** 인터뷰 질문을 받으면서 막연히 "가계부를 잘 쓰고 싶다"가 아니라 "입력 장벽이 문제"라는 진짜 원인을 찾을 수 있었다.

## 🧩 미션 2. 내 OS 기획
> 인터뷰 결과 + 세션 내용(흐민·배짱·키노) 활용해 기획
- **기획 내용:**
  - OS 이름: 가계부 OS
  - 핵심 흐름: 텔레그램 자연어 입력 → 카테고리 자동 분류 → CSV 저장 → 월별 현황 리포트
  - 카테고리 7종: 식비 / 외식 / 의료 / 교통 / 생활 / 육아 / 기타
  - 입력자 구분: "남편:"으로 시작하면 남편 지출로 별도 기록 → 부부 합산 가계부로 활용
  - 월말 리포트: 전월 대비 증감 + 가장 많이 늘어난 항목 자동 감지
- **막혔던 점 / 어떻게 풀었나:** OS의 행동 방식을 CLAUDE.md 규칙 파일로 정의하는 방법이 처음엔 낯설었다. "무엇을 어떻게 시킬지"를 문서로 먼저 정리하고 나니 구현이 자연스럽게 따라왔다.

## ⚙️ 미션 3. 내 OS 구현
> 실제로 만들어본 것 (클로드코드 '채널' 기능 활용 OK)
- **결과물:** 텔레그램으로 지출을 말하면 자동 분류·저장하는 가계부 OS
  - `CLAUDE.md` — OS 행동 규칙 정의 (카테고리 기준, 응답 형식, 입력자 구분)
  - `expenses.csv` — 날짜/카테고리/내용/금액/입력자 형식으로 지출 기록 누적
  - Claude Code 채널(텔레그램) 연동으로 실제 가동 중
  - "현황" 또는 "이번달" 입력 시 카테고리별 집계 + 전월 대비 리포트 자동 생성
- **링크 / 스크린샷:** (이미지는 `이미지첨부/` 폴더에 추가 예정)
