# 1주차 — 나만의 OS 만들기

## 🎯 미션 1. 내 OS 재료 찾기

"뭘 만들지"보다 **"나는 이상하게 ___가 거슬려"** 라는 질문에서 재료가 나왔다.

- 핵심 마찰: 시간이 *없어서*가 아니라 *내 것이 아니라서* 괴롭다 = **주도권의 부재**
- 내가 원하는 한 조각: "나를 위한 커피챗 — 내가 먼저, 내가 원해서"

---

## 💡 미션 2. 내 OS 기획

**커리어 OS — 회사 타겟 모드**
가고 싶은 회사가 생겼을 때, **리서치 → 핏 점수 → 액션(커피챗 or 지원)** 까지 한 흐름으로 끌어주는 OS.

- 표면 목적: "뭘 어떻게 해야 할지 몰라" 멈추는 순간을 없애기
- 숨은 목적: 어디에 끌렸는지가 쌓여 내 방향이 드러난다
- 구조: ①회사 리서치(5앵글) ②JD 핏 스코어링 ③유사 기업 + 산업 트렌드 ④커피챗 or 지원 액션 분기

**왜 이게 나인가:**
BD 경력의 무기인 "연결·명분 생성"을 그동안 회사를 위해서만 썼다. 이 OS는 그 능력을 처음으로 나 자신에게 쓰게 한다.

---

## ⚙️ 미션 3. 내 OS 구현

Next.js 웹앱 + Claude Code 스킬로 구현.

- **웹앱**: 이력서/PDF/스크린샷 입력 → 회사·사람 검색+확인 → 리서치·핏점수·액션플랜(또는 커피챗 액션) 자동 생성
- **스킬파일**: `커피챗-os/SKILL.md`

**핵심 설계:**
- 이력서 먼저 → 회사 검색 순서 (내가 누구인지 파악이 선행)
- 채용 URL 붙여넣으면 AI가 1차 소스로 사용 (할루시네이션 방지)
- STEP별 카드로 분리 출력 (리서치 / 핏점수 / 유사기업 / 액션)
- 검색·확인 단계 추가: 이름만 치면 실제 웹검색으로 후보를 찾아서 보여주고 내가 고름 (회사/사람 정확도, 동명이인 문제 해결)
- 회사 vs 사람 모드 분기: 사람(커피챗 대상)이면 4스텝이 아니라 리서치+액션 2스텝만
- 프로필 입력 화면에 PDF/이미지 첨부 → 텍스트 자동 추출해서 붙여넣기 대체

**삽질 과정:**
- 처음엔 하고 싶은 거 10개를 다 쏟아놓고 시작함(레쥬매 넣기, JD 스크리닝, 핏 점수화, 유사기업, 트렌드, 커피챗 명분까지 전부). 브레인스토밍하면서 "일단 1단계는 회사 타겟 모드까지만"으로 범위를 잘라냄.
- 1차로 만들고 써보니까 바로 회사 검색부터 들어가더라. "어? 근데 내가 누구인지도 모르는데 이게 되나?" 싶어서 이력서/프로필부터 받는 순서로 뒤집음.
- 회사 이름이 좀 낯선 거(예: tynapse) 넣었더니 AI가 지 맘대로 "시냅스"로 착각해서 딴 얘기를 지어냄. 이거 보고 소름 돋아서 — 회사 정보는 내가 직접 붙여넣으면 그걸 1차 소스로 무조건 쓰게 룰 박음. 안 그러면 그럴싸하게 헛소리하는 게 제일 위험하더라.
- 결과 나오는 동안 화면이 계속 아래로 밀려서 위에 내용을 못 읽는 것도 짜증나서 고침. 한 번에 다 쏟아내지 말고 STEP별로 카드처럼 끊어서 보여주게 바꿈.
- vercel에 배포하는데 프로젝트 이름 `Career_OS`가 대문자·언더바 써서 거부당함 — 소문자로 다시. API 키 환경변수도 Production에만 등록해놓고 왜 안되지 하다가 다시 등록.
- 파일도 처음엔 profile.md, README.md, SKILL.md, submission.md 이렇게 4개까지 늘어났었는데, "어차피 웹앱에 경력 쓰는 칸 있는데 이걸 왜 파일로 또 만들었지" 싶어서 SKILL.md 하나로 정리함. 개인정보 정리도 3~4번은 다시 훑었음.
- 근데 곰곰 생각해보니 애초에 AI가 실시간 웹검색 없이 그냥 "아는 척"으로 리서치하고 있었던 거였음. 그래서 검색 도구를 진짜로 붙이고, 이름 치면 후보 목록 보여준 다음 내가 고르는 확인 단계를 새로 만들었음 — 할루시네이션·동명이인 문제가 한 방에 정리됨.
- 사람(커피챗 대상)을 검색했는데 회사용 4스텝 카드가 그대로 나와서 어색했음. "핏 점수·유사기업"은 사람한텐 말이 안 되니까, 사람이면 리서치+액션 2스텝만 나오게 분기침.
- 정보가 거의 없는 사람 이름으로 테스트하다가 AI가 STEP1만 쓰고 "정보 더 주세요"라고 되묻고 멈추는 걸 발견 — 근데 매번 그러진 않고 가끔만 그럼(비결정적). 5번 연속 돌려서 재현 확인하고, "정보 없어도 STEP2까지는 무조건 채워라"를 프롬프트에 못 박아서 고침.
- worktree 정리하다가 실수로 로컬 `.env.local`을 지워서 API 키가 통째로 날아감. Vercel에 저장해뒀으니 괜찮을 줄 알았는데, "Sensitive" 타입으로 등록한 값은 한 번 설정하면 그 이후로 CLI로도 절대 다시 못 읽어온다는 걸 뒤늦게 알았음(`vercel env pull` 해도 빈 문자열만 나옴) — 결국 키 새로 발급받아서 로컬·Vercel 둘 다 다시 등록.
- PDF/이미지 첨부 기능은 반대로 순조로웠음. 처음부터 검증 로직을 순수 함수로 떼어내서 테스트 먼저 짜놓으니, 실제 API 라우트 붙일 때 타입 에러 한 번 없이 바로 통과됨 — 앞의 삽질들 덕에 패턴이 잡힌 느낌.
