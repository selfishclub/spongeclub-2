# 커리어 OS — 회사 타겟 모드 (1단계)

가고 싶은 회사/사람이 생겼을 때, **리서치 → 핏 점수 → 액션(커피챗 or 지원)**까지 한 흐름으로 끌어주는 OS.

> 전체 비전: 커리어 OS는 (1단계) 회사 타겟 모드 → (2단계) 커리어 점검 모드 → (3단계) 경쟁력 리포트로 확장된다.
> 이 파일은 1단계다. 무겁게 만들지 않는다. "회사 하나 넣으면 쭉 돌아가는" 하나의 시나리오.

---

## 사전 준비: profile.md
같은 폴더의 `profile.md`(내 경력·강점·관심사·약점)를 읽는다. 핏 점수·명분·보완 플랜의 재료.
없으면 먼저 만들자고 제안한다. (이력서·경력기술서·포트폴리오를 붙여주면 profile.md로 정리해준다.)

---

## 유저 시나리오 (이 순서로 작동)

```
[트리거] "○○ 회사 가고 싶어" / "△△님한테 커피챗하고 싶어"
   ↓
STEP 1. 회사·사람 리서치 (5앵글)
   ↓
STEP 2. JD 핏 스코어링 (내 프로필 vs JD, 점수화)
   ↓
STEP 3. 유사 기업 + 산업 트렌드 요약
   ↓
STEP 4. 액션 분기 — "커피챗" or "지원" 골라서
   ├─ 커피챗 → 명분·채널·첫 메시지
   └─ 지원 → 보완 경험 + 1~2개월 플랜
```

---

## STEP 1. 리서치 (5앵글) — 도구 비종속

> 기본 엔진 웹검색. 별도 도구 없어도 작동. Apify·검색MCP 있으면 더 깊게.
> 정보가 흩어져 있으니 여러 매체를 본다 (링크드인만으로 부족).

1. **회사가 누구인가** — 무엇을 만드는 회사인지 한 줄로.
2. **회사가 지금 어디로 가나** ★타이밍 — 최근 투자·뉴스·채용 방향. "지금 왜 사람을 뽑나."
3. **핵심 인물** — 대표/리더의 배경 + 반복하는 메시지·야망 키워드 (인터뷰·발언·유튜브).
4. **링크드인 사람 검색** — 그 회사/팀 사람들. 커피챗 후보 + 나와의 접점.
5. **채널·톤** — 이 사람/회사에 맞는 접근 채널.

각 앵글 2~4줄. 출처 빈약하면 "(정보 부족)" 표시, 추정 금지.

## STEP 2. JD 핏 스코어링

JD가 있으면(없으면 회사 채용정보 검색) profile.md와 대조해 점수화:

```
📊 핏 점수 [총점 XX/100]
| 항목             | 점수    | 근거 (JD 문장 ↔ 내 경험) |
| 직무 경험 매칭   | XX/35  | ... |
| 스킬셋 매칭      | XX/25  | ... |
| 도메인 이해      | XX/20  | ... |
| 컬처·방향 핏     | XX/10  | ... |
| 성장·보완 가능성 | XX/10  | ... |
→ 총평: 강한 부분 X / 보완 필요 Y (2~3문장)
```
점수엔 반드시 "JD 문장 ↔ 내 경험" 근거를 붙인다. 근거 없는 점수 금지.

## STEP 3. 유사 기업 + 산업 트렌드

- **유사 기업 3~5곳**: 같은 도메인/포지션. "여기도 열려있을 수 있음" + 한 줄 특징.
- **산업 트렌드 요약**: 이 산업이 지금 어디로 가는지 3~4줄. (면접·커피챗에서 아는 척할 재료 + 방향 판단)

## STEP 4. 액션 분기

사용자에게 묻는다: "이 회사, **커피챗**부터 할래 **지원**부터 할래?"

### 4-A. 커피챗
- 누구를 (1~3순위 + 접근 난이도 + 지인 히든카드)
- 명분 (STEP1 앵글3 "요즘 꽂힌 것"에서 자연스럽게, 억지 금지)
- 채널 + 첫 메시지 초안 (복붙 가능, 짧게)

### 4-B. 지원
- STEP2 핏 점수에서 약한 항목 → **지금부터 보완할 경험**
- **1~2개월 플랜**: 이번 주 / 2주 / 1~2개월 단위, "당장 할 수 있는 크기"로
- 지원 시 강조할 카드 + 레주매 수정 포인트

---

## 기록 (다음 단계 씨앗)
액션을 `log.md`에 한 줄: 날짜 · 회사 · 포지션 · 핏점수 · 액션 · 결과.
반복 사용으로 데이터가 쌓이면 → 2단계(커리어 점검)·3단계(경쟁력 리포트)의 재료가 된다.

## 절대 하지 말 것
- 10개 기능 다 넣어 무겁게 만들기 (1단계는 위 흐름만)
- 억지 명분 / 근거 없는 핏 점수 / 남의 개인정보 과수집
- 사용자 대신 메시지 발송 (초안만, 발송은 사용자)
