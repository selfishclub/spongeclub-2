# 2주차 — 내 OS 구현하기 🚀

## 🎯 미션 1. 내 OS 만들기

**✅ 선택:** 내 삶을 돕는 OS

---

### 한 줄 요약

**폰에서 텔레그램에 아무거나 던지면 — 링크·스샷·약속·생각 — 봇이 읽고 판단해서 캘린더/투두/옵시디언 볼트로 정리한다. 그리고 매주 일요일 아침, 쌓인 걸 요약해서 나한테 다시 돌려준다.**

기존에 쓰던 개인 COO 봇(`~/dev/dylan-os`)에 **세컨브레인을 확장 모듈로 붙였다.** 새 OS를 만들지 않았다.

---

### 📐 기획 — 합격 기준을 바꿨더니 만들 게 달라졌다

1주차 career-os가 OS로서 약하게 느껴졌다. 이유는 기능 부족이 아니라 **매일 안 써서**였다. 회사 리서치는 시간 붕 뜰 때만 하니까.

내 반복 함정이 이거다 — **새 걸 화려하게 벌이고 방치.** 그래서 이번 OS의 합격 기준을 "멋진가"가 아니라 **"매주 강제로 쓰게 되는가"** 로 잡았다.

이 기준이면 **세컨브레인을 새로 만드는 게 최악**이다. PKM은 "수집만 하고 다시 안 봄"이 방치의 원조 공식이니까. 그런데 dylan-os가 이미 3요소를 다 갖고 있었다.

| 세컨브레인 요소 | dylan-os에 이미 있던 것 |
|---|---|
| 입력 (던지면 정리) | 텔레그램 라우팅 — 캘린더/투두/inbox 자동 분류 |
| 수집함 | `inbox.md` |
| 재소비 출구 | 매일 아침 브리핑 |

새로 만들면 80% 중복. → **inbox를 옵시디언 볼트로 승격하는 확장**으로 갔다.

**설계 원칙 하나:** 세컨브레인의 진짜 과제는 "잘 모으기"가 아니라 **"쌓인 게 반드시 다시 튀어나오는 출구"** 다. 출구 = **주간 리뷰를 일요일 아침 브리핑에 합체.** 매주 일요일 = 방치가 구조적으로 불가능.

볼트를 만들고 나니 "폰으로 보다 만 유튜브·아티클·스샷"이 전부 볼트 밖에서 새고 있는 게 보여서, 입력을 3종 더 뚫었다.

---

### ⚙️ 구현

```
텔레그램에 아무거나 던짐
   ↓  (CLAUDE.md 라우팅 규칙이 판단)
   ├─ 날짜+시간 있는 약속      → 구글 캘린더 이벤트
   ├─ 마감 있는 할 일          → 캘린더([마감] 종일) + todos.md 둘 다
   ├─ 날짜 없는 할 일          → todos.md
   ├─ "~~했음"                 → todos.md에서 [x] 처리
   ├─ 유튜브 링크              → 자막 긁어 요약+함의 → vault/clips/
   ├─ 웹 아티클 링크           → 본문 fetch → 요약+함의 → vault/clips/
   ├─ 이미지·PDF 첨부          → 읽고 → 내용에 따라 위 어디로든
   ├─ 기억할 개인 신호         → vault/inbox.md + #주제 태그
   └─ 일회성 잡담              → 답만 하고 저장 안 함 (inbox 오염 방지)

매일 아침  → 모닝 브리핑 (일정 + 투두 + 프로젝트 진척)
매주 일요일 → 거기에 주간 리뷰 합체 (지난 7일 inbox → 3블록 요약 → vault/weekly/)
```

**특이한 부분:** 라우팅이 **코드가 아니라 `CLAUDE.md`라는 규칙 문서**다. 텔레그램 데몬(`claude --channels`)이 상시 떠 있고, 메시지가 오면 그 문서를 읽고 판단해서 파일을 조작한다. 그래서 "유튜브 링크도 처리해줘"가 **파서 짜기가 아니라 규칙 한 문단 추가**로 끝난다.

| 파일 | 역할 |
|---|---|
| `CLAUDE.md` (250줄) | OS의 뇌 — 라우팅 표 + 절차 4개 |
| `scripts/telegram-channel.sh` | 상시 데몬 래퍼 (크래시·sleep/wake 자동 복구) |
| `scripts/morning-briefing.sh` | launchd 15분 폴링, 하루 1회 dedup, 일요일 분기 |
| `scripts/extract-clip.sh` | 유튜브 자막 추출 (429 재시도 내장) |
| `vault/` | 옵시디언 볼트 — 그냥 md 폴더. 연동 코드 0줄 |

**공개 경계:** 볼트에 건강·돈·관계까지 들어가서, 레포를 나중에 공개해도 안 새게 화이트리스트 gitignore로 잘랐다. **구조·규칙·설계문서는 커밋, 실데이터는 제외.**

**📖 사용법 치트시트:** [Dylan 사유공간 사용법](https://claude.ai/code/artifact/7f07b9bc-50e5-4519-955a-02fc78243efb)

### 🧗 삽질 (요약)

이번 주 삽질은 대부분 **"내 맥북 위에서 봇을 24시간 살려두기"** 에서 나왔다.

**1. 제일 나빴던 것 — 맥이 자다 깨면 봇이 조용히 먹통 (실전 2일간)**
프로세스는 살아있는데 텔레그램 연결만 죽어 있었다. 죽은 걸 아무도 모름. 헬스체크에서 **"20초 자려 했는데 벽시계가 90초 넘게 점프 = 맥이 서스펜드됐던 것"** 으로 감지해 강제 재시작. (덤: 텔레그램이 24시간 버퍼링해줘서 다운타임 중 던진 것도 복구 후 자동 라우팅된다.)

**2. 다른 프로젝트 창만 열어도 봇이 죽음**
텔레그램 봇 API는 폴러가 하나만 가능하다. 딴 폴더에서 Claude Code 세션을 열기만 해도 그 세션이 폴링을 가로챘다. 최종: 데몬만 자기 전용 설정을 `--settings`로 명시 로드.

**3. 새벽 3시 생각이 아침 9시로 기록됨**
맥이 꺼져 있을 때 던진 메시지는 켠 뒤에 처리되는데, 처리 시각을 찍고 있었다. 세컨브레인에서 **언제 그 생각을 했는지**는 내용만큼 중요하다. 발신 시각(`ts`)을 KST로 변환해 찍게 고침.

**4. "첨부는 형식이지 의도가 아니다"**
이미지를 던지면 "이거 뭐 하려는 거야?" 되묻게 해놨더니, 약속 잡은 카톡 스샷을 **읽지도 않고** 되물었다. → **먼저 읽고, 읽은 내용으로 라우팅.** 컨테이너 타입이 아니라 내용물이 의도를 정한다.

**5. 만들고 나서야 "뭘 던져야 하는지"를 나도 모름**
폰 앞에서 멈칫했다. 써보면서 알아낸 제약 — 인용 답장이 안 먹히고(봇은 뭘 인용했는지 모른다), 세션이 재시작되면 앞 대화를 잊고, 맥이 24시간 넘게 꺼져 있으면 유실된다. 그래서 치트시트를 만들었다.

<details>
<summary>나머지 삽질 6개 (launchd pty, PATH, 알림 도배, 브리핑이 자기 봇 죽임, 유튜브 429, 빈 메모가 인사이트 죽임)</summary>

- **launchd로 띄우니 봇이 대화를 안 함** — pty가 없으면 `--print` 모드로 떨어진다. `script`로 가짜 pty를 물려 해결.
- **텔레그램 MCP가 안 뜸** — launchd가 셸 PATH를 안 물려줘서 bun을 못 찾음. PATH 통째로 export.
- **밤새 알림 도배** — sleep/wake마다 "🔁 채널 복구됨" 핑. 진짜 크래시일 때만 보내게 분리.
- **모닝 브리핑이 자기 봇을 죽임** — 브리핑 `claude -p`도 플러그인을 로드해 두 번째 폴러가 됐다. 브리핑은 텍스트만 생성, 전송은 스크립트가 Bot API로 직접.
- **유튜브 429** — 쿠키 우회하려다 키체인·승인 팝업 지옥. 재시도 루프를 스크립트 안으로 밀고 allowlist엔 그 한 줄만.
- **빈 칸 하나가 인사이트를 죽임** — 추출 노트의 "나에게의 함의"가 `project-status.md`를 읽는데, 메모가 빈 프로젝트는 연결이 안 됐다. 메모 한 줄을 필수로.

</details>

### 💡 인사이트

**1. 세컨브레인의 어려운 부분은 수집이 아니라 출구다.**
다들 잘 모은다. 근데 다시 안 꺼낸다. 그래서 수집 도구엔 시간을 거의 안 썼고(볼트 = 그냥 md 폴더, 연동 코드 0줄), **"매주 일요일 아침에 강제로 튀어나온다"** 는 출구 하나에 설계를 다 걸었다. 방치가 구조적으로 불가능해야 한다.

**2. "매일 쓰게 되는가"를 합격 기준으로 잡으면 만들 게 달라진다.**
1주차 OS가 약했던 건 기능 부족이 아니라 발동 조건이 드물어서였다. 이 기준을 세우니 "새로 만들지 말고 이미 매일 쓰는 것에 붙여라"가 자동으로 답이 됐다.

**3. 라우팅을 코드가 아니라 마크다운 문서로 두면 확장이 미친듯이 싸다.**
이번 주에 유튜브·웹·첨부 3종을 붙였는데 파서를 짠 게 아니라 규칙 문단을 3개 썼다. 코드로 짰으면 아직도 정규식 만지고 있었을 것.

**4. 미래의 나를 못 믿을 땐, 판단 시점을 내 기억 밖에 둔다.**
"언제 구조를 바꿔야 하지"를 내가 기억할 리 없다. 그래서 100개 넘으면 주간 리뷰가 알아서 알려준다.

**5. 크루들한테 특히 — 봇을 만들 거면 "조용히 죽는 경우"를 먼저 설계해라.**
크래시는 티라도 나는데, 자다 깨서 연결만 죽은 건 이틀을 몰랐다. **살아있는지 스스로 확인하고 알려주는 장치가 기능보다 먼저다.**

**6. "개인용"은 안전하다는 뜻이 아니라 아직 안 막았다는 뜻이었다.**
사용법 가이드에 봇 핸들을 적다가 아차 싶었다. dylan-os엔 인증이 없다 — "나만 쓰니까". 근데 **핸들을 아는 사람 누구나 내 COO에게 말을 걸 수 있고, 그게 내 볼트에 그대로 쓰인다.** 1주차 "링크만 알면 내 API 크레딧 무제한"과 똑같은 모양. (다음 작업: chat_id 화이트리스트.)

**7. 봇과 클로드 앱은 경쟁이 아니라 분업이다.**
**봇은 캡처와 실행** — 내 볼트·투두·캘린더를 알고, 남기고, 행동한다. 폰에서 한 탭. **앱은 사고와 생산** — 깊은 멀티턴, 글·코드·리서치. 대신 내 OS를 모른다.
다리: **앱에서 깊이 생각하고, 결론 한 줄만 봇에 던져 OS에 남긴다.**
