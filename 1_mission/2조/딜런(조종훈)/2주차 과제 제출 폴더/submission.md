# 2주차 — 내 OS 구현하기 🚀

## 🎯 미션 1. 내 OS 만들기

**✅ 선택:** 내 삶을 돕는 OS

### 📐 기획

#### 새 OS를 안 만들고, 있던 걸 확장하기로 함

1주차에 만든 career-os가 OS로서 뭔가 약하게 느껴졌다. 이유를 파보니 "기능이 부족해서"가 아니라 **매일 안 써서**였다. 회사 리서치는 시간이 붕 뜰 때만 하는 거니까.

내 반복 함정이 이거다 — 새 걸 화려하게 벌이고 방치. 자기소개에도 썼지만 "내가 만든 스킬들은 나만을 위한 거였고 데일리로 자주 쓰는 건 아니었다."

그래서 이번 OS의 합격 기준을 "멋진가"가 아니라 **"매주 강제로 쓰게 되는가"** 로 잡았다.

이 기준으로 보면 **세컨브레인을 새로 만드는 게 최악**이다. PKM은 "수집만 하고 다시 안 봄"이 방치의 원조 공식이니까. 그리고 마침 `~/dev/dylan-os`(텔레그램으로 던지면 판단해서 정리해주는 개인 COO 봇)가 이미 세컨브레인의 3요소를 다 갖고 있었다.

| 세컨브레인 요소 | dylan-os에 이미 있던 것 |
|---|---|
| 입력 (던지면 정리) | 텔레그램 라우팅 — 캘린더/투두/inbox로 자동 분류 |
| 수집함 | `inbox.md` |
| 재소비 출구 | 매일 아침 브리핑 |

새로 만들면 80% 중복이다. → **dylan-os의 inbox를 옵시디언 볼트로 승격하는 확장 모듈**로 갔다.

#### 핵심 설계 원칙 하나

세컨브레인의 진짜 과제는 "잘 모으기"가 아니라 **"쌓인 게 반드시 다시 튀어나오는 출구"** 다.

출구 = **주간 리뷰를 일요일 아침 브리핑에 합체.** 매주 일요일 = 방치가 구조적으로 불가능.

#### 그리고 입력을 3종 더 뚫음

볼트를 만들고 나니 "폰으로 보다 만 유튜브·아티클·스샷"이 전부 볼트 밖에서 새고 있다는 게 보였다. 각각 별도 브레인스토밍 → 스펙 → 계획 사이클을 돌려서 붙였다.

### ⚙️ 구현

**최종 구조:**

```
텔레그램에 아무거나 던짐
   ↓  (CLAUDE.md 라우팅 규칙이 판단)
   ├─ 날짜+시간 있는 약속      → 구글 캘린더 이벤트
   ├─ 마감 있는 할 일          → 캘린더([마감] 종일) + todos.md 둘 다
   ├─ 날짜 없는 할 일          → todos.md
   ├─ "~~했음"                 → todos.md에서 [x] 처리
   ├─ 유튜브 링크              → 자막 긁어 요약+함의 → vault/clips/
   ├─ 웹 아티클 링크           → 본문 fetch → 요약+함의 → vault/clips/
   ├─ 이미지·PDF 첨부          → 비전/PDF로 읽고 → 내용에 따라 위 어디로든
   ├─ 기억할 개인 신호         → vault/inbox.md + #주제 태그
   └─ 일회성 잡담              → 답만 하고 저장 안 함 (inbox 오염 방지)

매일 아침  → 모닝 브리핑 (일정 + 투두 + 프로젝트 진척)
매주 일요일 → 거기에 주간 리뷰 합체 (지난 7일 inbox → 3블록 요약 → vault/weekly/)
```

**구현 방식이 좀 특이한 부분:** 라우팅이 **코드가 아니라 `CLAUDE.md`라는 규칙 문서**다. 텔레그램 데몬(`claude --channels`)이 상시 떠 있고, 메시지가 오면 그 문서를 읽고 판단해서 파일을 조작한다. 그래서 "유튜브 링크도 처리해줘" 같은 기능 추가가 **파서 짜는 게 아니라 규칙 한 문단 추가**로 끝난다.

**실제로 만진 것:**

| 파일 | 역할 |
|---|---|
| `CLAUDE.md` (250줄) | OS의 뇌 — 라우팅 표 + 브리핑/주간리뷰/추출 절차 4개 |
| `scripts/telegram-channel.sh` | 상시 데몬 래퍼 (크래시·sleep/wake 자동 복구) |
| `scripts/morning-briefing.sh` | launchd 15분 폴링, 하루 1회 dedup, 일요일이면 주간 분기 |
| `scripts/extract-clip.sh` | 유튜브 자막 추출 (429 재시도 내장) |
| `vault/` | 옵시디언 볼트 — 그냥 md 폴더. 연동 코드 0줄 |

**공개 경계 (구조는 public, 사색은 로컬):**

볼트 내용이 건강·돈·관계 같은 것까지 다 들어가는 곳이라, 나중에 레포를 공개해도 절대 안 새게 화이트리스트 gitignore로 잘랐다.

| 커밋되는 것 (= 남이 배울 구조) | gitignore (= 내 데이터) |
|---|---|
| `CLAUDE.md`, `scripts/`, `launchd/` | `inbox.md`, `todos.md`, `project-status.md` |
| `*.example.md` (빈 템플릿) | `vault/inbox.md`, `vault/clips/*`, `vault/weekly/*` |
| `vault/README.md`, `docs/` 스펙·계획 | `.obsidian/` |

> 아직 remote 없이 로컬 git으로만 돌린다. 공개 준비(분리)만 끝내둔 상태.

### 🧗 과정에서의 삽질

이번 주 삽질은 대부분 **"내 맥북 위에서 봇을 24시간 살려두기"** 에서 나왔다.

**1. launchd로 띄우니 봇이 대화를 안 함**
`claude --channels`를 launchd로 올렸더니 채널 모드가 아니라 `--print` 모드로 떨어졌다. 터미널(pty)이 없으면 대화형이 아니라고 판단하는 거였음. `script` 명령으로 가짜 pty를 물려서 해결.

**2. 그다음엔 텔레그램 MCP가 안 뜸**
launchd는 내 셸 PATH를 안 물려준다. 텔레그램 플러그인이 bun으로 도는데 bun을 못 찾고 있었음. 스크립트에서 PATH를 통째로 export.

**3. 제일 골 아팠던 것 — 다른 프로젝트 창을 열면 봇이 죽음**
텔레그램 플러그인을 전역 설정에 켜놨더니, career-os 같은 **딴 폴더에서 Claude Code 세션을 열기만 해도 그 세션이 봇 폴링을 가로챘다.** (텔레그램 봇 API는 폴러가 하나만 가능 — 새 인스턴스가 뜨면 이전 걸 죽인다.) 세션을 며칠 켜두면 데몬이 계속 굶는 것.

프로젝트 설정으로 옮겼더니 이번엔 dylan-os 폴더에서 VSCode를 열면 똑같이 터졌다. `--channels` 플래그를 안 줘도 `enabledPlugins`에 있으면 플러그인을 로드해버린다(Claude Code #38098).

최종: 프로젝트 설정에서 플러그인을 **완전히 빼고**, 데몬만 자기 전용 `scripts/daemon-settings.json`을 `--settings`로 명시 로드. 12초 관찰하면서 bun이 뜨는지 확인해가며 "`--channels`만으로는 플러그인이 안 뜬다"를 검증했다. 이제 일반 세션은 어디서 열든 폴링을 못 뺏는다.

**4. 맥이 자다 깨면 봇이 조용히 먹통 (실전 2일간)**
프로세스는 멀쩡히 살아있는데 텔레그램 롱폴 연결만 죽어 있었다. 죽은 걸 아무도 모르니까 제일 나쁜 종류의 버그. 20초마다 도는 헬스체크에서 **"20초 자려고 했는데 벽시계가 90초 넘게 점프했으면 = 맥이 서스펜드됐던 것"** 으로 감지해서 세션을 강제 재시작하게 했다. (덤: 텔레그램이 못 받은 메시지를 24시간 버퍼링해줘서, 다운타임 중 던진 것도 재시작 후 자동으로 라우팅된다.)

**5. 그거 고쳤더니 밤새 알림 도배**
sleep/wake마다 세션이 재시작되고, 재시작마다 "🔁 채널 복구됨" 핑이 왔다. 자다 깨서 보니 알림이 쌓여 있음. 진짜 크래시일 때만 핑 보내게 분리.

**6. 모닝 브리핑이 자기 봇을 죽임**
브리핑 스크립트가 `claude -p`로 도는데, 얘도 텔레그램 플러그인을 로드하면 두 번째 폴러가 돼서 데몬의 연결을 뺏는다. 브리핑은 **텍스트만 생성**하게 하고, 전송은 스크립트가 Bot API로 직접 curl 때리게 바꿨다.

**7. 새벽 3시 생각이 아침 9시로 기록됨**
맥이 꺼져 있을 때 던진 메시지는 맥을 켠 뒤에 처리된다. 그런데 inbox에 `date`(처리 시각)를 찍고 있었으니 새벽 감상이 아침 9시 기록으로 남았다. 세컨브레인에서 **언제 그 생각을 했는지**는 내용만큼 중요한데. 인바운드 메시지에 붙어오는 `ts`(발신 시각)를 KST로 변환해서 찍게 고침.

**8. 유튜브가 429로 막음**
자막 긁으려니 yt-dlp가 429(rate limit). 브라우저 쿠키를 물려서 우회하려 했더니 이번엔 키체인 접근 문제 + 폰으로 승인 팝업이 매번 날아옴(`ls`, `cat`, `rm` 하나하나). 결국 **쿠키 안 쓰고** 재시도 루프를 `extract-clip.sh` 스크립트 안으로 밀어넣고, 데몬 allowlist엔 그 스크립트 **한 줄만** 등록. 승인 팝업도 같이 사라짐.

**9. "첨부는 형식이지 의도가 아니다"**
이미지를 던지면 일단 "이거 뭐 하려는 거야?" 되묻게 만들어놨었다. 근데 친구랑 약속 잡은 카톡 스샷을 던졌더니 봇이 내용을 **읽지도 않고** 되물었다. 스샷 안에 약속이 들어 있으면 그건 그냥 약속이다. → **먼저 읽고, 읽은 내용으로 라우팅**하게 뒤집었다. (컨테이너 타입이 아니라 내용물이 의도를 정한다.)

**10. 빈 칸 하나가 인사이트를 죽임**
추출 노트의 "💡 나에게의 함의"는 `project-status.md`를 읽어서 "이게 내 프로젝트에 무슨 의미인지"를 쓰는 부분이다. 근데 프로젝트를 추가할 때 `- 메모:`를 비워둔 게 있었더니 함의가 연결이 안 됐다. 프로젝트 추가 시 메모 한 줄을 **필수로** 되묻게 규칙을 박음.

**11. 만들고 나서야 "뭘 던져야 하는지"를 나도 모름**
다 만들어놓고 정작 폰 앞에서 멈칫했다. 이거 던져도 되나? 봇한테 답장(인용)하면 알아듣나? 실제로 써보면서 알아낸 제약들:

- 텔레그램 **인용 답장이 안 먹힌다.** 봇은 뭘 인용했는지 모른다. "ㅇㅇ" 말고 대상을 명시해야 함.
- 세션이 재시작되면 **앞 대화를 잊는다.**
- 맥이 **24시간 넘게** 꺼져 있으면 그 전 메시지는 유실(텔레그램 버퍼 한계).

그래서 [사용법 치트시트](https://claude.ai/code/artifact/7f07b9bc-50e5-4519-955a-02fc78243efb)를 한 장 만들었다. **내가 만든 OS의 사용법을 내가 문서로 정리해야 했다는 게 좀 웃겼는데**, 쓰면서 "잡담은 안 담는다" 같은 규칙이 오히려 선명해졌다.

- 입력 4종: 텍스트 · 유튜브 · 웹 아티클 · 이미지/PDF 첨부
- [사용법 치트시트](https://claude.ai/code/artifact/7f07b9bc-50e5-4519-955a-02fc78243efb) — 뭘 던지고 뭘 안 던지나, 어디로 가나

**inbox 형식** (내용은 예시로 대체):

```
- 2026-07-09 10:06: [영상] 기존의 마케팅 퍼널은 죽었다고 — 핵심 …
                    → vault/clips/2026-07-09-….md #콘텐츠
- 2026-07-08 22:14: (개인 신호 한 줄) #건강
```

주제 태그 사전 8개(`#크립토 #커리어 #콘텐츠 #사업 #건강 #관계 #재정 #단상`)로 고정 — 옵시디언에서 태그 클릭하면 스레드가 살아난다.

**주간 리뷰 노트 형식** (실내용은 사적이라 생략):

```
# 주간 리뷰 · 2026-W28
## 🔎 이번 주 관심사     ← 반복해서 등장한 주제
## 💡 배운 것 / 통찰      ← 관점 변화
## 🧭 다음 방향           ← 이게 가리키는 다음 액션
```

**아직 안 끝난 것 (정직하게):**

- **일요일 자동 실행은 아직 한 번도 안 돌았다.** 볼트를 수요일(7/8)에 만들어서 아직 일요일이 안 지나감. 지금 있는 W28 노트는 절차를 **수동으로 호출해서** 만든 것이고, 스크립트의 일요일 분기는 `DOW_OVERRIDE` 테스트 심으로만 검증했다. **첫 실주행은 7/12(일).**
- 레포 공개 안 함. 개인 데이터 분리만 끝내둔 상태.
- inbox가 100개 넘으면 "B단계(노트 파일 분리 + 위키링크) 할 때"라고 봇이 알아서 알려주게 해뒀다. 지금 17개.

### 💡 알게 된 인사이트 & 공유하고 싶은 내용

**1. 세컨브레인의 어려운 부분은 수집이 아니라 출구다.**
노션이든 옵시디언이든 다들 잘 모은다. 근데 다시 안 꺼낸다. 그래서 이번엔 수집 도구를 고민하는 데 시간을 거의 안 쓰고(볼트 = 그냥 md 폴더, 연동 코드 0줄), **"매주 일요일 아침에 강제로 튀어나온다"** 는 출구 하나에 설계를 다 걸었다. 방치가 구조적으로 불가능해야 한다.

**2. "매일 쓰게 되는가"를 합격 기준으로 잡으면 만들 게 달라진다.**
1주차 OS가 약했던 건 기능 부족이 아니라 발동 조건이 드물어서였다. 이 기준을 세우니까 이번엔 "새로 만들지 말고 이미 매일 쓰는 것에 붙여라"가 자동으로 답이 됐다.

**3. 라우팅을 코드가 아니라 마크다운 문서로 두니까 확장이 미친듯이 싸다.**
`CLAUDE.md`에 규칙 표 한 줄 추가하면 새 입력 타입이 붙는다. 이번 주에 유튜브·웹·첨부 3종을 붙였는데 새 파서를 짠 게 아니라 절차 문단을 3개 썼다. 파싱·분기를 코드로 짰으면 아직도 정규식 만지고 있었을 것.

**4. 미래의 나를 못 믿을 땐, 시스템에 알람을 박아둔다.**
inbox가 평평한 단일 파일이라 항목이 많아지면 연결이 무너진다. 근데 "언제 구조를 바꿔야 하지"는 내가 기억할 리가 없다. 그래서 100개 넘으면 주간 리뷰가 알아서 "B단계 할 때"라고 알려주게 했다. **판단 시점을 내 기억 밖에 두는 것.**

**5. 개인 OS일수록 공개 경계를 처음부터 그어야 한다.**
이 볼트엔 건강·돈·관계까지 들어간다. 근데 남이 클론해서 구조는 배웠으면 좋겠다. 그래서 `.env.example` 패턴을 그대로 가져와서 **"구조·규칙·설계문서는 커밋, 실데이터는 화이트리스트 gitignore"** 로 잘랐다. remote 붙이기 전에 해서 다행이었다 — 이미 3개 파일이 추적되고 있었다.

**6. 봇과 클로드 앱은 경쟁이 아니라 분업이다.**
처음엔 "봇으로 다 하면 되지 않나" 싶었는데 아니었다. **봇은 캡처와 실행** — 내 볼트·투두·캘린더를 알고, 남기고, 행동한다. 폰에서 한 탭. **앱은 사고와 생산** — 깊은 멀티턴, 글·코드·리서치. 대신 내 OS를 모르고 아무것도 안 남긴다.
다리는 이거다: **앱에서 깊이 생각하고, 결론 한 줄만 봇에 던져 OS에 남긴다.** 대화 전체를 옮길 필요가 없다.

**7. 크루들한테 특히 권하고 싶은 것:** 봇을 만들 거면 **"조용히 죽는 경우"** 를 먼저 설계해라. 크래시는 티라도 나는데, 맥이 자다 깨서 연결만 죽은 건 이틀을 몰랐다. 살아있는지 스스로 확인하고 알려주는 장치가 기능보다 먼저다.

**8. 개인 봇에도 인증은 필요하다 (이번 주에 또 배움).**
사용법 가이드를 쓰다가 봇 핸들을 적어놓고 아차 싶었다. dylan-os엔 인증 분기가 아예 없다 — "나만 쓰니까". 근데 **핸들을 아는 사람 누구나 내 COO에게 말을 걸 수 있고, 그게 내 볼트에 그대로 쓰인다.** 1주차에 "링크만 알면 내 API 크레딧 무제한"으로 데인 것과 똑같은 모양이다. 개인용이라는 말은 안전하다는 뜻이 아니라 **아직 안 막았다는 뜻**이었다. (다음 작업: chat_id 화이트리스트.)
