# 1주차 — 나만의 OS 만들기 🛠️

> 미션을 진행하며 **과정과 결과물**을 기록해주세요. (다 못 채워도 OK, 한 것 위주로!)

## 🎯 미션 1. 내 OS 재료 찾기
> 인터뷰 스킬(아이데이션)로 "내 삶에 필요한 게 뭔지" 찾기

- **과정 (어떻게 찾았나):**
  **내가 직접 만든 `service-blueprint` 스킬**(아이디어를 12단계로 발전시키는 서비스 기획 파이프라인)을 호출하는 것으로 시작했다. 재료는 내 일상에서 나왔다 — 나는 매일 혈당을 체크하고, 운동을 기록하고, 인바디를 재는 건강 관리형 인간이고, 친구들과 "한 달 12회 운동 채우기" 클럽을 운영 중이다. 이 둘을 조합하면 서비스가 되지 않을까? 스킬이 단계마다 던지는 질문(누구의 어떤 페인포인트인가, 무엇을 과감히 뺄 것인가)에 답하며 아이디어를 좁혀갔다.

- **결과:**
  기존 운동 앱의 페인포인트 3가지를 도출했다 — ①너무 복잡하다(기능·수치 과다) ②혼자라 재미없고 작심삼일 ③모르는 사람과의 과한 경쟁·무리. 여기서 "친구들끼리 서로 독려해서, 잘 못해도 꾸준히만 하면 되는" 가벼운 운동 기록 서비스라는 방향이 나왔다.

- **느낀 점:**
  처음엔 "내 건강 데이터 트래킹"이 핵심인 줄 알았는데, 내 스킬의 질문에 답하다 보니 진짜 원하는 건 "친구들의 응원으로 꾸준해지는 것"이었다. 스킬이 나에게 묻고, 내가 답하며 심장을 찾아가는 과정 — 이게 내 기획 OS가 돌아가는 방식이다.

## 🧩 미션 2. 내 OS 기획
> 인터뷰 결과 + 세션 내용(흐민·배짱·키노) 활용해 기획

- **기획 내용:**
  **AzaazA (아자아자)** — 종목 불문, 강도가 아닌 *빈도*만 기록하고, 아는 친구들끼리 하루 단위로 응원하는 운동 기록 웹 서비스.
  - 슬로건: **"혼자 말고, 같이 아자아자."**
  - 핵심 기능: 어떤 운동이든 한 번의 탭으로 기록 → 종목별 컬러 점이 쌓이는 "채워지는 달력" → 하루를 마치면 수동으로 "오늘 운동 등록" → 그룹 피드에 하루치 카드 1장 발행 → 친구들이 좋아요·댓글로 응원.
  - **내 `service-blueprint` 스킬의 12단계를 처음부터 끝까지 완주했다:** 가치 정의 → BM → 경쟁 벤치마킹(챌린지앱 vs 트레이너 CRM 사이 빈자리 발견) → 타겟 → 브랜딩·슬로건 → 디자인 톤(라임+퍼플+크림) → 데이터 구조 → 기능 정의 → IA → 와이어프레임 → 디자인 가이드 → 개발 핸드오프. 각 단계마다 스킬이 선택지와 리스크를 차려주고, 내가 결정하는 방식으로 진행했다.

- **막혔던 점 / 어떻게 풀었나:**
  - **메인 화면을 피드로 할지, 내 기록으로 할지** 갈렸다. → 스킬이 던진 "사용자가 매일 여는 이유가 뭔가?"라는 질문으로 판단. 내 동기부여의 원천은 친구들의 응원이므로 **피드를 첫 화면**으로 확정.
  - **댓글을 운동 하나하나에 달지, 하루 단위로 달지** 고민. → "한 사람의 하루 = 카드 1장"(DayLog) 구조로 정리하고, 응원은 하루 전체에 붙게 설계. 발행도 자동이 아닌 **수동 등록**으로 단순화.
  - 타겟이 중간에 PT 강사 쪽으로 흘렀다가, "내가 만들고 싶은 건 내 기록과 친구 응원"이라는 원점으로 돌아와 다시 잡았다. 기획은 직진이 아니라 왕복이었다.
  - **그리고 이 과정에서 스킬의 빈틈도 발견했다** — 슬로건을 잡는 단계가 없었고, 와이어프레임 다음에 뭘 해야 하는지 프로세스 안내가 없었다. (→ 미션 3에서 스킬 업그레이드로 해결)
