# 1주차 — 나만의 OS 만들기 🛠️

> 미션을 진행하며 **과정과 결과물**을 기록해주세요. (다 못 채워도 OK, 한 것 위주로!)

## 🎯 미션 1. 내 OS 재료 찾기
> 인터뷰 스킬(아이데이션)로 "내 삶에 필요한 게 뭔지" 찾기

- **과정 (어떻게 찾았나):**
  스폰지 인터뷰 스킬로 5스텝 인터뷰를 진행했다. "관계"에서 출발했는데, 처음엔 신념("관심 갖고 노력하면 관계가 좋아지고 성과는 따라온다")만 맴돌았다. 각도를 바꿔가며 파고드니, 막연한 믿음이 아주 구체적인 지점으로 좁혀졌다 — **"2조도 회사 팀원도, 챙기는 걸 혼자 다 짊어지고 내 책임이라 생각한다."** 지난주(발표·이기적공유·운영진/오프라인 모임이 겹친 주)엔 조원 챙김이 제일 먼저 밀려 미안함이 남았던 게 결정적 단서였다.

- **결과:** 🟡 내 OS 재료 카드

  | 칸 | 내용 |
  |----|------|
  | 영역 | 일 + 삶 둘 다 — 사람을 챙기고 이끄는 **관계·리더십 전반** |
  | 걸리는 지점 | 혼자서만 애쓰고 "내 책임"이라 다 떠안음. 바쁜 주엔 조원 챙김이 제일 먼저 밀림 |
  | 지금은 | 챙김이 전부 내 머릿속·손 하나에 걸려 있음 → 일 겹치면 밀리고 미안함이 남음 |
  | OS가 된다면 | **서로 챙기는 판** — 조원끼리 서로 안부·진행을 챙기게 만드는 구조. 내가 다 안 해도 굴러감 |
  | 한 문장 | *"서로가 서로를 챙길 때, 우리는 서로에게 더 좋은 사람·더 좋은 관계가 되고, 더 끈끈해지고 더 성장한다."* |
  | 첫 한 걸음 | 2조를 짝으로 묶어 서로 챙기게 하는 이벤트부터 |

- **느낀 점:**
  "서로가 서로를 위하면 해결된다"는 평소 신념이, 인터뷰를 거치며 **혼자 짊어지는 구조를 바꾸자**는 실행 과제로 또렷해졌다. 내가 열심히 하는 게 문제가 아니라, **모든 게 나 하나에 걸려 있는 구조**가 문제였다는 걸 알았다.

## 🧩 미션 2. 내 OS 기획
> 인터뷰 결과 + 세션 내용(흐민·배짱·키노) 활용해 기획

- **기획 내용: 「마음배달」 — 2조 마니또 이벤트 OS**

  재료 카드의 "서로 챙기는 판"을 실제 프로그램으로 구체화했다. 한 줄 정체성은 **"누군지 모른 채 챙기고, 마지막 날 밝혀지는 다정한 챙김."**

  - **무대·대상:** 슬랙(2조 채널) · 2조 12명 (확장 가능성 열어둠)
  - **형태:** 6주 중 2주차에 마니또 뽑기(온라인 조별모임) → 3주차 오프라인 모임에서 발표. 반응 좋으면 앙코르(2회차는 실시간 웹 버전으로 확장)
  - **두 가지 핵심 가치(척추):**
    1. **몰래 그러나 진심** — 익명이라는 장치가 부끄러움을 덜어, 오히려 더 솔직한 응원을 꺼내게 함
    2. **드러나는 재미** — "혹시 이 사람인가?" 추측하고 밝혀지는 과정 자체가 이벤트
  - **핵심 페르소나 & 엔진 구조:** 대상은 "마음은 깊은데 표현의 문턱에 막힌 사람". 먼저 움직이는 두 엔진(①조용한 성실러 ②리액션 요정)이 만든 따뜻하고 재밌는 판을, 관망하는 사람들(③완주 갈증러 ④신중러)이 보고 저절로 스며들게 설계 ("줄 서는 가게" 원리)
  - **진행 4단계:** ① 마니또 배정(웹봉투 DM) → ② 주간 앵커 미션 + 주중 깜짝 미션(채널 안 다정한 챙김) → ③ "누굴까?" 추측 스레드 → ④ 3주차 오프라인 발표(정체 공개 + 선물)
  - **성공 지표:** 매출·전환이 아니라 **참여자의 연결과 온기.** '누군가 나를 챙기고 있다'는 감각이 AI 공부를 혼자가 아니라 함께 완주하게 만든다.
  - 전체 운영 규칙은 Claude 프로젝트 지침 **「마음배달 운영 지침 v1.1」** 로 문서화했다.

- **막혔던 점 / 어떻게 풀었나:**
  - **익명 챙김이 '감시·스토킹'으로 읽힐 위험** → 톤 규칙으로 못박음: '지켜본다' 뉘앙스 금지, 프레임은 언제나 '마음을 배달하기'. '몰래'는 🤫·'살짝'·'한 스푼' 같은 장치로 귀여운 비밀로 전환.
  - **2주차라는 낯선 시점의 참여 저조 우려** → 관망하는 사람을 설득하지 않고, 먼저 즐겁게 움직이는 엔진 ①②가 판을 만들어 끌어들이는 구조로 해결.
