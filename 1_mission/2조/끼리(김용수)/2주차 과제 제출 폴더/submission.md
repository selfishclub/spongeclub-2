## 🎯 미션 1. 내 OS 만들기
> **[ 내 삶을 돕는 OS ]** 또는 **[ 콘텐츠 OS ]** 중 하나를 선택해 완성해주세요.

**✅ 선택:** 콘텐츠 OS (블로그 2개 운영 — 개인 블로그 / 끼리 블로그)

### 📐 기획
> 무엇을, 왜, 어떻게 만들지

두 개의 블로그를 각각 별도 OS로 설계했습니다.

1. **용인 인플루언서 OS (개인 블로그)**
   - 만다라트 캡처를 기반으로 "용인 지역 인플루언서"라는 목표를 확인하고 그에 맞춰 콘텐츠 체계를 재설계
   - 콘텐츠 카테고리를 여행/데이트코스, 맛집·카페, 부동산, 정책 4개로 확정
   - Notion DB 4개로 구조화: 콘텐츠 파이프라인 / 로컬자산 DB / 성과트래커 / 마일스톤 로드맵

2. **끼리 블로그 OS (별도 블로그, 수익화 목적)**
   - 기존에 용인정보 + 주식매매일지가 섞여 있던 콘텐츠를 수익화 중심으로 전환하기로 방향 확정
   - 첫 수익화 채널로 네이버 브랜드커넥트(쇼핑커넥트) 선정
   - Notion DB 3개로 구조화: 콘텐츠파이프라인 / 수익트래커 / 제휴협찬관리

### ⚙️ 구현
> 실제로 만든 것 (링크·스크린샷 — 이미지는 `이미지첨부/` 폴더에)

- 01~04, 06~08번 CSV 파일을 만들어 Notion에 `/import`로 DB 실제 생성
- 두 OS를 별도 Notion 페이지로 분리 정리 (처음에 파일을 잘못 첨부한 실수를 관계형(Relation) 연결로 옮겨서 수정)
- 각 DB 속성 타입 정리 (선택/날짜/숫자/체크박스/URL)
- DB 간 관계형(Relation) 연결, 칸반 보드·타임라인 뷰 추가
- 콘텐츠 생성 자동화용 프롬프트 템플릿 2종 제작
  - 05번: 네이버 블로그용 콘텐츠 초안 템플릿 (경험 기반, 사진 첨부 활용)
  - 09번: 끼리 블로그용 콘텐츠 초안 템플릿 (제휴 고지 문구 포함)
- 두 프로젝트 모두 Claude Project로 만들어 지침(instructions) 등록 완료

### 🧗 과정에서의 삽질
> 막혔던 지점, 시도한 방법, 어떻게 풀었는지 솔직하게

- Notion에 파일을 잘못 첨부하는 실수가 있어 관계형(Relation) 연결 방식으로 다시 정리
- 네이버 자동 포스팅을 검토했으나 진정성 문제와 기술 난이도 리스크가 있어 비추천으로 결론, 수동 붙여넣기 방식 유지하기로 결정
- 대안으로 Claude Cowork 예약 작업(Scheduled Tasks)을 활용해 아이디어 수집·DB 정리를 자동화하는 방안을 논의 (아직 설정 전, 다음 단계로 남겨둠)

### ✅ 결과물
> 완성한 것 / 작동 화면

- 용인 인플루언서 OS, 끼리 블로그 OS 각각 Notion 기반 뼈대(DB 구조 + 뷰) 완성
- 두 OS 모두 Claude Project로 연결되어 콘텐츠 초안 생성 준비 완료
- 다음 단계: 실제 콘텐츠 발행 + Cowork 예약 작업 설정

### 💡 알게 된 인사이트 & 공유하고 싶은 내용
> 하면서 깨달은 것, 크루들과 나누고 싶은 것

- 목적이 다른 콘텐츠(정보성 vs 수익화)는 하나의 블로그/DB에 섞기보다 처음부터 OS를 분리하는 게 관리가 훨씬 쉬웠음
- 자동화라고 다 좋은 건 아니고, 네이버 자동 포스팅처럼 플랫폼 정책·진정성 리스크가 있는 영역은 사람이 직접 하는 게 맞다는 판단
- Notion 관계형(Relation)을 활용하면 파일 첨부보다 훨씬 유연하게 DB 간 데이터를 연결할 수 있다는 걸 실수하면서 배움
