# 1주차 — 나만의 OS 만들기 🛠️

> 미션을 진행하며 **과정과 결과물**을 기록해주세요. (다 못 채워도 OK, 한 것 위주로!)

## 🎯 미션 1. 내 OS 재료 찾기
> 인터뷰 스킬(아이데이션)로 "내 삶에 필요한 게 뭔지" 찾기
- **과정 (어떻게 찾았나):** Claude Code 인터뷰 스킬로 "요즘 일상에서 가장 반복적으로 귀찮은 일"부터 물어가며 점점 구체화했다. 회사 업무보다 개인 일정 관리에서 진짜 문제를 발견했다 — 큰 일정(예: 친구 방문) 하나 안에 숨어있는 여러 하위 할일들(청소 예약, 장보기, 남편 일정 확인 등)을 빠짐없이 적절한 타이밍에 처리하는 게 힘들다는 것.
- **결과:** 지금은 큰 일정만 알림앱에 걸고 세부 할일은 카톡으로 적어두는데, 놓치진 않지만 계속 "놓칠까봐" 확인하는 피로감이 크고, 특히 급하지 않은 중요한 일(병원 정기검진 등)은 계속 미루게 된다는 걸 발견했다. 또한 알림이 와도 "지금 뭘 해야 하는지" 구체적 액션이 안 그려지면 미루게 된다는 심리적 패턴도 확인했다.
- **느낀 점:** 문제는 "일정을 잊어버리는 것"이 아니라, 하위 할일이 눈에 안 보이는 상태에서 매번 스스로 다시 판단해야 하는 부담이었다.

## 🧩 미션 2. 내 OS 기획
> 인터뷰 결과 + 세션 내용(흐민·배짱·키노) 활용해 기획
- **기획 내용:**
  - **한 줄 설명**: 큰 일정 하나를 던져두면, 그 안에 숨은 하위 할일들을 자동으로 찾아내고, 적절한 때에 구체적으로 리마인드하고, 미루면 어떤 위험이 있는지 알려주고, 진행하면서 쌓이는 결정들을 계속 반영해 계획을 정교하게 다듬어가는 개인 비서
  - **어떻게 작동하나 (사용 흐름)**
    1. 텔레그램에 "8/8 친구들 놀러옴, 남편도 나가있어야 함"처럼 큰 일정을 자연어로 보낸다
    2. 일정 분해 엔진이 이를 하위 할일(청소 대행 예약, 장보기, 남편 일정 확인 등)로 자동으로 쪼개서 기록해둔다
    3. 정해진 시점이 되면 "지금 청소 대행 예약 전화하세요"처럼 구체적인 액션 문장으로 텔레그램에 알려준다
    4. "완료" 또는 "미룰래"로 답하면서 그 김에 결정된 디테일(예: "완료, 남편 3시 출발")을 같이 보내면, 그 디테일이 기록에 쌓이고 관련된 다른 하위 할일의 타이밍도 자동으로 조정된다
    5. 미루기를 선택하면 미뤘을 때 뭐가 위험한지 한 줄로 알려준다
    6. 매주 월요일에는 이번 달 안에 마감인 하위 할일들을 소속 대일정과 함께 정리해 다이제스트로 보내준다 (다른 달 대일정의 선행 작업도 포함)
  - **구성 요소**
    - 이벤트 기록: 대일정별 제목·날짜·누적 디테일·하위 할일 목록(상태/리마인드 시점/액션 문장)을 담은 기록
    - 일정 분해 엔진: 특정 일정 유형에 종속되지 않는 범용 구조 (분해 결과는 언제든 대화로 수정 가능)
    - 리마인드 스케줄러: 주기적으로 오늘 처리할 하위 할일을 확인해 발송
    - 디테일 누적 & 재조정: 응답으로 들어온 디테일을 기록에 반영하고 관련 타이밍을 조정
    - 리스크 안내: 미룰 때 위험을 안내
    - 주간 다이제스트: 매주 월요일 이번 달 마감 항목을 소속 대일정과 함께 정리해 발송
- **막혔던 점 / 어떻게 풀었나:** 처음엔 범용 시스템과 시나리오 하나 사이에서 범위를 고민했으나, 분해 엔진 자체가 특정 일정에 종속되지 않는 구조라는 걸 확인하고 시스템은 범용으로 설계했다. 대신 이번 주 검증은 성격이 다른 두 예시(① 친구 방문 — 단기·손님맞이형, ② 9월 신혼여행 — 장기·준비형)로 진행해 실제로 다양한 일정에서 잘 작동하는지 확인하기로 했다.

## ⚙️ 미션 3. 내 OS 구현
> 실제로 만들어본 것 (클로드코드 '채널' 기능 활용 OK)
- **결과물:** Claude Code '텔레그램 채널' 기능으로 실제 작동하는 봇을 구현했다. `일정뽀개기OS/` 폴더에 이벤트 기록 형식과 동작 규칙(CLAUDE.md)을 만들고, 친구 방문(8/8)·9월 신혼여행 두 예시로 범용성까지 검증했다.
  - **일정 분해**: 자연어로 큰 일정을 보내면 하위 할일 + 리마인드 시점 + 구체적 액션 문장으로 자동 분해 (성격이 다른 두 일정 모두 성공, 심지어 소요시간·요일까지 스스로 고려)
  - **오늘 할 일 조회 & 완료/미루기**: `[오늘 할 일]` 형식으로 번호 매겨 보여주고, "완료-1" / "미룰래-2"처럼 번호로 답장하면 처리됨. 완료 시 디테일이 누적되고, 미루면 리스크를 안내
  - **자유 수정**: "청소업체 3일 전에 추가해주고 예약은 2주 전으로 해줘"처럼 대화로 언제든 하위 할일을 추가/수정 가능
  - **월간 일정**: 대일정별로 그룹지어 정리, 완료된 항목은 취소선으로 표시
  - **일일 자동 리마인드**: 매일 아침 9:33 자동 체크 예약 (단, Claude Code 세션이 켜져있는 동안만 작동하는 라이브 데모 버전 — 세션 종료 시 예약도 함께 사라짐. 상시운영은 다음 단계 과제)
- **링크 / 스크린샷:** `이미지첨부/` 폴더 참고
  1. `01_일정분해_친구들방문.jpeg` — 큰 일정 입력 → 자동 분해 결과
  2. `02_오늘일정_완료미루기.jpeg` — 오늘 할 일 조회 → 번호 응답으로 미루기 처리 + 리스크 안내
  3. `03_자유수정_하위할일추가.jpeg` — 대화로 하위 할일 추가/수정
  4. `04_월간일정_그룹핑.jpeg` — 월간 일정 (대일정별 그룹 + 완료 취소선)
  5. `05_주간일정_보너스.jpeg` — (설계에 없던 "주간 일정"도 즉석으로 물어봤는데 알아서 응답한 보너스 케이스)
