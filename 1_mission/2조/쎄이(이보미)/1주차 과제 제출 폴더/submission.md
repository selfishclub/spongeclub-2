## 🎯 미션 1. 내 OS 재료 찾기
> 인터뷰 스킬(아이데이션)로 "내 삶에 필요한 게 뭔지" 찾기
- **과정 (어떻게 찾았나):** Claude Code 인터뷰 스킬로 "요즘 일상에서 가장 반복적으로 귀찮은 일"부터 물어가며 점점 구체화했다. 회사 업무보다 개인 일정 관리에서 진짜 문제를 발견했다 — 큰 일정(예: 친구 방문) 하나 안에 숨어있는 여러 하위 할일들(청소 예약, 장보기, 남편 일정 확인 등)을 빠짐없이 적절한 타이밍에 처리하는 게 힘들다는 것.
- **결과:** 지금은 큰 일정만 알림앱에 걸고 세부 할일은 카톡으로 적어두는데, 놓치진 않지만 계속 "놓칠까봐" 확인하는 피로감이 크고, 특히 급하지 않은 중요한 일(병원 정기검진 등)은 계속 미루게 된다는 걸 발견했다. 또한 알림이 와도 "지금 뭘 해야 하는지" 구체적 액션이 안 그려지면 미루게 된다는 심리적 패턴도 확인했다.
- **느낀 점:** 문제는 "일정을 잊어버리는 것"이 아니라, 하위 할일이 눈에 안 보이는 상태에서 매번 스스로 다시 판단해야 하는 부담이었다.

## 🧩 미션 2. 내 OS 기획
> 인터뷰 결과 + 세션 내용(흐민·배짱·키노) 활용해 기획
- **기획 내용:**
  - **컨셉**: 큰 일정 하나를 텔레그램에 입력하면, 자동으로 하위 할일을 분해하고, 적절한 타이밍에 구체적 액션으로 리마인드하며, 미루면 리스크를 알려주고, 결정된 디테일이 쌓이면서 계획이 계속 정교해지는 개인 비서
  - **① 이벤트 기록**: 대일정마다 하나의 기록 (제목·날짜, 누적 디테일, 하위 할일 목록과 각각의 상태·리마인드 시점·구체 액션 문장)
  - **② 일정 분해 엔진**: 텔레그램으로 새 대일정이 자연어로 들어오면 Claude가 하위 할일로 자동 분해 — 특정 일정 유형에 종속되지 않는 범용 구조 (분해 결과는 언제든 대화로 수정 가능)
  - **③ 리마인드 스케줄러**: 주기적으로(cron) 오늘 처리할 하위 할일을 확인해 구체적 액션 문장으로 텔레그램 발송
  - **④ 디테일 누적 & 재조정**: "완료" 또는 "미루기" 응답에 디테일(예: "완료, 남편 3시 출발")을 함께 실어 보내면, 그 디테일이 이벤트 기록에 반영되고 관련 하위 할일의 타이밍이 자동 조정됨
  - **⑤ 리스크 안내**: 미루기 응답 시 미뤘을 때의 리스크를 한 줄로 안내
  - **⑥ 주간 다이제스트**: 매주 월요일, 이번 달이 마감인 하위 할일들을 소속 대일정과 함께 표시해 발송 (다른 달 대일정의 선행 작업도 포함)
  - **참고한 세션 내용**: 흐민 세션의 "인풋→저장→연결→제안→발행" 루프와 텔레그램+Claude Code 채널 연동 방식을 차용했고, 배짱의 "온맘" 사례(매일 정해진 시간 자동 메시지 발송)를 리마인드 스케줄러의 선례로 참고했다
- **막혔던 점 / 어떻게 풀었나:** 처음엔 범용 시스템과 시나리오 하나 사이에서 범위를 고민했으나, 분해 엔진 자체가 특정 일정에 종속되지 않는 구조라는 걸 확인하고 시스템은 범용으로 설계했다. 대신 이번 주 검증은 성격이 다른 두 예시(① 친구 방문 — 단기·손님맞이형, ② 9월 신혼여행 — 장기·준비형)로 진행해 실제로 다양한 일정에서 잘 작동하는지 확인하기로 했다.
