## 🎯 미션 1. 내 OS 만들기
> **[ 내 삶을 돕는 OS ]** 또는 **[ 콘텐츠 OS ]** 중 하나를 선택해 완성해주세요.

**✅ 선택:** 내 삶을 돕는 OS — 1주차에 만든 **일정뽀개기OS**를 이어서 "상시운영" 단계로 완성

### 📐 기획
> 무엇을, 왜, 어떻게 만들지

1주차에 만든 일정뽀개기OS는 아직 완벽하게 작동하지 않고, 자잘하게 고쳐야 할 기능적인 부분들이 남아 있다. 다만 일주일 써보니 가장 불편했던 건 **맥을 계속 켜놔야 한다는 것**이었다.

그래서 2주차는 기능 다듬기보다 이 문제부터 해결하기로 했다. 목표는 "맥을 신경 쓰지 않아도 매일 아침 리마인드가 오는 상태". 두 단계로 나눴다.

- **1단계 — 맥에서 상시운영**: 세션 없이도 맥이 정해진 시간에 스스로 Claude를 깨워서 리마인드를 보내게 하기 (launchd 활용)
- **2단계 — 맥 없이 운영 (리서치)**: 맥이 꺼져 있거나 내가 여행 중이어도 돌아가는 구조 찾기

### ⚙️ 구현
> 실제로 만든 것 (링크·스크린샷 — 이미지는 `이미지첨부/` 폴더에)

**1단계: launchd 기반 상시운영 (구현 완료, 일주일 실운영)**

맥의 예약 실행 시스템인 launchd에 매일 아침 9:30 작업을 등록했다. 구조는 두 파일이다:

- `~/Library/LaunchAgents/com.bomilee.dailyreminder.plist` — "매일 9:30에 이 스크립트를 실행해라"라는 예약
- `~/.claude/scripts/daily_reminder.sh` — 일정뽀개기OS 폴더로 이동한 뒤 Claude Code를 **headless 모드**로 실행하는 스크립트:

```bash
#!/bin/zsh
cd "/Users/bomilee/Desktop/claude/일정뽀개기OS"
claude -p "오늘 리마인드 체크해줘" --dangerously-skip-permissions
```

핵심은 `claude -p` (headless 실행)다. 세션을 열어두지 않아도 launchd가 매일 아침 Claude를 새로 실행하고, Claude는 `CLAUDE.md`의 동작 규칙과 `events/` 파일만 읽고 판단해서 텔레그램으로 리마인드를 보낸다. 데이터는 파일에, 규칙은 CLAUDE.md에 분리해둔 1주차 구조라서 대화 기억이 없는 새 세션에서도 동일하게 작동한다.

**2단계: 맥 없이 운영 — 리서치 & 테스트**

launchd 방식은 맥이 꺼져 있으면 실행되지 않는다. 그래서 여러 옵션을 비교하고 클라우드 구축도 직접 테스트해봤다.

- 클라우드에 구축하는 것을 테스트해본 결과, 원활하지 않음을 확인 — 텔레그램 채널이 결국 맥의 로컬 세션에 붙어 있어서, 이 구조로는 결국 맥을 켜놔야 했다
- 그래서 세 조각을 전부 클라우드로 옮기는 아키텍처로 결론:
  - **데이터**: GitHub 저장소 → GitHub 서버에 있음 (맥 아님)
  - **실행**: 루틴(스케줄 에이전트) → Anthropic 클라우드에서 새 세션이 뜸 (맥 아님)
  - **인박스**: 디스코드 → 디스코드 서버에 있음 (맥 아님)
- 대안으로 서버 구매(VPS)도 검토했으나, **우선 위 클라우드 조합으로 먼저 돌려보고 토큰을 얼마나 쓰는지 본 뒤 서버로 넘어갈지 결정**하기로 했다

### 🧗 과정에서의 삽질
> 막혔던 지점, 시도한 방법, 어떻게 풀었는지 솔직하게

1. **나흘간 리마인드가 안 옴**: 7/6 첫 자동 발송은 성공했는데, 7/7~7/10 나흘간은 리마인드가 안 왔다. 로그를 확인해보니 `API Error: Connection closed` / `Request timed out`으로 실행이 실패하고 있었고, 7/11에야 5건이 "지연됨"으로 몰아서 발송됐다. 아직 실패 시 재시도 같은 건 없어서, 예약 실행만으로는 매일 안정적으로 나간다고 보장할 수 없다는 걸 확인했다.

2. **클라우드 구축 테스트**: 맥 없이 돌리려고 클라우드 구축을 테스트했지만, 텔레그램 채널이 로컬 세션에 묶여 있어 결국 맥이 필요하다는 걸 확인했다. 실행만 옮기는 게 아니라 데이터·실행·인박스를 전부 옮겨야 해서, GitHub + Anthropic 클라우드 루틴 + 디스코드 조합으로 설계를 다시 잡았다.

### ✅ 결과물
> 완성한 것 / 작동 화면

- **세션 없이 돌아가는 일정뽀개기OS**: 매일 아침 9:30에 launchd가 `claude -p`를 실행해 텔레그램 리마인드가 자동 발송된다 (7/6부터 운영 중)
- 리마인드에 "1-1"(완료), "2-2"(미루기)로 답하면 기록 반영·리스크 안내까지 1주차 기능이 그대로 작동
- **맥 없이 운영하는 구조 설계**: GitHub(데이터) + Anthropic 클라우드 루틴(실행) + 디스코드(인박스)로 정리, 토큰 사용량을 본 뒤 VPS 이전 여부 결정

### 💡 알게 된 인사이트 & 공유하고 싶은 내용
> 하면서 깨달은 것, 크루들과 나누고 싶은 것

- 기능을 만드는 것과 그걸 자동으로 계속 돌아가게 하는 것은 별개의 작업이었다.
- `claude -p` (headless 모드)를 쓰면 세션 없이 명령 한 줄로 Claude를 실행할 수 있다. launchd/cron 같은 예약 실행과 조합하면 자동 운영이 가능하다. 단, 대화 기억이 없으므로 규칙은 CLAUDE.md에, 데이터는 파일에 분리해둬야 한다.
- 예약 실행을 걸어도 API 오류로 리마인드가 안 나가는 날이 있었다. 재시도나 실패 알림이 필요하다.
- 맥 없이 운영하려면 데이터·실행·인박스가 각각 어디에서 도는지 전부 확인해야 한다. 하나라도 로컬에 묶여 있으면 맥을 꺼둘 수 없다.
