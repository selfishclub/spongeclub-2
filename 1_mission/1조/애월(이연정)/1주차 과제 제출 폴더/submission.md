# 1주차 — 나만의 OS 만들기 🛠️

> 미션을 진행하며 **과정과 결과물**을 기록해주세요. (다 못 채워도 OK, 한 것 위주로!)

## 🎯 미션 1. 내 OS 재료 찾기
> 인터뷰 스킬(아이데이션)로 "내 삶에 필요한 게 뭔지" 찾기
- **과정 (어떻게 찾았나):**
  인터뷰 스킬을 따라가며 오늘 하루 중 시간을 제일 많이 잡아먹은 일부터 가볍게 떠올렸다. 그러다 바로 걸린 게 영어공부. 늘 해야 한다고 마음먹지만 어떻게 할지 막막한 영역이었다. 어디서 막히는지 파고들어 보니, 문제는 공부 자체가 아니라 그 앞에 있었다. 앱·유튜브를 잔뜩 다운받아 저장만 해두고 "뭐부터 하지?" 고민하다 손을 놓고, 어렵게 시작해도 2~3일 뒤 "이 방법 맞나? 늘긴 하나?" 의심하다 지속에 실패하는 패턴. 매번 반복되는 '결정 피로'와 '피드백 부재'가 진짜 마찰점이라는 걸 발견했다. "무엇을 만들까"가 아니라 "내 하루의 어디가 자꾸 걸리는가"에서 출발하니 재료가 또렷하게 잡혔다.
- **결과:**
  『**영어공부가 의지가 아니라 습관으로 알아서 굴러가는 루틴 OS**』라는 재료 카드가 나왔다.
  - **영역**: 삶 (영어공부 — 늘 마음먹지만 안 굴러가던 영역)
  - **걸리는 지점**: 뭘로 시작할지 못 정해 손 놓고, 시작해도 며칠 뒤 의심하다 지속 실패
  - **지금은**: 앱·유튜브 잔뜩 저장만 → 뭐부터 할지 고민하다 손 놓음 → 시작해도 의심 → 후회
  - **OS가 된다면**: ① 오늘 뭐 할지 정해주고 ② 복습·예습을 자동으로 굴리고 ③ 늘고 있다는 증거를 보여주는 '루틴 엔진'
  - **첫 한 걸음**: 매일 아침 "오늘의 10분"을 정해서 던져주는 '영어 루틴' 채널 하나부터
- **느낀 점:**
  딱히 만들 게 없는데 라고 생각했는데, 하루를 같이 들여다보니 매일 걸리던 게 분명히 있었다. 특히 내 문제가 '영어 실력'이 아니라 그 앞단의 '매번 뭘 할지 결정하는 피로'였다는 걸 알게 된 게 컸다. OS는 거창한 자동화가 아니라 내가 매번 쓰던 의지를 시스템이 대신 써주는 것이라는 감이 잡혔다. 결정을 시스템에 넘기고 나는 실행만 하면 된다고 생각하니, 막막하던 영어공부가 처음으로 '해볼 만한 것'처럼 느껴졌다.

## 🧩 미션 2. 내 OS 기획
> 인터뷰 결과 + 세션 내용(흐민·배짱·키노) 활용해 기획
- **기획 내용:**
  - 한 문장: "완벽하게 말하려다 멈추는 대신, 짧아도 맥락 맞게 티키타카하는 습관이 내 실제 업무 영어로 알아서 굴러간다."
  - 무엇을 위한 OS인가: 영어공부는 늘 마음먹지만 안 굴러가던 영역이었다. 실패는 세 군데서 반복됐다 — ① 뭘로 시작할지 못 정해 손 놓고, ② 시작해도 2~3일 뒤 "늘긴 하나" 의심하다 중단, ③ 복습이 자동으로 안 굴러 매번 처음부터. 그래서 의지가 아니라 습관으로 알아서 굴러가는 루틴 엔진을 만들기로 했다. 범위는 슬랙/줌 업무 영어로 좁혔다(보안 서비스 회사 UX/UI 디자이너 — 개발자·PM·디자이너와의 디자인 설명·크리틱). 정면 조준한 문제는 "완벽하게 말하려다 아예 말을 안 해버리는 것", 특히 내 디자인에 질문받고 답하는 순간.
  - 시스템 구조 (흐민식 3레이어 중 1층 + 가벼운 복습엔진부터)
    - 인터페이스: 텔레그램 봇 '영어 루틴' 채널 ↔ 항상 열어둔 전용 Claude Code 세션
    - 브레인: OS.md(운영 매뉴얼)를 읽고 매 접점에서 일관되게 행동하는 세션
    - 자산: 로컬 MD 파일 — expressions.md(간격반복 표현 로그), state.md(스트릭), profile.md, daily/, scripts/
  - 하루 루프 (아침·점심·밤 3방 + 상시 캡처)
    - 🌅 아침 — 전날 표현·복습 due를 한국어로 던지면 내가 영어로 인출(능동 회상) → 오늘의 표현 5개(뜻+예문)
    - 🍱 점심 — 디자인 Q&A·크리틱 텍스트 롤플레이 1건 ("완벽 금지, 짧아도 즉답")
    - 🌙 밤 — 재복습 → 완료 처리(🔥스트릭) → 막힌 표현 캡처 → 한 줄 회고
    - ⚡ 상시 — 아무 때나 막힌 표현을 던지면 자연스러운 영어로 바꿔 로그에 적재 → 다음 아침 복습에 편입
  - 차별점
    - 남의 교재가 아니라 내 실제 미팅 스크립트에서 표현을 뽑는다 → 전이(transfer)가 빠름
    - "완벽함"이 아니라 "짧아도 맥락 맞게 즉시 반응" 근육에 조준
    - 성장 증거를 눈에 보이게 — 스트릭 + 쌓이는 표현 로그로 "늘긴 하나" 의심을 끊음

- **막혔던 점 / 어떻게 풀었나:**
  - "어떤 공부가 나에게 맞는지" 감이 안 잡힘: 목적을 커리어/생활 전부가 아니라 슬랙·줌 업무 상황 하나로 좁힘. "완벽주의로 멈추는 것"이라는 진짜 병목을 인터뷰로 찾아내 OS의 조준점으로 삼음.
  - 레벨을 "잘 모르겠음": 정식 시험 대신 OS가 5분 캐주얼 진단(슬랙 대화 + 디자인 한 줄 설명 + 스크립트 관찰)을 먼저 돌려 시작점을 잡게 설계
  - 텔레그램이 답장을 안 받음 (이모지 ack는 뜨는데 응답 없음): 봇 프로세스·세션 바인딩을 추적 → 인바운드는 poller를 마지막에 띄운 세션 하나에만 주입된다는 걸 확인. 흐민식대로 전용 --channels 터미널 세션을 주역으로 세우니 수신이 살아남.
  - 매 답변마다 권한(permission) 허용을 눌러야 함: .claude/settings.json에 텔레그램·cron·english-os/** 파일 도구를 allowlist 등록 → 무인 자동 발사가 가능해짐(안 그러면 새벽 cron이 멈춤).
  - 예약(cron)이 세션 전용·7일 만료: 전용 세션이 켜질 때 CLAUDE.md 부팅 시퀀스가 3방 cron을 스스로 재등록하도록 설계 → 세션만 다시 켜면 자동 복구.
  - "시스템을 크게 만들다 지치는" 함정: 흐민의 "시스템보다 습관이 먼저"를 따라 1층 + 가벼운 복습엔진만 짓고, 위키(2층)·주간 리캡(3층)은 로드맵으로 미룸.


## ⚙️ 미션 3. 내 OS 구현
> 실제로 만들어본 것 (클로드코드 '채널' 기능 활용 OK)
- **결과물:** 클로드코드 '채널' 기능으로 실제 작동하는 "영어 루틴 엔진"을 구축함. 텔레그램 봇 ↔ 전용 Claude Code 세션 ↔ 로컬 MD 파일의 3요소로 돌아가고, 오늘(2026-07-05) 진단 1바퀴가 실제로 완료됨.
  - 1. 파일 시스템
telegram_test/
├── CLAUDE.md              # 세션을 '엔진'으로 부팅 + 3방 cron 자동 재등록 시퀀스
├── .claude/settings.json  # 권한 allowlist 9개 (무인 자동 발사용)
├── 이미지첨부/             # 스크린샷 폴더
└── english-os/
    ├── OS.md              # 운영 매뉴얼 (3접점·복습박스·표현추출·스트릭 규칙)
    ├── profile.md         # 진단 결과: 중급(B1), 교정 포인트=관사·전치사
    ├── state.md           # 스트릭 + 누적 표현 4개
    ├── expressions.md     # 표현 로그 (간격반복 box/다음복습일 포함) — 현재 4개
    ├── daily/             # 일일 기록
    └── scripts/raw·processed/  # 미팅 스크립트 파이프라인
  - 2. 텔레그램 채널 연동 — 봇을 전용 --channels 세션에 1:1 연결. 아웃바운드(브리핑 발송) + 인바운드(내 답장 수신) 양방향 작동 확인.
  - 3. 자동화 — 아침 9시·점심 1시·밤 11시(평일) 3방을 cron으로 예약. 권한 프롬프트 없이 무인 발사되도록 allowlist 설정.
  - 4. 실제 작동 증거 — 진단 대화가 굴러가 profile.md에 레벨(B1)이 기록되고, expressions.md에 표현 4개("It's coming along well.", "It came out of our last meeting.", "to zero in on…", "Let me think…")가 간격반복 메타와 함께 적재됨.
