## 삽질 과정

> 막혔던 지점, 시도한 방법, 해결 과정을 솔직하게 적어주세요.
대부분의 내용은 노션 페이지에 있습니다
**판정봇:**
- 슬랙은 실시간 push가 안 됐다. 공식 채널 플러그인은 텔레그램·디스코드·iMessage뿐이고 슬랙 커넥터는 pull 방식이라 "말 걸면 바로 받는" 형태가 불가능. 게다가 API 봇으로 만들면 월 2~3만원이 든다.
- 유료 API 봇 → 슬랙 pull 커넥터 → 루프 방식… "실시간"과 "공짜"를 둘 다 만족하는 건 없었다.
- 텔레그램이 공짜였던 이유가 "Claude Code(구독)가 뇌, 채널은 그냥 파이프"라 API 미터가 안 돈다는 걸 발견. 그대로 슬랙 Socket Mode MCP 채널 서버를 직접 만들어 공개 URL 없이 실시간 수신 + 스레드 답글을 구현. 스레드로 데이터를 보충하면 처음부터가 아니라 "무엇이 바뀌는지" 중심으로 판정을 업데이트(STOP→GO)하게 했다.

**SEO OS:**
- 발행 백엔드(Lovable+Supabase)에서 관리자인데도 스토리지 업로드가 계속 거부됐고, 브라우저 페이지로 이미지 바이트·본문 HTML을 넣는 경로가 전부 막혔다.
- 로컬 서버 fetch / 클립보드 / 파일 업로드 도구 / base64 손수 이동 — 채널마다 다른 이유로 다 막힘.
- 관리자 세션으로 만든 경로별·시간제한 서명 업로드 링크를 터미널에 넘겨, 터미널이 디스크 파일을 무손실로 업로드. 긴 본문 HTML도 같은 방식으로 스토리지를 경유시켰다.
