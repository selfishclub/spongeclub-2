# 1주차 — 나만의 OS 만들기 🛠️

> 미션을 진행하며 **과정과 결과물**을 기록해주세요. (다 못 채워도 OK, 한 것 위주로!)

## 🎯 미션 1. 내 OS 재료 찾기
> 인터뷰 스킬(아이데이션)로 "내 삶에 필요한 게 뭔지" 찾기

- **과정 (어떻게 찾았나):**
  인터뷰 스킬을 따라 5스텝으로 진행했다. ①오늘 하루 시간을 잡아먹은 일 → ②짜증 포인트 → ③일 밖의 영역 → ④무게 재기 → ⑤OS가 된다면 순서로, 질문을 하나씩 받으며 좁혀갔다.
  - 처음엔 "폼 제출 / Zoom 미팅 요약 / 제출된 폼을 여러 방식으로 확인" 같은 **기능** 단위로 떠올렸는데,
  - 파고들수록 진짜 마찰점은 기능이 아니라 **"매번 사이트에 로그인해서 들어가야 하는 것 자체"** 였다는 걸 발견했다.
  - 놓치기 싫은 것으로 **미팅 일정**이 나왔고, 결국 "데스크탑 캘린더 한 화면에서 다 본다"로 일·삶이 한 몸처럼 수렴했다.

- **결과:** 🟡 **내 OS 재료 카드**

  | 항목 | 내용 |
  |------|------|
  | 영역 | 일 (업무 OS) |
  | 걸리는 지점 | 폼 제출·Zoom 요약을 확인하려고 매번 회사 사이트에 로그인해서 들어가는 게 복잡함. 미팅 일정도 놓치기 싫음. |
  | 지금은 | 브라우저 열어 로그인 → 폼 목록 페이지 이동 → 항목 하나씩 클릭 → Zoom 요약은 또 따로 (여러 단계) |
  | OS가 된다면 | 데스크탑 캘린더 한 화면에, 폼 제출·미팅·Zoom 요약을 **로그인 없이** 날짜별로 전부 모아서 보여주기 |
  | 한 문장 | **"사이트에 안 들어가도, 켜기만 하면 오늘 할 일이 다 보인다"** |
  | 첫 한 걸음 | 데스크탑 캘린더/위젯에 '폼 제출 목록' 한 가지부터 로그인 없이 띄워보기 |

- **느낀 점:**
  기능부터 떠올렸는데, 인터뷰를 따라가다 보니 정작 나를 매일 붙잡는 건 "로그인해서 들어가는 마찰" 이라는 게 드러났다. 별거 아닌 것 같은 반복이 매일 쌓이면 진짜 무겁다는 걸 확인했다. OS를 "기능 목록"이 아니라 **"내 하루가 통과하는 길"** 로 보게 된 게 가장 큰 소득.

## 🧩 미션 2. 내 OS 기획
> 인터뷰 결과 + 세션 내용(흐민·배짱·키노) 활용해 기획

- **기획 내용:** 재료 카드를 3단계로 나눠, 이번 주는 **1단계**만 목표로 잡았다.

  > **한 줄 정의:** 회사 사이트에 로그인하지 않아도, 데스크탑에서 켜기만 하면 오늘 들어온 **폼 제출 목록**이 날짜별로 보이는 화면.

  - **1단계** — 폼 제출 목록을 캘린더에 로그인 없이 표시 *(이번 주 목표)*
  - **2단계** — 거기에 미팅 일정 + Zoom 요약 얹기
  - **3단계** — 세 개가 한 화면에 도는 완성형 데스크탑 OS

  **1단계 구성요소**

  | 조각 | 내용 | 이번 주 목표 |
  |------|------|-------------|
  | ① 데이터 가져오기 | 폼 제출 내용을 어디서 어떻게 받아올지 (API / CSV·JSON 내보내기 중 되는 것) | 폼 목록을 데이터로 한 번 뽑아내기 성공 |
  | ② 날짜별 정리 | 받아온 걸 제출일 기준으로 정렬 | 날짜순 표로 나열 |
  | ③ 한 화면 표시 | 데스크탑에서 로그인 없이 열리는 캘린더/리스트 뷰 | 창 하나로 목록이 뜨는 것 확인 |

  **이번 주 첫 걸음:** "폼 제출 데이터를 로그인 없이 한 번 손에 넣기". ①만 되면 ②③은 금방 붙는다. ①이 진짜 관문.

- **막혔던 점 / 어떻게 풀었나:**
  처음엔 만들 게 없다고 느껴 기능 나열에서 맴돌았다. 인터뷰의 "무게 재기" 스텝에서 일 쪽을 선택하며 범위가 확 좁아졌고, "전부 다"가 아니라 **폼 목록 하나부터**로 1단계를 쪼개니 이번 주에 실제로 손댈 수 있는 크기가 됐다.
