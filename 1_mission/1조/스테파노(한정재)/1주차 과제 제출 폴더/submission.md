# 1주차 — 나만의 OS 만들기 🛠️

> 미션을 진행하며 **과정과 결과물**을 기록해주세요. (다 못 채워도 OK, 한 것 위주로!)

## 🎯 미션 1. 내 OS 재료 찾기
> 인터뷰 스킬(아이데이션)로 "내 삶에 필요한 게 뭔지" 찾기

- **과정 (어떻게 찾았나):**
  인터뷰 스킬로 하루를 되짚으며 재료를 찾았다. 하루 대부분은 어린이 충치·교정 진료로 채워지는데,
  진료 자체는 대체 불가라 그 **주변의 반복 업무**로 각도를 좁혔다. 설명 반복 / 차트 정리 / 예약·리마인드가
  모두 걸렸고, 그중에서도 **교정 환자 사진 정리 + 이전에 메모해둔 것 정리**가 가장 자주 미뤄지고 쌓이는
  지점이라는 걸 확인했다. (지금은 구강 사진을 촬영·저장해두고 "나중에 몰아서" 편집·정리하는 흐름.)

- **결과 — 🟡 내 OS 재료 카드:**

  | 항목 | 내용 |
  |------|------|
  | 영역 | 일 (진료 주변의 반복 업무) |
  | 걸리는 지점 | 교정 환자 사진 정리 + 이전에 메모해둔 것 정리 |
  | 지금은 | 구강 사진 촬영 → 저장 → 나중에 몰아서 편집·정리 (미뤄지고 쌓임) |
  | OS가 된다면 | 사진 넣으면 → 환자별 자동 분류 → before/after 배치 → 메모까지 붙고 → 반복 편집 일괄 처리 |
  | 한 문장 | **"미뤄둔 사진 정리가 알아서 끝나 있다"** |
  | 첫 한 걸음 | 클로드 코드로 '교정사진-정리' 채널을 만들어, 사진 묶음을 환자별·날짜별 폴더로 분류하는 것부터 |

- **느낀 점:**
  "딱히 불편한 게 없다"고 생각했는데, 하루를 같이 뜯어보니 진료 뒤에 쌓이는 사진·메모 정리가
  매번 나를 붙잡고 있었다. 거창한 자동화가 아니라 "매번 반복되는데 시간 잡아먹는 것" 하나만
  집어도 OS의 재료가 된다는 걸 알게 됐다.

## 🧩 미션 2. 내 OS 기획
> 인터뷰 결과 + 세션 내용(흐민·배짱·키노) 활용해 기획

- **기획 내용 — 교정 환자 사진 정리 OS (3단계 로드맵):**

  기존 폴더 체계:
  ```
  [이름_번호_주소_교정방법]        ← 메인 폴더 (환자 1명)
     ├── 진단
     └── [촬영날짜]               ← 날짜별 서브폴더
  ```

  | 단계 | 모습 |
  |------|------|
  | **1단계** (이번 주) | 새 사진 → 이름·번호 보고 → 해당 환자 메인 폴더의 촬영날짜 서브폴더로 자동 분류 (없으면 새로 생성) |
  | **2단계** | 초진/최근 before·after 나란히 배치 + 메모 붙이기 |
  | **3단계** | 전체 아카이브 → 편집 일괄 처리까지, 환자 1명 = 한 화면 |

  **이번 주 1단계 시작점:** 클로드 코드로 '교정사진-정리' 채널(폴더)을 만들고, 새로 찍은 사진 10장을
  넣어 *"이름·번호 보고 기존 환자 폴더의 날짜 서브폴더에 정리해줘"* 로 테스트 → 규칙이 맞으면 전체로 확장.

- **막혔던 점 / 어떻게 풀었나:**
  ①~④(자동 분류 / before·after / 메모 통합 / 편집) 전부 하고 싶었지만 한 번에 다 만들면 무거워진다.
  → 가장 자주 걸리는 **자동 분류(1단계)** 한 조각으로 좁혔다.
  또한 폴더에 이름·주소 같은 **환자 개인정보(민감정보)**가 포함되어, 작업은 **로컬 PC 안에서만** 돌리고
  외부 업로드 없이 진행하는 방향으로 정리했다.
