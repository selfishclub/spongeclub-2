## 결과물

> 이번 주는 앱 세 개를 각각 굴리며 다듬었다. 따로 만든 앱들이 구글 캘린더·텔레그램·맥미니를 통해
> **서로 이어지는 하나의 생활 OS**가 되기 시작했다.

### 📸 ① i_Photo (구 환자사진변환) v1.3 → v1.4.6 — "만든 앱"에서 "매일 쓰는 도구"로

> 발표자료: [i_Photo-발표.pdf](발표자료/i_Photo-발표.pdf)

이번 주는 새 기능을 벌리기보다, **실제 진료실에서 매일 쓰면서 걸리는 부분을 하나씩 없애는 주간**이었다.
이름도 「환자사진변환」에서 **i_Photo**로 바꿨다.

![새 앱 아이콘 — 흑백(전)·컬러(후) 인화지를 겹친 '스택 프린트'](이미지첨부/새%20앱아이콘.png)

- **파일탐색기형 UI 통일 (v1.4.0)** — '사진 가져오기'와 '전후 비교'를 같은 파일 브라우저(폴더 트리 + 폴더 이름 검색 + 날짜순 썸네일 격자)로 통일. 데스크 선생님들이 쓰던 파일탐색기와 같은 감각으로, 설명이 필요 없는 UI가 목표.
- **전후 비교 완성** — 사진 2~8장 선택 → 전체화면 비교, 더블클릭 = 그 부위만 전 날짜 동시 확대, 드래그 영역 확대, ✏️펜으로 설명 그리기. 저장 시 병원 로고 헤더 + 자동 파일명.
- **앱 밖으로 나갈 일 없애기 (v1.4.5–1.4.6)** — 폴더 바로가기(홈·데스크탑·네트워크 + 사용자 지정), 우클릭 파일 관리(복사·이동·삭제), 보드 편집 상단 이름·촬영일(EXIF 자동) 입력줄, 저장 후 보드 자동 초기화.
- **혼자 쓰는 앱 → 남에게 줄 수 있는 앱** — 새 앱 아이콘, 페이지별 상징색 탭, 홈 화면 병원 로고, 설정에 사용 설명서 + 업데이트 내역 내장. 오늘 **Windows v1.4.6 빌드**로 병원 PC 배포 완료. (환자 사진은 여전히 전부 로컬 처리, 원본 비파괴)

*(실제 화면은 환자 얼굴·구강 사진이 담겨 개인정보 보호를 위해 제출본엔 생략)*

### 📅 ② i_Day — 10년 쓴 유료 캘린더를 하루 만에 대체 (7/11~12)

> 발표자료: [i_Day-발표.html](발표자료/i_Day-발표.html)

10년 쓴 마이캘린더(유료앱)의 **장점은 계승, 단점은 수리, 저장은 구글에** — 를 목표로 만든 나만의 캘린더 웹앱.

- **구조**: Vercel(호스팅) + Supabase(로그인·설정) + **Apps Script 브리지**(구글이 무료로 돌려주는 나만의 API 서버) + 구글 캘린더(유일한 저장소 = 동기화 문제 원천 차단). **전부 무료 티어.**
- **발명품 — 한 글자짜리 시계 뱃지**: 월 달력 한 칸은 좁아서 "13:30"은 못 넣는다 → **숫자 = 시(時), 배경이 시계처럼 찬 만큼 = 분(分)**. 이 뱃지가 앱 아이콘까지 이어지는 i_Day의 시그니처가 됨.
- **입력 혁명 — 말하면 일정이 된다**: ＋버튼 길게 누르고 "내일 10시 반에 서현역" → 날짜·시간·내용 자동 분해. 음성의 함정(붙여쓰기·공백·조사)까지 **테스트 38개**로 고정한 한국어 파서.
- **반복 일정 3택 수정**: 이 일정만 / 이후 전체 / 전체 — "이후 전체"는 기존 반복을 그 자리에서 쪼개(UNTIL + 새 반복) 구글 캘린더로 왕복 검증 완료.
- 그 외: 더블탭 날짜 팝업(휴일 지정), 스와이프 삭제, D-day, 음력, 주간/야간 테마, 일정 검색, 사용 설명서 내장.

### 🧠 ③ i_Memo — 텔레그램으로 채우는 두 번째 뇌, 캘린더와 이어지다

> 발표자료: [i_Memo-시스템-발표.html](발표자료/i_Memo-시스템-발표.html) · [i_Memo-v1.1-발표.html](발표자료/i_Memo-v1.1-발표.html)

애플 노트를 대체하려 만든 개인 메모 시스템. **메모 8,460개·사진 46GB 이관**, 맥미니 24시간 상시 구동.

- **입력은 텔레그램**: 글·사진에 `#메모`(로컬) 또는 `#메모s`(클라우드)만 붙여 보내면, 맥미니의 Claude Code가 내용을 읽고 **제목·분류를 스스로 정해** 저장. 새 폴더를 만들지 않고 기존 8,000여 분류 중 가장 맞는 곳으로.
- **두 버전, 하나의 뇌**: 로컬 i_Memo(맥미니 파일, 완전 내 소유) + 클라우드 i_Memo_S(Supabase, 로그인 하나로 어디서나).
- **이번 주 핵심 — 구글 캘린더 시너지 4종**:
  1. **아침 브리핑** — 전일 메모 요약 + 오늘 일정을 한 화면으로
  2. **자동 등록** — "다음주 목요일 3시 세미나" 메모가 저장과 동시에 캘린더 일정으로
  3. **리마인더** — 일정 시작 전, 그 주제와 관련된 내 메모를 찾아 함께 알림
  4. **말로 수정·삭제** — "치과 예약 16일 2시로 옮겨줘" 한마디로 (모호하면 되묻기)
- OAuth·클라우드 콘솔 없이 **Apps Script 웹앱을 개인 브리지**로 3분 배포 — 이 브리지를 i_Day와 공유한다.

**세 앱의 연결**: i_Memo가 적고 → 구글 캘린더에 쌓이고 → i_Day가 보여주고 → 텔레그램이 아침마다 브리핑한다.

## 삽질 과정

**i_Photo**
- **한글 IME 마지막 글자 누락**: 이름 입력 직후 저장하면 마지막 글자가 빠짐("홍길동" → "홍길ㄷ"). 조합 중인 글자가 확정 전이라 생기는 문제 — 저장 직전 조합 강제 확정(`commit_ime`)으로 해결. 내보내기에서 잡았는데 보드 편집에서 또 나와 두 번 잡았다.
- **검색한 폴더가 트리에 안 보임**: 트리가 비동기로 로드되며 스크롤이 계속 밀림 → 조상 폴더를 먼저 펼치고 로드 타이밍마다 스크롤을 여러 번 재보정.
- **더블클릭/우클릭 혼동**: 확대·축소 제스처가 섞여 헷갈림 → 더블클릭=확대 전용, 우클릭=처음으로 복귀로 역할 분리. 우클릭 시 사진이 넘어가버리는 버그도 이때 발견해 수정.

**i_Day**
- 👻 **일정이 유령 캘린더로 사라진 사건**: 앱은 저장 성공인데 구글 캘린더엔 없음 → Apps Script 브리지가 **다른 구글 계정(naver) 소유**였다. 1년치 일정이 아무도 안 보는 캘린더에! 교훈: 브리지 문제는 로그인 계정부터 의심.
- 🎤 **음성 받아쓰기의 함정**: "십오분"(한글) → "15 분"(공백) → "…분에"(조사), 고칠 때마다 다른 형태로 재발 → 결정타는 실제 입력 칸 스크린샷 한 장. "안 돼요"보다 실제 데이터 한 장이 백 배 빠르다.
- 🎙️ **꺼지지 않는 마이크**: iOS 웹킷 버그로 닫아도 마이크 표시 잔류 → stop 대신 abort + 마이크를 잠깐 잡았다 놓는 우회로 해제.
- 📦 **고쳐도 안 고쳐지는 폰**: 서버는 최신인데 폰은 옛날 동작 → 범인은 캐시된 옛 파서 파일. `?v=` 버전표식으로 즉시 반영. "배포했는데 안 바뀌어요"의 8할은 캐시.

**i_Memo**
- **"된다"와 "계속 된다"는 다르다**: 깨진 토큰(특수문자), 노트북·맥미니 중복 수신, TTY 없는 백그라운드에서 죽는 프로세스 — screen+launchd 상시 구동과 단일 수신으로 안정화해야 진짜 완성.

## 인사이트

> 앱은 만들었을 때가 아니라 **남이(미래의 나 포함) 설명 없이 쓸 수 있을 때** 완성된다.
> 그리고 앱이 세 개가 되니 알겠다 — 각 앱의 가치보다, **서로 이어질 때 생기는 가치가 더 크다.**
> 저장소를 구글 캘린더 하나로 모은 선택이 세 앱을 OS로 묶어줬다.
