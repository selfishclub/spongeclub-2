## 결과물

> **교정 진단 사진 앱 「환자사진변환」 v1.3 → v1.4.6** — 이번 주는 새 기능을 벌리기보다,
> **실제 진료실에서 매일 쓰면서 걸리는 부분을 하나씩 없애는 주간**이었다.

![새 앱 아이콘 — 흑백(전)·컬러(후) 인화지를 겹친 '스택 프린트'](이미지첨부/새%20앱아이콘.png)

**① 파일탐색기형 UI 통일 (v1.4.0)**
- '사진 가져오기'와 '전후 비교'를 **같은 파일 브라우저**(폴더 트리 + 폴더 이름 검색 + 날짜순 썸네일 격자)로 통일.
- 병원 데스크 선생님들이 쓰던 파일탐색기와 같은 감각으로 쓰게 — 설명이 필요 없는 UI가 목표.

**② 전후 비교 완성**
- 사진 2~8장 선택 → 전체화면 비교, **더블클릭 = 그 부위만 전 날짜 동시 확대**, 드래그 영역 확대, ✏️펜으로 설명 그리기.
- 비교 이미지 저장에 병원 로고 헤더 + 자동 파일명(`이름 초기날짜-마지막날짜`).

**③ 앱 밖으로 나갈 일 없애기 (v1.4.5–1.4.6)**
- 폴더 **바로가기**(홈·데스크탑·다운로드·네트워크 + 사용자 지정), **우클릭 파일 관리**(복사·이동·삭제)까지 앱 안에서.
- 보드 편집 상단에 **이름·촬영일(EXIF 자동) 입력줄**, 저장하면 보드 자동 초기화 → 다음 환자 바로 준비.

**④ 혼자 쓰는 앱 → 남에게 줄 수 있는 앱**
- **새 앱 아이콘**(흑백 전 / 컬러 후 인화지를 겹친 디자인), 페이지별 상징색의 상단 탭, 홈 화면에 병원 로고·병원명.
- 설정에 **사용 설명서 + 업데이트 내역 내장**(도움말 다이얼로그) — 내가 없어도 데스크 선생님이 쓸 수 있게.
- 오늘 **Windows v1.4.6 빌드**를 만들어 병원 PC 배포까지 완료. (환자 사진은 여전히 **전부 로컬 처리**, 원본 비파괴)

*(실제 화면은 환자 얼굴·구강 사진이 담겨 개인정보 보호를 위해 제출본엔 생략)*

## 삽질 과정

- **한글 IME 마지막 글자 누락**: 이름을 입력하고 바로 저장하면 마지막 글자가 빠짐(예: "홍길동" → "홍길ㄷ"). 한글 조합 중인 글자가 확정 전이라 생기는 문제 — 저장 직전에 조합을 강제 확정(`commit_ime`)해서 해결. 내보내기에서 잡았는데 보드 편집에서 또 나와서 두 번 잡았다.
- **검색한 폴더가 트리에 안 보임**: 폴더 검색으로 깊은 폴더를 찾아도 트리가 비동기로 로드되며 스크롤이 계속 밀림 → 조상 폴더를 먼저 펼치고, 로드 타이밍마다 스크롤을 여러 번 재보정하는 식으로 해결.
- **더블클릭/우클릭 혼동**: 비교 뷰어에서 확대·축소 제스처가 섞여 오히려 헷갈림 → 더블클릭=확대 전용, 우클릭=처음으로 복귀로 역할을 갈라서 정리. 우클릭하면 사진이 넘어가버리는 버그도 이때 발견해 수정.
- **기능을 벌리고 싶은 유혹**: 얼굴검출 자동 배치 같은 큰 기능 대신, 실사용에서 나온 자잘한 요청(바로가기·파일관리·도움말)을 우선했다. Windows 8.1 호환(순수 Pillow, 무거운 ML 미도입) 원칙도 그래서 유지.

## 인사이트

> 앱은 만들었을 때가 아니라 **남이 설명 없이 쓸 수 있을 때** 완성된다.
> 이번 주 커밋의 대부분은 새 기능이 아니라 실사용에서 나온 다듬기였는데, 체감 완성도는 이게 제일 크게 올렸다.
