## 🎯 미션 1. 내 OS 만들기

> **[ 내 삶을 돕는 OS ]** 또는 **[ 콘텐츠 OS ]** 중 하나를 선택해 완성해주세요.

**✅ 선택:** 콘텐츠 OS

### 📐 기획

평소 떠오른 생각을 Apple 메모, 음성 메모, 종이, AI 대화 등 여러 곳에 나누어 기록해 왔다.

문제는 기록한 사실만 어렴풋이 기억할 뿐, 시간이 지나면 어디에 무엇을 적었는지 다시 찾기 어려웠다는 것이다. 좋은 아이디어를 적어두고도 끝까지 발전시키지 못한 경우도 많았다.

그래서 이번 주에는 생각을 빠르게 기록하고, 나중에 검색해서 다시 꺼내볼 수 있는 개인용 콘텐츠 OS를 만들기로 했다.

핵심 방향은 다음과 같다.

- 말이나 손글씨로 부담 없이 기록한다.
- 음성과 손글씨 원본을 그대로 보존한다.
- 음성 전사와 OCR은 검색을 돕는 용도로만 사용한다.
- 기록할 때부터 복잡한 분류를 요구하지 않는다.
- 쌓인 생각을 나중에 글이나 콘텐츠로 발전시킬 수 있게 한다.

AI가 새로운 생각을 대신 만들어주는 시스템보다는, 내가 이미 했던 생각을 잃지 않고 다시 발견하게 해주는 시스템에 가깝다.

### ⚙️ 구현

iPhone에서 사용할 수 있는 SwiftUI 네이티브 앱을 제작했다.

현재 구현한 주요 기능은 다음과 같다.

#### 음성 기록

- iPhone에서 음성 녹음
- 일시 정지 후 이어서 녹음
- 원본 음성 파일 보존
- 저장한 음성의 한국어 전사
- 전사 결과 수정
- 원본 음성 다시 듣기

음성 파일 저장과 전사를 분리해, 전사가 실패하더라도 원본 기록은 남도록 했다.

#### 손글씨 기록

- 종이에 쓴 메모 촬영
- 원본 이미지 보존
- 한국어 OCR
- OCR 결과 수정
- 원본 이미지 다시 보기

손글씨 OCR 결과는 완성된 문서가 아니라 ‘검색용 자동 초안’으로 표시했다.

#### 검색과 보관

- 전사문과 OCR 텍스트 검색
- 날짜와 출처 검색
- 검색 결과에서 원본 열기
- 기록 보관과 복원
- 보관 직후 실행 취소
- 수정 내용을 저장하지 않고 나갈 때 경고

앞으로는 외부에서 발견한 자료와 내 생각을 연결하고, 발전시킬 가치가 있는 기록은 글이나 콘텐츠로 이어갈 수 있도록 확장할 계획이다.

### 🧗 과정에서의 삽질

#### 1. 웹앱과 네이티브 앱 사이의 고민

처음에는 모바일 웹앱으로 구현하는 방안도 고려했다.

하지만 음성 녹음, 한국어 전사, 카메라, OCR처럼 iPhone 기능과 밀접하게 연결된 부분이 많았다. 잠금 화면이나 단축어에서 빠르게 기록하는 기능까지 고려하면 네이티브 앱이 더 적합하다고 판단했다.

기존에 만들었던 웹 프로토타입은 화면 흐름을 참고하는 용도로만 사용하고, 앱은 SwiftUI로 새로 구현했다.

#### 2. 한국어 손글씨 OCR의 한계

실제 손글씨로 테스트했을 때 OCR이 글자를 정확히 읽지 못하는 경우가 있었다.

같은 영역을 확대해 다시 인식하면 일부 결과가 좋아졌지만, 취소선이나 화살표, 연한 글씨까지 안정적으로 복원하기는 어려웠다.

문맥을 이용해 자동으로 고치는 방법도 생각했지만, 사용자가 쓰지 않은 문장을 AI가 그럴듯하게 만들어낼 위험이 있었다.

그래서 OCR의 역할을 다음처럼 제한했다.

- OCR 결과를 정답으로 취급하지 않는다.
- 검색을 돕는 초안으로만 사용한다.
- 사용자가 직접 수정할 수 있게 한다.
- 실제 기록의 기준은 항상 원본 이미지로 둔다.

#### 3. 분류가 기록보다 복잡해지는 문제

기록할 때마다 주제, 유형, 프로젝트를 고르게 하면 앱을 사용하는 것 자체가 귀찮아질 수 있었다.

그래서 새 기록은 우선 수집하고, 실제로 다시 읽거나 활용할 때만 정리하는 방식으로 방향을 바꿨다.

### ✅ 결과물

현재 앱을 실제 iPhone에 설치해 실행하고 있다.

완료한 항목은 다음과 같다.

- iPhone 실기기 설치와 실행
- 한국어 음성 녹음과 전사
- 원본 음성 파일 보존
- 손글씨 촬영과 OCR
- 원본 이미지 보존
- 전사·OCR 결과 수정
- 키워드 검색
- 원본 다시 열기
- 기록 보관과 복원
- 앱 업데이트 후 기존 기록 유지
- 테스트 7개 통과

남아 있는 실사용 점검 항목은 장시간 녹음, 화면 잠금, 전화나 알람으로 녹음이 중단되는 상황이다.

작동 화면 및 스크린샷:

```text
이미지첨부/ 폴더에 추가

- 앱 홈
- 음성 녹음 화면
- 음성 전사 결과
- 손글씨 원본과 OCR 결과
- 검색 화면
- 기록 상세 화면
```

### 💡 알게 된 인사이트 & 공유하고 싶은 내용

#### 1. AI 시대에는 생성만큼 기록이 중요하다

AI를 사용하면 결과물은 빠르게 만들 수 있다.

하지만 내가 무엇을 경험했고 어떤 판단을 했는지 남기지 않으면, 시간이 지난 뒤에는 결과만 남고 생각의 과정은 사라질 수 있다.

그래서 이번에는 AI가 생각을 대신하는 기능보다, 과거의 내 생각을 다시 찾게 해주는 기능에 집중했다.

#### 2. 원본과 검색용 데이터는 분리해야 한다

음성 전사와 OCR은 편리하지만 원본을 완전히 대신할 수는 없다.

특히 손글씨의 취소선, 화살표, 반복된 표현은 단순한 오류가 아니라 생각이 바뀐 과정일 수 있다.

검색을 위한 텍스트와 실제 원본을 함께 보존하는 것이 중요했다.

#### 3. 기록할 때는 가볍고, 정리는 나중에 해야 한다

기록하는 순간부터 완벽하게 분류하려고 하면 기록 자체를 하지 않게 된다.

일단 빠르게 남기고, 나중에 다시 발견했을 때 가치 있는 내용만 정리하는 방식이 더 현실적이었다.

#### 4. 모든 생각이 콘텐츠가 될 필요는 없다

일기, 감정, 짧은 관찰까지 모두 결과물로 완성할 필요는 없다.

대신 발전시킬 가치가 있다고 판단한 생각만 글이나 콘텐츠로 이어갈 수 있게 하는 것이 중요하다.

#### 5. 기능보다 먼저 제품의 원칙을 정해야 한다

OCR 정확도를 높이기 위해 자동 교정을 추가할 수도 있었지만, 그것이 원본을 왜곡할 가능성이 있다면 사용하지 않는 편이 맞았다.

무엇을 더 만들 것인지보다, 무엇을 하지 않을 것인지 정하는 과정이 필요했다.
