# 1주차 — 나만의 OS 만들기 🛠️

> 미션을 진행하며 **과정과 결과물**을 기록해주세요. (다 못 채워도 OK, 한 것 위주로!)

## 🎯 미션 1. 내 OS 재료 찾기
> 인터뷰 스킬(아이데이션)로 "내 삶에 필요한 게 뭔지" 찾기
- **과정 (어떻게 찾았나):** `sponge-and-interview` 인터뷰 스킬 5스텝을 그대로 따라갔다. 어제 하루를 떠올리는 것부터 시작해서, 반기리포트 준비할 때의 마찰점을 짚어냈다.
- **결과:** 🟡 내 OS 재료 카드
  - **영역**: 일
  - **걸리는 지점**: 반기리포트 준비할 때, 여러 미디어·채널에 흩어진 데이터를 항목별로 모으고 인사이트 뽑는 것
  - **지금은**: 미디어/채널 사이트에 하나씩 들어가서 데이터를 엑셀에 손으로 복붙
  - **OS가 된다면**: 다른 미디어, 다른 문서형식, 브랜드별로 흩어진 데이터를 리포트 항목에 맞춰 자동으로 정리해주는 것
  - **한 문장**: "반기리포트 준비 시간을 줄이고, 야근을 덜 한다"
- **느낀 점:** 시작이 항상 오래 걸리는 사람이었는데, 인터뷰 스킬 쓰니 5분도 안 걸림.

## 🧩 미션 2. 내 OS 기획
> 인터뷰 결과 + 세션 내용(흐민·배짱·키노) 활용해 기획
- **기획 내용:** 수퍼파워 brainstorming 스킬로 재료 카드를 실제 설계까지 구체화했다.
  - **입력·출력 확정**: 입력은 채널별 CSV/엑셀 파일(손 복붙 X, 다운로드 O), 출력은 PPT 보고서에 넣을 **표 + 인사이트 초안**까지.
  - **범위 선택(3개 접근 중)**: ①정리·인사이트까지만 ②PPT 슬라이드 자동생성 ③데이터 수집까지 자동화 → 이 중 **①을 1단계로 선택**. ②는 다음 단계로, ③은 로그인·크롤링 리스크가 커서 뺐다(YAGNI).
  - **구조 설계**: `01_raw`(원본) → `02_정리결과`(표) → `03_인사이트`(초안)로 폴더를 분리해 원본 훼손을 막고, OS의 규칙은 전부 `CLAUDE.md`(하네스)에 모았다. → 규칙만 고치면 동작이 바뀌는 구조.
  - **핵심 규칙 정의**: 숫자 창작 금지 / 정리 전 인식 확인 / 애매하면 멈추고 질문 / 출력 3종 세트.
  - **단계 계획**: 1단계=채널 1개(인스타)부터 안정화 → 2단계=다른 채널 확장 → 3단계=PPT 자동생성.
- **막혔던 점 / 어떻게 풀었나:** 처음엔 "전체를 한 번에 자동화해야 하나" 막막했다. brainstorming이 질문을 하나씩 던지며 **범위를 '접근 A · 채널 1개 · 이번 주'로 좁혀줘서** 시작점이 명확해졌다. 또 "구현=웹앱 만들기"인 줄 알았는데, 클로드코드 터미널 + 파일 구조 자체가 OS라는 걸 알게 되면서 부담이 줄었다.

## ⚙️ 미션 3. 내 OS 구현
> 실제로 만들어본 것 (클로드코드 '채널' 기능 활용 OK)
- **결과물:** **반기리포트 데이터 정리 OS (1단계 MVP)** 를 만들었다. 채널에서 받은 CSV/엑셀을 `01_raw/` 폴더에 넣으면, 리포트 항목에 맞춰 표 + 인사이트 초안으로 자동 정리해주는 OS.
  - **핵심 원리**: OS의 규칙을 `CLAUDE.md`(하네스)에 고정 → 규칙만 고치면 동작이 바뀜. 세션에서 배운 "AI는 무수한 모래라 범위를 좁혀줘야 한다"를 그대로 적용.
  - **정리 규칙**: ①숫자 창작 금지(없는 값은 빈칸) ②정리 전 인식 확인 ③애매하면 멈추고 질문 ④출력 3종 세트(정리표·제외항목·확인필요).
  - **폴더 구조**: `01_raw`(원본) → `02_정리결과`(표) → `03_인사이트`(초안)로 입력·결과 분리.
  - **테스트 통과**: T1 정상 CSV / T2 빈값(숫자 안 지어냄) / T3 매핑에 없는 열(넘겨짚지 않고 질문) 3종 검증 완료.
  - **접근**: 데이터 수집·PPT 자동생성은 뺐고(YAGNI), "채널 1개가 믿고 쓸 수 있게 도는 것"을 1단계 목표로 함. 다음 단계는 채널 확장 → PPT 자동화.
