> 여기에는 **무엇을 어떻게 만들었는지**만 기록합니다. (광고주는 "브랜드B"로 익명화)

## 🎯 미션 1. 내 OS 만들기
> **[ 내 삶을 돕는 OS ]** 또는 **[ 콘텐츠 OS ]** 중 하나를 선택해 완성해주세요.

**✅ 선택:** 내 삶을 돕는 OS — **반기리포트 OS**
(마케팅 대행사 기획/PM으로서, 반기마다 제 시간을 제일 많이 먹는 반기리포트를 줄이려고 만든 업무 OS)

> **결론부터 — 처음엔 완전히 망했어요.** "AI에 끌려다니다 운전대를 잡기까지"가 이번 2주였습니다.

### 📐 기획
> 무엇을, 왜, 어떻게 만들지

- **왜 시작했나:** 반기리포트는 반기마다 돌아오는 하반기 최대 일정. 지금까지는 매번 손으로 ①흩어진 데이터 긁어모으기 → ②표 정리 → ③인사이트 쓰기 → ④PPT로 옮기기. 이걸 줄이려고 "AI한테 시키자"고 생각했어요.
- **무엇을:** raw 데이터(엑셀/CSV)를 넣으면 → 정리표 · 인사이트 · **웹 대시보드** · PPT까지 이어지는 파이프라인.
- **어떻게(2주차에 바로잡은 핵심):** 도구를 바꾸는 게 아니라 **내가 원하는 것을 먼저 정의**한다. 리포트 항목·필요한 인사이트 관점·문서의 목적을 내가 정해두고 AI에게 시킨다.

### ⚙️ 구현
> 실제로 만든 것

**설계가 3단계 → 6단계로 자랐다**
- 1주차: `raw → 정리 → 인사이트`
- 2주차: `raw → 정리 → 인사이트 → 리포트 → 대시보드 → 어드민`
- 실제로 굴려보니 리포트·대시보드·어드민 페이지까지 생김 → OS가 "내 업무가 통과하는 길"로 자랐다.

**결과물 ① 대시보드 (광고주 기준으로)**
- 내 기준이 아니라 **광고주가 보고 싶은 것** 중심으로 재구성
- 세 가지 비교를 한눈에: 월별/연도별 비교 · 경쟁사 퍼포먼스 · **PPT 파일 출력(상부 보고용)**
- 상반기 요약 스탯(노출수·도달수·평균 참여율)을 헤드라인으로

**결과물 ② 데이터 입력 — 항목마다 다르게**
- "다 똑같이 할 필요가 없더라." 손이 제일 덜 가는 방식을 항목별로 선택:
  - **웹 폼** — KPI·유입·매체 성과: 화면에서 직접 입력 → DB 바로 반영
  - **구글 스프레드시트** — 팔로워 추이·목표: 시트에 쌓으면 자동 동기화
  - **CSV 파일** — 메타 광고 로우 데이터: 받은 파일 그대로 업로드

> 📸 대시보드·어드민 화면 캡처는 `이미지첨부/` 폴더에 (민감 수치 가린 버전)

### 🧗 과정에서의 삽질
> 막혔던 지점, 시도한 방법, 어떻게 풀었는지 솔직하게

**1주차 — 완전히 망했다**
- "AI로 만들자"는 생각 하나뿐, 정작 **이걸 설명하지 못했다.** 무엇을 넣을지(리포트 항목), 어떤 인사이트를(필요한 관점), 무슨 목적의 문서인지(정체성) — 셋 다 정의 못 함.
- 나온 건 **정체불명 보고서**: 대시보드도 데이터도 없는 메타광고 보고서…? 내가 원한 항목·형식·인사이트 아무것도 아니었다.
- 가장 아이러니: 정작 **진짜 반기리포트는 AI 못 쓰고 손으로 제출**했다. 제출일은 오는데 OS는 못 쓰는 상태 → 결국 손으로.
- **여기서 배운 것:** AI가 문제가 아니라, **내가 뭘 원하는지 몰랐던 게 문제였다.** 도구를 바꿀 게 아니라 내가 먼저 정해야 했다.

**2주차 — 이번엔 거꾸로 갔다**
- 리포트를 제출하고 시간이 나서, 조모임 피드백을 그대로 적용: "실제 데이터를 넣어봐" / "대시보드를 만들어".
- 실제 엑셀·발표 PPT·1주차 OS 문서를 통째로 주고, **"내가 원하는 것"을 먼저 정의**한 뒤 시작.
- (기술적 삽질) 병합셀 엑셀 파싱이 자꾸 밀림 → 정형 입력 템플릿으로 전환. 좌우 병렬 표가 다른 브랜드인 줄 알았더니 전년도 비교였음. 빈값을 0으로 채웠다가 차트 왜곡 → 불확실한 값은 빈칸(`-`)으로.

### ✅ 결과물
> 완성한 것 / 작동 화면

- ✅ 광고주 대시보드 (홈 + 상세 섹션, 월별/연도별·경쟁사 비교, 라이트/다크) — 로컬 검증 완료
- ✅ 데이터 입력 3방식 (웹 폼 / 구글시트 / CSV)
- ✅ PPT 실시간 다운로드 (상부 보고용)
- ✅ 어드민 페이지 (직접 입력 → DB 반영)
- ✅ 이 워크플로우를 개인 스킬 `report-dashboard-os`로 저장
- ⬜ 호스팅(광고주 공유용) — 다음 단계

**보너스 · 텔레그램** — 1주차에 만들어놓고 쓸 줄 몰라 접어뒀던 봇을 살렸다. 쌓인 데이터를 **노트북 없이 폰으로 꺼내 보는 창구**로. 점심 먹다 광고주가 갑자기 물어도 폰으로 바로 답한다.

### 💡 알게 된 인사이트 & 공유하고 싶은 내용
> 하면서 깨달은 것, 크루들과 나누고 싶은 것

- **AI에 끌려가지 않게, 뭐가 필요한지 내가 계속 고민하고 채우는 과정이 진짜 일이었다.** 도구가 아니라 정의가 먼저.
- **거꾸로 접근이 통했다.** 만들고 싶은 걸 먼저 정의 → 실제 데이터로 → 한눈에 보이는 대시보드로.
- **자동화 만능이 아니다.** 형식 없는 엑셀·PPT를 던지니 필요한 열이 다 안 잡혔다. 항목마다 손이 덜 가는 입력 방식을 따로 고른 게 오히려 효율적이었다.

**아직 부족한 것 (솔직하게)**
- AI가 만든 raw의 항목 부족·데이터 오류·누락 (형식 없는 원본을 던진 탓)
- 디자인 — 2주차에 배운 디자인 시스템으로 디벨롭 예정
- 어드민 페이지 불안정 — 클로드랑 계속 대화하며 고쳐 나갈 것

**다음 단계 — 리포트 OS를 넘어 업무 OS로**
- 이미지 소재 업로드(소재까지 함께 관리), 일·주·월 업무 현황 기록 → 리포트를 넘어 업무 전체로.
