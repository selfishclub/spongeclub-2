# 예시 실행 — 회원가입(온보딩) 화면 개편

`피드백정리-OS` 스킬을 실제 상황에 돌려본 예시.

## 입력 (결 안 맞는 채로 통째로 붙여넣음)

```
- (임원) "요즘 다 그렇게 하던데, 시작하자마자 화려한 메인 배너 크게 넣자"
- (현업/영업) "가입 단계가 너무 많대요. 고객들이 중간에 나가요"
- (사용자 데이터) 가입 퍼널 3번째 단계(휴대폰 인증)에서 이탈률 42%
- (우리 UX 방향) 첫 화면은 가치 제안 한 줄 + CTA 하나로 단순하게 가고 싶음
- (임원) "경쟁사 A는 색을 보라색 쓰던데 우리도 바꾸는 게 어때?"
- (현업/CS) "비밀번호 규칙 안내가 없어서 문의가 많이 와요"
- (사용자 데이터) 가입 완료자의 70%가 소셜 로그인(카카오) 사용
```

## 출력 (스킬이 뱉어준 분류표)

| 들어온 피드백 | 출처 | 분류 | UX 원칙 부합? | 데이터 근거 | 권장 처리 |
|---|---|---|---|---|---|
| 시작하자마자 화려한 메인 배너 크게 | 임원 | 취향/트렌드 | ❌ (가입 흐름 방해, 우리 방향과 충돌) | 없음 | 기각 |
| 가입 단계가 너무 많다 | 현업·영업 | 사용성(이탈) | ⭕ | 있음 — 3단계 이탈 42% | 반영 |
| 휴대폰 인증 단계 이탈률 42% | 사용자 데이터 | 우선순위(핵심 병목) | ⭕ | 있음 | 반영 |
| 첫 화면 = 가치제안 + CTA 하나 | 우리 UX 방향 | 단순성 | ⭕ | 간접(이탈 데이터) | 반영 |
| 경쟁사 따라 보라색으로 변경 | 임원 | 판단필요(브랜드) | 판단필요 | 없음 | 별도검토 |
| 비밀번호 규칙 안내 없음 | 현업·CS | 사용성(에러방지) | ⭕ | 간접(문의량) | 반영 |
| 가입자 70%가 카카오 로그인 | 사용자 데이터 | 우선순위 | ⭕ | 있음 | 반영 |

### 한눈 요약
- ✅ **집중할 곳**: 인증 단계 이탈(42%) · 카카오 로그인 우선 · 가입 단계 축소 → 여기에 시안 집중
- ⏸️ **보류/기각**: 화려한 배너(취향·근거없음) → 흔들려도 이 표로 방어
- 🔎 **별도검토**: 보라색 변경 → 브랜드 의사결정 영역(UX 밖)

## 무엇이 달라지나

임원의 "배너 크게"가 `기각` 칸에 들어가 있는 것 — 매번 말로 방어하던 걸 **표 한 장이 대신** 해준다.
이게 한 문장 *"설득에 쓰던 시간을, 디자인에 쓴다"* 가 실제로 작동하는 모습.
