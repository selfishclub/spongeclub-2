# 피드백정리-OS

디자인 시안/기획에 쏟아지는 피드백을, 결이 안 맞는 채로 그냥 붙여넣어도
**① UX 원칙·데이터 기준으로 자동 분류**하고 → **② 채택·기각 결정과 설득 근거까지 담은 "결정서" 한 장**으로 만들어주는 스킬.

> 만든 사람: 달빛그린(윤지윤) · 스폰지클럽 2기 1주차
> 한 문장: **"설득에 쓰던 시간을, 디자인에 쓴다."**
> 현재 구현: **2단계까지** (분류표 → 처리 결정서)

## 언제 쓰나

- 회의·슬랙·메일·임원 코멘트·사용자 데이터에서 피드백이 한꺼번에 쏟아질 때
- "이거 반영해야 하나, 말아야 하나"가 매번 애매할 때
- 컨셉이 자꾸 바뀌고 비전문가 피드백까지 들어와 프로젝트가 산으로 갈 때
- 채택/기각 결정을 **말이 아니라 표로** 방어하고 싶을 때

## 입력 — 그냥 통째로 붙여넣기

결을 맞출 필요 없다. 들어온 그대로 붙여넣으면 된다. 출처가 적혀 있으면 더 정확하다.

```
- (임원) 시작하자마자 화려한 메인 배너 크게 넣자
- (현업) 가입 단계가 너무 많대요
- (데이터) 3단계 인증에서 이탈률 42%
- (우리 UX 방향) 첫 화면은 가치제안 한 줄 + CTA 하나로 단순하게
```

## 처리 규칙 (Claude가 따를 것)

각 피드백을 한 줄씩, 아래 5개 칸으로 분류한다.

1. **출처** — 임원 / 현업 / 사용자 데이터 / 우리(UX 방향) / 기타. 안 적혀 있으면 문맥으로 추정하고 `(추정)` 표시.
2. **분류** — 다음 중 하나로 라벨링:
   - `사용성` (이탈·에러·혼란 등 실제 사용 문제)
   - `일관성` (디자인 시스템·패턴 통일)
   - `접근성` (가독성·대비·터치영역 등)
   - `단순성` (흐름·정보 줄이기)
   - `취향/트렌드` (근거 없는 선호, "요즘 다 그렇게 하던데")
   - `우선순위` (데이터가 가리키는 집중 지점)
   - `판단필요` (별도 검토가 필요한 건 — 예: 브랜드 컬러)
3. **UX 원칙 부합?** — `⭕` / `❌` / `판단필요`. 우리 UX 방향과 충돌하거나 사용 흐름을 해치면 `❌`.
4. **데이터 근거** — `있음`(직접 데이터) / `간접`(문의량·정성 신호) / `없음`. 있으면 근거를 짧게 병기.
5. **권장 처리** — `반영` / `보류` / `기각` / `별도검토`. ②③④를 종합해 한 단어로.

### 출력 형식 (항상 이 표로)

| 들어온 피드백 | 출처 | 분류 | UX 원칙 부합? | 데이터 근거 | 권장 처리 |
|---|---|---|---|---|---|
| (원문 그대로) | … | … | ⭕/❌/판단필요 | 있음/간접/없음 | 반영/보류/기각/별도검토 |

표 아래에는 **"한눈 요약"** 3줄을 붙인다:
- ✅ **집중할 곳** (데이터로 뒷받침되는 진짜 UX 문제)
- ⏸️ **보류/기각** (취향·트렌드라 근거 없는 것 — 흔들려도 이 표로 방어)
- 🔎 **별도검토** (브랜드 등 UX 밖 의사결정이 필요한 것)

### 2단계 — 피드백 처리 결정서 (분류표 다음에 이어서 출력)

분류표를 만든 뒤, **회의에 그대로 들고 갈 수 있는 "결정서" 한 장**을 이어서 만든다.
`반영`/`보류`/`기각`/`별도검토`가 하나라도 있으면 아래 형식으로 생성한다.

각 피드백마다 **"결정 + 한 줄 근거"** 를 붙인다. 근거는 반드시 **데이터나 UX 원칙**에 기대고,
상대(임원·현업)를 존중하는 톤으로 쓴다. 감정이 아니라 사실로 방어하는 게 목적.

```
# 📋 피드백 처리 결정서 — {프로젝트명} / {날짜}

## ✅ 반영 (근거가 분명해 바로 진행)
- **{피드백}** → 반영
  근거: {데이터/원칙 기반 한 줄}. (예: "인증 단계 이탈률 42% 데이터가 직접 뒷받침")

## ⏸️ 보류 (지금은 아님 · 조건부)
- **{피드백}** → 보류
  근거: {왜 지금이 아닌지 + 어떤 조건이면 다시 볼지}

## ❌ 기각 (근거 없음 · 흐름 저해)
- **{피드백}** → 기각
  근거: {UX 원칙/우리 방향과 어떻게 충돌하는지}. 정중하지만 단호하게.

## 🔎 별도검토 (UX 밖 의사결정 필요)
- **{피드백}** → 별도검토
  근거: {누가/어떤 관점에서 따로 판단해야 하는지}

---
**요약 한 줄**: {이번 라운드 핵심 결정 한 문장}
```

결정서의 목적: 매번 말로 방어하던 걸 **문서 한 장으로 대신**하게 만드는 것.
같은 피드백이 다음에 또 흔들려도, 이 결정서를 그대로 다시 보여주면 된다.

## 톤

- 판단을 대신 내려주되, 단정 짓지 않는다. 근거(데이터/원칙)를 항상 함께 보인다.
- 임원/상사 피드백이라도 근거 없으면 `취향/트렌드 · 없음`으로 분류한다. (그게 이 스킬의 존재 이유)

## 단계 로드맵 (이 스킬의 성장 방향)

- **1단계**: 피드백 → 분류표. ✅ 구현됨
- **2단계 (현재)**: 채택/기각 + **설득 근거 초안(결정서)** 자동 생성. ✅ 구현됨
- **3단계 (다음)**: 결정서가 `decisions/`에 날짜별로 쌓여 **시안 변경 히스토리**로 추적·방어
