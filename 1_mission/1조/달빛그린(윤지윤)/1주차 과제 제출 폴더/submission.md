# 1주차 — 나만의 OS 만들기 🛠️

> 미션을 진행하며 **과정과 결과물**을 기록해주세요. (다 못 채워도 OK, 한 것 위주로!)

## 🎯 미션 1. 내 OS 재료 찾기
> 인터뷰 스킬(아이데이션)로 "내 삶에 필요한 게 뭔지" 찾기

- **과정 (어떻게 찾았나):**
  `sponge-and-interview` 인터뷰 스킬로 5스텝 인터뷰를 진행했다. 오늘 하루 떠올리기 → 짜증 포인트 → 일 밖의 나 → 무게 재기 → OS가 된다면, 순으로 가볍게 따라가니 막연하던 게 점점 또렷해졌다. 일(디자인 시안 작업)과 삶(부동산 갈아타기) 두 후보가 나왔고, **일이 지금 하루의 90%를 잡아먹는 병목**이라 일 쪽으로 좁혔다.

- **결과:** 🟡 내 OS 재료 카드

  | 항목 | 내용 |
  |---|---|
  | 영역 | 일 (하루의 90%를 잡아먹는 병목) |
  | 걸리는 지점 | 디자인 시안을 만드는데 컨셉이 계속 바뀌고, UX 전문가가 아닌 사람들의 피드백까지 반영되면서 프로젝트가 자꾸 산으로 감 |
  | 지금은 | 네 갈래 입력(사용자 데이터·임원 피드백·현업 피드백·우리가 원하는 UX 방향)이 한꺼번에 쏟아지는데, 뭐가 진짜 중요한지 매번 손으로 판단하고 안 맞는 건 근거 만들어 설득·방어한 뒤 시안 수정 |
  | OS가 된다면 | ② 들어온 피드백이 UX 원칙에 맞는지/데이터로 뒷받침되는지 **자동 분류** + ④ 채택·기각 결정이 **시안 변경 히스토리로 추적** |
  | 한 문장 | **"설득에 쓰던 시간을, 디자인에 쓴다."** |
  | 첫 한 걸음 | 클로드 코드로 '피드백 정리' 스킬 하나 — 피드백을 붙여넣으면 UX 원칙/데이터 기준으로 분류해주는 것부터 |

- **느낀 점:**
  "딱히 불편한 게 없다"고 생각했는데, 막상 하루를 들여다보니 *매일 같은 마찰*이 분명히 있었다. 특히 객관식 보기로 펼쳐주니 내가 못 떠올린 선택지(②번 자동 분류, ④번 히스토리)에서 "이거다" 싶었다. 문제를 말로 하면 거대해 보였는데, 카드로 쪼개니 "이번 주에 만들 수 있는 한 조각"으로 줄어든 게 가장 좋았다.

## 🧩 미션 2. 내 OS 기획
> 인터뷰 결과 + 세션 내용(흐민·배짱·키노) 활용해 기획

- **기획 내용:** **피드백 정리 OS** — 디자인/기획에 쏟아지는 피드백을 결 안 맞는 채로 붙여넣으면, UX 원칙·데이터 기준으로 자동 분류해 한 장의 표로 정리해주는 시스템.

  3단계 로드맵으로 설계하고, 이번 주는 **1단계만** 구현:
  - **1단계 (이번 주)** — 피드백 → 분류표 (출처/분류/UX 부합/데이터 근거/권장 처리)
  - **2단계** — 채택·기각 + 설득 근거 초안까지 자동 생성
  - **3단계 (전체 OS)** — 결정이 시안 변경 히스토리로 쌓여 "왜 이렇게 됐는지" 추적·방어

- **막혔던 점 / 어떻게 풀었나:**
  처음엔 "어떻게 입력하고 테스트해야 할지" 막막했다. 완벽한 입력 양식을 먼저 만들려다 멈췄는데, **"결 안 맞아도 그냥 통째로 붙여넣기"**로 입력 부담을 없애는 쪽으로 방향을 바꿔 풀었다. 또 기능을 다 넣으려다 산으로 갈 뻔해서, 채택/기각·설득자료는 2단계로 미루고 *"분류해서 보이게 만들기"* 하나로 1단계 범위를 좁혔다. (내가 겪던 "프로젝트가 산으로 가는" 문제를, 기획 자체에도 똑같이 적용한 셈)
