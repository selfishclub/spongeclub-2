# 1주차 — 나만의 OS 만들기 🛠️

> 미션을 진행하며 **과정과 결과물**을 기록해주세요. (다 못 채워도 OK, 한 것 위주로!)

## 🎯 미션 1. 내 OS 재료 찾기
> 인터뷰 스킬(아이데이션)로 "내 삶에 필요한 게 뭔지" 찾기

- **과정 (어떻게 찾았나):**
  클로드 코드에 스폰지클럽 1주차 인터뷰 스킬(`sponge-and-interview`)을 설치하고, 5스텝 인터뷰를 따라갔다.
  하루를 되짚으며 시간을 많이 쓴 일 → 그중 반복되고 짜증나는 지점 → 일 밖의 삶 영역 → 무게 재기 → OS가 대신할 한 가지 순서로 좁혀갔다.
  - STEP 1 (오늘 하루): ① 일정 확인 ② AI 기술 트렌드·자료 찾고 정리 ③ 메일·메시지로 상대 눈높이 맞춰 설득 ④ 결과를 임원 보고자료로 만들기
  - STEP 2 (마찰점): ②(트렌드 자료 수집·정리)와 ④(임원 보고자료 제작) — 사실 하나로 이어진 파이프라인
  - STEP 3 (삶 영역): 수술·병가 후 복귀 초기라 재활 운동·수면(4시간 이상) 리듬 지키기 / 주말에 생각 정리해 글쓰기
  - STEP 4 (무게 재기): **일 쪽**을 선택 → 지금은 ①수집(뉴스·커뮤니티 리뷰·논문) → ②인사이트 추출(경쟁사 강약점·당사 대응방안·시사점) → ③임원 눈높이로 재구성 → ④슬라이드/문서/대시보드로 매번 손수 정리
  - STEP 5 (OS가 될 한 가지): **③ 보고 포맷 변환** — 정리된 인사이트를 임원용 슬라이드·대시보드로 자동 변환

- **결과:** 🟡 내 OS 재료 카드

  ```
  🟡 내 삶을 돕는 OS · 재료 카드

  · 영역      : 일 — AI 트렌드·전략 보고 업무
  · 걸리는 지점 : 흩어진 AI 트렌드 자료를 모아 임원 보고자료로 만드는 반복 작업
               (특히 마지막 '보고 포맷 변환'이 제일 소진됨)
  · 지금은    : ① 뉴스·소비자 리뷰(커뮤니티)·논문 수집 →
               ② 인사이트 추출(경쟁사 강약점·당사 대응방안·시사점) →
               ③ 임원 눈높이로 요약·재구성 →
               ④ 슬라이드/문서/대시보드로 매번 손수 정리
  · OS가 된다면 : 정리된 인사이트를 넣으면 → 임원용 슬라이드·대시보드
               형태로 자동 변환해주는 한 칸
  · 한 문장   : "보고 자료를 만드느라 소진되던 나의 삶은, 이제 성장을 위한
               생각에만 집중하며 매일 전진하게 될 것이다"
  · 첫 한 걸음 : 클로드 코드로 '보고 포맷 변환' 스킬 하나 —
               정리된 인사이트 텍스트를 넣으면 임원용 슬라이드 초안으로 뽑아주기
  ```

- **느낀 점:**
  "딱히 대단한 걸 만들 게 있나" 싶었는데, 하루를 4단계로 쪼개 보니 매일 반복하는 파이프라인이 선명하게 보였다.
  특히 인사이트를 '고민하는 앞부분'과 슬라이드로 '옮기는 뒷부분'을 분리하고 나니, 내가 정말 넘기고 싶은 건 뒷부분(포맷 변환)이라는 게 또렷해졌다. 앞부분(사고)은 오히려 내가 쥐고 싶은 일이었다.

## 🧩 미션 2. 내 OS 기획
> 인터뷰 결과 + 세션 내용(흐민·배짱·키노) 활용해 기획

- **기획 내용:**

  **OS 이름(가제): "보고 포맷 변환기" — 인사이트는 내가, 슬라이드는 OS가**

  전체를 3단계로 나누고, 이번 주는 1단계만 완성한다.

  | 단계 | 무엇을 | 크기 |
  |------|--------|------|
  | **1단계 (이번 주)** | 정리된 인사이트 텍스트를 넣으면 → 임원용 슬라이드 초안으로 뽑아주기 | 스킬 1개 |
  | **2단계** | 인사이트 추출도 도와주기(자료 넣으면 "경쟁사 강약점·대응방안·시사점" 틀로 1차 초안) + 대시보드 포맷 | 스킬 +1~2개 |
  | **3단계** | 수집(뉴스·커뮤니티·논문)까지 앞단 자동화 → ①~④ 전체가 하나의 길로 | 전체 OS |

  **1단계 — 이번 주에 만들 것:**
  클로드 코드에 `보고-포맷-변환` 스킬을 하나 만든다. 이미 뽑아둔 인사이트 텍스트(경쟁사 강약점 / 당사 대응방안 / 시사점)를 넣으면, "제목 한 줄 + 핵심 3점 + So what(시사점)" 구조의 임원용 슬라이드 초안으로 변환해준다. 지금 ④번에서 매번 손으로 하던 레이아웃·요약·재구성을 OS가 초안까지 끌어다 놓고, 나는 다듬기만 한다.

- **막혔던 점 / 어떻게 풀었나:**
  처음엔 "수집부터 보고까지 전부 자동화"를 떠올려서 범위가 너무 커졌다. 인터뷰의 STEP 5(딱 하나만 대신한다면?)에서 **가장 소진되는 마지막 ④번(포맷 변환)만** 떼어내 1단계로 좁히니 이번 주에 실제로 만들 수 있는 크기가 됐다.

## ⚙️ 미션 3. 내 OS 구현
> 실제로 만들어본 것 (클로드코드 '채널' 기능 활용 OK)

- **결과물:** 클로드 코드 스킬 **`보고-포맷-변환` (report-format-converter)** 을 실제로 만들었다.
  - 정리된 인사이트 텍스트를 넣으면 → `결론 한 줄 → 경쟁구도 → 대응방안(단기/중기) → 실행 요약(Ask)` 순의 임원 슬라이드 초안으로 변환.
  - 각 장은 `핵심 3점 + So what(시사점)` 구조. 입력에 없는 수치는 지어내지 않고 `[확인 필요]`로 표시.
  - 임원 보고 톤(명사형 압축 + 결론 먼저)까지 자동 적용.
  - 파일: 이 폴더의 [`보고-포맷-변환_스킬/`](./보고-포맷-변환_스킬/) (`SKILL.md` + 설치·사용법 `README.md`)

- **동작 테스트 (샘플 입력 → 출력):**
  - 입력(메모 수준): "경쟁사 OpenAI가 기업용 AI 에이전트 플랫폼 발표. 강점=개발자 생태계·빠른 배포 / 약점=온프레미스 보안·규제 산업 대응 느림. 당사는 금융·의료 규제 특화 보안으로 차별화. 단기=규제 대응 레퍼런스 확보, 중기=산업별 전용 에이전트 템플릿. 시사점='범용 최강' 경쟁 대신 '규제 산업 1등' 포지셔닝."
  - 출력(발췌):
    ```
    [슬라이드 1] 요약 — '규제 산업 1등'으로 정면 차별화
    [슬라이드 2] 경쟁 구도 — OpenAI 강점(생태계·속도) vs 빈틈(규제·보안)
    [슬라이드 3] 당사 대응 — 단기: 규제 레퍼런스 / 중기: 산업별 템플릿 → So what: 자원 집중
    [슬라이드 4] 실행 요약(Ask) — 리소스 재배분 승인 / 예산 [확인 필요]
    ```
  - 결과: 메모 수준 인사이트가 임원 보고 4장 구조로 자동 변환됨을 확인. ✅
