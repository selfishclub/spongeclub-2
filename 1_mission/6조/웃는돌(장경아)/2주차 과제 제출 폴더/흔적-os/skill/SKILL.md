# 흔적-분류-복원 (흔적 OS의 두뇌)

> 원칙: **내가 시스템을 돌보는 게 아니라, 시스템이 내 흔적을 돌본다.**
> 사용자는 던지기만 한다. 정리·분류·복원은 이 스킬이 대신한다.

## 역할(고정 4개 + 예비)

| 역할 | 뜻 | 예시 신호어 |
|------|-----|-----------|
| `본업` | 회사/직장 업무 | 회의, 보고, 상사, 프로젝트명, 마감 |
| `콘텐츠` | 글쓰기·콘텐츠 발행 | 글감, 제목, 발행, 블로그, 뉴스레터, 초안 |
| `사이드` | 사이드 프로젝트/창업 | PoC, 기획서, 아이디어, 비즈니스, 실험 |
| `엄마` | 육아·가정 | 아이, 어린이집, 준비물, 병원, 저녁, 가족 |
| `미분류` | 위에 안 걸리면 임시 보관 | 애매하면 여기 (버리지 않는다) |

애매하면 **버리지 말고** `미분류`로 담고, 사용자에게 한 줄로만 되물어본다. (되묻기는 최대 1회, 안 하고 미분류로 둬도 OK — 무관리 원칙)

## 두 가지 모드

### 1) 담기 (Capture → Classify)
입력: 흘려보낸 한 줄 (또는 `흔적-os/inbox.md` 안의 미처리 줄들)

각 줄마다:
1. **역할** 판단 (본업/콘텐츠/사이드/엄마/미분류)
2. **종류** 판단 — `할일` / `생각` / `글감` / `자료` 중 하나
   - 동사·마감·"해야"·물음표 없는 지시 → `할일`
   - 링크·URL·출처 → `자료`
   - "~쓰면 좋겠다"·제목 후보·소재 → `글감`
   - 나머지 관찰·아이디어 → `생각`
3. **한 줄 정제** — 원문 뜻은 유지하되 나중에 알아볼 수 있게 다듬기 (지어내기 금지)
4. 해당 `흔적-os/roles/{역할}.md`의 알맞은 섹션에 **한 줄 append** (형식은 아래)
5. 처리한 줄은 inbox에서 제거(또는 `- [x]` 처리)

**역할 파일 저장 형식** (각 `roles/{역할}.md`):
```
## 🔓 열린 루프 (할일 / 결정 대기)
- [ ] (YYYY-MM-DD) 정제된 한 줄

## 💭 생각 · 글감
- (YYYY-MM-DD) 정제된 한 줄  ·  [글감]

## 📎 자료
- (YYYY-MM-DD) 정제된 한 줄 — 링크
```
- 할일 → `열린 루프` / 생각·글감 → `생각·글감` / 자료 → `자료`
- 항상 **최신이 위로**. 날짜는 캡처 시각.

### 2) 꺼내기 (Restore — 역할 전환 시 '한 화면 복원')
입력: 전환하려는 역할 이름 (예: "본업 복원", "지금 콘텐츠 상태")

해당 역할 파일을 읽고 **딱 한 화면**으로 압축해 보여준다:
```
🟢 [{역할}] 복원 — {오늘 날짜}

▶ 지금 열린 루프 (먼저 처리)
  - [ ] ... (최대 5개, 오래된 것/마감 임박 우선)

▶ 최근 생각·글감 (최근 3개)
  - ...

▶ 마지막으로 만진: {가장 최근 날짜}
▶ 한 줄 브리핑: "이 역할에서 지금 중요한 건 __ 이다"
```
- 목표: **맥락 재부팅 없이** 이 화면만 보면 그 역할로 바로 이어가기.
- 5개/3개를 넘기면 잘라내고 "…외 N개"로 표시. 화면을 넘기지 않는다.

## 하지 말 것
- 없는 내용 지어내기 (원문에 없는 수치·사실 금지)
- 애매하다고 버리기 (→ 미분류로 보관)
- 한 화면을 넘기는 장황한 복원 (요약이 핵심)
