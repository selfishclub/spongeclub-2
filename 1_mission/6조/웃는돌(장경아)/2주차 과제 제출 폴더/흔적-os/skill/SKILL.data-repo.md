# 흔적-분류-복원 (흔적 OS의 두뇌)

> 실행 위치: **`imprint-os-data` 저장소** (루트에 `inbox.md`, `roles/`).
> 원칙: 사용자는 던지기만 한다. 정리·분류·복원은 이 스킬이 대신한다. (무관리)

## 역할(고정 4개 + 예비)

| 역할 | 뜻 | 신호어 |
|------|-----|-----------|
| `본업` | 회사/직장 업무 | 회의, 보고, 상사, 프로젝트명, 마감 |
| `콘텐츠` | 글쓰기·콘텐츠 발행 | 글감, 제목, 발행, 블로그, 뉴스레터, 초안 |
| `사이드` | 사이드 프로젝트/창업 | PoC, 기획서, 아이디어, 비즈니스, 실험 |
| `엄마` | 육아·가정 | 아이, 어린이집, 준비물, 병원, 저녁, 가족 |
| `미분류` | 위에 안 걸리면 임시 보관 | 애매하면 여기 (버리지 않는다) |

## 두 가지 모드

### 1) 담기 (인박스 정리 — inbox.md → roles/)
1. `inbox.md` 를 읽어 아직 처리 안 된 줄(`- [ ]`)들을 모은다.
2. 각 줄마다:
   - **역할** 판단 (본업/콘텐츠/사이드/엄마/미분류)
   - **종류** 판단 — `할일` / `생각` / `글감` / `자료`
     - 동사·마감·"해야" → 할일 / 링크·URL → 자료 / 제목후보·"~쓰면" → 글감 / 나머지 → 생각
   - **한 줄 정제** (뜻 유지, 지어내기 금지)
   - `roles/{역할}.md` 의 알맞은 섹션에 한 줄 append (파일 없으면 새로 만들기)
3. 처리한 줄은 `inbox.md` 에서 지우거나 `- [x]` 처리.

**roles/{역할}.md 형식** (없으면 이 골격으로 생성):
```
# {역할} · 흔적 스토어

## 🔓 열린 루프 (할일 / 결정 대기)
- [ ] (YYYY-MM-DD) 정제된 한 줄

## 💭 생각 · 글감
- (YYYY-MM-DD) 정제된 한 줄  ·  [글감]

## 📎 자료
- (YYYY-MM-DD) 정제된 한 줄 — 링크
```
- 할일→열린 루프 / 생각·글감→생각·글감 / 자료→자료. 항상 최신이 위로. 날짜는 캡처일(줄 앞의 날짜) 사용.

### 2) 꺼내기 (역할 복원 — '한 화면')
입력: 역할 이름 ("본업 복원", "지금 콘텐츠 상태")
`roles/{역할}.md` 를 읽어 한 화면으로 압축:
```
🟢 [{역할}] 복원 — {오늘}

▶ 지금 열린 루프 (먼저 처리)
  - [ ] ... (최대 5개, 오래된/마감임박 우선)

▶ 최근 생각·글감 (최근 3개)
  - ...

▶ 마지막으로 만진: {가장 최근 날짜}
▶ 한 줄 브리핑: "이 역할에서 지금 중요한 건 __ 이다"
```
- 5개/3개 넘으면 잘라 "…외 N개". 화면을 넘기지 않는다.

## 하지 말 것
- 없는 내용 지어내기 / 애매하다고 버리기(→미분류) / 한 화면 넘는 장황한 복원
