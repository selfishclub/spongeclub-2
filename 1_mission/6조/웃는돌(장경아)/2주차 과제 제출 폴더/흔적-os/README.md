# 흔적 OS 🫧

> **"내가 시스템을 돌보는 게 아니라, 시스템이 내 흔적을 돌본다."**
> 흘러가는 하루(본업·콘텐츠·사이드·엄마)가 저절로 쌓이는 개인 OS.

## 왜
역할을 오갈 때마다 머릿속을 **재부팅**하는 비용(맥락 다시 불러오기)을 없앤다.
던지기만 하면 시스템이 줍고·분류하고·복원한다.

## 최소 루프 (캡처 → 분류 → 복원)

```
① 캡처   inbox.md 에 한 줄 던진다 (30초 이내)
   │      (2단계에서 텔레그램 한 줄 → 자동으로 inbox 유입)
   ▼
② 분류   "인박스 정리해줘" → 흔적-분류-복원 스킬이
   │      역할별(본업/콘텐츠/사이드/엄마)로 roles/*.md 에 쌓는다
   ▼
③ 복원   "본업 복원" → 그 역할의 '현재 상태'를 한 화면으로
```

## 폴더 구조
```
흔적-os/
├── README.md          ← 지금 이 파일
├── inbox.md           ← 던지는 곳 (캡처)
├── roles/             ← 역할별로 쌓이는 곳 (분류 결과)
│   ├── 본업.md
│   ├── 콘텐츠.md
│   ├── 사이드.md
│   ├── 엄마.md
│   └── 미분류.md
└── skill/
    └── SKILL.md       ← 흔적 OS의 두뇌 (분류·복원 규칙)
```

## 쓰는 법
1. **던지기**: `inbox.md`에 `- [ ] 한 줄` 추가 (또는 클로드에게 "이거 담아줘: …")
2. **분류**: 클로드에게 `"인박스 정리해줘"` → 역할별로 자동 정리
3. **복원**: 역할 전환할 때 `"본업 복원"` / `"콘텐츠 지금 뭐였지"` → 한 화면 브리핑

## 로드맵
- [x] **1단계** — 분류·복원 두뇌(스킬) + 역할 저장 구조  ← 2주차 지금
- [ ] **2단계** — 텔레그램 봇 → inbox 자동 유입 (30초 캡처 완성)
- [ ] **3단계** — 역할 전환 감지 시 복원 화면 자동 푸시

## 설치 (클로드 코드)
`skill/SKILL.md`를 클로드 코드 스킬로 등록하면, 위 "담기/꺼내기" 명령을 알아듣는다.
(1주차 `보고-포맷-변환` 스킬과 같은 방식)
