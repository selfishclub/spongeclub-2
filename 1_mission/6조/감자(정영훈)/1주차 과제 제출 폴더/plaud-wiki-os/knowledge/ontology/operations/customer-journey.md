# 우리 회사 고객 여정 & 파이프라인 설계서

> 관련 원칙: [[mentor-k-sales]], [[mentor-k-systems]]

---

## 1. 고객 여정 맵

### 전체 흐름

```
광고유입 ─┐
대행사건 ─┼→ 상담중 → 미팅예정 → 제안중 → 계약완료 → 서비스중 → 종료
소개유입 ─┘                                                  ↑
                                                            │
콜드보관 ──── (재연락 시) ──→ 상담중 ─────────────────────────────┘
```

### 입구 3개 (리드 소스별)

| 입구 | 대상 | 정산 |
|------|------|------|
| **광고유입** | Meta 광고, 진단 폼, 홈페이지, 이북 | 없음 |
| **대행사건** | 협력업체8 등 영업대행사 | 수익쉐어 (대행사별 조건) |
| **소개유입** | 소개인1, 소개인2 등 개인 소개 + 직접 문의 | 마진쉐어 또는 없음 |

### 단계별 상세

| # | 단계 | 고객 상태 | 우리 회사 행동 | 기간 기준 |
|---|------|----------|----------------|----------|
| 1 | **광고유입/대행사건/소개유입** | 리드 생성됨, 미접촉 | 24시간 이내 1차 연락 | 즉시 |
| 2 | **상담중** | 1차 대화 진행 중 | 니즈 파악, 진단 리포트 전달, 미팅 잡기 | 7일 이내 |
| 3 | **미팅예정** | 미팅 일정 확정 | 고객 GBP 사전 분석, 제안 초안 | - |
| 4 | **제안중** | 미팅 완료, 견적 검토 중 | 견적서 발송, Q&A 대응, 반론 처리 | 14일 이내 |
| 5 | **계약완료** | 계약 서명 + 입금, 온보딩 | GBP 권한 이관, 작업자 배정, 초기 세팅 | 7일 이내 |
| 6 | **서비스중** | 정상 서비스 진행 | 주간/월간 리포트, 최적화, 업셀 기회 포착 | 계약 기간 중 (자동 연장) |
| 7 | **종료** | 해지/거절 | 사유 기록, 3~6개월 후 리터칭 | - |

> 계약은 자동 연장 조항 있음. 별도 리뉴얼 단계 불필요. 업셀은 서비스중에서 알림으로 처리.

### 이탈 경로 (어디서든 빠질 수 있음)

| 이탈 시점 | 이유 | 처리 |
|----------|------|------|
| 유입 후 무응답 | 관심만 있었음 | 7일 후 → 콜드보관 |
| 상담 중 무응답 | 타이밍 안 맞음, 가격 부담 | 14일 후 2차 팔로업, 30일 후 → 콜드보관 |
| 제안 후 거절 | 경쟁사 선택, 예산 부족, 필요성 불인정 | 거절 사유 기록 → 종료, 3개월 후 리터칭 |
| 서비스 중 해지 | 효과 불만, 예산 삭감 | 이탈 사유 기록 → 종료 |

### 재진입 (콜드보관/종료 → 복귀)

| 상황 | 처리 |
|------|------|
| 콜드보관 고객이 재연락 | 기존 의뢰를 **상담중**으로 이동 (새 의뢰 안 만듦) |
| 종료 고객이 재문의 | 기존 의뢰를 **상담중**으로 이동 |
| 분기 리터칭 응답 | **상담중**으로 복귀 |

## 2. pluuug 파이프라인 단계 정의서

### 신규 단계 구성 (10단계)

현재 11단계 → **10단계** (입구 3개 + 파이프라인 4개 + 계약 후 2개 + 종료 1개 + 콜드보관)

```
광고유입 ─┐
대행사건 ─┼→ 상담중 → 미팅예정 → 제안중 → 계약완료 → 서비스중 → 종료
소개유입 ─┘                                                  ↑
                                                            │
콜드보관 ──── (재연락 시) ──→ 상담중 ─────────────────────────────┘
```

#### 입구 1: 광고유입

| 항목 | 내용 |
|------|------|
| **정의** | Meta 광고, 진단 폼, 홈페이지, 이북 등을 통해 유입된 리드. |
| **정산** | 없음 (자체 마케팅 비용만) |
| **해야 할 일** | 24시간 이내 1차 연락 (전화 또는 카톡) |
| **이탈 조건** | 7일 무응답 → 콜드보관 |
| **다음 단계** | 연락 성공 → **상담중** |

#### 입구 2: 대행사건

| 항목 | 내용 |
|------|------|
| **정의** | 협력업체8 등 영업대행사가 가져온 건. |
| **정산** | 수익쉐어 (대행사별 조건 상이) |
| **해야 할 일** | 대행사로부터 고객 정보 인수 → 24시간 이내 1차 연락 |
| **이탈 조건** | 7일 무응답 → 콜드보관 |
| **다음 단계** | 연락 성공 → **상담중** |

#### 입구 3: 소개유입

| 항목 | 내용 |
|------|------|
| **정의** | 소개인1, 소개인2 등 개인 소개 또는 직접 문의. |
| **정산** | 마진쉐어 또는 없음 |
| **해야 할 일** | 24시간 이내 1차 연락 |
| **이탈 조건** | 7일 무응답 → 콜드보관 |
| **다음 단계** | 연락 성공 → **상담중** |

#### 단계 4: 상담중

| 항목 | 내용 |
|------|------|
| **정의** | 1차 연락 완료. 니즈 파악 및 미팅 유도 중. |
| **진입 조건** | 고객과 최초 대화 성공 (전화, 카톡, 이메일) |
| **해야 할 일** | 진단 리포트 발송, 니즈 파악, 미팅 일정 잡기 |
| **이탈 조건** | 14일 무응답 → 1회 추가 팔로업, 30일 무응답 → 콜드보관 |
| **다음 단계** | 미팅 일정 확정 → **미팅예정** |

#### 단계 5: 미팅예정

| 항목 | 내용 |
|------|------|
| **정의** | 미팅(대면/화상/전화) 일정이 확정된 상태. |
| **진입 조건** | 구체적 미팅 일시 합의 |
| **해야 할 일** | 미팅 준비 (고객 GBP 사전 분석, 경쟁사 분석, 제안 초안) |
| **이탈 조건** | 미팅 노쇼 2회 → 콜드보관 |
| **다음 단계** | 미팅 완료 → **제안중** |

#### 단계 6: 제안중

| 항목 | 내용 |
|------|------|
| **정의** | 미팅 완료. 견적서/제안서 발송 후 결정 대기. |
| **진입 조건** | 미팅 진행 완료 (대면, 전화, 화상 모두 포함) |
| **해야 할 일** | 견적서 발송 (미팅 당일 또는 익일), 3일 후 팔로업, 반론 처리 |
| **이탈 조건** | 명시적 거절 → 사유 기록 후 **종료**. 21일 무응답 → 콜드보관 |
| **다음 단계** | 계약 동의 → **계약완료** |

#### 단계 7: 계약완료

| 항목 | 내용 |
|------|------|
| **정의** | 계약서 서명 + 입금 확인. 온보딩 포함. |
| **진입 조건** | 계약서 체결 + 첫 입금 확인 |
| **해야 할 일** | pluuug 계약 생성, 작업자 배정, GBP 권한 요청, 초기 세팅 |
| **체류 기간** | 최대 7일 |
| **다음 단계** | 초기 세팅 완료 → **서비스중** |

#### 단계 8: 서비스중

| 항목 | 내용 |
|------|------|
| **정의** | 정상 서비스 진행. 자동 연장 조항으로 별도 리뉴얼 불필요. |
| **진입 조건** | 초기 세팅 완료 + 주간 작업 루틴 시작 |
| **해야 할 일** | 주간/월간 리포트, 최적화 작업, 업셀 기회 포착 (Basic→Standard 등) |
| **이탈 조건** | 해지 요청 → 사유 기록 후 **종료** |

#### 단계 9: 종료

| 항목 | 내용 |
|------|------|
| **정의** | 의뢰가 최종 종결된 상태. 거절, 해지 모두 포함. |
| **진입 조건** | 명시적 거절, 계약 해지 |
| **기록** | 종료 사유 필수 기록 (가격, 효과 불만, 경쟁사, 내재화, 기타) |
| **리터칭** | 거절: 3개월 후 / 해지: 6개월 후 |

#### 단계 10: 콜드보관

| 항목 | 내용 |
|------|------|
| **정의** | 무응답으로 활성 파이프라인에서 제외된 리드. 데이터 보존. |
| **진입 기준** | 유입 7일 / 상담중 30일 / 미팅예정 노쇼 2회 / 제안중 21일 무응답 |
| **재활성화** | 고객 재연락 시 → **상담중**으로 복귀 (기존 의뢰 이동, 새로 안 만듦) |
| **일괄 리터칭** | 분기 1회, 콜드리드 대상 진단 리포트 재발송 |

### 단계 비교: AS-IS vs TO-BE

| AS-IS (현재 11단계) | 문제 | TO-BE (신규 10단계) |
|--------------------|------|-------------------|
| 유입 | 소스 구분 없음 | 삭제 → **광고유입/대행사건/소개유입** 3개로 분리 |
| 광고 유입 | 유지 | **광고유입** (이름 유지 가능) |
| 소개건 | 대행사와 개인 혼재 | **소개유입** (개인 소개 + 직접 문의) |
| (없음) | 대행사 구분 없음 | **대행사건** 신설 |
| 1차 컨택 | 상담 과정의 일부 | **상담중** (1차 연락~미팅 전) |
| 콜백 | 상담 과정의 일부 | **상담중**에 통합 |
| 미팅 예정 | 유지 | **미팅예정** (기존 유지) |
| 미팅 완료 | 견적과 분리 불필요 | **제안중** (미팅 후~계약 전) |
| 견적 요청 | 미팅 완료와 중복 | **제안중**에 통합 |
| 무응답 | 콜드와 중복 | 삭제 → **콜드보관**으로 통합 |
| 최종 거절 | 종료 상태 | **종료**에 통합 |
| 미팅 후 이탈 | 종료 상태 | **종료**에 통합 |
| (없음) | 계약 후 관리 없음 | **계약완료, 서비스중** 신설 |

## 3. 리드 소스 관리 방안

### 원칙: 입구 단계 = 리드 소스

입구를 3개(광고유입/대행사건/소개유입)로 나눴으므로, 의뢰가 어디서 왔는지는 **최초 진입 단계**로 추적한다. 상담중 이후로 넘어가도 pluuug에 이력이 남아있으므로 별도 태그 불필요.

### 소스별 전환율 추적 (월간)

```
| 소스 | 리드 수 | 상담 전환 | 미팅 전환 | 계약 전환 | CPL | CAC |
|------|--------|----------|----------|----------|-----|-----|
| 광고유입 | - | - | - | - | ~n원 | - |
| 대행사건 | - | - | - | - | 0원 | 수익쉐어 |
| 소개유입 | - | - | - | - | 0원 | 마진쉐어 또는 0 |
```

> 이 표는 pipeline.md의 월간 리포트에 추가한다.

## 4. 콜드리드 처리 규칙

### 콜드 판정 기준

| 현재 단계 | 콜드 기준 | 최종 시도 |
|----------|----------|----------|
| 광고유입/대행사건/소개유입 | 7일 무응답 (3회 연락 시도 후) | 진단 리포트 발송 |
| 상담중 | 30일 무응답 | "혹시 다른 결정 하셨나요?" 메시지 |
| 미팅예정 | 노쇼 2회 | 일정 재조율 요청 |
| 제안중 | 21일 무응답 | "견적 관련 궁금하신 점 있으신가요?" |

### 콜드 처리 프로세스

```
무응답 감지 → 최종 팔로업 메시지 → 응답 없음 → 콜드보관으로 이동
                                    ↓
                                응답 있음 → 원래 단계 유지
```

### 콜드리드 리터칭 (분기 1회)

1. 콜드보관 리스트 추출
2. 최신 진단 리포트 자동 생성 (GBP 현재 상태 기반)
3. "3개월 전 문의하셨던 OO원장님, GBP 현황이 이렇게 변했습니다" 형태로 발송
4. 응답 시 → 신규유입 또는 상담중으로 복귀

### 현재 n건 중 콜드 판정 대상

- 과거 Meta 광고 유입 → 4개월+ 무응답 → **전원 콜드보관**
- 현재 n건이 30일+ → 이 중 최근 30일 이내 접촉 이력 있는 건만 활성 유지
- 예상: 활성 n건 유지, 나머지 n건 콜드보관

## 5. 자동화 포인트

### 자동화 가능 (pluuug API)

| 트리거 | 자동화 내용 | 구현 방법 |
|--------|-----------|----------|
| 신규 리드 유입 | 의뢰 자동 생성 + "신규유입" 단계 배정 | Meta 웹훅 → pluuug POST /inquiries |
| 콜드 판정 | 30일 무응답 의뢰 → 콜드보관 단계 이동 | 매일 크론: 의뢰 목록 조회 → 마지막 활동일 체크 → PATCH |
| 단계 변경 알림 | 의뢰 단계 변경 시 iMessage 알림 | pluuug 웹훅 또는 폴링 → iMessage |
| 리뉴얼 알림 | 계약 만료 30일 전 알림 | 매일 크론: 계약 만료일 체크 → iMessage 알림 |
| 리드 소스 집계 | 월간 소스별 전환율 자동 리포트 | 월 1회 크론: 의뢰 + 계약 조회 → 집계 |

### 반자동 (트리거 자동 + 실행 수동)

| 트리거 | 알림 | 수동 작업 |
|--------|------|----------|
| 신규유입 24시간 경과 | "OO 리드 아직 미접촉" 알림 | 전화/카톡 |
| 상담중 7일 경과 | "OO 팔로업 필요" 알림 | 팔로업 메시지 |
| 제안 후 3일 경과 | "OO 견적 팔로업" 알림 | 전화/카톡 |
| 리뉴얼 시점 도달 | "OO 만료 30일 전" 알림 | 성과 리포트 + 리뉴얼 미팅 |

### 수동 (자동화 불가)

| 작업 | 이유 |
|------|------|
| 미팅 진행 | 대면/화상 상담 |
| 견적서 작성 | 고객별 맞춤 (pricing.md 기반이지만 커스텀 요소 있음) |
| 계약 체결 | 서명, 입금 확인 |
| 업셀 판단 | 고객 상태, 관계, 성과 기반 인간 판단 |

### 우선순위

1. **지금 당장**: 콜드리드 일괄 이동 (마이그레이션)
2. **1주 내**: 콜드 자동 판정 크론 (매일 실행)
3. **2주 내**: 리뉴얼 알림 크론 (만료 30일 전)
4. **1개월 내**: 신규유입 미접촉 알림, 팔로업 알림

## 6. 마이그레이션 계획 (현재 → 신규)

### Step 0: pluuug 웹 UI에서 신규 단계 생성

pluuug 단계 추가는 API로 불가, 웹에서만 가능. 아래를 새로 생성:

1. **대행사건** (신규)
2. **상담중** (신규)
3. **제안중** (신규)
4. **계약완료** (신규)
5. **서비스중** (신규)
6. **종료** (신규)
7. **콜드보관** (신규)

기존 유지:
- **광고 유입** (기존 유지)
- **소개건** → "소개유입"으로 이름 변경 (가능하면)
- **미팅 예정** (기존 유지)

> 기존에서 삭제할 단계: 유입, 1차 컨택, 콜백, 미팅 완료, 견적 요청, 무응답, 최종 거절, 미팅 후 이탈
> 모든 의뢰 이동 완료 후 삭제한다.

### Step 1: n건 분류 + 이동

| 분류 | 기준 | 이동할 단계 |
|------|------|-----------|
| 계약 진행 중 (status=I) | contract 연결 + 활성 | **서비스중** |
| 계약 대기 (status=P) | contract 연결 + 대기 | **계약완료** |
| 계약 완료/해지 (status=C,T) | contract 종료 | **종료** |
| 최근 30일 활성 (계약 없음) | 현재 단계 기반 매핑 | 광고유입/소개유입/상담중/제안중 |
| 30일+ 무응답 (계약 없음) | 콜드 리드 | **콜드보관** |

### Step 2: 기존 단계 정리

모든 의뢰 이동 완료 후:
1. 사용하지 않는 기존 단계 삭제 (유입, 1차 컨택, 콜백, 미팅 완료, 견적 요청, 무응답, 최종 거절, 미팅 후 이탈)
2. "소개건" → "소개유입"으로 이름 변경 (가능하면)
