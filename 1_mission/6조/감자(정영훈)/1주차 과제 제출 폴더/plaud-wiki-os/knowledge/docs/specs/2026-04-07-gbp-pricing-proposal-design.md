# GBP 프라이싱 제안서 설계 스펙

## 목적
신규 병원 대상 콜드 영업용 제안서. 시장 논리 → 플랫폼 인사이트 → 솔루션 → 사례 → 프라이싱 순서로 설득.

## 타겟
외국인환자 유치 병원 (성형외과, 피부과, 치과, 한의원 등). GBP 관리를 아직 안 하거나 체계적으로 못 하고 있는 곳.

## 포맷
- **HTML 슬라이드 프레젠테이션** (reveal.js 기반)
- 16:9 비율, 풀스크린
- 키보드 화살표 네비게이션
- PDF 내보내기 (Ctrl+P 또는 ?print-pdf 쿼리)
- 단일 HTML 파일 (CDN 의존)

## 톤앤매너
- 다크 네이비 (#0a1628) + 화이트 + 블루 악센트 (#3b82f6)
- 클린, 미니멀, 프리미엄 느낌
- 한국어 본문 + 영어 섹션 라벨 (MARKET INSIGHT, PLATFORM INSIGHT 등)
- 폰트: Pretendard (본문) + 영문 볼드 산세리프

## 슬라이드 구성 (13장)

### Part 1: 시장 논리 (Why Now)

**01 — 커버**
- 우리 회사 로고/워드마크
- "Google Maps Growth for Medical Tourism"
- 서브: "데이터 기반 글로벌 환자 유치 시스템"

**02 — 시장 변화 (MARKET INSIGHT 01)**
- 제목: "의료관광 시장의 구조적 전환"
- 내수 의료 시장 성장 둔화 vs 외국인 환자 급증
- 핵심 수치: 방한 외래 관광객 1,845만명 (전년 대비 15.2% 증가), 외국인 관광소비 15.7조 (20.5% 증가)
- 의료관광 환자 60만명+ 돌파

**03 — 환자 여정 (MARKET INSIGHT 02)**
- 제목: "외국인 환자는 어떻게 병원을 찾는가?"
- 여정 플로우: Google 검색 → Google Maps → 리뷰 확인 → 예약/방문
- 네이버/카카오맵은 외국인 접근 불가 (언어, 인증 장벽)
- 구글 지도가 유일한 터치포인트

**04 — 데이터 근거 (MARKET INSIGHT 03)**
- 제목: "데이터로 입증되는 K-의료 수요"
- 한국 의료관광 검색량 추이 차트 (Google Trends 스타일)
- 서울 아시아 재방문 희망 여행지 3위
- 의료관광 성장률 연간 추이

### Part 2: 플랫폼 인사이트 (Why Google Maps)

**05 — 왜 구글 지도인가 (PLATFORM INSIGHT)**
- 4가지 이유 카드 레이아웃:
  1. 글로벌 1위 지도 플랫폼 (MAU 10억+, 256개국)
  2. 국내 플랫폼의 구조적 한계 (언어/인증 장벽)
  3. 환자 동선 직접 타겟팅 (Walk-in + 예약)
  4. 수요 대비 압도적 저경쟁 (블루오션)

**06 — 의료관광 특화 문제 (PROBLEM)**
- 현재 대부분 병원의 GBP 현실:
  - 다국어 리뷰 관리 부재 → 악성 리뷰 방치
  - GBP 프로필 미최적화 → 검색 노출 저조
  - 경쟁사 대비 콘텐츠 부족 → 신뢰도 열위
  - 데이터 기반 의사결정 불가 → 감으로 운영

### Part 3: 솔루션 (What We Do)

**07 — 풀퍼널 시스템 (CORE STRATEGY 01)**
- 3단계 플로우:
  - Phase 1: 알고리즘 장악 (Organic SEO) — GBP 최적화, 키워드 선점
  - Phase 2: 수요 선제 타격 (Google Ads) — 상권 내 잠재 환자 타겟팅
  - Phase 3: 이탈 제로화 (Conversion) — 다국어 랜딩, 예약 직결

**08 — 8단계 통합 프로세스 (CORE STRATEGY 02)**
- 8개 카드 그리드:
  1. 프로필 최적화 세팅
  2. 다국어 콘텐츠 기획 (영/중/일)
  3. 타겟팅 광고 세팅 (Google Ads)
  4. 전환율 관리 (다국어 랜딩)
  5. 브랜드 평판 방어 (리뷰 모니터링)
  6. 핵심 키워드 최적화
  7. 고품질 트래픽 모니터링
  8. 월간 데이터 분석 리포트

**09 — 우리 회사 차별점 (WHY US)**
- 다른 대행사와 다른 점:
  - AI 기반 자동 리포트 (PPT 자동 생성)
  - 실시간 순위 트래킹 대시보드 (클라이언트 포털)
  - 다국어 악성 리뷰 자동 감지 + 텔레그램 알림
  - 경쟁사 프로필/리뷰 모니터링
  - 데이터 기반 키워드 전략 (DataForSEO 연동)

## 기술 구현

### 파일 구조
```
company-proposal/
├── index.html          # 단일 HTML (reveal.js CDN)
├── assets/
│   └── images/         # 차트 이미지, 로고 등
└── README.md
```

### 의존성
- reveal.js 5.x (CDN)
- Pretendard 폰트 (CDN)
- Chart.js 또는 인라인 SVG (차트용)

### PDF 내보내기
- `?print-pdf` 쿼리 파라미터로 인쇄 레이아웃 전환
- Ctrl+P → PDF 저장

## 데이터 슬롯 관리
- 사례 데이터: HTML 내 `data-case="1"` 등 마커로 표시
- 가격 데이터: `[TBD]` 텍스트로 표기, 나중에 일괄 치환
- 차트 데이터: 인라인 변수로 분리하여 수정 용이하게
