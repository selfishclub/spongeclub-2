# ourcompany.kr 스토리텔링 홈페이지 재설계 스펙

## 개요

기존 기능 나열형 홈페이지를 **스토리텔링 기반**으로 전면 재설계.
growth-engine.co.kr의 내러티브 흐름을 참고하되, 우리 회사만의 세일즈 포인트를 담는다.

**핵심 메시지:** "원장님은 진료에만 집중하세요. 마케팅은 시스템이 합니다."

**디자인 톤:** 내러티브 미니멀. 화이트 배경, 큰 텍스트, 넉넉한 여백. 카피가 주인공. 사진/스크린샷으로 신뢰 보강.

## 피드백 반영 사항

- 담당자 피드백: 진단 CTA를 히어로로 올리기, 레퍼런스 기반 디자인, AI 생성 티 제거
- 사용자 피드백: 콘텐츠 밀도 낮추기, 사진 많이, 스토리 차별화 ("우리 잘한다"만으론 부족)

## 사이트 구조

```
ourcompany.kr/                    ← 메인 홈 (스토리 아크 + 진단 히어로)
ourcompany.kr/google-maps         ← 구글 지도 스토리
ourcompany.kr/youtube             ← 유튜브 스토리
ourcompany.kr/instagram           ← 인스타·숏폼 스토리
ourcompany.kr/search-ads          ← 검색광고 스토리
ourcompany.kr/naver               ← 네이버 플레이스 스토리
ourcompany.kr/website             ← 홈페이지 제작 스토리
ourcompany.kr/case-studies        ← 성공사례 (숫자 중심)
ourcompany.kr/diagnostic          ← 무료 진단 (기존 wizard 리디자인)
ourcompany.kr/blog                ← 블로그 (기존 유지)
ourcompany.kr/pricing             ← 가격 (홈에서 분리)
ourcompany.kr/faq                 ← FAQ (홈에서 분리)
ourcompany.kr/privacy             ← 개인정보처리방침 (기존 유지)
```

## 1. 메인 홈페이지 (/)

### 스토리 아크 (9섹션)

| # | 섹션 | 역할 | 핵심 카피 |
|---|------|------|----------|
| 1 | 히어로 | 궁금증 유발 + 진단 입력 | "구글에서 우리 병원, 환자 눈에 보이고 있을까?" + 큰 검색창 |
| 2 | 공감 | 원장님 고민 대변 | "블로그도, 광고도 하는데 왜 환자는 안 늘까요?" |
| 3 | 문제 재정의 | 착각 깨기 (다크 반전) | "네이버 1등, 릴스 10만뷰... 이게 진짜 성공일까요?" |
| 4 | 인사이트 | 환자 여정 시각화 | "환자는 한 채널에서 결정하지 않습니다" (구글→유튜브→인스타→예약) |
| 5 | 우리 방식 | 솔루션 제시 | "전체 채널을 하나의 시스템으로" + 채널 필 목록 |
| 6 | 증거 | 성과 숫자 | 케이스 3개 핵심 수치 (상세는 /case-studies) |
| 7 | 대표 메시지 | 진정성 | 대표 사진 + 인용문 |
| 8 | CTA | 행동 유도 | "우리 병원 마케팅, 한번 확인해 보세요" + 진단 버튼 |
| 9 | 푸터 | 네비게이션 | 서비스 링크, SNS, 연락처 |

### 히어로 진단 입력

- 병원 이름 입력 → /diagnostic으로 이동
- 예시 버튼: "협조병원1 피부과", "협조병원2 치과" 등
- 구글 GBP 데이터 기반 진단 (DataForSEO API, 기존 연동)
- 포지셔닝: "구글 검색 노출 상태 확인" (네이버 약속 안 함)

### 사진 필수 영역 (3곳)

1. 히어로 아래: 대시보드 스크린샷 또는 팀 작업 사진
2. 환자 여정 섹션: 구글맵 검색 결과 실제 스크린샷
3. 증거 섹션 아래: 실제 리포트/대시보드 (public/images/proof/에서 선별)

## 2. 서비스 페이지 (6개)

모든 서비스 페이지는 동일한 스토리 아크를 공유:

**공감 → 재정의 → 인사이트 → 우리 방식 → 증거 → CTA**

### /google-maps — 구글 지도 최적화

| 섹션 | 내용 |
|------|------|
| 공감 | "구글 지도에서 우리 병원, 환자 눈에 보이고 있나요?" 내수 단가 낮고 경쟁 치열 → 외국인이 답 |
| 재정의 | 구글 지도 정보 반출 → 네이버 플레이스 신뢰도 하락 → 구글 점유율 상승 |
| 인사이트 | 외국인 환자 여정은 구글 안에서 완결. 객단가 내수 3~10배, 경쟁은 블루오션 |
| 우리 방식 | GBP 프로필 최적화, 다국어 리뷰 관리, 키워드 최적화, 월간 리포트 |
| 증거 | 협조병원1 홍대 +539%, 협조병원2 치과 키워드 1위 |
| CTA | "우리 병원이 구글에서 어떻게 보이는지, 무료로 확인하세요" + 진단 입력창 |

### /youtube — 유튜브

| 섹션 | 내용 |
|------|------|
| 공감 | "원장님 영상 하나 없이, 환자 신뢰를 어떻게 얻으시겠어요?" |
| 재정의 | 유튜브 = 가장 강력한 설득. 롱폼 끝까지 본 환자는 무조건 온다. 그리고 결제한다. |
| 인사이트 | 영상 하나 = 영업사원 100명. 24시간 설득, 광고비 0원 |
| 우리 방식 | 기획→촬영→편집→배포. 원장님은 촬영만. |
| 증거 | 실제 제작 영상 임베드 + "이 영상 보고 왔어요" 후기 |
| CTA | "우리 병원 영상, 어떻게 만들면 좋을지 상담받기" |

### /instagram — 인스타·숏폼

| 섹션 | 내용 |
|------|------|
| 공감 | "뭘 올려야 할지 모르겠어요" |
| 재정의 | 숏폼 = 알고리즘이 환자를 찾아가는 유도탄 (롱폼=설득, 숏폼=발견) |
| 인사이트 | 숏폼 발견 → 프로필 → 롱폼 설득 → 예약 |
| 우리 방식 | 월 콘텐츠 캘린더, 촬영·편집·업로드 대행, 릴스/숏츠 동시 배포 |
| CTA | "우리 병원 SNS, 어떻게 살릴 수 있을지 상담받기" |

### /search-ads — 검색광고

| 섹션 | 내용 |
|------|------|
| 공감 | "광고비만 나가고, 진짜 환자가 오는 건지 모르겠어요" |
| 재정의 | 건당 6만원 디비 광고 vs 의도 있는 검색 환자. 검색광고가 합리적. |
| 인사이트 | 외국인 타겟 시 구글 지도 최적화와 시너지 폭발 (검색 상단 + 지도 동시 노출) |
| 우리 방식 | 키워드 설계 → 전환 추적 → 매주 최적화. 클릭이 아니라 예약을 추적. |
| 증거 | diagnostic.ourcompany.kr 실제 데이터 + ROAS |
| CTA | "현재 광고비 대비 효율, 무료로 분석해 드립니다" |

### /naver — 네이버 플레이스

| 섹션 | 내용 |
|------|------|
| 공감 | "네이버 플레이스에서 우리 병원이 밀리고 있어요" |
| 재정의 | 상위 노출 보장. 안 되면 돈 안 받음. |
| 우리 방식 | 플레이스 + 블로그 + 지도 연동, 네이버 검색 생태계 전체 노출 |
| CTA | "우리 병원 네이버 순위, 지금 확인하기" / 보장: 상위 노출 안 되면 비용 0원 |

### /website — 홈페이지 제작

| 섹션 | 내용 |
|------|------|
| 공감 | "홈페이지 있긴 한데, 거기서 예약이 들어온 적이 있나요?" |
| 재정의 | ❌ 글로 벽 만든 홈페이지 ❌ 전환 추적 없는 홈페이지 ❌ 페이지 수로 견적 뻥튀기 |
| 인사이트 | 홈페이지 = 24시간 영업사원. 예약 동선 설계가 본질. |
| 우리 방식 | 마케터+디자이너 함께 설계, 전환 추적, Clarity 히트맵, 매달 데이터 기반 개선 |
| CTA | "현재 홈페이지 전환율, 무료로 진단해 드립니다" |

## 3. 성공사례 (/case-studies)

### 구조

1. **히어로** — "숫자가 증명합니다."
2. **요약 숫자 리본** — N+병원, 최대 노출 증가율, 최대 클릭 증가율, 채널 수
3. **케이스 카드** (반복) — 각 케이스마다:
   - 헤드라인 (숫자 중심)
   - 실제 스크린샷 (필수)
   - Before → After 패널
   - 변화량 필 (+539%, #1 등)
   - 스토리 3단: 상황 → 우리가 한 것 → 결과
   - 원장님 추천사 (수집 필요)
4. **CTA** — "다음 성공사례의 주인공이 되세요"

### 현재 데이터 (4건)

- 협조병원1 홍대점: 프로필 조회수 +539% (1개월), 영어 키워드 1위
- 협조병원1 명동점: 웹사이트 클릭 +895.7%, 상호작용 +720% (YoY)
- 협조병원2 잠실: "porcelain veneers" 1위 (SoLV 100), "dental implants" 1위 (SoLV 96)
- C의원 동대문: 조회수 +177%, 중문 키워드 로컬팩 안착

### 서비스 페이지 연동

서비스 페이지 증거 섹션에서 관련 케이스를 필터해서 표시.
"자세히 보기" → /case-studies#{케이스id} 앵커 링크.

## 5. 디자인 시스템

### 톤

- 라이트 배경 (#FAFAF8, #fff)
- 문제 재정의 섹션만 다크 반전 (#111)
- CTA 섹션 다크 (#111)
- 나머지는 화이트 기반

### 타이포그래피

- 한글: Pretendard (기존 유지)
- 영문/숫자: 기존 Plus Jakarta Sans 또는 변경 검토
- 큰 텍스트 위주 (h1 clamp 32~52px)
- 여백 넉넉히 (section padding 120px)

### 사진 원칙

- AI 생성 이미지 사용 금지
- 실제 대시보드 스크린샷, 구글맵 캡처, 리포트 이미지 사용
- 팀/대표 사진 (사용자 촬영 필요)
- public/images/proof/에 기존 23장 활용

### 컴포넌트 재사용

서비스 페이지 6개가 같은 구조이므로 공통 컴포넌트:
- `StorySection` — 스토리 아크 섹션 래퍼 (라벨 + 제목 + 본문)
- `BeforeAfter` — Before/After 비교 패널
- `ProofCard` — 성과 숫자 카드
- `ServiceCTA` — 서비스별 CTA (입력창 or 버튼)
- `CasePreview` — 케이스스터디 미리보기 (서비스 페이지용)
