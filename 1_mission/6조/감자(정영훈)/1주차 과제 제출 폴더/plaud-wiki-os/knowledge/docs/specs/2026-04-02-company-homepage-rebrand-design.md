# 우리 회사 홈페이지 리브랜딩 설계 스펙

## 개요

우리 회사(ourcompany.kr) 홈페이지 전면 개편. 기존 아임웹 기반 제너릭 마케팅 에이전시 사이트를 병의원 구글 지도 SEO 전문 에이전시로 리포지셔닝.

## 문제 정의

현재 홈페이지 5가지 핵심 문제:
1. **포지셔닝 실종** — 구글 지도 SEO, 외국인 환자 유치 언급 없음
2. **타깃 희석** — 병의원+전문직+요식업, 실제 포커스 불명확
3. **Social proof 전무** — 클라이언트 로고, 후기, 케이스스터디 없음
4. **CTA 빈약** — "지금 바로 문의하세요" 1줄, 리드마그넷 없음
5. **강점 숨김** — GridRank, 다국어, 실제 케이스 실적 미노출

## 기술 스택

| 항목 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | Next.js (SSG) | Claude Code 직접 수정 가능, 오케스트레이터 연동 |
| 호스팅 | Vercel | 무료, 자동 배포, 99.99% 가동률 |
| 폼/리드수집 | company-pipeline 통합 | 기존 진단 폼+웹훅+리포트 생성 파이프라인 재사용 |
| 도메인 | ourcompany.kr 유지 | 기존 URL 보존 |
| 자동 업데이트 | company-orchestrator → git push → Vercel 자동 배포 |

### 플랫폼 선택 근거
- 사용자가 디자인 툴을 사용하지 않고 Claude Code로 모든 수정을 위임
- 오케스트레이터가 레퍼런스/상품 변경 시 자동으로 홈페이지 코드 수정 → 배포 가능
- 정적 사이트로 서버 에러 가능성 극히 낮음

## 페이지 구조 (14섹션)

### 고정 네비게이션 바
- 우리 회사 로고 (좌측)
- 서비스 7개 앵커 링크: 구글 지도 | AEO | 유튜브 | 네이버 지도 | 퍼포먼스 광고 | 인플루언서 | 홈페이지
- 가격 앵커 링크
- "무료 진단" CTA 버튼 (우측 고정)
- 모바일: 햄버거 메뉴로 축소

### 섹션 1: 히어로
- 상단 라벨: "GOOGLE MAPS SEO AGENCY"
- 메인 카피: "구글 지도에서 외국인 환자가 먼저 찾는 병원으로"
- 서브 카피: "다국어 SEO · 데이터 기반 · 실시간 순위 추적"
- 핵심 숫자 3개: +539% 조회수 증가 / 구글맵 1위 / N+ 병의원 파트너
- 듀얼 CTA: "무료 GBP 진단받기" (primary) + "서비스 소개서" (secondary)

### 섹션 2: 문제 제기 (Pain Point)
- "외국인 환자가 검색하는데, 원장님 병원이 안 보입니다"
- 3가지 구체적 pain point 제시:
  1. "dermatologist near me" 검색 → 경쟁 병원만 노출
  2. 영어/중국어/일본어 프로필 미비 → 외국인 환자 이탈
  3. 순위 변동을 모니터링하지 않으면 경쟁사에 밀림

### 섹션 3: 구글 지도 최적화 (핵심 서비스)
- 서비스 내용: 다국어 GBP 최적화 (한/영/중/일), GridRank 실시간 추적, 주간 리포트, 리뷰 관리
- 인라인 케이스: 협조병원1 +539%, 협조병원2 1위
- 가격 기준: n만원/언어

### 섹션 4: AEO (AI 검색 최적화)
- 서비스 내용: ChatGPT, Perplexity, Google AI Overview에서 병원이 추천되도록 최적화
- 메시지: "서울 피부과 추천해줘 → AI가 원장님 병원을 답변에 포함"
- SEO와의 시너지: 구글 지도 = 지금 검색, AEO = AI 시대 검색

### 섹션 5: 유튜브 제작
- 서비스 내용: 롱폼 4편 + 숏폼 8편/월, 병원 전문 촬영팀
- 메시지: 검색 → 지도 → 영상 3채널 시너지
- 프로덕션 파트너: 협력업체1

### 섹션 6: 네이버 지도 상위노출
- 서비스 내용: 보장형 모델, 키워드별 맞춤 견적
- 메시지: 구글+네이버 동시 공략으로 국내+외국인 환자 모두 커버

### 섹션 7: 퍼포먼스 광고
- 서비스 내용: Google Ads + Meta Ads, SEO×광고 듀얼 채널
- 인라인 케이스: 협조병원1 홍대 CPA n원, PMax CTR 11.11%

### 섹션 8: 인플루언서 체험단
- 서비스 내용: 인플루언서 매칭, 체험 캠페인 기획/운영, 숏폼 콘텐츠 연계
- G엔터 파트너십 기반

### 섹션 9: 홈페이지 제작
- 서비스 내용: 병의원 전용 랜딩페이지, 다국어 대응, 예약 폼 내장, SEO+AEO 최적화 구조
- 메시지: "GBP에서 유입 → 홈페이지에서 전환. 유입과 전환을 동시에"

### 섹션 10: 실적 & Social Proof
- 케이스스터디 카드 (전/후 숫자):
  - 협조병원1 홍대: 조회수 +539% (1개월), "english speaking dermatologist" 1위
  - 협조병원1 명동: 웹사이트 클릭 전년비 +895.7%
  - 협조병원2 잠실: "porcelain veneers" 1위 (SoLV 100)
  - C의원 동대문: 조회수 전년비 +177%
- 클라이언트 로고 슬라이드
- 원장님 추천사 1~2개

### 섹션 11: 프로세스
- 5단계 타임라인: 무료 진단 → 전략 수립 → 다국어 최적화 → 주간 리포트 → 성과 측정
- 각 단계 아이콘 + 한줄 설명

### 섹션 12: 가격표
- 3-tier 패키지 카드:
  - BASIC n만원: 구글 지도 SEO 2개 언어 + 월간 리포트
  - STANDARD n만원: 4개 언어 + Google Ads + 숏폼 2편/월 + 주간 리포트
  - PREMIUM n만원: 풀 유튜브 패키지 + 위 전부
- 개별 견적 서비스: AEO, 홈페이지 제작, 네이버 지도, 인플루언서
- 각 플랜 CTA: "무료 진단받기"

### 섹션 13: FAQ
- 5~7개 아코디언:
  - "결과까지 얼마나 걸려요?" — 3개월 기준 가시적 성과
  - "다국어가 왜 중요해요?" — 외국인 환자 검색어의 90%가 영어/중국어/일본어
  - "AEO가 뭐예요?" — AI 검색엔진 최적화 쉽게 설명
  - "기존 GBP가 있어도 되나요?" — 기존 프로필 최적화 가능
  - "계약 기간은?" — 최소 3개월, 이후 월 단위

### 섹션 14: 최종 CTA + Footer
- CTA 블록: "지금 무료 진단 받기" 대형 버튼 + 카카오톡 + 전화번호
- Footer: 회사명, 대표자, 사업자등록번호, 통신판매업 신고번호, 주소, 개인정보처리방침 링크, 소셜 링크

## 폼/리드 파이프라인 (company-pipeline 통합)

기존 company-pipeline의 진단 폼 + 웹훅 + 리포트 생성을 Next.js 앱에 통합:

### 기존 파이프라인 구조
1. **진단 폼 3단계**: 병원 검색 → GBP 분석 → 연락처 입력
2. **웹훅 엔드포인트**: `/webhook/diagnostic-preview`, `/webhook/diagnostic-analyze`, `/webhook/diagnostic-complete`
3. **리포트 자동 생성**: HTML 리포트 → `/reports/{report_id}.html`로 직접 접근
4. **CRM 연동**: pluuug API 자동 문의 등록
5. **알림**: iMessage + Telegram 실시간 알림
6. **메타 광고**: `/webhook/meta-leadgen` 웹훅 연동

### 통합 방식
- Next.js API Routes로 웹훅 엔드포인트 마이그레이션 (또는 기존 서버를 API로 유지)
- 진단 폼 UI를 새 디자인 시스템으로 리디자인 (다크 프리미엄 톤 적용)
- "무료 GBP 진단받기" CTA → 인라인 모달 또는 전용 `/diagnostic` 페이지로 연결
- 리포트 페이지도 새 디자인으로 통일

### 언어
- 한국어 단일 (타깃: 한국 병원 원장님/마케팅 담당자)

### 법적 요구사항
- 개인정보처리방침 페이지 신설 (폼에서 이름/전화번호 수집 → PIPA 의무)
- 통신판매업 신고번호 Footer 표시

## 구현 방식: 멀티 에이전트 워게임

사용자 요청에 따라, 구현 단계에서 다음 역할의 서브에이전트가 이터레이션을 반복:

| 역할 | 담당 | 모델 |
|------|------|------|
| UI/UX 설계자 | 와이어프레임, 정보 구조, 사용자 흐름 | Opus |
| 디자이너 | 비주얼 톤, 컬러, 타이포, 스페이싱 | Opus |
| 코더 | Next.js 구현, 반응형, 성능 | Sonnet |
| 검수자 (QA) | 빌드 확인, 링크, 반응형, 접근성 | Sonnet |
| 베타 테스터 | 실 고객(원장님) 관점에서 메시지/전환 흐름 평가 | Opus |

워게임 루프: 코더 구현 → QA 검수 → 디자이너 리뷰 → 베타 테스터 평가 → 피드백 반영 → 반복 (사용자 만족까지)

## 오케스트레이터 연동

배포 후 company-orchestrator에서 자동 업데이트 파이프라인 구축:
- 클라이언트 추가/제거 시 로고 섹션 자동 갱신
- 케이스스터디 데이터 변경 시 숫자 자동 반영
- 가격 변경 시 가격표 자동 수정
- git push → Vercel 자동 배포
