# GBP 키워드 순위 측정 방법론

DataForSEO 등 순위 체크 API를 쓸 때 **단일 좌표 순위**가 아닌 **반경 확장 방식**으로 측정한다.

## 규칙

1. 거래처 좌표 기준 **0.1 mile 반경**에서 키워드 순위 체크 시작
2. 해당 키워드가 **Local Pack (상위 3위) 진입 확인**되면 0.1 mile 추가 (0.2 mile)
3. Pack 진입이 유지되는 동안 계속 0.1 mile씩 확장
4. Pack에서 빠지는 반경에서 멈춤 → **최대 Pack 진입 반경**이 해당 키워드의 실제 영향력

## 왜

- 0.1 mile은 거의 상호 바로 앞 반경 — 거기서 Pack 못 들어가면 이름/카테고리 자체 문제
- 반경이 넓어질수록 커버리지·유입 규모 커짐 → 진짜 실력은 "몇 mile까지 Pack 유지하냐"
- 순위 1등이어도 0.1 mile에서만 1등이면 의미 없음. 1 mile에서 3위가 훨씬 가치 있음

## How to apply

- 대시보드 키워드 순위 표시에 `순위 + Pack 진입 반경` 병기 (예: "잠실 치과 — 2위 · 0.4mi")
- "리뷰 투입 요청" 큐 트리거: 0.1 mile 조차 Pack 밖일 때 (긴급)
- "이름 변경 실험" 큐 트리거: 0.1 mile에서 Pack 진입 안 되고 리뷰·포스트 투입 4주 경과
- "반경 확장 실험" 큐 트리거: 현재 반경에서 Pack 진입 3주 연속 안정 → 다음 반경 시도

## GBP 최적화 본질 (사용자 경험칙)

GBP 순위 올리는 작업 비중:
- **이름 바꾸기 = 50%** (상호에 키워드 넣기, 실험·확정)
- **키워드 소팅 = 50%** (어떤 키워드를 잡을지 선정)
- **리뷰·답글·포스트 = 0%** (이름+키워드 끝나면 따라오는 결과물. 작업자 외주)

→ **대시보드는 의사결정 도구**: "어떤 이름 실험할지, 어떤 키워드 추가/제거할지" 결정.
→ 작업자에게 전달되는 **작업 지시서 출력**이 중요 기능. 사용자가 직접 리뷰 답글 쓰지 않음.
→ A안 "할 일 큐"의 카드는 사용자용(결정)과 작업자용(실행)을 구분해야 함.
