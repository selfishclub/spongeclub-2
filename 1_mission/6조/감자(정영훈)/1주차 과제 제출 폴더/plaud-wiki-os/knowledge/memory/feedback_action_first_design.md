---
name: 액션 우선 UI 설계
description: 대시보드/도구 UI 설계 시 입력/행동 영역을 최상단에 배치
type: feedback
---

대시보드나 도구 UI를 설계할 때 "행동하는 것"이 가장 중요하다고 봄.

**Why:** "계획을 아무리 이쁘게 짜도 행동하는 게 중요해" — 목표 대시보드에서 프로그레스 바보다 입력 폼이 먼저 보여야 한다고 판단.

**How to apply:** 대시보드/투두/트래커 같은 도구를 설계할 때, 데이터 시각화보다 사용자 액션(입력, 기록, 실행)을 화면 최상단에 배치할 것.
