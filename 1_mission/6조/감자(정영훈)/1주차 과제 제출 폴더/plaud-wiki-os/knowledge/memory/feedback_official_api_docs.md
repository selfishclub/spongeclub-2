type: feedback
---

API 연동 개발 시 반드시 공식 API 문서를 기반으로 개발해라. 추측으로 구현하지 마라.

**Why:** 클로드코드는 최신 API 문서를 가져오지 못하는 경우가 많다. 공식 문서 없이 추측으로 개발하면 인증 방식(헤더 vs form-data), 응답 구조, 엔드포인트 경로 등을 틀려서 시간 낭비가 반복됨. Local Falcon API에서 실제로 이 문제가 발생함.

**How to apply:**
1. API 연동 시작 전, Context7/WebFetch로 공식 문서를 먼저 시도해라
2. 가져오지 못하면 빠르게 포기하고 사용자에게 "공식 API 문서 URL이나 파일을 직접 붙여주세요"라고 요청해라 — 여러 번 시도하며 시간 낭비하지 마라
3. 문서가 확보되면 프로젝트 docs/ 폴더에 저장해서 이후 참조 가능하게 해라
4. 절대 API 응답 구조를 추측하지 마라 — 실제 API 호출로 확인하거나 문서를 기반으로 해라
