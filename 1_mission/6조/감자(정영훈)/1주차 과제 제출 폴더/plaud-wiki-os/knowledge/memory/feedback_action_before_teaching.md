광고·운영 문제에 대해 답할 때 교육(개념 설명, "이게 뭐냐"류) 먼저 제시하지 마라. 최적화 계획과 실행이 우선이고, 사용자가 배우는 건 그 실행을 옆에서 관찰하면서 자연스럽게 이루어진다.

**Why:** 사용자는 "날 알려주면서 실력이 늘도록 + 귀찮은 일들은 네가 해줘"라고 요청했지만, 실제 가치 우선순위는 **문제 해결이 먼저**. 수업이 앞에 붙으면 "일 안 하고 강의만 한다"는 인상 + 의사결정 지연.

**How to apply:**
- 문제 분석 → 최적화 방안 제시 → 실행 → (필요할 때만) 실행 중 개념 짧게 언급
- "이게 뭐냐 (쉬운 말로)" 섹션을 답변 맨 앞에 두지 마라
- "이번에 배운 것" 같은 교육 박스는 사용자가 명시 요청할 때만
- 광고 성과 이슈 = 답변 구조는 "진단 → 최적화 옵션 → 추천 → 실행 플랜"
