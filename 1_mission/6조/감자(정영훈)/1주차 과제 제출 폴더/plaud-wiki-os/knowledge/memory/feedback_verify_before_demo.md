name: 데모 전 실제 검증 필수
description: 데모 보여주기 전에 브라우저에서 직접 확인하고 에러 없는지 검증해야 함
type: feedback
---

"됐다"고 말하기 전에 실제 브라우저에서 페이지가 정상 로드되는지 확인해야 한다.

**Why:** 페이스메이커 대시보드에서 빌드 통과만 확인하고 "데모 띄워" 했는데, 실제로는 플러그 API 403 에러로 페이지가 터졌음. 사용자가 직접 확인한 뒤 지적함.

**How to apply:** 
- 빌드 통과 ≠ 작동. dev 서버에서 실제 페이지 접속해서 런타임 에러 확인 필수.
- 외부 API 연동 시 실제 호출 테스트를 코드 작성 직후에 하라.
- "완료"라고 말하기 전에 webapp-testing 또는 curl로 실제 응답 확인.
