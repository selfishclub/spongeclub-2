PPT·카드뉴스·홈페이지·랜딩·리포트·제안서 등 **외부로 메시지가 나가는 산출물**을 만들 때는 반드시 **모든 텍스트 내용을 사용자에게 검토받은 후** 디자인 빌드·발행으로 진행한다.

**Why:**
번역투·어색한 표현·부정확한 표현이 외부에 노출되면 회사 톤이 무너지고 거래처 신뢰가 깎인다. AI가 만든 표현은 자체 검수로 잡히지 않는 미묘한 어색함이 많아 사람의 눈이 반드시 필요하다.

**How to apply:**
1. 산출물에 들어가는 **모든 텍스트**(타이틀·부제·본문·표 셀·캡션·메타·CTA)를 정리해 사용자에게 전수 검토 요청
2. 검토 받지 않은 상태에서 디자인 도구(Pencil·Figma·HTML 등)에 텍스트를 채우지 말 것
3. 검토 후 수정 사항 반영 → 사용자 최종 OK → 그 다음 빌드 시작
4. 빌드 도중 추가/변경되는 텍스트도 동일 — 새 문구 넣기 전 먼저 보고

**적용 범위:**
- PPT·키노트 슬라이드 (Pencil 등)
- 카드뉴스·SNS 콘텐츠
- 홈페이지·랜딩 페이지 카피
- 외부 발송 리포트·제안서·인수인계 자료
- 광고 카피·크리에이티브
- 이메일·SMS·DM 발송 본문

**적용 제외:**
- 내부 메모·노트·기록
- 코드 주석·로그·내부 도구 UI
- 테스트용 더미 텍스트 (단, 빌드 전 사용자 검토 필요한 자리는 placeholder 명시)
