사용자가 산출물(.pen·.fig·코드 등)을 **직접 손으로 수정**한 사항은 우리 빌드 스크립트나 자동 생성 출력보다 **우선**한다. 사용자의 직접 수정은 "이렇게 해달라"는 명시적 지시다.

**Why:**
사용자 직접 수정은 우리 안이 잘못됐거나 부족해서 사용자가 직접 정정해 보여준 것. 그걸 빌드로 덮어쓰면 사용자 작업이 사라지고 같은 잘못을 반복하게 됨.

**How to apply:**
1. 빌드 스크립트 다시 돌리기 전에, 사용자가 그 사이 직접 편집했는지 먼저 확인 (파일 mtime·크기·내용 변동)
2. 직접 수정 흔적이 있으면 빌드로 전체 덮어쓰지 말고 **부분 수정만**
3. JSON 같은 구조화된 파일은 Python으로 특정 키·텍스트만 정확히 패치
4. 사용자가 직접 수정한 표현·구조·문구는 우리 1차 안과 충돌해도 사용자 안을 기준으로 진행
5. 충돌 발생 시 사용자에게 짧게 확인 ("이건 의도된 수정이에요? 아니면 우리 안으로 돌릴까요?")

**예외:**
사용자가 명시적으로 "내 수정 무시하고 다시 빌드해" 또는 "초기화" 요청한 경우만 전체 덮어쓰기 가능.

**적용 범위:**
.pen / .fig / .pptx / .docx / .xlsx / .html / 코드 파일 등 사용자가 직접 편집할 수 있는 모든 산출물
