산출물 파일(`.pen`, `.pptx`, `.pdf`, `.docx`, `.xlsx` 등 외부 공유용 결과물)은 작업 디렉토리(`/Users/user/Desktop/claude code`) 안에 두지 말고 **외부 GDrive 공유드라이브 적절한 폴더**에 직접 생성/저장한다.

**Why:** 작업 디렉토리에 산출물이 쌓이면 나중에 외부 공유/저장이 번거로움. 처음부터 최종 위치에 만들면 정리 부담 0.

**경로 베이스:**
`/Users/user/Library/CloudStorage/GoogleDrive-[연락처]/공유 드라이브/우리 회사 문서함/`

**최상위 구조:**
- `01_계약-법무` — 회사 계약/법무 자료
- `02_영업` — 영업 자료
- `03_클라이언트` — 거래처별 폴더
- `04_보고서` — 회사 차원 보고서
- `05_회사_내부` — 운영/내부 자료
- `06_학습-레퍼런스` — 학습/벤치마크 자료
- `_아카이브` — 아카이브

**`03_클라이언트/{거래처명}_{지점명}/` 하위 5종 (모든 거래처 폴더 동일):**
- `견적서` — 견적서 (PDF/Excel)
- `계약` — 계약서/계약 관련
- `기획안` — 제안서/기획안/디자인 시안 (.pen, .pptx, .docx)
- `보고서` — 거래처 발송 보고서 (GBP 진단/SEO/광고 리포트, PDF/.docx)
- `기타` — 위 4개 외 작업물

**산출물 → 폴더 매핑:**
| 산출물 | 폴더 |
|---|---|
| 견적서 | `03_클라이언트/{거래처}/견적서` |
| 계약서 | `03_클라이언트/{거래처}/계약` |
| 제안서/기획안/디자인 시안 (.pen, .pptx, .docx) | `03_클라이언트/{거래처}/기획안` |
| 거래처 발송 보고서 (.pdf, .docx) | `03_클라이언트/{거래처}/보고서` |
| 위 외 클라이언트 작업물 | `03_클라이언트/{거래처}/기타` |
| 회사 차원 보고서 (특정 거래처 X) | `04_보고서/` |
| 운영/내부 자료 | `05_회사_내부/` |
| 학습/레퍼런스 자료 | `06_학습-레퍼런스/` |

**How to apply:**
- 산출물 생성 직전 거래처명 + 산출물 종류로 GDrive 경로 결정
- 거래처 폴더에 새 거래처면 `03_클라이언트/`에 폴더 생성 + 5종 하위 폴더 생성 후 진행
- 파일명은 기존 규칙(`feedback_file_organization.md` — `YYMMDD_상호명_지점_채널_유형`) 적용
- 작업 중간 산출물(분석 노트, 임시 파일)은 작업 디렉토리 OK. **최종 산출물만 GDrive 직행**
- 사용자가 명시적으로 작업 디렉토리에 저장하라고 하면 그 지시 우선
