---
name: 기술 용어 대신 액션 위주 설명
description: 기술스택/파일명 나열 금지, 사용자가 해야 할 행동 위주로 쉽게 설명
type: feedback
---

사용자에게 작업 결과를 보고할 때 파일 목록, 타입 정의, 스키마 변경 등 기술 디테일을 나열하지 마라.

**Why:** 사용자는 비개발자(마케팅 대행사 대표). 기술스택 설명은 이해 불가. "무슨 말인지 모르겠다"는 피드백 받음 (2026-03-31).

**How to apply:**
- "뭘 만들었는지"가 아니라 "사용자가 뭘 할 수 있게 됐는지"로 설명
- 사용자가 직접 해야 하는 액션이 있으면 번호 매겨서 구체적으로 안내
- 기술 용어는 괄호 안 부연으로만 (예: "데이터베이스 구조 변경(마이그레이션)")
- 완료 보고는 3줄 이내 요약 → 사용자 액션 리스트 순서로
