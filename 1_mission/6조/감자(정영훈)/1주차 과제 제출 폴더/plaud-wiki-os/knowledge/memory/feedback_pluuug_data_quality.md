pluuug API 데이터를 분석할 때 전체 건수를 그대로 쓰면 안 된다.

**Why:** 작년 11~12월 광고로 유입된 리드가 파이프라인에 남아있지만 이미 죽은 DB. 이걸 "팔로업 대상"으로 분석해서 완전히 틀린 조언을 했음 (2026-04-03).

**How to apply:**
- 의뢰 데이터 분석 시 반드시 유입 날짜 확인 — 최근 30일 이내만 "활성 리드"로 취급
- 오래된 리드는 "과거 DB" 또는 "콜드 리드"로 별도 분류
- 파이프라인 현황 보고 시 "활성 리드 X건 (최근 30일)" 형태로 명시
- 계약은 자동 연장 조항이 있으므로 "만료 임박" 경고 불필요
