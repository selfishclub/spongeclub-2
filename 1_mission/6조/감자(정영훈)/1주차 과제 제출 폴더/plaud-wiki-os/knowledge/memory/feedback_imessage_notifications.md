type: feedback
---

알림 채널이 텔레그램에서 iMessage self-chat으로 이전됨 (2026-03-31).
- chat_id: `any;-;[연락처]`
- 텔레그램 불안정으로 사용자가 iMessage로 전환 결정

**알림 허용 조건:**
1. 사용자 권한/승인이 필요한 경우 (결정 필요, 에러로 중단 등)
2. 사용자가 명시적으로 지정한 반복 스케줄 알림
3. 모든 작업/프로세스 완료 시 결과 알림

**금지:**
- 세션 종료, 컨텍스트 압축 등 시스템 상태 알림
- "참고로 알려드립니다" 류의 정보성 알림

**Why:** 사용자가 자리를 비울 수 있으므로 작업 완료를 iMessage로 받아야 함. 단, 시스템 상태 알림은 여전히 불필요.

**How to apply:**
- 인세션 알림 → `mcp__plugin_imessage_imessage__reply` (chat_id: `any;-;[연락처]`)
- 작업 완료 시: 결과 1줄 요약 + 후속 액션 필요 여부만 간결하게
- 보내기 전 자문: "이건 작업 결과인가, 시스템 잡음인가?" → 잡음이면 보내지 마라
