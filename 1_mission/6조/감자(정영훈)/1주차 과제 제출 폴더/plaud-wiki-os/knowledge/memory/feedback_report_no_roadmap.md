업주 발송 리포트는 **진단·분석 문서**다. 로드맵·결정 요청·부록 섹션 만들지 마.

**Why:**
- "P0(즉시) → P1(2주) → P2(1개월)" 같은 우선순위 분류는 우리 워크플로우와 안 맞음. 현실성 떨어짐. 사용자가 자기 일정으로 결정.
- "결정 요청 사항" 섹션은 사용자가 결정. 묻지 말 것.
- "부록 — 외부 데이터 출처 목록" 같은 URL 풀은 본문에서 분리하면 죽은 정보. 본문 인라인 하이퍼링크로만.

**금지 섹션:**
- ❌ 액션 플랜 P0/P1/P2 (또는 즉시/2주/1개월 같은 우선순위 분류)
- ❌ 결정 요청 사항 / 업주 결정 사항 / 본점 검토 항목
- ❌ 부록 A/B/C (LocalFalcon 측정 결과·외부 채널 절차·출처 목록 등 모든 부록)
- ❌ "다음 액션" 섹션

**대체:**
- 진단 카드 끝에 "이 결과는 X를 의미합니다" 정도까지만 (행동 결정은 사용자가)
- 근거 URL은 본문 흐름 안에 인라인 `[텍스트](URL)` 하이퍼링크로
- LocalFalcon 측정 결과는 진단 본문 안에 자연 문장으로 녹임

**적용 범위:** 모든 외부 발송 리포트 (H대표 v4 톤 / Gem 양식 둘 다)
