우리 회사 영업 병목을 하나씩 제거한다. 광고 유입 리드 응대, 구글 지도 진단 요청 후속 조치, 미팅 세팅, 미응답 리드 회수, SOLAPI/Kakao/Pluuug CRM 누락을 매번 확인하고, 실제 고객 발송·CRM 변경·광고 설정 변경 전에는 반드시 사용자 승인을 받는다.

**Why:** 매출 병목은 "새 광고"보다 "이미 들어온 리드 응대 지연"에서 먼저 샌다. 광고는 돌고 있는데 응대·후속·미팅 세팅이 흘러내리는 게 실제 새는 구멍. 2026-05-08 사용자 정의.

**How to apply:**
- 매번 병목 1개만 잡는다 (광고 유입 응대 / 진단 요청 후속 / 미팅 세팅 / 미응답 회수 / SOLAPI·Kakao 응대 누락 / Pluuug 단계 정리 / 광고→문의→상담 전환 점검 중 택1)
- 사용자가 먼저 현황 읽고 오늘 처리 액션 3개로 압축 → Claude는 짧은 판단(이거 보내/이건 보류/미팅 제안 이걸로) 보조
- 자율 모드 아님. 사용자 주도, Claude는 데이터 정리·문구 초안·승인 대기
- 컨펌 필수 액션: SOLAPI 발송, Kakao 메시지, Pluuug 레코드 생성/수정, 광고 ON/OFF·예산, 고객 발송 최종 문구
- 첫 진입 추천: "최근 광고 유입/진단 요청 중 미응답·후속조치 누락 리드부터"
