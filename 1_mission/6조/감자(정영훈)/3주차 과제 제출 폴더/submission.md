# 3주차 — 내 OS 최종 완성 🏁

> 미션을 진행하며 과정과 결과를 기록해주세요. (다 못 채워도 OK, 한 것 위주로!)

## 🎯 미션 1. 내 삶을 돕는 OS 최종 완성

이번 주는 새 OS를 하나 만들기보다, 실무에서 이미 매일 돌아가는 **OS 3개를 최종 완성해 정리**했다. 세 개가 각각 성격이 다르다 — 콘텐츠를 뽑는 것, 회사 안에서 일을 처리하는 것, 밖에 파는 상품이 되는 것. 그래서 하나의 글에 몰지 않고 프로젝트별로 문서를 나눴다.

- **완성한 것 (무엇을):**
  1. [카드뉴스 자동화 스튜디오](카드뉴스-자동화-스튜디오.md) — 인스타 카드뉴스를 기획→카피→이미지까지 뽑는 로컬 웹 도구.
  2. [구글지도 최적화 OS](구글지도-최적화-OS.md) — 거래처의 구글 지도 키워드를 발굴·검증·선정하는 사내 업무 도구.
  3. [구글맵부스터 — 구글지도 최적화 SaaS](구글맵부스터-지도-SaaS.md) — 위 사내 도구를 병원 원장이 직접 쓰는 상품으로 만든 것.

- **피드백 반영한 점:**
  - 카드뉴스: 스폰지클럽 2회차 '캐러셀 OS' 강의를 듣고, 디자인/내용을 별도 문서로 분리하는 개념과 'HTML 템플릿으로 글자 정확하게 찍는 방식'을 도구에 더했다. (자세한 내용은 각 문서에)
  - 구글지도 최적화 OS: 예전에 '검증을 건너뛰고 키워드를 넣어버린 사고'가 있었다. 그 피드백을 사람의 주의력이 아니라 코드와 데이터베이스로 막게 다시 설계했다.
  - 구글맵부스터: 사내 도구를 만들며 실제로 데였던 함정(API·언어코드)을 상품 코드에 미리 반영해, 같은 실수를 반복하지 않게 했다.

- **결과물 (링크·스크린샷):** 위 3개 문서 참고. (스크린샷은 각 문서에서 필요 시 `이미지첨부/` 폴더로 연결)

- **알게 된 인사이트:**
  같은 뿌리(구글 지도 키워드 검증)에서 사내 도구 → 외부 상품이 자라났다. 내부에서 충분히 데이고 규칙을 코드로 굳혀 두니, 그 규칙이 그대로 상품의 품질 보증이 됐다. "도구를 만든 경험이 곧 상품의 원가·리스크 계산서가 된다"는 걸 이번에 체감했다.
