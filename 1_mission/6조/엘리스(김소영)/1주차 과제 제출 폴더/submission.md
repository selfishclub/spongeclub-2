> 미션을 진행하며 **과정과 결과물**을 기록해주세요. (다 못 채워도 OK, 한 것 위주로!)

## 🎯 미션 1. 내 OS 재료 찾기
> 인터뷰(아이데이션)로 "내 삶·일에 필요한 게 뭔지" 찾기

- **과정 (어떻게 찾았나):**
  내 하루와 일을 되짚어 보니, 나는 이미 **같은 요청을 매일 반복**하고 있었다 (매일 수동으로 돌리는 브리핑, 반복되는 데이터 정리, 반복 서버 실행 등). 동시에 떠오른 게 있었다 — 회사에서 AI 활용을 여러 번 발표했는데도, **1년이 지나도 AI로 아무것도 못 하는 동료들**이 있다. 그들도 AI를 쓰긴 쓴다. 이 두 개가 처음엔 따로였는데, 파다 보니 **하나의 뿌리**로 모였다: *"내 반복을 자동화하는 것"* 과 *"동료가 AI를 실제로 쓰게 만드는 것"* 은 같은 문제였다 — **맥락**.

- **결과 — 🟡 내 OS 재료 카드:**

  | 항목 | 내용 |
  |---|---|
  | 영역 | 일 + 팀 (내 반복 + 팀 전체의 AI 격차) |
  | 걸리는 지점 | ① 내 반복 요청이 스킬로 안 남음 ② 동료 AI는 맥락이 없어 못 씀 — 프롬프트를 줘도 세팅이 안 되고 그 컴퓨터에서 안 돌아감. **내 AI와 동료 AI가 다른 건 "아는 정보"가 달라서** ③ 중요한 일(지속·디벨롭 가치)이 데이터가 아니라 술자리·목소리 큰 사람으로 정해짐 |
  | 지금은 | 일일이 알려주고 프롬프트를 나눠주지만 전달이 안 됨. 중도 포기 |
  | OS가 된다면 | **활동수집만 켜면** → 내 반복 감지 후 "스킬로 만들까요?" 제안 + 팀 공통이슈를 "요즘 팀은 이걸 고민해요" 먼저 브리핑 + 이슈 연결로 진짜 가치를 데이터로 |
  | 한 문장 | "AI 못하는 사람의 바닥을 올리고, 중요한 일이 정치가 아니라 데이터로 드러나게" |
  | 첫 한 걸음 | 내 AI 로그에서 **반복 요청을 감지하는 엔진** 하나 |

- **느낀 점:**
  진짜 문제는 "프롬프트를 모른다"가 아니라 **"맥락이 없다"** 였다. 프롬프트를 아무리 나눠줘도 각자의 AI에 맥락이 없으면 안 돌아간다. 그래서 필요한 건 프롬프트 배포가 아니라 **맥락(내 활동 + 팀 공통이슈)을 자동으로 심어주는 그릇**이었다.

## 🧩 미션 2. 내 OS 기획
> 무엇을, 왜, 어떻게

- **기획 내용 — "길잡이" (팀 범용 AI 동반자):**

  대상은 **AI를 잘 못 쓰는 사람**. MCP도 스킬도 모르고, 클로드랑 대화만 할 줄 알고, 회사 계정으로 클로드 코드에 들어갈 줄은 아는 사람. 그런 사람도 이 팀 범용 스킬 하나면, 헤매지 않고 "우리 팀이 요즘 공통으로 겪는 것"을 **먼저 알아챈다.**

  **두 축:**
  - **① (개인)** 내 반복 요청을 감지 → *"이거 매주 반복하시네요, 스킬로 만들까요?"* → 스킬 생성
  - **② (공통)** 팀 공통 이슈를 수집 → *"요즘 팀은 이걸 고민해요"* 먼저 브리핑 → 내 질문·보고서 니즈와 **연결해 방향 제시**

  **양면 플랫폼:**
  | | 누구에게 | 무엇을 |
  |---|---|---|
  | 아래 | AI 못하는 동료 | 맥락을 심어 AI가 실제 작동 → **바닥 올리기** |
  | 위 | 팀장·실장·리더 | 흩어진 활동·이슈를 연결해 **진짜 가치를 데이터로 가시화** |

  이 둘을 잇는 축이 **활동수집**. 아래에서 켜지면 위에서 자동으로 데이터가 된다.

  **원칙:** 기록은 기본 **내 로컬에만**, 필요할 때만 골라서 공유(opt-in). 세팅은 **한 걸음(활동수집)**. 가장 작은 조각부터.

- **막혔던 점 / 어떻게 풀었나:**
  "반복감지 + 공통이슈 + 연결 + 가치가시화 + 온보딩"을 다 그리니 그림이 너무 커졌다. → **가장 작은 조각(내 반복을 감지하는 한 루프)** 부터 만들기로 범위를 좁히니 손에 잡혔다.
