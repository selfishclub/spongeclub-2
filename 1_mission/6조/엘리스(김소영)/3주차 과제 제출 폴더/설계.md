# WeStack — 팀 AI 공동자산 사무실 (설계)

> 1·2주차의 반복감지 엔진을 **ZEP풍 2D 픽셀 AI 가상 사무실**로 시각화·확장한 3주차 결과물의 설계.
>
> ⚠️ **발표·데모용입니다.** 실제 회사 데이터·서비스명·지표는 포함하지 않으며, 조직/인물/서비스는 모두 제네릭 예시(서비스1~4 등)입니다.

## 1. 문제의식 — 혼자 성장 vs 같이 성장

각자 AI 도움만 받으면 개인은 편해져도 팀은 성장하지 않는다. 서로 뭘 하는지 모르고, "그 방법 공유해줘"라고 부탁하기도 민망하다.

→ 누군가 만든 방법(스킬)을 **공공자산**으로 두고, 팀이면 누구나 열람·복사하게 하면 각자 일하던 팀이 자연스럽게 **같이 성장**한다.

## 2. 핵심 메커니즘 (데모가 증명하는 흐름)

1. **반복 업무 발견** — 내 봇이 매주 반복하는 일을 감지 (이번 주 N번·이번 달 M번 빈도까지 알려줌)
2. **스킬로 만들기** — 작업하면서 도구로 자동화 → 내 봇 레벨업
3. **공유 범위 선택 (개인 → 파트 → 팀 3단계)**
   - 먼저 **나만 쓰기 / 파트에 공유하기** 선택 (공유하면 파트장이 검토)
   - 파트장이 다시 결정: **우리 파트만 / 팀 전체 공유**
4. **전파** — 팀 전체면 📚 공공 자료실에 등록 → 다른 파트장이 *필요한 것만* 가져가 자기 업무에 맞게 바꿔 씀 / 파트만이면 파트 안에서만 나눠 씀
5. **함께 성장** — 팀 평균 레벨 ▲

## 3. 원칙

- **방법만 공유, 실제 데이터는 각자** — 자료실엔 방법·신호·함정만. 실제 수치·개인정보는 각자 공간에 남긴다.
- **필요한 사람만 가져가기(pull), 자동으로 뿌리지 않음** — 원치 않는 확산·오염 방지.
- **승인 게이트** — 공유는 내가 고른 것만. 파트/팀 확산은 파트장이 결정.

## 4. 화면 구성

- **가상 사무실(canvas)**: 파트별 자리(CS운영파트·서비스1~4파트) · 회의실 · 사내카페 · 분석실 · 팀장/실장실 · TF 회의실 · 라운지 · 📚 공공 자료실. 캐릭터는 자기 자리 주변을 자유롭게 배회한다.
- **캐릭터 클릭** → **전문 분야 + 소개 + 보유 도구(개수·목록) + 담당 업무**. 누가 뭘 잘하는지 한눈에 보인다.
- **상단 콕핏**: 오늘 브리핑(자동/직접 구분) · 스케줄 보드 · 전달함 · 팀 제안 · 팀 성장 바.
- **엘리스 상태바(ZEP식)**: 자리·식사·휴식·화장실·퇴근 — 상태를 바꾸면 해당 공간으로 이동. **퇴근 전엔 오늘 하루 브리핑 + 내일 할 일 체크**를 거친다.
- **공공 자료실**: 책(스킬)을 클릭하면 🎯 이 도구가 하는 일 / ♻️ 재활용 가능성 / ⚠️ 주의점 + [나에게 적용하기].

## 5. 구현

- 단일 self-contained `office.html` + `<canvas>`. **빌드·서버 0**, 브라우저로 바로 실행.
- MODI 디자인 토큰(primary `#2c6edb`, 흰 카드 + 얇은 테두리) + Wanted Sans, 라이트/다크 모드 지원.
- **[데모 재생 ▶]** 버튼이 위 메커니즘 전 과정을 자동 연출(시네마틱). 각 캐릭터·자료실은 직접 클릭해 상호작용.

## 6. 발표용 / 실제 적용 경계

- 지금 공개하는 것은 **메커니즘·설계·데모**뿐이다. 실제 서비스명·지표·회사 데이터는 포함하지 않는다.
- 실제 업무 적용 시엔 이 데모 뒤에 사내 데이터·활동 수집과 연동하는 **배선**을 별도(비공개)로 둔다. 데모의 `SERVICE_MAP` 등은 그 자리를 비워둔 연결점이다.
