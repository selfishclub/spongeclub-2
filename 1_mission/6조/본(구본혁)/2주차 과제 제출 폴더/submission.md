## 결과물

**내 OS: PAZX** — "생각이 만나는 순간, 파직." 1주차 텔레그램 수집함을 웹으로 옮겨, **모아서(수집) → Clap(합치기) → Bam(새 아이디어)** 한 문장으로 모든 화면이 돌아가는 아이디어 반응로.

- **동작하는 라이브 서비스**: https://clapbam.vercel.app (로그인 → 아이디어 입력 → 웹 레퍼런스 결합 → 새 컨셉·이미지)
- **탐색 로딩 애니메이션 데모**(로그인 없이): https://clapbam.vercel.app/beamtest

이번 주에 실제로 만든 것:
1. **홈 리믹스 스튜디오** — 자연어로 아이디어를 던지면 우측에 실시간 웹 레퍼런스를 고감도 이미지 위주 masonry로 보여주고, 조각을 골라 "결합"하면 Claude가 하나의 새 컨셉으로 재구성 + **Pollinations(무료)로 새 이미지까지 생성** → 스와이프 덱으로 보관.
2. **"검색"에서 "크리에이티브 컨설팅"으로 개편** — 키워드 매칭이 아니라 구어("제안서 컨셉 디벨롭해줘")를 방향으로 해석하도록 프롬프트·레이아웃을 전면 개편.
3. **PAZX 리브랜딩을 코드까지** — 실물 로고/엠블럼(말풍선 구름+라임 눈), 파비콘, 전용 폰트, 라임 #C6FF00 단일 포인트, Clap 융합 콜리전·로고 인트로·테마 토글 모션.
4. **아이디어 탐색 로딩 = 캐릭터 애니메이션** — UFO 구름이 서치라이트 빔으로 좌우를 두리번거리고, 별·번개 오브제가 빔을 타고 구름에 빨려 들어가는 로딩(브랜드 캐릭터 그대로, GPU transform만, reduced-motion 대체 포함).

## 삽질 과정

- **구글 이미지 검색 API 403 지옥** — 레퍼런스를 더 꽉 채우려고 Google Custom Search(이미지)를 붙였다. 검색엔진 생성 버튼이 계속 비활성(사이트 필드 필수), 크롬 확장프로그램이 reCAPTCHA를 깨는 문제, 그리고 키·프로젝트·결제까지 다 정상인데도 `403 PERMISSION_DENIED`가 20분+ 안 풀리는 벽을 만났다. 콘솔과 씨름하다 결론: **코드를 안전 폴백 구조로** 짰다 — 구글이 실패하면 조용히 원래 Claude 웹검색만으로 동작하고, 키가 살아나면 자동으로 결과가 늘어난다. 덕분에 API가 안 뚫려도 앱·시연은 멀쩡했다.
- **로딩 애니메이션이 "안 움직이던" 진짜 원인** — 구름 좌우 이동이 통 안 보였는데, 알고 보니 한 요소에 이동(translateX)과 부유(translateY) 두 애니메이션이 **같은 transform을 두고 충돌**해 부유가 이동을 통째로 덮어쓰고 있었다. 이동/부유를 다른 래퍼로 분리하니 살아났다.
- **모션은 말이 아니라 움직임으로만 검증됨** — "너무 딱딱하다 / 오브제가 작다 / 빔과 바닥 광원 위치가 어긋난다"를 스크린샷 크리틱으로 여러 번 다듬었다. 특히 빔 회전과 바닥 광원 위치를 삼각함수(apex→bottom × sin θ)로 계산해 동기화하고, 회전 방향 부호를 반대로 넣은 버그까지 잡고서야 맞아떨어졌다.

## 인사이트

수집과 생성은 다른 제품이다 — 모으는 기능보다 "부딪쳐 새 걸 뽑는 순간"에 가치가 있고, 외부 API·모션 같은 불확실한 조각은 **처음부터 폴백을 깔고 움직임으로 검증**해야 무너지지 않는다.
