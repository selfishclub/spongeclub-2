# 1주차 — 나만의 OS 만들기 🛠️

> 미션을 진행하며 **과정과 결과물**을 기록해주세요. (다 못 채워도 OK, 한 것 위주로!)

## 🎯 미션 1. 내 OS 재료 찾기
> 인터뷰 스킬(아이데이션)로 "내 삶에 필요한 게 뭔지" 찾기
- **과정 (어떻게 찾았나):** 현재 워크플로우를 3단계로 뜯어봤다. ① 보험사에서 발급하는 통합 계약 조회 리포트(PDF, 신정원 표준 담보명 정리) 받기 → ② 원자료를 자체 제작한 "컨설팅 전/후 비교" 엑셀 템플릿에 옮기고 유지·해지·신규로 재구성 → ③ 매달 갱신하는 보험사별 상품·보험료 비교표를 보며 고객 예산 안에서 보장을 직접 비교·결정. 이 중 어디에 반복적으로 손이 많이 가는지를 짚었다.
- **결과:** "설계(보험 상품 비교·제안서 작성)에 드는 시간을 줄이고 싶다." 불편함의 핵심은 업무 자체의 비효율에 있었다. 반복적으로 손이 가는 지점은 (1) 원자료의 담보 누락·순서 문제 보정, (2) 컨설팅 전/후 비교표에 데이터 옮겨 적기, (3) 상품 비교표 매달 최신화. 반면 3단계(신규 상품 최종 선정)는 보험사마다 조건이 다르고 매달 바뀌어서 의도적으로 자동화 대상에서 제외했다.
- **느낀 점:** 자동화 여지가 큰 영역(원자료 → 비교 엑셀 옮기기, 비교표 관리)과, 의도적으로 자동화하지 않는 영역(상품 조합 결정)이 명확히 나뉜다는 걸 알게 됐다.

## 🧩 미션 2. 내 OS 기획
> 인터뷰 결과 + 세션 내용(흐민·배짱·키노) 활용해 기획
- **기획 내용:** 흐민 세션의 "3층 구조 · 5단계 선순환 루프" 개인용 OS 모델을 내 업무에 매핑했다 — 1층·사고 레이어(리포트 업로드) → 정리·저장(담보명 표준화·재정렬 → 비교 엑셀 자동 반영) → 2층·지식 레이어(상품 비교표와 자동 대조해 부족 보장 발견) → 3층·자산화 레이어(비교표·제안서 초안 자동 생성) → 사람이 발행 결정(신규 상품 조합 최종 확정은 반드시 직접 판단). "0 to 100 복사 금지 — 내 불편함 하나에서 시작" 원칙에 따라 5단계를 한 번에 만들지 않고, 가장 반복적으로 손이 가는 앞의 두 단계(리포트 업로드 → 정리)부터 먼저 자동화하기로 했다.
- **막혔던 점 / 어떻게 풀었나:** 3단계(상품 비교·선정)까지 자동화 범위에 넣을지 고민했지만, 보험사 조건과 상품이 매달 바뀌어 판단 영역으로 의도적으로 남겨뒀다. 실행 방식도 별도 개발 환경 대신 claude.ai "프로젝트" 기능으로 가볍게 시작하기로 했다 — 텔레그램 연동이나 상시 구동 서버는 이 단계엔 과하다고 판단.

## ⚙️ 미션 3. 내 OS 구현
> 실제로 만들어본 것 (클로드코드 '채널' 기능 활용 OK)
- **결과물:** claude.ai에 "보험 설계 OS" 프로젝트를 만들고, 프로젝트 지침(리포트 PDF가 올라오면 기존 계약 열은 자동 채움 + 계약일자순 정렬, 해지·신규 열은 절대 채우지 않음)과 프로젝트 지식(빈 보장분석 템플릿 엑셀, 판단 규칙 모음)을 세팅했다. 실제 고객 사례 두 건으로 테스트: 첫 사례에서는 25칸 중 17칸(68%)이 손으로 정리한 값과 정확히 일치했고, 나머지는 특약이 여러 개 겹칠 때(CI보장특약 처리 등) 판단이 필요해 템플릿에 별도 행으로 분리하도록 구조를 개선했다. 두 번째 사례(새 고객)에서는 총보험료·항목별 합계가 원자료와 정확히 일치했고, 애매한 항목(한도가 여러 세부 조건으로 나뉘는 경우)은 채우지 않고 되묻도록 만들었다.
